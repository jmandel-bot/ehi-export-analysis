Export Zip Overview

The export zip contains files to help a consumer understand and navigate the folder tree and files contained within.

File

Description
README.txt Contains a description of the export zip file
SCHEMA.txt Contains Documentation URL (directs to this page) and Product Version
JSONTOC.txt Contains metadata for every data file in the export zip to aid in machine readability
  • Format is ndjson
  • DocumentReference resources conform to the EHI Export API Implementation Guide (draft).
  • More details below
Note: the file type is .txt but the contents inside are ndjson.

Section Information

Each section exists as a folder or file within the export zip.  This table provides details about what is contained within each section.

Section

Details

Patient Data (CSV) Files

 Folder: AANNNNNNNNNN - The folder name represents the internal patient’s medical record identifier.
  • File: AaAaaAaaa
    • Each file name represents the namespace where the included data was documented. 
    • File format is .CSV
CSV tables/columns included
  • Client/Server Acute & Ambulatory
  • MAGIC Acute & Ambulatory
  • MPM 6.08 Ambulatory

FHIR Resource Bundle

 File: FHIR Resources.json, FHIR Ambulatory Resources.json
  • Contains all available FHIR resources for the exported patient leveraging Patient $everything
    • FHIR R4: US Core STU 3.1.1 Implementation Guide
    • More information at fhir.meditech.com

Clinical Reports/Documents

The contents in this section will vary by MEDITECH platform.


MEDITECH Client/Server

Folder: AANNNNNNNNNN_Patient_Name - the folder name represents the account number concatenated with the patient name

  • Folder: Imaging and Phys Doc
    • File: ServiceDate_ReportName~#.<file type>
      • Contains physician documentation and radiology reports/documents
      • File type can be .DOC or .PDF
  • Folder: Pathology
    • File: ReceivedDate_Pathology Report_NN.<file type>
      • Contains Pathology report documents suffixed by the date the specimen was received into the Pathology Department (YYYYMMDD format) and prefixed by an incremental report counter (NN)
      • File type can be .DOC or .PDF
  • Folder: Nursing Image Documentation
    • File: Nursing_Images.pdf
      • Contains Image documentation associated to Nursing Interventions


MEDITECH MAGIC

  • Folder: Imaging and Phys Doc
    • File: ServiceDate_ReportName~#.<file type>
      • Contains physician documentation and radiology reports/documents
      • File type will be .TXT
    • File: ServiceDate_ReportName~#_Image#.<file type>
      • Contains images associated with physician documentation and radiology reports/documentation  
      • File type will be .PNG
  • Folder: Departmental and Phys Doc
    • File: ServiceDate_ReportName~#.<file type>
      • Contains physician documentation and departmental reports/documents
      • File type will be .TXT
    • File: ServiceDate_ReportName~#_Image#.<file type>
      • Contains images associated with physician documentation and departmental reports/documentation.
      • File type will be .PNG
  • Folder: Radiology Reports
    • File: Date_ExamName~#.<file type>
      • Contains radiology reports/documents
      • Date will display the report's dictation date if available, else the report/addendum date will be used
      • File type will be .TXT
  • Folder: Pathology
    • Contains Pathology report documents and image files suffixed by the date the specimen was received into the Pathology Department (YYYYMMDD format) and prefixed by an incremental report counter (NN). Includes any images from the Pathology Report and an additional Manifest file with additional details about each image. 
    • File: ReceivedDate_Pathology Report_NN.txt
      • Contains Pathology report document
    • File: ReceivedDate_Pathology Report_NN-IMAGE#.<file type>
      • Contains Pathology report images
      • File type can be .jpg or .bmp
    • File: ReceivedDate_Pathology Report_NN-MANIFEST.txt
      • Contains the image manifest for all image files from within the Pathology Report


MEDITECH MAGIC Ambulatory

Folder: AmbDocs

  • Folder: <Document number>-<PBR Database>
    • File: <Document Name>
      • Contains the text from the physician document or nurse note
      • File type is .TXT
    • File: <DocumentName>_ImageN
      • Contains images that were included in the physician document or nurse note
      • File Type is .PNG


MEDITECH MPM 6.08 Ambulatory

Folder: AANNNNNNNNNN_Patient_Name - the folder name represents the account number concatenated with the patient name.

Folder: Imaging and Phys Doc

  • File: ServiceDate_ReportName~#.<file type>
    • Contains the text from the physician document or nurse note
    • File type is .PDF

Provider and Patient Messages

The contents in this section will vary by MEDITECH platform

MEDITECH Client/Server

  • File: Provider Messages.pdf
    • Contains Provider to Provider and Provider to Patient messaging


MEDITECH MAGIC

  • File: Provider Messages.txt
    • Contains Provider to Provider and Provider to Patient messaging
  • File: Provider Messages_Image#.<file type>
    • Images which were embedded in the Provider and Provider to Patient messaging
    • File type will be .PNG

Financial Reports

 File: FinancialEHI.txt
  • Contains a Patient Accounting report that includes financial transactions for the patient

Structured Clinical Documents

 Folder: CCDA
  • Contains all structured Consolidated-CDA documents that have been created and possibly sent outside of the organization. The document format will match one of the following releases.
    • C-CDA R2.1: C-CDA Templates for Clinical Notes R2.1 Companion Guide, Release 2
    • C-CDA R1.1: HL7 Implementation Guide: S&I Framework Transitions of Care Companion Guide to Consolidated-CDA for Meaningful Use Stage 2, Release 1 – US Realm

External Documents/Images

The contents in this section will vary by MEDITECH platform

MEDITECH Client/Server Ambulatory and MPM 6.08 Ambulatory

Folder: AmbScans

  • File: <Date.Time>-<document description>-<internal document identifier>-<page number>.<file extension>

  • File: <Date.Time>-<document description>-<internal document identifier>-<page number>-<PBR database>.<file extension>
    • Contains external documents scanned or imported into the ambulatory setting
    • File formats may include .PDF, .DOCX, .DOC, .TXT, .MTDD, .HTML, .HTM, .RTF, .ODT, .TIF, .TIFF, .PNG, .JPG, .JPEG, .BMP, and .GIF

 Folder: MPM POC Scans
  • Contains scanned documents


JSONTOC Schema

The ndjson file contains an array of file metadata (FHIR Document Reference resources) for all of the patient data files contained within the export zip file.  The pseudo-JSON below explains what each field represents.

{    
   "resourceType":"DocumentReference",
   "id": <<Identifier of this DocumentReference>>,
   "meta": {
      "id": <<Identifier of this metadata>>,
      "versionId": <<Identifier of this version>>,
      "lastUpdated": <<When this was last updated>>,
      "source": <<Where the resource came from>>,
      "profile": [
         "http://fhir.org/argonaut/ehi-api/StructureDefinition/ehi-document-reference"
      ],
      "tag": [
         {
            "code": "ehi-export",
            "display": "generated as part of an ehi-export request"
         }
      ]   
   },
   "implicitRules": <<A set of rules under which this content was created>>,
   "language": <<Language of the resource content>>,
   "status": "current",
   "docStatus": <<Status options - preliminary | final | amended | entered-in-error>>,
   "type": <<CodeableConcept indicating kind of document>>,
   "subject": {
      "reference": <<Patient Reference, resource is available in the US Core FHIR Resources.json>>
   },
   "date": <<Date the file was created>>,
   "description": <<File description>>,
   "content": [
      {
         "attachment": {
            "id": <<Identifier of this attachment>>,
            "contentType": <<File type>>,
            "url": <<Pathway to the location of the file>>,
            "size": <<File size>>,
            "title": <<Title>>,
            "creation": <<Date the file was created>>
         }
      }
   ],
   "context": {
      "id": <<Identifier of this context>>,
      "encounter": [
         {
            "reference": <<Encounter Reference, resource is available in the US Core FHIR Resources.json>>
         }
      ],
      "period": {
         "start": <<Beginning of the context period>>,
         "end": <<End of the context period>>
      }
   }
}

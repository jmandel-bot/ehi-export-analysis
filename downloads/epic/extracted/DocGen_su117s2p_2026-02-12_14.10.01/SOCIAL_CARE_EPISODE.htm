SOCIAL_CARE_EPISODE
Description:
This table stores episode information for Compass Rose episodes.

Primary Key
Column Name Ordinal Position
EPISODE_ID 1

Column Information
Name Type Discontinued?
1 EPISODE_ID NUMERIC No
The unique identifier for the episode record.
2 EPISODE_DEF_ID_EPISODE_DEF_NAME VARCHAR No
This column displays the name of the episode / block definition record.
3 EPI_STATUS_C_NAME VARCHAR No
The status category ID for the episode.
May contain organization-specific values: No
Category Entries:
Active
Resolved
Deleted
4 EPISODE_START_DATE DATETIME No
The date the episode was started.
5 EPISODE_END_DATE DATETIME No
The date the episode was ended.
6 EPISODE_PAT_ENC_CSN_ID NUMERIC No
The unique contact serial number for a special contact which stores Social Care information at episode-level. This number is unique across all patient encounters in your system. If you use IntraConnect, this is the Unique Contact Identifier (UCI).
7 DECISION_FIN_ASST_CASE_ID NUMERIC No
The unique ID of the case which contains all the decisions created in the episode.
8 RESPONSIBLE_USER_ID VARCHAR No
The unique ID of the responsible user for the episode.
9 RESPONSIBLE_USER_ID_NAME VARCHAR No
The name of the user record. This name may be hidden.
10 INIT_REFERRAL_ID NUMERIC No
The unique ID of the referral that initiated the episode.
11 PROGRAM_EPISODE_ID NUMERIC No
The unique ID of the program episode for a service episode. If the episode is voided, it will contain a link to the parent record at the time the episode was voided.

13 EPISODE_DEPARTMENT_ID_EXTERNAL_NAME VARCHAR No
The external name of the department record. This is often used in patient correspondence such as reminder letters.
14 EPISODE_SUMMARY VARCHAR No
An overview of a Social Care episode.
15 CMGMT_ENROLL_DATE DATETIME No
The date when the episode became active. Refer to CMGMT_ENROLL_CALC_DATE for more robust reporting.
16 CMGMT_ENROLL_CALC_DATE DATETIME No
The date that the user documented as the enrollment date or the date the status was originally changed to Active if no enrollment date is documented. If the overall episode status is Enrolling, the value will be blank.

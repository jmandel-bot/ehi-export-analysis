IDENTITY_POC_ID_HX
Description:
This table contains the system Master Person Index ID history for your Plan of Care. Each Plan of Care may have multiple Master Person Index IDs. A line number is used to identify each identification number for a Plan of Care.

Primary Key
Column Name Ordinal Position
POC_ID 1
LINE 2

Column Information
Name Type Discontinued?
1 POC_ID NUMERIC No
The unique identifier for the plan of care record.
2 LINE INTEGER No
The line number for the information associated with this record. Multiple pieces of information can be associated with this record.
3 MPI_HX VARCHAR No
The unique ID of the Master Patient Index (MPI) identifier before an audit event occurred.
4 MPI_HX_INSTANT_DTTM DATETIME (Attached) No
The instant at which the audit event was logged for the identifier.
5 MPI_HX_ID_TYPE_ID NUMERIC No
The unique ID of the ID type of the Master Patient Index (MPI) identifier.
6 MPI_HX_ID_TYPE_ID_ID_TYPE_NAME VARCHAR No
The name of the ID Type.
7 MPI_HX_USER_ID VARCHAR No
The unique ID of the user that triggered an audit event for the Master Patient Index (MPI) identifier.
8 MPI_HX_USER_ID_NAME VARCHAR No
The name of the user record. This name may be hidden.
9 MPI_HX_AUDIT_TYPE_C_NAME VARCHAR No
The audit event category ID for the plan of care that updated the identifier.
May contain organization-specific values: No
Category Entries:
NEW
CHANGE
DELETED
MERGE
UNMERGE
EXPIRED
10 MPI_HX_NEW_ID VARCHAR No
The unique ID of the Master Patient Index (MPI) identifier after an audit event has occurred.
11 MPI_HX_DOTONE VARCHAR No
The internal Chronicles ID of the record for which this audit event was logged.
12 MPI_HX_FROM_DATE DATETIME No
The date the Master Patient Index (MPI) identifier is or was effective from.
13 MPI_HX_TO_DATE DATETIME No
The date the Master Patient Index (MPI) identifier is or was effective until.

EHI Data Export

NextGen Healthcare understands the importance of ensuring patients have timely, secure and easy access to electronic health information (“EHI”) to empower them in managing their own health and well-being.

On this page, you will find database dictionaries that define the schema of the EHI data exports for each of NextGen Healthcare’s certified health IT products: NextGen® Enterprise EHR, NextGen® Office, and NextGen® Direct Messaging, and Mirth Connect.

Key Information About the Exported Data:

  • The extracted data is presented in a machine-readable format.
  • The database dictionaries on this page outline the structure of each exported table and its corresponding data elements.
  • While certain EHI, such as images, documents, reports, and audio files are referenced in the tables, they are located in the corresponding file folders in the extract.
  • If the electronic format outlined in the data dictionary does not suit your intended purpose for extracting your EHI; or if it is not compatible with the health IT product targeted to receive and use this data; or if you require your EHI in a human-readable format, you may need to utilize an additional third-party product and/or request that your healthcare provider provide the data in an alternate format. This could include a Consolidated Clinical Document Architecture (CCDA) document which contains both human and machine-readable information.

Database Dictionaries

NextGen® Enterprise EHR

This database dictionary reflects the latest version of the NextGen® Enterprise EHR and clinical specialty content. This EHI export functionality is only available in v6.2021.1 Cures and higher. Depending on the specialty product and version used by your provider the tables found in your extract may differ slightly from those listed in the dictionary. This documentation describes the EHI schema as of June 27, 2025.
Database dictionary

NextGen® Office

NextGen Office provides EHI exports for patients as a ZIP archive containing various files. ZIP files can be opened and extracted using a variety of free products (WinZip, 7-Zip, etc.).
Files will be either:

  • CCDA Format – An XML file following the CCDA specification here https://www.hl7.org/implement/standards/.
  • CSV – Comma-Separated Variable - A file of text data with each field delimited by a comma. Each record includes one or more fields.
  • Binary files – Files of binary type, such as images and PDFs.
  • HTML files

NextGen® Direct Messaging

NextGen® Direct Messaging version 2.14 and higher provides EHI exports for patients as a ZIP archive containing various files. ZIP files can be opened and extracted using a variety of free products.
Files will be either:

  • CDA Format – An XML file following the CDA specification here https://www.hl7.org/implement/standards/.
  • Binary files – Files of binary type, such as images and PDFs.

Mirth Connect

Mirth Connect provides the ability to export messages with their attachments in XML format. If the export is encrypted, then the encryption method is included. The attachments are always base64 encrypted.

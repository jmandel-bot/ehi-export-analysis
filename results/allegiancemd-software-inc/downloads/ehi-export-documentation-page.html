EHI Export Documentation

 

Introduction

Data is exported in CBOR format (Concise Binary Object Representation) RFC 8949. The Concise Binary Object Representation (CBOR) is an open data format whose design goals include the possibility of extremely small code size, fairly small message size, and extensibility without the need for version negotiation. It offers best result and avoid formatting issues that can exist in JSON.

Please refer to https://cbor.io/impls.html for opensource tools and examples for the creation of quick scripts.

Please click here to see detailed data structure of the objects. Documents and Clinical data will be included in the zip file. Each document has the corresponding .meta file that describes each document.

EHI Export Documentation: CareVue complies with 170.315(b)(10)-EHI Export by creating XML files containing all patient-relevant record information in a single standard format. You can view the EHI Export XSD file format for an explanation of the .xml file format or download the EHI Export XSD file for use in interpreting the files.

A single EHI Export file will be created for each patient with a file naming convention combining up to the first 10 characters of a patient’s last name with the patient’s medical record number to facilitate output file identification. Each file will have a .xml extension. For information detailing the type of data each field in an EHI Export .xml file may contain you may view the data dictionary for more details.

EHI Export Documentation:

ChartLogic enables EHI export for both individual and patient populations in the format of Consolidated-Clinical Document Architecture (C-CDA), additionally, documents and images that have been scanned into the system can be exported as PDFs.

Currently, the C-CDA complies with the United States Core Data for Interoperability (USCDI), Version 1. The specifications for the C-CDA can be located on the HL7 website:

 HL7 Implementation Guide for CDA Release 2 Consolidation CDA Templates for Clinical Notes (US Realm), Draft Standard for Trial Use Release 2.1 C-CDA 2.1 with Errata, August 2015, June 2019 (with Errata)

HL7 CDA R2 IG: C-CDA Templates for Clinical Notes R2.1 Companion Guide, Release 2, October 2019, IBR approved for 170.205(a)(5).


Costs

The certified product version may require ongoing monthly costs to support computerized provider order entry provided through Change Healthcare Clinical Exchange and ePrescribe services provided through DrFirst Rcopia. ePrescribe services check for drug-drug and drug-allergy interactions. The ongoing monthly costs are flat rate per provider.

The certified product version may require one-time costs to establish interfaces for reporting to health information exchanges, immunization registries, and other public health agencies. The one-time cost is charged at the practice level per interface.

EHI Export Documentation:

Exporting single-patient information

Patients can export a complete set of their medical data in consolidated clinical Document Architecture (C-CDA) format.

Log into an account, select the desired patient, and select visits.

Click Visit and select Download My Data, when the choices appear, select CCDA.

Alternatively, a practice staff member can export the information on behalf of a patient. Only staff members who have been granted this access can perform this function.

Select the Connect to Patients option from the menu bar on the left side of the screen

Use the search to find specific patients or select from the list.

Once the patient is selected, click Visit and select Download My Data, when the choices appear, select CCDA.

The user interface allows exporting only one patient at a time, and you must know which patient you wish to export.

To obtain a bulk export in the format of zipped CCD-A files, submit a request to support@micro-officesystems.com.

The specifications for the CCDA can be found on the HL7 website.

Health Level 7 (HL7) Implementation Guide for CDA Release 2: IHE Health Story Consolidation, DSTU Release 1.1 (US Realm) Draft Standard for Trial Use July 2012

EHI (ePHI) Export

EHI – Electronic protected health information (ePHI) Export functionality allows Qlinical to do an export of health data for one or more patients. Qlinical supports the data export in CCD/C-CDA and ndjson formats. The C-CDA file uses stylesheets which is native to EndoSoft and provides a better user experience in viewing the patient data.

End-user can define what kind of data that they want to export. Below are the available sections that a user can choose.

  • Allergies
  • Encounter
  • Immunizations
  • Medications
  • Plan of treatment
  • Referral reason
  • Active problems
  • Reason for visit
  • Implants
  • Health concerns
  • Procedures
  • Functional status
  • Results
  • Social history
  • Vitals
  • Goals
  • Discharge instructions
  • Assessments
  • Cognitive status
  • Media
  • Diagnostic Report
  • Documents
  • Service Request

Qlinical offers two methods for data export.

  • Single/multi patient export
  • Bulk EHI export

Single/multi patient export

Qlinical uses HL7 C-CDA 2.1 guidelines to export the data. The file name format would be.

File Name Description
*.xml Discrete data elements
CDA.xsl Stylesheet

Qlinical supports both “All patient information” as well as “Single patient information” data export.

Bulk EHI Export

Qlinical uses FHIR APIs to export the data. More information can be found here.

Table-level Detail
Description: Result
Definition: Stores the current result status for a discrete task assay that is associated with a given patient's order.
Table Type: ACTIVITY





Column Detail - RESULT


Column Name Type Null? Definition
BB_CONTROL_CELL_CD NUMBER N A unique code value that identifies a specific control cell or the ID of the product of the antibody or antigen test. Blood bank-specific attribute.
BB_GROUP_ID NUMBER N This field stores the unique identifier of the Blood Bank group (rack) used for testing.
BB_RESULT_ID NUMBER N A unique, internal, system-assigned number that is used to group the results for a product or a control cell. Blood bank-specific attribute.
BIOLOGICAL_CATEGORY_CD NUMBER N Denotes the origin of the sequence of the part of the DNA or RNA being tested and the species of the organism being tested (animal, bacteria, or virus).
CALL_BACK_IND NUMBER Y Indicates whether a result needs to be called back to the area requesting the procedure to be performed. (Currently not implemented)
CATALOG_CD NUMBER N A unique code value that identifies a specific order catalog procedure associated with a result.
CHARTABLE_FLAG NUMBER Y Defines if the result should be used for charting/posting to Clinical Events. (Currently not implemented)
COMMENT_IND NUMBER Y Indicates whether a result comment is associated with the result. A value of 0 means there is not a chartable comment attached to the result. A value of 1 means there is a chartable comment attached to the result. (Currently not implemented)
CONCEPT_CKI VC2(255) N Concept CKI is the functional Concept Identifier; it is the codified means within Millennium to identify key medical concepts to support information processing, clinical decision support, executable knowledge and knowledge presentation. Composed of a source and an identifier. For example, if the concept source is "SNOMED" and the concept identifier is "123", then the CKI is "SNOMED!123".
DISPLAY_SEQUENCE_NBR NUMBER N Defines the sequence in which the result will be displayed in result entry applications. This attribute is originally derived from the profile_task_r table when the result row is inserted.
EVENT_ID NUMBER N A unique, internal, system-assigned number that identifies a specific event ID that relates the result to the clinical event tables. (Currently not implemented)
LOT_INFORMATION_ID NUMBER N This field stores the unique identifier of the reagent lot used for testing.
NC_COMMENT_IND NUMBER Y Indicates whether a result footnote is associated with the result. A value of 0 indicates there are no non-chartable footnotes for the result. A value of 1 indicates there are non-chartable footnotes for the result. (Currently not implemented)
ORDER_ID NUMBER N A unique, internal, system-assigned number that identifies a specific order. Creates a relationship to the orders table.
PENDING_IND NUMBER N Indicates whether this result must be performed before the order it is associated with should be considered complete. This attribute is originally derived from the profile_task_r table when the result row is written.
PERSON_ID NUMBER N This is the value of the unique primary identifier of the person table. It is an internal system assigned number.
PRIMARY_IND NUMBER N The result chosen as most accurate among all results for a given assay concept. Determines what displays in the person level viewer for inherited results.
REPEAT_NUMBER NUMBER Y Defines the number of times the procedure has been repeated to produce a result. (Currently not implemented)
RESTRICT_DISPLAY_IND NUMBER N Restricts the display of the result in result entry to only those that have results associated with them. This attribute is originally derived from the profile_task_r table when the result row is written.
RESULT_ID NUMBER N A unique, internal, system-assigned number that identifies a specific result.
RESULT_PROCESSING_CD NUMBER N A unique code value used to determine result processing. For Helix result entry applications, this field determines the column in which the result appears.
RESULT_STATUS_CD NUMBER N A unique code value that identifies the current status of the result, such as performed, verified, corrected, and so on.
SECURITY_LEVEL_CD NUMBER N A unique code value that identifies a specific security level for the result. (Currently not implemented)
TASK_ASSAY_CD NUMBER N A unique code value that identifies a specific discrete task assay associated with the result.
UPDT_APPLCTX NUMBER N The application context number from the record info block.
UPDT_CNT NUMBER N Set to 0 on insert. Incremented by 1 on update. Used to recognize update conflict where data in a row updated by one application is at risk of being lost by a second application attempting to update the row.
UPDT_DT_TM DATE N The date and time the row was last inserted or updated.
UPDT_ID NUMBER N The person_id of the person from the personnel table (prsnl) that caused the last insert or update of the row in the table.
UPDT_TASK NUMBER N The registered (assigned) task number for the process that inserted or updated the row.

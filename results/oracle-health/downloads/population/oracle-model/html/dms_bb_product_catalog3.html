Table-level Detail
Description: Transfusion Requirements
Definition: A reference of valid transfusion requirements that can pertain to patients. This includes antibodies as well as other requirements, such as irradiated blood needed, CMV negative blood needed, etc.
Table Type: REFERENCE





Column Detail - TRANSFUSION_REQUIREMENTS


Column Name Type Null? Definition
ACTIVE_IND NUMBER Y The table row is active or inactive. A row is generally active unless it is in an inactive state such as logically deleted, combined away, pending purge, etc.
ACTIVE_STATUS_CD NUMBER N Indicates the status of the row itself (not the data in the row) such as active, inactive, combined away, pending purge, etc.
ACTIVE_STATUS_DT_TM DATE Y The date and time that the active_status_cd was set.
ACTIVE_STATUS_PRSNL_ID NUMBER N The person who caused the active_status_cd to be set or change.
ANTI_D_IND NUMBER Y Indicates whether a particular transfusion requirement is the one for Anti-D. This determines how the antigen/antibody validation should work at the time of dispense. If this indicator is set, then the validation will check the Rh type of the product being dispensed, for being negative.
CHART_NAME VC2(40) Y The chart name of the transfusion requirement. (Currently not used).
CODESET NUMBER Y The number of the codeset from which this transfusion requirement originated, since a transfusion requirement could be an antibody. The two valid code sets for this column are 1611 and 1613. Codeset 1611 is for transfusion requirements that are not antibodies, and codeset 1613 is for antibodies.
DESCRIPTION VC2(50) Y A textual description of the transfusion requirement, as entered by the user in the database tool "AntibodyTool".
REQUIREMENT_CD NUMBER N The actual transfusion requirement, whether it be an antibody or other type of transfusion requirement (ex. "CMV Negative", "Irradiation", etc.). This value comes from codeset 1613 if is it an antibody, as indicated by the CODESET column on this table.
SIGNIFICANCE_IND NUMBER Y Indicates whether this antibody is clinically significant or not, used during an electronic crossmatch. A value of 1 indicates clinical significance. A value of 0 indicates no clinical significance.
UPDT_APPLCTX NUMBER N The application context number from the record info block.
UPDT_CNT NUMBER N Set to 0 on insert. Incremented by 1 on update. Used to recognize update conflict where data in a row updated by one application is at risk of being lost by a second application attempting to update the row.
UPDT_DT_TM DATE N The date and time the row was last inserted or updated.
UPDT_ID NUMBER N The person_id of the person from the personnel table (prsnl) that caused the last insert or update of the row in the table.
UPDT_TASK NUMBER N The registered (assigned) task number for the process that inserted or updated the row.

Table-level Detail
Description: Transfusion Requirement Relationship
Definition: A reference of the association between items in the Transfusion Requirements and the special testing items which satisfy the requirements.
Table Type: REFERENCE





Column Detail - TRANS_REQ_R


Column Name Type Null? Definition
ACTIVE_IND NUMBER Y The table row is active or inactive. A row is generally active unless it is in an inactive state such as logically deleted, combined away, pending purge, etc.
ACTIVE_STATUS_CD NUMBER N Indicates the status of the row itself (not the data in the row) such as active, inactive, combined away, pending purge, etc.
ACTIVE_STATUS_DT_TM DATE Y The date and time that the active_status_cd was set.
ACTIVE_STATUS_PRSNL_ID NUMBER N The person who caused the active_status_cd to be set or change.
ALLOW_OVERRIDE_IND NUMBER Y Indicates whether the Dispense application should allow the user to override this transfusion requirement when the product does not have the attribute necessary for this transfusion requirement. An example is the transfusion requirement of the antibody "Kell" on the patient, when the product to be dispensed has not been typed "Kell Negative".
RELATIONSHIP_ID NUMBER N The primary key of this table. An internal system-assigned number that ensures the uniqueness of each row.
REQUIREMENT_CD NUMBER N The specific transfusion requirement for which this relationship is defined. Refers to items built on the TRANSFUSION_REQUIREMENT table.
SPECIAL_TESTING_CD NUMBER N The attribute associated with the transfusion requirement that must be present on the product to be dispensed to a patient with the transfusion requirement defined on this row . These attributes may be red cell antigens or other types of special testing. Examples are "Kell Negative" and "CMV Negative".
UPDT_APPLCTX NUMBER N The application context number from the record info block.
UPDT_CNT NUMBER N Set to 0 on insert. Incremented by 1 on update. Used to recognize update conflict where data in a row updated by one application is at risk of being lost by a second application attempting to update the row.
UPDT_DT_TM DATE N The date and time the row was last inserted or updated.
UPDT_ID NUMBER N The person_id of the person from the personnel table (prsnl) that caused the last insert or update of the row in the table.
UPDT_TASK NUMBER N The registered (assigned) task number for the process that inserted or updated the row.
WARN_IND NUMBER Y Indicates whether the user should be warned of this transfusion requirement when it is not met by the product being dispensed. If this indicator is set, then the Dispense application may or may not allow this warning to be overridden depending on the value of the ALLOW_OVERRIDE_IND.

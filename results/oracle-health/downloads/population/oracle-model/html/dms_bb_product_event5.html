Column Name Type Null? Definition
ACTIVE_IND NUMBER Y The table row is active or inactive. A row is generally active unless it is in an inactive state such as logically deleted, combined away, pending purge, etc.
ACTIVE_STATUS_CD NUMBER N Indicates the status of the row itself (not the data in the row) such as active, inactive, combined away, pending purge, etc.
ACTIVE_STATUS_DT_TM DATE Y The date and time that the active_status_cd was set.
ACTIVE_STATUS_PRSNL_ID NUMBER N The person who caused the active_status_cd to be set or change.
ALPHA_TRANSLATION_ID NUMBER N Identifies the alpha translation used to create the product number for this product. Only applies to blood products. Only applies to products received from suppliers for which alpha translation is required.
BB_SUPPLIER_ID NUMBER N Identifies the originating supplier of this product. Only applies to blood products. This is the originating supplier, not the intermediate supplier, which is stored on the product table.
ELECTRONIC_RECEIPT_IND NUMBER N Received through FSI Inventory Transfer Interface
ORIG_INTL_UNITS NUMBER Y This column applies only to derivative products. It is the number of international units originally entered at the time the derivative batch was received and entered into inventory.
ORIG_RCVD_QTY NUMBER Y This column applies only to derivative products. It is the quantity of the batch originally entered at the time the derivative batch was received and entered into inventory.
PRODUCT_EVENT_ID NUMBER N The primary key of this table. It is an internal system-assigned number that ensure the uniqueness of each row. There is a corresponding row on the PRODUCT_EVENT table with this same value as the primary key, for the received event.
PRODUCT_ID NUMBER N The primary key to the PRODUCT table. An internal system-assigned number. On this table, it identifies the product that was received into the blood bank's inventory.
SHIP_COND_CD NUMBER N The original condition in which the product was shipped to the transfusion service (ex. "Dry Ice").
TEMPERATURE_DEGREE_CD NUMBER N Indicates the shipment temperature degree unit of measure at the time the product was received.
TEMPERATURE_VALUE NUMBER Y Indicates the shipment temperature at the time the product was received.
UPDT_APPLCTX NUMBER N The application context number from the record info block.
UPDT_CNT NUMBER N Set to 0 on insert. Incremented by 1 on update. Used to recognize update conflict where data in a row updated by one application is at risk of being lost by a second application attempting to update the row.
UPDT_DT_TM DATE N The date and time the row was last inserted or updated.
UPDT_ID NUMBER N The person_id of the person from the personnel table (prsnl) that caused the last insert or update of the row in the table.
UPDT_TASK NUMBER N The registered (assigned) task number for the process that inserted or updated the row.
VIS_INSP_CD NUMBER N The original visual inspection performed on the product at the time it was received into the blood bank's inventory (ex. "OK", "Bag leaking", etc.). An unsatisfactory visual inspection can cause the product to be quarantined automatically.

Column Name Type Null? Definition
ACTIVE_IND NUMBER Y The table row is active or inactive. A row is generally active unless it is in an inactive state such as logically deleted, combined away, pending purge, etc.
ACTIVE_STATUS_CD NUMBER N Indicates the status of the row itself (not the data in the row) such as active, inactive, combined away, pending purge, etc.

ORIG_TRANSFUSED_QTY NUMBER Y This column applies only to derivative products. It is the quantity of the derivative batch that was originally recorded as being in a transfused state for the patient.
PERSON_ID NUMBER N This is the value of the unique primary identifier of the person table. It is an internal system assigned number.
PRODUCT_EVENT_ID NUMBER N The primary key to this table, as well as the PRODUCT_EVENT table. An internal system-assigned number that ensure the uniqueness of each row. For every row on this table, there is a corresponding row on the PRODUCT_EVENT table with the same Product_Event_ID.

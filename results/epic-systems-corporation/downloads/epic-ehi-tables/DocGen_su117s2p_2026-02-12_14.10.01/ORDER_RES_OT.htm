ORDER_RES_OT
Description:
This table stores tracked recommendation information and Blood result dispense status for orders-related results. This table can also store Dermatology Skin Findings.

Primary Key
Column Name Ordinal Position
FINDING_ID 1
CONTACT_DATE_REAL 2

Column Information
Name Type Discontinued?
1 FINDING_ID NUMERIC No
The unique ID of the finding record corresponding to this result.
2 CONTACT_DATE_REAL FLOAT No
This is a numeric representation of the date of this encounter in your system. The integer portion of the number specifies the date of the encounter. The digits after the decimal point indicate multiple visits on one day.
3 CONTACT_DATE DATETIME No
The date of this contact in calendar format.
4 REC_STATUS_C_NAME VARCHAR No
The recommendation status category ID for a result finding.
May contain organization-specific values: Yes
Category Entries:
Needs Follow-up
Closed
Resolved
5 ADMINPX_ORD_DAT VARCHAR No
A unique contact date in decimal format from which the corresponding order's tracked result originated. The integer portion of the number indicates the date of contact. The digits after the decimal distinguish different contacts on the same date and are unique for each contact on that date. For example, .00 is the first/only contact, .01 is the second contact, etc.
6 ADMINPX_BLOODSTAT_C_NAME VARCHAR No
The blood product status history category ID for the order, as sent via a results interface from a blood bank.
May contain organization-specific values: Yes
Category Entries:
Requested
Selected
Issued
Transfusing
Transfused
Returned
Expired/Released
7 LESION_REMOVE_CMT VARCHAR No
The comments entered by the user when a lesion was removed from a patient's record.
8 LAST_DOC_DTTM DATETIME (UTC) No
The date and time when this finding was last documented on.
9 LESION_REMOVE_RSN_C_NAME VARCHAR No
The removal reason category ID for the lesion.
May contain organization-specific values: Yes
Category Entries:
Other

The name of the diagnosis.

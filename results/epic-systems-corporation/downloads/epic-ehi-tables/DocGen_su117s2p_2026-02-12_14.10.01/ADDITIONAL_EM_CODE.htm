ADDITIONAL_EM_CODE
Description:
This table holds all information related to additional evaluation and management (E/M) codes.

Primary Key
Column Name Ordinal Position
PAT_ENC_CSN_ID 1
LINE 2

Column Information
Name Type Discontinued?
1 PAT_ENC_CSN_ID NUMERIC No
A unique serial number for this encounter. This number is unique across all patients and encounters in the system.
2 LINE INTEGER No
The line number for the information associated with this contact. Multiple pieces of information can be associated with this contact.
3 EM_CODE_ADDL_ID_PROC_NAME VARCHAR No
The name of each procedure.
4 EM_CODE_MOD_ID VARCHAR No
Modifiers associated with the additional E/M code. Stored in comma-delimited format with up to four modifiers.
5 EM_CODE_BILPROV_ID_PROV_NAME VARCHAR No
The name of the service provider. This item may be hidden in a public view of the CLARITY_SER table.
6 EM_CODE_UNIQUE_NUM VARCHAR No
Unique number associated with each additional E/M code. The number is only unique within the contact.
7 EM_NO_CHG_REASON_C_NAME VARCHAR No
The reason that a charge was not triggered for an E/M code. This item being populated does not imply any issues with system integrity or system build; it will be set both for legitimate reasons that a charge was not triggered as well as non-legitimate reasons.
May contain organization-specific values: No
Category Entries:
Unspecified reason
Charge Ready to Trigger flag not set
Order Instructions set to not trigger charges
Procedure not specified
Order class set to not trigger charges
Charge triggering disabled at resulting lab
Charge filtered through discrimination programming point
No charge triggering method specified
Order is canceled
Holding until close encounter
Patient not specified
Visit not specified
Invalid LCM build
Charge trigger failed to execute
Non-chargeable external result
Orders in transmittal
8 AR_EM_CODE_DX VARCHAR No
The associated diagnosis information for additional E/M code data generated by an AR interface.

Table ccmr_initassesments_addldata
Description This table stores selected assessment details for the patient.
Column DataType Description
Id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3
ccmrId int Identifier for the CCMR record associated with the initial assessment. Foreign key relation to the CCMR table (ccmr_careplan.ccmrId). Example: 101, 102, 103
initAssementTypeId int This stores assessment type id. This column refers to items.itemID.
initAssementType varchar Description of the initial assessment type. Example: 'Physical Examination', 'Mental Health Assessment', 'Nutritional Assessment'
UpdDate datetime Date and time when the record was last updated. Example: '2021-09-01 12:34:56'
deleteFlag tinyint Flag to indicate if the record is deleted (1) or not (0). Example: 0, 1
userId int Identifier for the user who created or updated the record. Foreign key relation to the user table (users.uid). Example: 1001, 1002, 1003
userName varchar Name of the user who created or updated the record. Example: 'John Doe', 'Jane Smith', 'Alice Johnson'
locked int Flag to indicate if the record is locked (1) or not (0). Example: 0, 1
lockedDate datetime Date and time when the record was locked. Example: '2021-09-01 12:34:56'
encounterid int This column store encounterid for which the data is being recorded on the application. This column refers to encounterid column in enc table.
lockedBy int This column stores id of a provider/staff who locked the record on the application. Column refers to uid column in user table.
isStarted tinyint Flag to indicate if the initial assessment has been Not started(1) or in progress(0)
addedDtTime datetime Date and time when the record was added to the table. Example: '2021-09-01 12:34:56'
lockedData varchar Not Available

Table edi_837healthcarepricing
Description This table store about Health care pricing Info
Column DataType Description
Id (Primary Key) int Unique identifier for each record in the table. (e.g., 1, 2, 3)
pricingMethodology varchar A 5-character code representing the pricing methodology used. (e.g., 'ABCDE')
RepricedAllowedAmt varchar The repriced allowed amount for the service, represented as a string. (e.g., '100.00', '250.50')
RepricedSavingAmt varchar The repriced saving amount for the service, represented as a string. (e.g., '20.00', '50.50')
RepricingOrgIdentifier varchar A 60-character identifier for the repricing organization. (e.g., 'RepricingOrg123')
RepricingPerDiemOrFlatRateAmt varchar The repricing per diem or flat rate amount, represented as a string. (e.g., '150.00', '300.00')
RepricedApprovedAPGCode varchar A 60-character code representing the repriced approved APG (Ambulatory Payment Classification). (e.g., 'APG12345')
RepricedApprovedAPGAmount varchar The repriced approved APG amount, represented as a string. (e.g., '120.00', '240.00')
RejectReasonCode varchar A 5-character code representing the reason for rejection, if any. (e.g., 'R1234')
PolicyComplianceCode varchar A 5-character code representing the policy compliance status. (e.g., 'P1234')
Exceptioncode varchar A 5-character code representing any exceptions that may apply. (e.g., 'E1234')
InvId int An integer representing the invoice ID, which can be used as a foreign key to reference another table with invoice details. (e.g., 101, 102)
InvCptId int An integer representing the invoice CPT (Current Procedural Terminology) ID, which can be used as a foreign key to reference another table with CPT details. (e.g., 201, 202)
ClaimorCptFlag tinyint A tiny integer (0 or 1) indicating whether the record is for a claim (0) or CPT (1). (e.g., 0, 1)
deleteFlag tinyint A tiny integer (0 or 1) indicating whether the record has been deleted (1) or not (0). (e.g., 0, 1)

EHI Export Schema
Table fhir_user_audit_log
Description This table stores the audit logs of user actions related to FHIR resources, including the type of action, the resource type, and other relevant information.
Column DataType Description
id (Primary Key) int Unique identifier for each audit log entry. Example: 1, 2, 3.
resourceType varchar The type of FHIR resource involved in the action. Example: "Patient", "Observation", "Encounter".
fhirQuery varchar The FHIR query string used for the action, if applicable. Example: "Patient?_id=123", "Observation?subject=456".
patientId int The identifier of the patient associated with the action, if applicable. Example: 101, 102, 103.
providerId int The identifier of the provider associated with the action, if applicable. Example: 201, 202, 203.
vendorId int The identifier of the vendor associated with the action, if applicable. Example: 301, 302, 303.
appTypeId int The identifier of the application type associated with the action. Foreign key relation: `fhir_app_type.id`. Example: 401, 402, 403.
actionTypeId int The identifier of the action type performed. Foreign key relation: `fhir_audit_action_type.id`. Example: 501, 502, 503.
dateReceived datetime The date and time when the action was received. Example: '2021-08-01 12:34:56', '2021-08-02 14:56:23'.
audit_status tinyint A tiny integer representing the status of the audit log entry, if applicable. Example: 0, 1, 2.

EHI Export Schema
Table emcoder_mdm_chartb
Description This table stores information related to medical decision-making charts, including various review and assessment details, as well as information about the user who added or updated the record.
Column DataType Description
id (Primary Key) int This is the primary key for the table, which is an integer value that uniquely identifies each record. It is an auto-incrementing value. Example: 1, 2, 3, etc.
encid int Identifier for the associated encounter. This is a foreign key that refers to the `enc` table's `encounterID` column. Example: 101, 102, 103.
rev_ext_notes int It stores value for "review of prior external notes" from each unique source
rev_test_res int It stores value for "Review of the results" from each unique source
order_test int It stores value for orderes from each unique source
ind_asmt_hx int It stores boolean value for obtaining old records or history from another one.
ind_intr_td int It stores boolean value for Independent interpretation of tests performed or not
mgmt_disc int It stores boolean value for tests have been discussed with external physician or not
chartlevel int This column sotres data for the mdm chart level as integer value.
addedby int Identifier for the user who added the record. Example: 1001, 1002, 1003.
addeddate datetime Date and time when the record was added. Example: '2022-01-01 12:34:56'.
updatedby int Identifier for the user who updated the record. Example: 1001, 1002, 1003.
updateddate datetime Date and time when the record was last updated. Example: '2022-01-02 12:34:56'.
deleteflag tinyint This column stores value 0 / 1 indicating this appointment is deleted. This is soft delete. 0 - Not Deleted , 1 -Deleted

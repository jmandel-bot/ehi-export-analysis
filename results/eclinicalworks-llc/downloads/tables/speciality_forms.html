EHI Export Schema
Table speciality_forms
Description This table stores information about speciality forms, including their type, data, and related patient and encounter information.
Column DataType Description
id (Primary Key) int This is an auto incremented primary key of This column .
pid int This column stores id of the patient for which speciality form is added. This column refers to column UID from user table.
encid int This column stores id of the encounter of the patient. Column refers to encounterid column in enc table.
modified_date datetime Date and time when the speciality form was last modified. Example: '2021-09-01 12:34:56'.
comment varchar This column is not in use.
form_type_id int This column stores id of the speciality form type. This column refers to column id from speciality_form_type table.
form_data text This column stores data containing the contents of the speciality form in xml format
form_url text This columns stores the URL of the Specialty / Smart forms when gets saved from the form.
locked char This column stores value Y/N indicating if the speciality form is locked. Y - Locked , N - Unlocked.
created_date datetime Date and time when the speciality form was created. Example: '2021-08-01 10:20:30'.
SF_data text This column stores the structured data stored in xml format used for smart forms.
pagenum int This column is not in use.
delFlag tinyint This column stores value 0 / 1 indicating if the document (speciality form) is deleted or not, this is soft delete. 0 - Not Deleted , 1 -Deleted.
docid int This column stores id of the document (speciality form) used for the patient. Column refers to docid column in document table.
sign_data nvarchar This column stores the data of the sign done on specialty forms through signature pad.

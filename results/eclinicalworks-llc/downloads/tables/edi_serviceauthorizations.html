Table edi_serviceauthorizations
Description This table stores information about EDI (Electronic Data Interchange) service authorizations, including details about the sender, receiver, and various other attributes related to the authorization process.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
date datetime Date when the service authorization was created. Example: '2021-09-30', '2021-10-01'.
SubFlag int An integer flag indicating the submission status. Example: 0 (not submitted), 1 (submitted).
deleteFlag int An integer flag indicating if the record is deleted. Example: 0 (not deleted), 1 (deleted).
vmid varchar A varchar field storing a unique identifier.
Description varchar A brief description of the service authorization. Example: 'Authorization for medical service', 'Authorization for dental service'.
Receiverid int An integer representing the unique identifier of the receiver. Example: 101, 102.
NumEncs int An integer representing the number of encounters associated with the service authorization. Example: 2, 5.
GS06 int An unsigned integer representing the group control number. Example: 1001, 1002.
ISA15 char A char field representing the interchange usage indicator. Example: 'U' (for production data), 'T' (for test data).
ISA13 int An unsigned integer representing the interchange control number. Example: 2001, 2002.
ISA08 varchar A varchar field representing the receiver's interchange id. Example: 'REC123', 'REC456'.
ISA06 varchar A varchar field representing the sender's interchange id. Example: 'SEN123', 'SEN456'.
updateFlag int An integer flag indicating if the record has been updated. Example: 0 (not updated), 1 (updated).
timestamp datetime A time stamp indicating the last time the record was updated. Example: '2021-09-30 12:34:56', '2021-10-01 14:56:23'.
Received tinyint A tinyint flag indicating if the service authorization has been received. Example: 0 (not received), 1 (received).
Response text A longtext field storing the response message related to the service authorization.
Submitted tinyint A tinyint flag indicating if the service authorization has been submitted. Example: 0 (not submitted), 1 (submitted).
Insid int An integer representing the unique identifier of the insurance. Example: 301, 302.
ErrorMessage text A longtext field storing any error message related to the service authorization.
InsSeqNo tinyint A tinyint field representing the insurance sequence number. Example: 1, 2.
FileName varchar A varchar field storing the name of the file associated with the service authorization. Example: 'file1.txt', 'file2.txt'.
submitterid int An integer representing the unique identifier of the submitter. Example: 401, 402. This could be a foreign key referring to another table's column, but the table and column are not specified in the provided schema.
isReferral tinyint A tinyint flag indicating if the service authorization is a referral. Example: 0 (not a referral), 1 (a referral).

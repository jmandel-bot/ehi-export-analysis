EHI Export Schema
Table bh_historystructuredata
Description This table stores the history of structured data related to patient encounters, including details about categories, items, and values.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
encounterid int identifier for the patient encounter. It has a foreign key relation with the `enc` table on the `encounterid` column. Example: 1001, 1002, 1003.
catid int identifier for the category of the structured data. It has a foreign key relation with the `items` table on the `itemid` column. Example: 10, 11, 12.
itemid int identifier for the item within the category. It has a foreign key relation with the `items` table on the `itemid` column. Example: 101, 102, 103. Default value is NULL.
detailid int identifier for the detail of the structured data. It has a foreign key relation with the `structdatadetail` table on the `id` column. Example: 10001, 10002, 10003.
VALUE varchar Text value of the structured data. Example: "High blood pressure", "Diabetes", "Asthma".
valueid varchar identifier for the value of the structured data. Example: "val_1001", "val_1002", "val_1003". Default value is an empty string.
notes varchar Additional notes related to the structured data. Example: "Patient has a family history of diabetes", "Patient is a smoker".
isBehavioralRisk tinyint A boolean flag indicating if the structured data represents a behavioral risk. Example: 0 (false), 1 (true). Default value is 0.
chartdisplay tinyint A boolean flag indicating if the structured data should be displayed on the chart. Example: 0 (false), 1 (true). Default value is 1.
displayIndex int An integer representing the display order of the structured data. Example: 0, 1, 2. Default value is 0.
deleteflag tinyint A boolean flag indicating if the record has been deleted. Example: 0 (false), 1 (true). Default value is 0.
lastUpdate datetime time stamp of the last update made to the record. Example: '2021-09-01 12:34:56'.

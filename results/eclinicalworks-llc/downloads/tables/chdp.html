EHI Export Schema
Table chdp
Description This table stores information about child health and disability prevention (CHDP) exams, including patient details, exam results, and fees.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table.
encounterid int This column stores the encounterid from enc table. To map the chdp to that encounter.
patId int It stores patient's id
name varchar Patient's full name.
medicalRecordNo varchar Patient's medical record number.
LAcode char 3-character code representing the local agency.
DOB_month char Month of patient's date of birth (e.g., "Jan", "Feb").
DOB_Date char Day of patient's date of birth (e.g., "01", "15").
DOB_Year varchar Year of patient's date of birth (e.g., "2000", "2010").
Age varchar Patient's age at the time of the exam (e.g., "5", "10").
sex char Patient's sex (e.g., "M" for male, "F" for female).
county varchar Patient's county of residence.
countyCode char 3-character code representing the patient's county.
phone varchar Patient's phone number.
next_CHDP_exam_month char Month of the next scheduled CHDP exam (e.g., "Jan", "Feb").
next_CHDP_exam_Date char Day of the next scheduled CHDP exam (e.g., "01", "15").
next_CHDP_exam_year varchar Year of the next scheduled CHDP exam (e.g., "2000", "2010").
respName varchar Name of the responsible party for the patient.
respStreet varchar Street address of the responsible party.
respApt varchar Apartment number of the responsible party.
respCity varchar City of the responsible party.
respZip varchar Zip code of the responsible party.
Ethnic char 2-character code representing the patient's ethnicity.
DOS_month char Month of the date of service (e.g., "Jan", "Feb").
DOS_date char Day of the date of service (e.g., "01", "15").
DOS_year varchar Year of the date of service (e.g., "2000", "2010").
History_A char Various exam results, each with a 3-character code (e.g., "Yes", "No").
Urine_D char Various exam results, each with a 3-character code (e.g., "Yes", "No").
History_Fees varchar Fees associated with each exam section (e.g., "10.00", "25.00").
TB_Fees varchar Fees associated with each exam section (e.g., "10.00", "25.00").
ReferTo1 varchar Name of the first referral provider.
ReferTo1_Tel varchar Phone number of the first referral provider.
ReferTo2 varchar Name of the second referral provider.
ReferTo2_Tel varchar Phone number of the second referral provider.
Comments text Additional comments about the exam.
LabCode1 char Various lab and other exam codes and descriptions, each with a 3-character code (e.g., "Yes", "No").
OtherCode4_Desc varchar Various lab and other exam codes and descriptions, each with a 3-character code (e.g., "Yes", "No").
Height_inches varchar Various patient measurements (e.g., "60" for height in inches, "120" for weight in lbs).
Blood_Lead varchar Various patient measurements (e.g., "60" for height in inches, "120" for weight in lbs).
ImmuneCode1 varchar Immunization codes, descriptions, results, and fees (e.g., "MMR" for immune code, "10.00" for fees).
Immunization7_Fees varchar Immunization codes, descriptions, results, and fees (e.g., "MMR" for immune code, "10.00" for fees).
Dental char 2-character code indicating if the patient received dental services (e.g., "Y" for yes, "N" for no).
FosterChild char 3-character code indicating if the patient is a foster child (e.g., "Yes", "No").
Icd1 varchar International Classification of Diseases (ICD) codes for the patient's diagnoses.
Icd2 varchar International Classification of Diseases (ICD) codes for the patient's diagnoses.
Quest1_Yes char It stores boolean value if user has selected yes for question 1 in the form
Quest3_No char It stores boolean value if user has selected no for question 3 in the form
PatVisit_Type1 char 2-character codes for patient visit and screening types.
Screen_Type2 char 2-character codes for patient visit and screening types.
TotalFees varchar Total fees for the exam (e.g., "100.00").
PatCounty char 3-character code representing the patient's county.
PatAID varchar Patient's aid identification number.
PatIdNum varchar Patient's identification number.
deleteFlag tinyint This column stores value 0 / 1 indicating this appointment is deleted. This is soft delete. 0 - Not Deleted , 1 -Deleted
Enrol_WIC char 2-character codes for enrollment in WIC, referral to WIC, partial screening, and recheck screening (e.g., "Y" for yes, "N" for no).
Recheck_Screen char 2-character codes for enrollment in WIC, referral to WIC, partial screening, and recheck screening (e.g., "Y" for yes, "N" for no).
PM160Date varchar Date of the PM160 form.
HealthPlanCode varchar Health plan code for the patient.
POS varchar Place of service code.
ServiceLocation1 varchar Service locations for the exam.
ServiceLocation4 varchar Service locations for the exam.
ProviderType int Provider type code.
rendPrvdr varchar Rendering provider's name.
Pt_eligible1 char 2-character codes for patient eligibility (e.g., "Y" for yes, "N" for no).
Pt_eligible2 char 2-character codes for patient eligibility (e.g., "Y" for yes, "N" for no).

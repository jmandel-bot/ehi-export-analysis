EHI Export Schema
Table ip_rh_items
Description This table stores categories and items configured for clinical content in the application.
Column DataType Description
itemID (Primary Key) int The primary key and unique identifier for each item. Identity auto increments itemID value. The data type is int because the items can be dynamically created and the items created could be huge in number. The column cannot be NULL as it will always have an itemid.
itemName varchar It stores the item name. The datatype is varchar(255) with a reasonable size constraint i.e., 255 characters as per business requirement. The column has a default value '' and cannot be NULL.
itemDesc varchar It stores item description entered by a user. The datatype is varchar(255) with a reasonable size constraint i.e., 255 characters as per business requirement. The column has a default value '' but cannot be NULL.
parentID int This column stores id of a parent item under which new item has been created. The parent item is stored in ip_rh_items table only. However, there cannot be a foreign key relationship because it refers to the legacy table, items table where the parentId can be stored as 0 and accordingly it is used in the code also.
xmlPath varchar This column stores path of the item from parent item. It displays the hierarchy of the item under which it is created.
itemType char This column stores the type of an item. It can have the following bold values.C - It is a category I - It is a category Item N - It is a Narrative item S - It is a Structured item If value is '', it means it is a free form item.
createdDate datetime Stores the date and time the item was created. As certain items will be shipped through the queries without the createdDate column, for these items the createdDate column will be the default value i.e 1901-01-01 00:00:00. This default value is used in the code. Hence, this column cannot be changed to NULL. For items created through application, this column will be populated with created Date and Time.
modifiedDate datetime Stores the date and time the item was last modified. As certain items will be shipped through the queries, for these items the modifiedDate column will be NULL. For items created through application, this column will be populated with the last modified date and time.
displayIndex int The order index for displaying the item in a list. The data type is smallint as compared to int is because the expected items will not exceed the limitation provided by the smallInt datatype. The column is NOT NULL and has a default value of 0.
deleteflag tinyint This column stores flags 1/0 to indicate whether item has been marked as deleted or not. The datatype is tinyint. The column is NOT NULL and has a default value of 0.Value 0 indicates the item is not deleted and Value 1 indicates the item has been deleted.
keyName varchar It stores the key name associated with the item. The datatype is varchar(80) as the expected value to be stored in the keyName column will not exceed 80 characters.. The column cannot be NULL and has a default value ''.
modifiedBy int Stores the user ID of the person who last modified the item. For items shipped through the queries, the modifiedBy column will be NULL. For items created through the application , this column will be populated with user id of the user who last modified the item. It refers to uid column in users table.

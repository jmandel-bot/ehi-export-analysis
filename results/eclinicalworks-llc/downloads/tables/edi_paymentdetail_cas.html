EHI Export Schema
Table edi_paymentdetail_cas
Description This table stores the payment details and adjustment reasons for Electronic Data Interchange (EDI) transactions for ERA payment.
Column DataType Description
PmtDtlCASid (Primary Key) int Unique identifier for each payment detail record. Example: 1, 2, 3.
PaymentDetailid int identifier for the payment detail, with a default value of 0. This can be a foreign key referring to another table's payment detail id column.
GroupCode varchar A code representing the group associated with the payment detail, with a default value of an empty string. Example: 'GRP01', 'GRP02'.
ReasonCode varchar A code representing the reason for the payment detail, with a default value of an empty string. Example: 'R01', 'R02'.
amount decimal The amount of the payment detail, with a default value of 0.00. Example: 100.50, 250.00.
deleteFlag tinyint A flag indicating whether the record has been deleted (1) or not (0), with a default value of 0.
CreatedBy int identifier for the user who created the record, with a default value of 0.
CreatedDate datetime The date the record was created. Example: '2021-09-01', '2021-09-02'.
CreatedTime datetime The time the record was created. Example: '12:30:00', '14:45:00'.
ModifiedBy int identifier for the user who last modified the record, with a default value of 0.
ModifiedDate datetime The date the record was last modified. Example: '2021-09-10', '2021-09-11'.
ModifiedTime datetime The time the record was last modified. Example: '15:30:00', '16:45:00'.
timestamp datetime The time stamp of when the record was last updated. Example: '2021-09-10 15:30:00', '2021-09-11 16:45:00'.
IsERAPostedCASCode tinyint A flag indicating whether the CAS code has been posted to ERA (1) or not (0), with a default value of 0.
PostedAs tinyint A code representing the posting status, with a default value of 0. Example: 1, 2, 3.

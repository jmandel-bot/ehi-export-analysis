Table ip_rh_ptorders_detail
Description This table appears to store detailed information about patient orders, specifically focusing on the discontinuation of orders and secondary cosigning details.
Column DataType Description
orderDetailId (Primary Key) int Acts as the primary key for the table, uniquely identifying each record. This column is essential for indexing and referencing specific order details.
orderId int Foreign key that links to the main order table to associate the detail record with the corresponding order. Likely refers to an ip_rh_ptorders table.
discontinueReasonId int Stores a reference to the reason why an order was discontinued. Helps in reporting and analyzing why orders are discontinued.
discontinueDateTime datetime Records the date and time when the order was discontinued. This field is optional as it is set to NULL which means not every order detail must have a discontinued date and time.
secondaryCosignedBy int Indicates the ID of the secondary individual who cosigned the order. This column references an users table uId in the context of the application. This is an optional field and be used for audit trials or additional verification processes.
secondarycosignDateTime datetime Records the date and time when the order was cosigned by the secondary individual. Similar to discontinueDateTime, this field is optional and provides a timestamp for when the secondary cosigning occurred.

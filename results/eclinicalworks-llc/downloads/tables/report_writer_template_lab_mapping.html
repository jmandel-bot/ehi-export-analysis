Table report_writer_template_lab_mapping
Description This table stores the mapping between report templates, sentences, and lab reports, along with any associated notes and preview sentence notes.
Column DataType Description
id (Primary Key) int This is the primary key for the table, an auto-incrementing integer that uniquely identifies each mapping record. Example: 1, 2, 3, etc.
templateid int This column is used to store the id of report_writer_template table.  Documented report writer should have templateid. Hence, not NULL constraint is suitable here. This column is a foreign key that references to report_writer_template (templateid). Templateid column of report_writer_template is INT type. Hence, INT datatype is suitable here.
sentenceid int This column is used to store the id of report_writer_sentence table.  Hence, not NULL constraint is suitable here. This column is a foreign key that references to report_writer_sentence (sentenceid). Sentenceid column of report_writer_sentence is INT type. Hence, INT datatype is suitable here.
reportid int This column is used to store reportid of labdata table. Report writer result can associate with other entities apart from reportid. Hence, added this column as NULL. This column is a foreign key that references to labdata (reportid). Reportid column of labdata is INT type. Hence, INT datatype is suitable here.
notes varchar This column is used to store the sentence notes details. The default value of this column is not NULL because user can not save empty/NULL report from UI. As per requirement, user can add up to 40000 characters from UI. Hence, Text/Varchar(Max) is suitable here.
previewSentenceNote varchar This column is used to store the sentence notes details in HTML format. The default value of this column is not NULL because user can not save empty/NULL report from UI. As per requirement, user can add up to 60000 characters from UI. Hence, Text/Varchar(Max) is suitable here.

Table prisma_socialhistory_component
Description This table stores the components of a patient's social history, including various codes, displays, and systems related to the social history.
Column DataType Description
id (Primary Key) bigint A unique identifier for each record in the table. Example: 1, 2, 3.
socialHistoryId bigint The identifier of the related social history record. This is a foreign key that refers to the `id` column in the `prisma_socialhistory_details` table.
code varchar A code representing social history detail.
display varchar A human-readable description of the code. Example: "Smoking status", "Alcohol consumption", "Exercise frequency".
codeSystem varchar The coding system used for the `code` column. Example: "LOINC", "SNOMED-CT".
valueQty varchar The quantity value associated with the component. Example: "20", "3", "5".
valueQtyUnit varchar The unit of measurement for the `valueQty` column. Example: "cigarettes/day", "drinks/week", "hours/week".
valueQtyDisplay varchar A human-readable description of the quantity value and unit. Example: "20 cigarettes per day", "3 drinks per week", "5 hours of exercise per week".
valueQtySystem varchar The system used for the `valueQty` and `valueQtyUnit` columns. Example: "UCUM", "SI".
valueCode varchar A code representing the value of the component. Example: "1", "2", "3".
valueDisplay varchar A human-readable description of the `valueCode`. Example: "Current smoker", "Occasional drinker", "Regular exerciser".
valueSystem varchar The coding system used for the `valueCode` column. Example: "LOINC", "SNOMED-CT".

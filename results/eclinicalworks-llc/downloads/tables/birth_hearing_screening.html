Table birth_hearing_screening
Description This table stores information about the Hearing screening test results for a patient during birth.
Column DataType Description
id (Primary Key) int Unique identifier for each hearing screening record. Example: 1, 2, 3, etc.
birthvitalsid int identifier for the corresponding birth vitals record. This is a foreign key that refers to the `id` column in the `birth_vitals` table. Example: 101, 102, 103, etc.
NAME varchar This column stores the name of the screening eg Screening1, Screening2. Hence, varchar(12) is selected. This field will always have a value hence defined as NOT NULL
rightearresult tinyint This column stores the screening result for right ear. It can have value as 0 (blank), 1 (Pass), 2(Fail), 3(Declined). Hence, the datatype tinyint is selected.This field will always have a value hence defined as NOT NULL.
righteartestmethod tinyint This column stores the screening test method for right ear. It can have value as 0 (blank), 1 (ABR), 2(AOE). Hence, the datatype tinyint is selected.This field will always have a value hence defined as NOT NULL.
leftearresult tinyint This column stores the screening result for left ear. It can have value as 0 (blank), 1 (Pass), 2(Fail), 3(Declined). Hence, the datatype tinyint is selected.This field will always have a value hence defined as NOT NULL.
lefteartestmethod tinyint This column stores the screening test method for left ear. It can have value as 0 (blank), 1 (ABR), 2(AOE). Hence, the datatype tinyint is selected.This field will always have a value hence defined as NOT NULL.

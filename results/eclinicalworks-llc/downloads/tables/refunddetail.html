EHI Export Schema
Table refunddetail
Description This table stores information about refund details like PaymentId, claimid, refundid and amount.
Column DataType Description
id (Primary Key) int Unique identifier for each refund detail entry. Example: 1, 2, 3.
RefundId int Identifier for the refund associated with this detail. Example: 101, 102, 103.
dtlType int Integer representing the type of refund detail. Example: 0, 1, 2.
PaymentId int Identifier for the payment associated with this refund detail. Example: 1001, 1002, 1003.
ClaimId int Identifier for the claim associated with this refund detail. Example: 2001, 2002, 2003.
amount decimal Decimal value representing the amount of the refund detail. Example: 100.00, 50.00, 25.00.
notes text Text field containing any additional notes or comments about the refund detail. Example: "Refund for damaged item", "Partial refund due to late delivery".
deleteFlag int Integer representing whether the refund detail has been deleted or not. 0 means not deleted, 1 means deleted. Example: 0, 1.
parentDelete int Integer representing whether the parent refund has been deleted or not. 0 means not deleted, 1 means deleted. Example: 0, 1.
TrUserId int Identifier for the user who made the transaction related to this refund detail. Example: 3001, 3002, 3003.
modifiedDate datetime time stamp indicating the last time the refund detail was modified. Example: '2021-08-01 12:34:56'.

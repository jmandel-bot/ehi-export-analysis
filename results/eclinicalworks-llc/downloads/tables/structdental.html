EHI Export Schema
Table structdental
Description This table stores information related to dental structure records for each patient encounter, including the category, item, and detail of the dental structure, as well as the answers and notes provided by the dentist.
Column DataType Description
id (Primary Key) int This column stores id value of the structdental record. It is auto generated and primary key of This column . This field is kept as integer(11) so it will be consistent with all dental related tables and it is defined as NOT NULL constraint as it is a primary key.
encounterid int This column stores encounterid value for which this dental history documented. This column refers to encounterid column in enc table. This field is kept as integer(11) as it referring column encounterid from enc table. Hence data type, size and constraint is kept similar.
catid int This column stores categoryid value for history category which answered question belongs. This column refers to itemid column in items table. This field is kept as integer(8) as it allows value up to 2147483647 digits value and in real time scenario this field value will not exceed the maximum limit. This column is also kept NOT NULL constraint as it is referring itemid of items table.
itemid int This column stores itemid value for "Dental History" item. This column refers to itemid column in items table. This field is kept as integer(11) as it referring column itemid of items table. Hence data type, size and constraint is kept similar.
detailid int This column stores questionid value for which this answer documented. This column refers to id column in structdatadetail table. This field is kept as integer(8) as it referring column id of structdatadetail table. Hence data type and constraint is kept similar. Size is kept as integer(8) because in real time scenario this field value will not exceed the maximum limit.
answer text This column stores answer value of the question. This field is kept as text data type as it allows to store free text answer value and not NULL constraint as it will enforce to have not NULL value. This column allows empty string as default value.
answerid varchar This column stores answerid value for answer selected. This column refers to optionid column of structdataoptions table.  This field is kept as varchar(255) as it contains comma separated values of selected option's optionid and it is defined as NOT NULL constraint as it will enforce to not allow NULL value. This column allows empty string as default value as for the free text answer, this value can be empty. 
notes text This column stores free text note added for this question. This field is kept as text data type as it allows to store free text answer notes and not NULL constraint as it will enforce to not allow NULL value. This column allows empty string as default value. 
displayIndex tinyint This column stores display index for this question. This field is kept as tinyint(1) as it allows value up to 255 and in real time scenario this field value will not exceed 255 digits. This column allows zero as default value.

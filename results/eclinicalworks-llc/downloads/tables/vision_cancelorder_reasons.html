Table vision_cancelorder_reasons
Description This table stores a list of cancellation reasons for the pick-up in collection queue.
Column DataType Description
id (Primary Key) int This is the primary key column, which stores a unique integer value for each reason. It is auto-incremented, meaning that the value increases automatically as new reasons are added to the table. Example: 1, 2, 3, etc.
Description varchar This column stores a brief description of the reason for canceling an order. It is a variable character field with a maximum length of 255 characters. Example: "Customer changed their mind", "Product out of stock", etc.
isdeleted tinyint This is a tiny integer column that indicates whether the reason has been deleted from the system or not. It stores a value of 0 for not deleted and 1 for deleted. Example: 0, 1.
createdby int This column stores the logged in user id who added this reason. This column refers column UID from user table.
createdon datetime This column stores the date and time when the cancel order reason was created. It is a datetime field. Example: '2022-01-01 12:34:56'.
updatedby int This column stores an integer value representing the user who last updated the cancel order reason. It refers to the `id` column in the user table (not provided). Example: 1, 2, 3, etc.
updatedon datetime This column stores the date and time when the cancel order reason was last updated. It is a datetime field. Example: '2022-01-02 14:56:32'.

EHI Export Schema
Table documentbreakglass
Description This table stores information about instances when a user breaks the glass to access a patient's document, including the user, patient, document, reason, accessing date, status, and IP address.
Column DataType Description
id (Primary Key) int Unique identifier for each break glass event. Example: 1, 2, 3
userid int identifier for the user who initiated the break glass event. Foreign key relation: `users` table, `UID` column. Example: 101, 102, 103
patientid int identifier for the patient whose document was accessed during the break glass event. Example: 1001, 1002, 1003
documentid int identifier for the document that was accessed during the break glass event. Foreign key relation: `document` table, `docid` column. Example: 2001, 2002, 2003
reason varchar Reason provided by the user for initiating the break glass event. Example: "Emergency access", "Patient consented"
accessingdate datetime Date and time when the break glass event occurred. Example: '2021-08-01 12:34:56'
STATUS varchar Status of the break glass event, such as "Active" or "Resolved". Example: "Active", "Resolved"
ipAddress varchar IP address of the user who initiated the break glass event. Example: '192.168.1.1', '10.0.0.2'

EHI Export Schema
Table fhir_write_back_audit_request
Description This table stores information about FHIR write-back audit requests, including the request ID, vendor ID, request status, and creation time.
Column DataType Description
id (Primary Key) int An auto-incrementing integer that serves as the primary key for the table and uniquely identifies each record. Example: 1, 2, 3, etc.
request_id varchar A varchar(45) column that stores the unique ID of the audit request. Example: "REQ123", "REQ124", etc.
vendor_id varchar A varchar(45) column that stores the ID of the vendor associated with the audit request. This column can be NULL if there is no vendor associated with the request. Example: "VND001", "VND002", etc.
request_status tinyint A tinyint(4) column that stores the status of the audit request. The default value is '1'. Example: 1 (Pending), 2 (Completed), 3 (Failed), etc.
creation_time datetime A datetime column that stores the time stamp when the audit request was created. Example: "2022-01-01 12:34:56", "2022-01-02 14:23:45", etc.

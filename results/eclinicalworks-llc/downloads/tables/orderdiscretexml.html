EHI Export Schema
Table orderdiscretexml
Description This table stores information about discrete XML values associated with specific lab tests for patients during their encounters.
Column DataType Description
id (Primary Key) int This is the primary key column that uniquely identifies each record in the table. It is an integer value that is automatically incremented for each new record. Example: 1, 2, 3, etc.
Encounterid int This column stores the encounter id associated with the lab test. It is a foreign key that refers to the `id` column in the `encounters` table. Example: 101, 102, 103, etc.
Labid int This column stores the lab id associated with the lab test. It is a foreign key that refers to the `id` column in the `labs` table. Example: 201, 202, 203, etc.
Testid int This column stores the test id associated with the lab test. It is a foreign key that refers to the `id` column in the `tests` table. Example: 301, 302, 303, etc.
Patientid int This column stores the patient id associated with the lab test. It is a foreign key that refers to the `id` column in the `patients` table. Example: 401, 402, 403, etc.
DiscreteXmlValue text This column stores the discrete XML value associated with the lab test. It is a text column that can store large amounts of text data. Example: `45mg/dL`.

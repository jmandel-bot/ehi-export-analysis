EHI Export Schema
Table emcoder_historypfsh
Description This table stores the history of a patient's past, family, and social history (PFSH) information for the encounter for E&M Computation purposes when E&M Coder V2 is used for calculations. This table contains user-entered information for E&M Computation.
Column DataType Description
id (Primary Key) int This is the primary key for the table, which is an integer value that uniquely identifies each record. It is an auto-incrementing value. Example: 1, 2, 3, etc.
encid int The `encounterId` associated with the `encoder_historyfsh` is a foreign key that refers to the `encounterId` column in the `enc` table. Example: 101, 102, 103, etc.
past int A Integer column stores the 0/1 for checked or unchecked values.
family int A Integer column stores the 0/1 for checked or unchecked values.
social int A Integer column stores the 0/1 for checked or unchecked values.
addedby int Identifier of the user who added the record. Foreign key relation with the `users` table (assuming it exists). Example: 1001, 1002, 1003
addeddate datetime Date and time when the record was added. Example: '2021-09-01 12:34:56'
updatedby int Identifier of the user who last updated the record. Foreign key relation with the `users` table (assuming it exists). Example: 1001, 1002, 1003
updateddate datetime Date and time when the record was last updated. Example: '2021-09-15 14:23:45'
deleteflag tinyint Tiny integer value indicating whether the record has been deleted or not. Example: 0 (Not deleted), 1 (Deleted)

Table encclaims
Description This table stores information about medical encounters, including details about the patient, doctor, date, time, and facility.
Column DataType Description
encounterid (Primary Key) int Unique identifier for each encounter. Foreign key relation: `enc.encounterid' (e.g., 1, 2, 3).
patientid int Unique identifier for the patient involved in the encounter. Foreign key relation: `patients.pid` (e.g., 101, 102, 103).
doctorid int Unique identifier for the doctor involved in the encounter. Foreign key relation: `Users.UID` (e.g., 201, 202, 203).
date datetime Date of the encounter (e.g., '2021-09-01', '2021-09-02', '2021-09-03').
startTime datetime Time when the encounter started (e.g., '09:00:00', '14:30:00', '16:45:00').
VisitTypeid int identifier for the type of visit (e.g., 1 for regular checkup, 2 for emergency). For example: 1, 2, 3.
deleteFlag tinyint A flag indicating if the encounter record has been deleted (0 for not deleted, 1 for deleted). For example: 0, 1.
encType tinyint Type of encounter (e.g., 1 for in-person, 2 for telemedicine). For example: 1,2.
encLock tinyint A flag indicating if the encounter record is locked (0 for unlocked, 1 for locked). For example: 0,1.
facilityid int Unique identifier for the facility where the encounter took place (e.g., 301, 302, 303).
POS tinyint Place of service code representing the location where the encounter occurred (e.g., 11 for office, 21 for inpatient hospital). For example: 11, 21, 22.
visitstscodeid int Unique identifier for the visit status code (e.g., 1 for scheduled, 2 for completed, 3 for canceled). For example: 1, 2, 3.

Table rpm_healow_fitness_data
Description This table stores fitness data collected from various devices and sources for patients.
Column DataType Description
fd_id (Primary Key) int Contains auto incremented id, datatype is INT. This column stores integer value which could grow with number of RPM healow fitness data added and hence INT is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index.
UID int This column stores unique id of a Patient which references the UID column of the user table. The data type chosen is INT. This field will always have a value hence defined as NOT NULL.
device_action int This column stores the Vendorid from whom the devicehas been purchased.Contains " device_action " value referenced from id column of the rpm_vendors_list table, datatype is INT.
parameter_id int Contains "parameter_id" value referenced from id column of the rpm_healow_tracker_list table, datatype is INT. This field will always have a value hence defined as NOT NULL.
measure_datetime datetime Contains " measure_datetime " value at time when fitness data is measured, datatype is DATETIME. This field will always have a value hence defined as NOT NULL.
parameter_value varchar Contains the name of the fitness parameter measured and datatype is VARCHAR(255) and default value is NULL. This column stores characters which cannot exceed 255 characters hence VARCHAR(255) is selected.
modifiedDateTime datetime Contains " modifiedDateTime" value at time when user logs into healow, datatype is DATETIME.  This field will always have a value hence defined as NOT NULL.
delflag tinyint Flag indicates fitness data is deleted or not, datatype is tinyint - 0 - Not Deleted, 1 - Deleted. This field will always have a value hence defined as NOT NULL. By default, fitness data should not be in Deleted status hence default 0 is chosen.
reviewed tinyint Contains value that specifies if the fitness data is reviewed or not and data type is TINYINT. This field will always have a value hence defined as NOT NULL and default value is 0.  Values 0 :Readings has not been reviewed by Provider 1-Readings has been reviewed by provider
reviewedByid int Contains the UID value who reviewed the rpm healow fitness data which refers from UID column of user table and data type is INT.
reviewedDateTime datetime Contains the date and time when fitness data is reviewed by the reviewer, datatype is DATETIME and default value is NULL.
isManuallyAdded tinyint Contains the flag that specifies if the fitness data was measured by tracker or manually added and datatype is TINYINT . By default, fitness data should not be manually added hence default 0 is chosen. It could either have values 0 or 1. 0 means its not manually added, 1 is for manually added entry
device_id int Contains " device_id" value referenced from device_id column of the rpm_supported_devices_list table, datatype is INT.
source_id int Contains " source_id" value referenced from id column of the rpm_healow_tracker_source table, datatype is INT
isRPMData tinyint This column stores a boolean value (0 or 1) indicating whether the RPM reading are received from RPM prescribed device or not. For example: 0 for not prescribed device reading, 1 for RPM prescribed device reading

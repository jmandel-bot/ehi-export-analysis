Table ip_rh_dischargemedication
Description This table stores the details of discharge medications for a patient
Column DataType Description
dismedid (Primary Key) int This column serves as the unique identifier for each order record. The integer data type is suitable for this purpose, as it provides efficient storage and quick querying of filter order records. This column is not part of UI, but it helps maintain order records. Constraint defined for this column is PK_ip_rh_dischargemedication
masterencounterid int This column is used to store associated encounter id. The integer data type is used for efficient storage and querying. The column has a foreign key constraint (FK_ip_dischargemedication_masterencounterid) referencing the [encounterID] from the [enc] table. This column is NOT NULL as it is required to maintain reconciliation detail. This column is not part of UI, but it helps maintain order records.
itemid int This column is used to store associated item id. The integer data type is used for efficient storage and querying. The column has a foreign key constraint (FK_ip_dischargemedication_itemid) referencing the [itemid] from the [items] table. This column is NOT NULL as it is required to maintain reconciliation detail. This column is not part of UI, but it helps maintain order records.
doctorsflag int This column is used to store doctor flag value which represents the discharge medications status. The TINYINT data type is chosen for efficient storage and sorting. NOT NULL constraint ensures doctor flag should not be null. Expected values are as below: 0 - STOP medication 1 - START medication 2 - CONTINUE medication
strength varchar This column is used to store strength for inpatient or home medication order. VARCHAR data type allows for flexibility in action type descriptions while limiting storage usage. It stores values like 10mg, 20 mg etc.
dose varchar This column is used to store dose for inpatient or home medication order. The data type used for this column is VARCHAR, size is 30. The varchar data type helps by providing enough space for descriptive detail while maintaining efficient storage and retrieval. This column is NULLABLE as it is not mandatory. Sample value : ‘1 tablet' This value is derived from ip_rh_drugformulary : dosesize table and it can contain value of length 30 as per formulary side.

The varchar data type helps by providing enough space for descriptive detail while maintaining efficient storage and retrieval. This column is NULLABLE as it is not mandatory Sample value : ‘injection' free text box This value is derived from ip_rh_drugformulary : doseUom table and it can contain value of length 30 as per formulary side.

The varchar data type helps by providing enough space for descriptive detail while maintaining efficient storage and retrieval. This column is NULLABLE as it is not mandatory Sample value : ‘by mouth' This value can be derived from ip_rh_drugformulary_routes : routedesc OR ip_rh_homemedication: route from table and it can contain value of length 255

It can either derive from ip_rh_ptorders table - orderid column or ip_rh_homemedication table - homemedid column. To identify whether id is of inpatient or homemed, refmedtype column is used.

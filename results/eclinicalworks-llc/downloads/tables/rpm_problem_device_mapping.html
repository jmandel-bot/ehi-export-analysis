Table rpm_problem_device_mapping
Description This table stores the mapping between devices, and the problems (ICD Codes) they are facing for a patient. When the user wants to prescribe one or more devices for a patient in RPM, he has to select one or more ICD codes from the patient problem list to map with that device.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT. This column stores integer value which could grow with number of problem device mappings added and hence integer is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index.
patientid int Contains patientid which is referenced using the Primary Key column UID of user table. It stores the unique id of a Patient and data type is INT. This field will always have a value hence defined as NOT NULL
deviceid int Contains patientid which is referenced using the Primary Key column UID of user table. It stores the unique id of a Patient and data type is INT. This field will always have a value hence defined as NOT NULL
tracker_mapping_id int Contains tracker_mapping_id which is referenced by the Primary Key column id of rpm_device_tracker_mapping table to link the device with Tracker Type and data type is INT. This field will always have a value hence defined as NOT NULL.
ICDCode varchar This column stores the problem linked to prescribed device and the data type is VARCHAR(60). This column stores characters which cannot exceed 60 characters hence VARCHAR(60) is selected .This field can have a NULL value. The default value of this field is Blank String.
itemid int This column stores the Primary Key of items table which manages all the ICD Codes.Contains itemid value which is referenced from itemid column od items table and data type is INT. This field will always have a value hence defined as NOT NULL.
modifiedBy int This column stores the Provider id who has modified the record last and data type is INT. Contains modifiedBy value which is referenced from UID column of user table. This field will always have a value hence defined as NOT NULL.
modifiedDatetime datetime This column stores the Date Time when the record was last modified and data type is DATETIME. This field will always have a value hence defined as NOT NULL.
deleteFlag tinyint Flag indicates the record added is deleted, datatype is tinyint - 0 - Not Deleted, 1 - Deleted. This field will always have a value hence defined as NOT NULL. By default, this column should not be in Deleted status hence default 0 is chosen.
rpm_enrollment_id int This column contains enrollmentid value referenced from id column of the rpm_enrollment_details table, datatype is INT. This field will always have a value hence defined as NOT NULL.

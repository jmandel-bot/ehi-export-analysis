EHI Export Schema
Table asc_vitals_batch_device
Description This table stores information about batches of vital signs data collected from devices, including details about the encounter, the person who reported the data, and the import status.
Column DataType Description
batchid (Primary Key) int An integer that serves as the unique identifier for each batch of vital signs data. It is auto-incremented and set as the primary key. Example: 1, 2, 3.
encounterid int An integer representing the unique identifier of the encounter associated with the batch of vital signs data. It has a default value of 0.NOT NULL. Example: 101, 102, 103.
reportedBy int An integer representing the unique identifier of the user who reported the batch of vital signs data. It has a default value of 0.NOT NULL. Example: 1001, 1002, 1003.
reportedTime datetime A datetime value indicating the date and time when the batch of vital signs data was reported. It can be NULL. Example: '2022-01-01 12:34:56', '2022-01-02 14:30:00'.
notes text A text field for storing any additional notes or comments related to the batch of vital signs data. Example: 'Patient was feeling dizzy', 'Device malfunctioned during measurement'.
deleteFlag tinyint A tinyint(1) value indicating whether the batch of vital signs data has been deleted (1) or not (0). It has a default value of 0. Example: 0, 1.
isVitalFromDevice tinyint A tinyint(1) value indicating whether the vital signs data is from a device (1) or not (0). It has a default value of 1.NOT NULL. Example: 0, 1.
importedBy int An integer representing the unique identifier of the user who imported the batch of vital signs data. It can be NULL. Example: 2001, 2002, 2003. This column has a foreign key relation with the `users` table, specifically the `UID` column.
isImported tinyint A tinyint(1) value indicating whether the batch of vital signs data has been imported (1) or not (0).NOT NULL. It has a default value of 0. Example: 0, 1.

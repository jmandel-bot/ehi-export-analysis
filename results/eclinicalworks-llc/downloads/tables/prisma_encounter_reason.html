Table prisma_encounter_reason
Description This table stores information about the reasons for a specific encounter in a healthcare setting.
Column DataType Description
id (Primary Key) bigint A unique identifier for each encounter reason record. Example: 1, 2, 3.
encounterDetailId bigint A foreign key that refers to the `id` column in the `prisma_encounter_details` table, representing the specific encounter associated with this reason. Example: 101, 102, 103.
reasonTypeCode varchar A short code representing the type of reason for the encounter. Example: 'INJ', 'ILL', 'CHK'.
reasonType varchar A more descriptive text for the reason type. Example: 'Injury', 'Illness', 'Checkup'.
date varchar The date when the reason for the encounter was recorded, in a string format. Example: '2021-09-01', '2021-09-02', '2021-09-03'.
reasonCode varchar A short code representing the specific reason for the encounter. Example: 'F10', 'G20', 'H30'.
reasonName varchar A more descriptive text for the specific reason. Example: 'Fracture', 'Gastroenteritis', 'Headache'.
reasonTypeSystem varchar A system or standard used to define the reason type codes. Example: 'ICD-10', 'SNOMED-CT', 'LOINC'.
reasonTypeText varchar A text description of the reason type system. Example: 'International Classification of Diseases, Tenth Revision', 'Systematized Nomenclature of Medicine - Clinical Terms', 'Logical Observation Identifiers Names and Codes'.
reasonReferenceCode varchar A code that references an external system or standard for the specific reason. Example: 'A123', 'B456', 'C789'.
reasonReferenceDisplay varchar A display text for the external system or standard referenced by the reason code. Example: 'External System A', 'External System B', 'External System C'.

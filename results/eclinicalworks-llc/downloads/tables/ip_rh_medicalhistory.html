Table ip_rh_medicalhistory
Description Table is used to store past medical history details for patient encounter.
Column DataType Description
id (Primary Key) int This column stores auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer (Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
uId varchar This column stores vm id string value. We are using VARCHAR (60) as data type because this column's value will not be more than 60 characters. This column will always have a value hence defined as NOT NULL.
encounterID int This column is used to store patient episodic level encounter id against which medical history items get documented. This column is FOREIGN KEY which refers to encounterid column of enc table. This column stores a high range numeric value hence integer is selected, and this column will always have a value hence defined as NOT NULL.
pasthistory text This column stores past history item value. We are using VARCHAR (100) as data type because this column's value will not be more than 100 characters. This column will not always have a value hence defined as NULL.
historyicd varchar This column stores icd code value. We are using VARCHAR (10) as data type because this column's value will not be more than 10 characters. This column will not always have a value hence defined as NULL.
historyicdid int This column stores icdcode id added for patient encounter. This column stores itemid of items table. This column is FOREIGN KEY which refers to itemid column of items table This column stores a high range numeric value hence integer is selected, and this column will not always have a value hence defined as NULL DEFAULT NULL.
deleteFlag tinyint This will store 0 and 1 value. 0-not deleted and 1-deleted. This column is used to identify whether row is in use or not. We are using TINYINT as datatype as we just want to store 0 or 1 value. This column will always have a value hence defined as NOT NULL. By default, row will be in use, so we keep default value is 0.
createdBy int This column stores user id of user. This column is used to identify which user has added record (medical history item) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store a high range of numeric value hence integer is selected. This column will always have a value hence defined as NOT NULL.
createdDateTime datetime This Column stores datetime as we want to store a datetime when the item is added, so we are using DATETIME as data type of column. This column will always have a value hence defined as NOT NULL.
modifiedBy int This column stores user id of user. This column is used to identify which user has the last modified record (medical history item) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store a high range of numeric value hence integer is selected. This column will not always have a value hence defined as NULL DEFAULT NULL.
modifiedDateTime datetime This Column stores datetime as we want to store a datetime when the item is added, so we are using DATETIME as data type of column. This column will not always have a value hence defined as NULL DEFAULT NULL.

EHI Export Schema
Table emcoder_mdm_chartc
Description This table stores information related to medical charts, including chart data, chart level, and details about who added or updated the information.
Column DataType Description
id (Primary Key) int This is the primary key for the table, which is an integer value that uniquely identifies each record. It is an auto-incrementing value. Example: 1, 2, 3, etc.
encid int The encounter ID associated with the chart. This is a foreign key that refers to the `encounterID` column in the `enc` table. Example: 101, 102, 103, etc.
chartc_data varchar It is storing data as json string for chart c of Emcode 2021 guidelines.
chartlevel int This column sotres data for the mdm chart level as integer value.
addedby int The user ID of the person who added the record. Example: 1001, 1002, 1003, etc.
addeddate datetime The date and time when the record was added. Example: '2021-08-01 12:34:56'
updatedby int The user ID of the person who last updated the record. Example: 1001, 1002, 1003, etc.
updateddate datetime The date and time when the record was last updated. Example: '2021-08-02 14:23:45'
deleteflag tinyint This column stores value 0 / 1 indicating this appointment is deleted. This is soft delete. 0 - Not Deleted , 1 -Deleted

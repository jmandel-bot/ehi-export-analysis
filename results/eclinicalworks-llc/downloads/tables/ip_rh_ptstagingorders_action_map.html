Table ip_rh_ptstagingorders_action_map
Description The 'ip_rh_ptstagingorders_action_map' table store mappings of actions related to staging orders. Each record is uniquely identified by an 'actionMapId' and contains details about the staging order, user, order status, and some flags indicating the state of the order.
Column DataType Description
actionMapId (Primary Key) int This is the auto increment primary key for the table.
stagingOrderId int This is an Integer data type and cannot be NULL. This field holds the identifier for the staging order associated with the action map. It references the stagingOrderId column in the ip_rh_ptstagingorders_action_map table.
orderId int This is an integer and CANNOT be NULL. The default value is 0. This field is to hold the identifier for the order associated with the action map. It references the orderId column in the ip_rh_ptorders table.
userId int Integer data type, CANNOT be NULL. This field holds the identifier of the user who performed the action. It references the uId column in the users table.
orderStatus int int data type, CANNOT be. This field represents the status of the order at the time of the action. This is FK that rorderStatus value mark as 1 when place a new order, orderStatus value mark as 2 when signoff order, orderStatus value mark as 3 when modify existing order, orderStatus value mark as 5 when discontinue order, orderStatus value mark as 4 when modified order signoff, orderStatus value mark as 10 when modified order discontinue.efers to id coumn of ip_rh_orderstatus_master table.
isSignedOffFlag tinyint This is a tinyint, a smaller range integer, and CANNOT be NULL. This column indicates whether the order has been signed off or not. The default value is 0 (indicating not signed off).
isReconciled tinyint This is another tinyint column and CANNOT be NULL. This column indicates whether the order has been reconciled or not. The default value is 0 (indicating not reconciled).
deleteFlag tinyint This is a tinyint, a smaller range integer. It cannot be NULL and has a default value of 0. This column indicates whether a patient's staging order action has been marked for deletion or not. deleteFlag value 0 mark as not deleted and value 1 mark as deleted.

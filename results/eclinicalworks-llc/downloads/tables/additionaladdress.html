Table additionaladdress
Description This table stores additional addresses for patients, including details such as address lines, city, Zip Code, state, country, and the type of address.
Column DataType Description
id (Primary Key) int Unique identifier for each additional address entry. Example: 1, 2, 3.
patientid int Foreign key referring to the patient's unique identifier in the `patient` table. Example: 101, 102, 103.
AddressLine1 text First line of the additional address. Example: "123 Main St", "456 Elm St".
AddressLine2 text Second line of the additional address, typically used for apartment or suite numbers. Example: "Apt 4B", "Suite 300".
city varchar City where the additional address is located. Example: "New York", "Los Angeles".
zip varchar Zip code of the additional address. Example: "10001", "90001".
state varchar State where the additional address is located. Example: "New York", "California".
country varchar Country where the additional address is located. Example: "United States", "Canada".
isPrimary tinyint This field stores 0 and from patientdemographics it can not be change to 1
typeofaddress varchar Type of the additional address, such as "home", "work", or "billing". Example: "home", "work".

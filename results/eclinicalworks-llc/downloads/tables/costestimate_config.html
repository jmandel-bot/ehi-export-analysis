Table costestimate_config
Description This table stores configuration data for cost estimates, including the estimate description, date, and creator information.
Column DataType Description
id (Primary Key) int This is the primary key for the table, which is an auto-incrementing integer. It uniquely identifies each cost estimate configuration. Example: 1, 2, 3, ...
estid int This is an integer column that stores the estimate ID, which can be used to link this configuration to other tables. (Foreign key relation: `patient_costestimate_header.estid`) iExample: 101, 102, 103.
est_desc varchar This is a varchar(50) column that stores a short description of the cost estimate. Example: "Building construction", "Software development", "Marketing campaign", ...
estdate datetime This is a date column that stores the date when the cost estimate was created. It has a default value of '0000-00-00'. Example: '2021-08-01', '2021-08-02', '2021-08-03', ...
createdby int This is an integer column that stores the ID of the user who created the cost estimate. It has a default value of '0'. (Foreign key relation: `Users.UID`) Example: 1, 2, 3.
void char This is a char(1) column that indicates whether the cost estimate configuration is void or not. It has a default value of 'N'. Example: 'Y' (void), 'N' (not void)
deleteflag tinyint This is a tinyint(4) column that indicates whether the cost estimate configuration has been deleted or not. It has a default value of '0'. Example: 0 (not deleted), 1 (deleted)

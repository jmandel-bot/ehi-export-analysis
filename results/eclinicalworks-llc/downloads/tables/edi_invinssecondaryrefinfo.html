EHI Export Schema
Table edi_invinssecondaryrefinfo
Description This table stores information about secondary reference codes and their corresponding names used in Electronic Data Interchange (EDI) for inventory and insurance purposes.
Column DataType Description
id (Primary Key) int This is the primary key, an integer value that uniquely identifies each record in the table. It is an auto-incrementing column, meaning that its value will automatically increase for each new record inserted. Example values: 1, 2, 3, etc.
Code varchar This is a unique varchar(4) column that stores the secondary reference code used in EDI transactions. It has a maximum length of 4 characters and cannot be NULL. Example values: 'A001', 'B002', 'C003', etc.
Name varchar This is a varchar(255) column that stores the descriptive name corresponding to the secondary reference code. It has a maximum length of 255 characters and cannot be NULL. Example values: 'Inventory Code A', 'Insurance Code B', 'Shipping Code C', etc.

EHI Export Schema
Table todo_list
Description This table stores information about tasks or activities that need to be performed for patients.
Column DataType Description
id (Primary Key) int Primary key column for the table that is unique for each record.
patient_id int This column stores integer patient id. Required to store patient id for To Do list that is foreign key for UID column in the user table. Also, used as integer as it's an id and never remains NULL.
type_id tinyint This column stores tiny int value for source id. Required to store type id for To Do list that is foreign key for id column in todo_type table. Also, used as tiny int as there are 9 types and never remains NULL.
source_id tinyint This column stores tiny int value for source id. Required to store source id for To Do list that is foreign key for id column in todo_source table. Also, used as tiny int as there are 3 types and never remains NULL.
description varchar This column stores varchar description. Comes from an external source and can be NULL for items added from quick search. Required to use with size varchar(255) as more characters are required to store when it comes from external source.
reason varchar This column stores varchar reason details. It's a free text that user can enter, while deleting an item and can be NULL for one of the use case.  Required to use with size varchar(max) as more characters are required as it's an user entered free text.
status tinyint This column stores tiny int status details. Used to status for To Do list and it's tiny int field as statuses are PENDING (0), IMPORTED (1), and SKIPPED (2), as of now.
timeZone varchar This column stores varchar value for time zone. Cannot rely on master table.
performed_by int This column stores integer value for performed by. Required to store performed by id for To Do list that is foreign key for UID column in user table. Also, used as integer as it's an id and never remains NULL.
performed_on datetime This column stores date time value for performed on. Used to store date time value for To Do list and can be NULL in a use case.
deleteflag tinyint This column stores tiny int value for delete flag. Used to represent a record in use or not required only 2 value 0 and 1 that requires tiny int type. Value 1 represents item is discarded by user. Value 0 represents item present on EMR and not yet discarded by user.
dataOrigin varchar The origin of the data for the task. Example: EMR, Patient Portal, Phone Call.
dueDate date The date when the task is due. Example: 2022-01-10, 2022-01-15, 2022-01-20.

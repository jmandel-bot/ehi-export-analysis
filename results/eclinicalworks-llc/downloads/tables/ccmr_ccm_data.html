EHI Export Schema
Table ccmr_ccm_data
Description This table stores information about time added by staff / provider for a ccm / cpo / rpm enrolled patient in given billing month. It also stores details about claim / out of office encounter created or claim failure reason in case of failure.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer(Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
PTCO_id int This column is used to identify enrollment details of patient. This is foreign key column. It refers to primary key of ptacoprograms table. Data type is int(11).
Patientid int This column stores unique id of a Patient which references the UID column of the user table. The data type chosen is INT. This column allows a NULL value.
Claimid int When batch is processed and claim is generated , the claim id is stored in this column. This field can not have a NULL value. The default value of this field is 0. The data type chosen is INT as this field is pointing to id column of edi_invoice table which is also int.
ClaimDate datetime Contains date time when the batch is processed. The data type of this field is datetime. This field can have a NULL value.
CreatedDate datetime Contains date time when the timer record was created and datatype is DATETIME. This column store datetime hence time stamp is selected.
UpdatedDate datetime Contains date time when the timer record was last updated and datatype is DATETIME. This column store datetime hence time stamp is selected.
Encounterid int When batch is processed and encounter is generated , the encounter id is stored in this column. This field can not have a NULL value. The default value of this field is 0. references to to encounterid column of enc table. The data type chosen is INT as this field is pointing to encounterid column of enc table which is also int.
BillingDate datetime Contains date of specific billing month for which the batch is processed. The data type of this field is datetime. This field can not have a NULL value.
claimFailureReason varchar Contains error description in case of claim failure. The data type is text and it can have a NULL value.

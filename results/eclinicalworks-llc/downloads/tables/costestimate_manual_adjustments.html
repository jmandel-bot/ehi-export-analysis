Table costestimate_manual_adjustments
Description This table stores manual adjustments made to cost estimates by users, including the adjustment amount and the user who made the adjustment.
Column DataType Description
id (Primary Key) int This is the primary key for the table, which is an integer value that uniquely identifies each record. It is an auto-incrementing value. Example: 1, 2, 3, etc.
estid int This is a foreign key that refers to the `estid` column in the `patient_costestimate_header` table. It represents the cost estimate that the manual adjustment is associated with. Example: 101, 102, 103, etc.
userId int This is a foreign key that refers to the `uid` column in the `users` table. It represents the user who made the manual adjustment. Example: 1001, 1002, 1003, etc.
amount decimal This is a decimal column that stores the amount of the manual adjustment made to the cost estimate. It has a precision of 9 digits and a scale of 2 decimal places. Example: 150.25, -75.00, 200.00, etc.
modifiedOn datetime This is a datetime column that stores the date and time when the manual adjustment was made. Example: '2021-08-01 14:30:00', '2021-08-02 10:15:00', etc.
deleteflag tinyint This column stores value 0 / 1 indicating this appointment is deleted. This is soft delete. 0 - Not Deleted , 1 -Deleted

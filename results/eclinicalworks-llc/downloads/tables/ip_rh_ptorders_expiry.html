Table ip_rh_ptorders_expiry
Description This table appears to manage the expiry details of patient orders. It includes information about the order's expiry, renewal, and associated flags.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. This column does not refer to another table but is likely referenced by other tables as a foreign key.
orderId int Associate the expiry record with a specific order. Likely refers to an ip_rh_ptorders table, specifically the id column, indicating which order this expiry pertains to.
renewedBy int Indicates which user or process renewed the order. Likely refers to a users table, specifically the uid column, to identify the user who renewed the order.
expireDateTime datetime Records the date and time when the order will expire or has expired. The default date '1901-01-01 00:00:00' is a sentinel value that can be used to easily identify records where the actual expiry date has not been set.
expiryFlag int A flag indicating the expiry status of the order. The default value of 0 likely indicates that the order is active or not expired when the record is first created.
renewedDateTime datetime Records the date and time when the order was last renewed. This column is used to track when an order's expiry has been extended, which is important for maintaining accurate records of order validity.

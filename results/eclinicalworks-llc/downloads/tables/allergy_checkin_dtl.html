Table allergy_checkin_dtl
Description This table stores information about patients' allergy check-in details, including their encounter, facility, and any reactions or notes related to the check-in.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
PATIENTid int Unique identifier for the patient. Foreign key relation to the patient table (e.g., `patient.id`). Example: 101, 102, 103.
ENCOUNTERid int Unique identifier for the patient's encounter. Foreign key relation to the encounter table (e.g., `encounter.id`). Example: 1001, 1002, 1003.
SHOT_FAC_TYPE tinyint A small integer representing the type of shot facility. Example: 0, 1, 2.
FACILITYid int Unique identifier for the facility where the check-in occurred. Example: 501, 502, 503.
FACILITYNAME varchar Name of the facility where the check-in occurred. Example: "Allergy Clinic A", "Allergy Clinic B".
PT_CHKIN_STATUS tinyint A small integer representing the patient's check-in status. Example: 0 (not checked in), 1 (checked in).
DELETEFLAG tinyint A small integer indicating whether the record has been deleted. Example: 0 (not deleted), 1 (deleted).
PT_ADMIN_STATUS tinyint A small integer representing the patient's administration status. Example: 0 (not administered), 1 (administered).
DIS_CONDITION varchar A string describing the patient's condition at the discharge. Example: "Mild allergy", "Severe allergy".
DIS_IS_PATIENT_WAITED tinyint A small integer indicating whether the patient waited during the check-in. Example: 0 (no), 1 (yes).
DIS_REASON varchar A string describing the reason for the patient's discharge. Example: "Routine check-up", "Allergic reaction".
DIS_NOTES varchar A string containing any additional notes related to the patient's discharge: "Patient was anxious", "Patient had a mild reaction".
NA_REASON varchar A string describing the reason for not administering the shot. Example: "Patient refused", "Medical contraindication".
IS_SYS_REACTION tinyint A small integer indicating whether there was a systemic reaction. Example: 0 (no), 1 (yes).
SYS_REACTION varchar A string describing the systemic reaction, if any. Example: "Anaphylaxis", "Urticaria".
SYS_REACTION_TIME datetime The date and time when the systemic reaction occurred. Example: "2021-08-01 14:30:00".
SYS_NOTES varchar A string containing any additional notes related to the systemic reaction. Example: "Patient was treated with epinephrine", "Reaction resolved within 30 minutes".
SYS_USERid int Unique identifier for the user who recorded the systemic reaction. Example: 10001, 10002, 10003.
SYS_USERNAME varchar The username of the user who recorded the systemic reaction. Example: "userA", "userB".
DIS_TIME datetime The date and time when the patient was discharged. Example: "2021-08-01 15:00:00".
DIS_USERid int Unique identifier for the user who recorded the patient's discharge. Example: 10001, 10002, 10003.
DIS_USERNAME varchar The username of the user who recorded the patient's discharge. Example: "userA", "userB".
SHOT_ADMIN_TIME datetime The date and time when the shot was administered. Example: "2021-08-01 14:00:00".

Table physiciannotes
Description This table stores the notes created by physicians during patient encounters.
Column DataType Description
PhysicianNotesid (Primary Key) int Unique identifier for each physician note entry. (Example: 1, 2, 3)
Encounterid int identifier for the patient encounter to which the note is related. This is a foreign key that refers to the `Encounter` table's `Encounterid` column. (Example: 101, 102, 103)
Selected int Indicates whether the note is selected or not, with 1 representing selected and 0 representing not selected. (Example: 0, 1)
Notes text Contains the text of the physician's notes. (Example: "Patient reports mild headache.")
createDate datetime time stamp for when the note was created, stored in datetime format. (Example: '2021-09-15 14:30:00')
notes2 text Contains additional text for the physician's notes. (Example: "Advise patient to take ibuprofen.")
deleteFlag int Indicates whether the note has been deleted or not, with 1 representing deleted and 0 representing not deleted. (Example: 0, 1)
trans_locked int Indicates whether the note is locked for editing or not, with 1 representing locked and 0 representing not locked. (Example: 0, 1)
trans_status varchar Contains the status of the note's transaction, stored as a string. (Example: 'COMPLETED', 'PENDING')

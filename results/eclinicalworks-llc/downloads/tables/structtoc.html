EHI Export Schema
Table structtoc
Description This table stores structured data related to patient's TCM for an encounter.
Column DataType Description
id (Primary Key) int It is auto-generated unique primary key
patientid int This column stores the patient id for which structured TCM data have been documented. This column refers to column UID from user table.
catid int This column stores the itemid of the category for which structured TCM data have been documented. This column refers to column itemid from items table.
itemid int This column stores the id of the Symptom displayed in the User Interface. This column refers to column propid of properties table.
detailid int This column stores the id of the structured question associated with the TCM of the patient. This column refers to column id from structdatadetail table.
answer text This column stores value of an answer to the structured question documented by end user. This column refers to column name from structdataoptions table.
answerid varchar This column stores the id of an answer selected by the end user while documenting structured questions. This column refers to column optionid from structdataoptions table.
notes text This column stores the note added by the end user for the respective structured question.
displayIndex tinyint Stores order in which structured questions are listed on the user interface.
notificationid int identifier for the notification associated with the entry. This could be a foreign key referring to the `notification` table's `id` column. Example: 3001, 3002, 3003.

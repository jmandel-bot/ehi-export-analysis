EHI Export Schema
Table claimsuppressederrors
Description This table stores information about suppressed errors related to claims.
Column DataType Description
id (Primary Key) int Unique identifier for each suppressed error entry. It is an integer value and is the primary key for the table. Example: 1, 2, 3, etc.
TrUserid int identifier for the user who triggered the suppressed error. It is an integer value and can be NULL. This can be a foreign key referring to the `id` column in a `users` table. Example: 101, 102, 103, etc.
ClaimNo int identifier for the claim associated with the suppressed error. It is an integer value and can be NULL. This can be a foreign key referring to the `ClaimNo` column in a `claims` table. Example: 1001, 1002, 1003, etc.
ErrDesc text Description of the suppressed error. It is a text field and can store a detailed explanation of the error. Example: "Invalid claim date", "Missing claimant information", etc.
deleteFlag tinyint A flag indicating whether the suppressed error entry has been deleted or not. It is a tinyint(1) value with a default of '0'. '0' means not deleted, and '1' means deleted. Example: 0, 1.

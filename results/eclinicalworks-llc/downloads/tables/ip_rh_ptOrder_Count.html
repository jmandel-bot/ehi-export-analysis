Table ip_rh_ptOrder_Count
Description The table 'ip_rh_ptOrder_Count' appears to store count data related to various aspects of patient orders in a healthcare setting. Each record is uniquely identified by an 'id' and contains details about the patient encounter and counts of overdue nursing orders, co-signatures, acknowledgements, and overdue and due medications.
Column DataType Description
id (Primary Key) int This is the primary key for the table, providing a unique identifier for each record. It is an integer data type, auto incrementing from 1, and cannot be NULL, ensuring a unique and not NULL value for every record.
episodeEncId int This is an integer and CANNOT be NULL. It holds the identifier for the patient encounter associated with the order counts. It has a foreign key constraint referencing the 'encounterID' column in the 'enc' table.
nursingOverdueCnt int This is an integer and cannot be NULL. The default value is 0. It holds the count of overdue nursing orders.
cosignCnt int This is an integer and cannot be NULL. The default value is 0. It holds the count of co-signatures.
ackCnt int This is an integer and cannot be NULL. The default value is 0. It holds the count of acknowledgements.
medicationOverdueCnt int This is an integer and cannot be NULL. The default value is 0. It holds the count of overdue medications.
medicationDueCnt int This is an integer and cannot be NULL. The default value is 0. It holds the count of due medications.
updatedDateTime datetime This field is a datetime data type and can be NULL. It represents the date and time the count data was updated.

EHI Export Schema
Table bh_reviewlog
Description This table stores the review logs for each patient's ACO program, including the reviewer's information and the review data.
Column DataType Description
id (Primary Key) int This is the primary key for the table, and it is an auto-incrementing integer. It uniquely identifies each review log entry. Example: 1, 2, 3, etc.
ptAcoid int This is a foreign key that refers to the `id` column in the `ptacoprograms` table. It represents the patient's ACO program that is being reviewed. Example: 101, 102, 103, etc.
reviewerid int This is a foreign key that refers to the `UID` column in the `users` table. It represents the user who reviewed the patient's ACO program. Example: 1001, 1002, 1003, etc.
reviewedTime datetime This is a datetime column that stores the time stamp when the review was conducted. It has a default value of the current time stamp. Example: '2021-09-01 12:34:56', '2021-09-02 14:30:00', etc.
lockStatus tinyint This is a tinyint column that stores the lock status of the review log. It has a default value of '0'. A value of '0' means the review log is unlocked, and '1' means it is locked. Example: 0, 1.
data varchar This is a text column that stores the actual review data or comments made by the reviewer. Example: 'Review completed successfully', 'Patient needs follow-up', etc.

Table ip_rh_ptADTOrders
Description The table 'ip_rh_ptADTOrders' appears to store information related to patient Admission, Discharge, and Transfer (ADT) orders in a healthcare setting. Each record is uniquely identified by an 'id' and contains details about the order, admission, discharge, providers involved, and other related information.
Column DataType Description
id (Primary Key) int This is the primary key for the table, providing a unique identifier for each record. It is an integer data type, auto incrementing from 1, and cannot be NULL, ensuring a unique and not NULL value for every record.
orderId int This is an integer and CANNOT be NULL. It holds the identifier for the order associated with the ADT event. It references the orderId column in the ip_rh_ptorders table.
admitAsId int This is an integer, CANNOT be NULL and has a default value of 0. It holds the identifier for the admission type.
levelOfCareId int This is an integer, CANNOT be NULL and has a default value of 0. It holds the identifier for the level of care for the patient.
attendingProviderId int This is an integer, CAN be NULL. It holds the identifier for the attending provider. It references the uId column in the users table.
deleteFlag tinyint This is a tinyint, a smaller range integer, CANNOT be NULL and has a default value of 0. This column indicates whether an patient's ADT order has been marked for deletion or not. deleteFlag value 0 mark as not deleted and value 1 mark as deleted.
dischargeDispositionId int This is an Integer data type, can be NULL. This field holds the identifier for the discharge disposition. This is FK and refers id column in ip_rh_dict_disposition.
dischargeProviderId int This is an Integer data type, can be NULL. This field hold the identifier of the provider who discharged the patient.
patientWaitingFor text This is a varchar data type and cannot be NULL. It holds information about what the patient is waiting for.
dischargeContingencies text This is a varchar data type and cannot be NULL with a default value empty. It holds information about any contingencies related to the patient's discharge.
admittingProviderId int This is an Integer data type, can be NULL. This field hold the identifier of the provider who admitted the patient. It references the uId column in the users table.
accommodationTypeId int This is an Integer data type, can be NULL. This field hold the identifier for the type of accommodation assigned to the patient.
reasonForTransfer text This is a varchar data type and cannot be NULL with default value empty. It holds information about the reason for the patient's transfer.
doNotCreateSubEncFlag tinyint This is a tinyint, a smaller range integer, CANNOT be NULL and has a default value of 0. This column indicates whether a sub-encounter should be created or not.
numOfMidNightStay int This is an Integer data type, can be NULL. This field hold the number of midnights stays for the patient.
admitDateTime datetime This field is a datetime data type and can be NULL. It represents the date and time the patient was admitted.
anticipatedDischargeDate date This is a date data type and can be NULL. It represents the anticipated discharge date for the patient.
dischargeDateTime datetime This field is a datetime data type and can be NULL. It represents the date and time the patient was discharged.

EHI Export Schema
Table transworldcollhistory
Description This table stores the history of transactions and collection statuses for patients in a medical practice.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table (e.g., 1, 2, 3).
Patientid int The identifier for the patient associated with the transaction. This could be a foreign key referring to the id column in the Patient table (e.g., 101, 102, 103).
PrevBalance decimal The patient's balance before the transaction (e.g., 100.00, 200.00, 300.00).
Balance decimal The patient's current balance after the transaction (e.g.,150.00, 250.00, 350.00).
CollectionStatus varchar The status of the collection process for the patient's balance (e.g., Pending, Paid, Overdue).
date datetime The date when the transaction occurred (e.g., 2021-10-01, 2021-10-02, 2021-10-03).
time datetime The time when the transaction occurred (E.g., 14:30:00, 15:45:00, 16:00:00, etc.).
modifieddatetime datetime The timestamp for when the record was last modified. This automatically updates when the record is modified (e.g., 2021-10-01 14:30:00, 2021-10-02 15:45:00, 2021-10-03 16:00:00).
fileid int The identifier for the file associated with the transaction. This could be a foreign key referring to the id column in the `transworldfilelog` tables. (e.g., 1001, 1002, 1003).
practiceid int This is the identifier for the medical practice associated with the transaction. This could be a foreign key referring to the id column on the Practice table (e.g., 1, 2, 3).

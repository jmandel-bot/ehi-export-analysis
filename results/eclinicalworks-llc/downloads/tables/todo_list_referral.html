Table todo_list_referral
Description This table stores information about referrals related to To Do Lists, including the referral reason, priority, and notes.
Column DataType Description
id (Primary Key) int This is the primary key for the table, and it is an auto-incrementing integer. It uniquely identifies each referral entry. Example: 1, 2, 3, etc.
todo_list_id int This is a foreign key that refers to the id column in the todo_list table. It represents the id of the todo list that the referral is associated with. Example: 1, 5, 10, etc.
referral_to_id int This is a foreign key that refers to the UID column in the users table. It represents the id of the user to whom the referral is made. This column can have a NULL value if the referral is not directed to a specific user. Example: 1, 3, 7, etc.
priority tinyint This is a tiny integer column that represents the priority level of the referral. The default value is 0. Example: 0, 1, 2, 3, etc.
speciality_id int This is a foreign key that refers to the `id` column in the edi_speciality table. It represents the id of the speciality associated with the referral. This column can have a NULL value if the referral is not associated with a specific speciality. Example: 1, 4, 8, etc.
referral_reason varchar This is a text column that stores the reason for the referral. Example: Patient needs a specialist consultation, Request for a second opinion, etc.
notes varchar This is a text column that stores any additional notes or comments related to the referral. Example: Urgent referral, Follow up in 2 weeks, etc.

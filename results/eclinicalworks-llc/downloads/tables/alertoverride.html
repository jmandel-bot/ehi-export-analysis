EHI Export Schema
Table alertoverride
Description This table stores the suppression status of CDSS Alerts for patients.
Column DataType Description
id (Primary Key) int This column is an auto incremented primary key for this column.
Patientid int This column stores an id of the patient for whom alerts are suppressed. This column refers to the column UID from the user table.
Measureid varchar This column stores the id of the measure/alert which is suppressed for a patient. This column refers to the column Measureid from the measureconfiguration table.
DueDate datetime This column stores the date until which the alert in the Measureid column is suppressed. The format used is YYYY-MM-DD 00:00:00.
NeverRemind tinyint This column stores flag 0 or 1. 1 - permanent suppression of alerts, and 0 - temporary suppression of alerts.
Reason text This column stores the free text reason documented by the end user as to why the alert in Measureid column is suppressed.
status varchar This column stores one of the below pre- defined values as status. 1P, 2P, 3P and ERRONEOUS. These values are mapped with the values in statusLabel column respectively. 1P - MEDICAL REASON, 2P - PATIENT REASON, 3P - SYSTEM REASON & ERRONEOUS - ERRONEOUSDIAGNOSES.
statusLabel varchar This column stores one of the below pre-defined values as status label. MEDICAL REASON, PATIENT REASON, SYSTEM REASON & ERRONEOUS DIAGNOSES. These values are mapped with the values in status column respectively. 1P - MEDICAL REASON, 2P - PATIENT REASON, 3P - SYSTEM REASON & ERRONEOUS - ERRONEOUSDIAGNOSES.
reset tinyint This column stores flags 0 or 1. 1 - if the suppression has been reset, and 0 - if the suppression has not been reset.

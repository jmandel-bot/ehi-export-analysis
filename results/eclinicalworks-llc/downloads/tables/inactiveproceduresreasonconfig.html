Table inactiveproceduresreasonconfig
Description This table stores the configuration of reasons for marking procedures as inactive, along with information about who created and modified the records and when.
Column DataType Description
id (Primary Key) smallint A unique identifier for each record in the table. It is a small integer that auto-increments with each new entry. Example: 1, 2, 3, etc.
NAME varchar A short name for the reason of marking a procedure as inactive. It is a variable character string with a maximum length of 25 characters. Example: "Outdated", "Deprecated", "Error".
description varchar A detailed description of the reason for marking a procedure as inactive. It is a variable character string with a maximum length of 256 characters. Example: "This procedure is no longer in use due to updated gUIDelines."
invalidFlag tinyint A flag indicating whether the reason is valid or not. It is a tiny integer with a default value of '0'. Example: 0 (valid), 1 (invalid).
createdBy int The user id of the person who created the record. It is an integer that refers to the `UID` column in the `users` table as a foreign key. Example: 101, 102, 103, etc.
createdDate datetime The date and time when the record was created. It is a datetime value. Example: "2021-09-01 12:34:56".
modifiedBy int The user id of the person who last modified the record. It is an integer that can be NULL and refers to the `UID` column in the `users` table as a foreign key. Example: 101, 102, 103, etc.
modifiedDate datetime The date and time when the record was last modified. It is a datetime value that can be NULL. Example: "2021-09-10 15:30:00".
deleteFlag tinyint A flag indicating whether the record has been deleted or not. It is a tiny integer with a default value of '0'. Example: 0 (not deleted), 1 (deleted).

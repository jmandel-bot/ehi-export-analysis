EHI Export Schema
Table patient_rafscore
Description This table stores the RAF (Risk Adjustment Factor) scores for patients, along with the year and the last updated timestamp.
Column DataType Description
RAF_id (Primary Key) int This is the primary key for the table and is an integer that uniquely identifies each record in the table. It is an auto-incrementing value, meaning it will automatically increase by 1 for each new record added to the table. Example: 1, 2, 3, etc.
RAF_Patientid int The unique identifier for the patient associated with the RAF score. This refers to the `Patientid` column in a `patients` table. Example: 101, 102, 103, etc.
RAF_Score float The RAF score for the patient, which is a float value. Example: 1.2, 2.5, 3.1, etc.
RAF_Year datetime The year for which the RAF score is applicable, stored as a datetime value. Example: '2022-01-01 00:00:00', '2023-01-01 00:00:00', '2024-01-01 00:00:00', etc.
RAF_LastUpdated datetime The time stamp when the RAF score was last updated, stored as a datetime value. Example: '2022-07-15 14:30:00', '2022-08-10 10:45:00', '2022-09-01 16:00:00', etc.
RAF_DeleteFlag tinyint A tinyint value indicating whether the RAF score entry has been deleted (1) or not (0). Example: 0, 1.

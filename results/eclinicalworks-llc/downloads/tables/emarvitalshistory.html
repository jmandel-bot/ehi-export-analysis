EHI Export Schema
Table emarvitalshistory
Description This table stores the history of vital signs for patients in an eMAR (electronic Medical Administration Record) system.
Column DataType Description
Id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
emarID int Foreign key referring to the emar_medication table's ID column. Represents the eMAR record associated with the vital sign history. Example: 101, 102, 103.
edasID int Foreign key referring to the emar_direct_admin_schedule table's ID column. Represents the eMAR record associated with the vital sign history. Example: 101, 102, 103.
vitalID int Foreign key referring to the items table's ID column. Represents the vitaltype being recorded. Example: 301, 302, 303.
propID int Foreign key referring to the Properties table's ID column. Represents the property associated with the vital sign.
vitalValue varchar The value of the vital sign being recorded. Stored as a string. Example: "98.6", "120/80", "18".
vitalValueHtm text The value of the vital sign being recorded, formatted in HTML. Stored as a text. Example: "98.6", "120/80", "18".
UpdatedBy int Foreign key referring to the User table's ID column. Represents the user who updated the vital sign history record. Example: 501, 502, 503.
UpdatedTime datetime time stamp of when the record was last updated. Example: "2021-08-01 12:34:56".
deleteflag tinyint A tiny integer value indicating whether the record has been deleted (1) or not (0). Example: 0, 1.

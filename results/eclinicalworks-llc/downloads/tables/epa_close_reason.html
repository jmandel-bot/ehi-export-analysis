EHI Export Schema
Table epa_close_reason
Description This table stores the close reasons for EPA (Electronic Prior Authorization) cases, along with their associated reason codes, descriptions, vendor IDs, and deletion flags.
Column DataType Description
id (Primary Key) tinyint This is the primary key and an auto-incrementing tinyint(4) column, which uniquely identifies each close reason record in the table. Example: 1, 2, 3, etc.
reason_code varchar This is a varchar(10) column that stores the unique reason code for each close reason. Example: "RC001", "RC002", "RC003", etc.
reason_description varchar This is a varchar(500) column that stores a detailed description of the close reason. Example: "Case resolved successfully", "Case closed due to lack of evidence", "Case transferred to another department", etc.
vendor_id tinyint This is a tinyint(4) column that stores the ID of the vendor associated with the close reason. This could potentially be a foreign key referring to a `vendor` table's primary key (e.g., `vendor.id`). Example: 1, 2, 3, etc.
delete_flag tinyint This is a tinyint(4) column with a default value of '0', which indicates whether the close reason record has been deleted (1) or not (0). Example: 0 (not deleted), 1 (deleted).

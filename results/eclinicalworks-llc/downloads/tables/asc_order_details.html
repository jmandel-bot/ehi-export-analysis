EHI Export Schema
Table asc_order_details
Description This table stores the details of orders in a medical context, specifically related to different types of notes (pre-operative, intra-operative, post-operative, and discharge).
Column DataType Description
orderDetailid (Primary Key) int This column is auto-generated unique primary key.
orderid int This column is used to store primary key of particular order type. e.g if AUC order type is lab/DI/Proc, then it will store reportid column of labdata table, if AUC order type is RX, then it will store oldrxid of rxdetails table.
noteType varchar This column is used to describe the provider/Anesthesia note type i.e Pre/Intra/Post for which order have been placed.
tableName varchar This column is used to store table name like labdata.
entryMethod tinyint This column stores tinyint value, It can be either 0 or 1.
deleteFlag tinyint This column store delete flag. It will be set to 1 if entry is deleted, otherwise 0.
orderPriority tinyint This is a tinyint column that represents the priority of the order. It has a default value of 0. Example: 0, 1, 2, etc.

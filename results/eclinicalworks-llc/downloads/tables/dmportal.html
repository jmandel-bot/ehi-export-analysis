EHI Export Schema
Table dmportal
Description This table stores the patient document metadata that are published to portal by the provider. The documents in this table are uploaded to the patient portal for patient access through a scheduled job.
Column DataType Description
id (Primary Key) int This column is an identity column with auto-increment value.
pid int This column stores the patient id of the document.
truserid int This column stores the user id of the logged in user who published the document for portal access. It refers to users.UID column
docid int This column stores the document id of the document. It refers to document.docid column
docname varchar This column stores the document name along with extension.
doctype int This column stores the type of the document. This column is copied from document.doc_type which refers to documentfolders.id column.
publishDate datetime This column stores the date and time this document was published to portal by the end user.
path varchar This column stores the storage path of the document in FTP.
size int This columns store the storage size of the document
websync int This column indicates if the document has been uploaded to the patient portal. 0 - not web synchronized 1- web synchronized
delflag int This columns stores the delete flag of the document. 0 - not deleted 1 - soft deleted
customName varchar This column stores the custom name an end user enters for the document.
source varchar This column indicates the source/module from which the document was published. eg. For patient documents the source is "publishToPortal"
ftpDeleted tinyint This column indicates if the document has been deleted from FTP. Available for international clients

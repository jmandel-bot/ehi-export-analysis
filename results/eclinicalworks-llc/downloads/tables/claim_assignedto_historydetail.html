Table claim_assignedto_historydetail
Description This table stores the history details of claims assigned to different users, including actions taken, notes, and other related information.
Column DataType Description
detailid (Primary Key) int Unique identifier for each history detail entry. Example: 1, 2, 3.
ClmHistoryid int identifier for the claim history to which the detail entry belongs. This can be a foreign key referring to the claim_assignedto_history table primary key. Example: 101, 102, 103.
actionid int Reference of claim_actions table of id primary column
notes varchar Notes or comments related to the action taken on the claim. Example: "Approved after verification", "Rejected due to insufficient documents".
ModifiedDate datetime Date and time when the history detail entry was last modified. Example: '2021-09-15 14:30:00'.
EnteredBy int identifier for the user who entered or modified the history detail entry. This can be a foreign key referring to the user table's primary key. Example: 1001, 1002, 1003.
resultCodeid int Reference of resultcodes table of id primary column
notetype tinyint Type of note associated with the history detail entry. Example: 0 (General), 1 (Internal), 2 (External).
fromAppeal tinyint Indicates whether the history detail entry is related to an appeal or not. Example: 0 (Not from appeal), 1 (From appeal).

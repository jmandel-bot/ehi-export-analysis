EHI Export Schema
Table emessagelogs
Description This table stores information about portal messages sent to patients by campaigns.
Column DataType Description
id (Primary Key) int Unique identifier for each message log entry. Example: 1, 2, 3.
patientid int This column stores the patient id to whom message has been sent. This column refers to pid column in patients table.
apuid int This column stores the APU id of the practice.
facilityid int This column stores the id of the facility from where the message has been sent. This column refers to Id column in edi_facilities table.
reportid int Identifier for the report associated with the message. Foreign key relation to the `reports` table (`id` column). Example: 2001, 2002, 2003.
inserttime datetime time stamp for when the message log entry was created. Example: '2021-09-01 12:34:56'.
campaignid int This column stores the campaign id for appointment reminder, Rx reminder, Lab reminder etc.
campaign varchar column not in use/ inactive
SUBJECT text This column stores the subject line of emessage.
Body text This column stores the actual content of the emessage.
dtReceived datetime time stamp for when the message was received. Example: '2021-09-01 13:45:00'.
readflag tinyint Indicates whether the message has been read (1) or not (0). Example: 0, 1.
dtRead datetime time stamp for when the message was read. Example: '2021-09-01 14:30:00'.
healthMaintenance tinyint Indicates whether the message is related to health maintenance (1) or not (0). Example: 0, 1.
messageid int This column store the message id associated with row.
policyid int This column stores policy id for all the reminders like appointment, rx, lab etc.
policy varchar column not in use/ inactive
doctorId int This column stores the provider id. This column refers to uid column in user table.

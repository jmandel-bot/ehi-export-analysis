EHI Export Schema
Table jointexaminationresult
Description This table stores the joint examination results for patients, including various assessment scores and related information.
Column DataType Description
JointExaminationResultid (Primary Key) int • This column will store the id of the Joint Examination Result data. • The datatype of the column would be INT to accommodate large number of different Joint Examination Result data that can be added. • This would be unique and auto incremented and so PRIMARYKEY constraint is added
encounterid int • This column stores the encounter id where the results are being documented. • This is a reference to the column encounterid of table enc.
tenderJointCount tinyint • This column stores the count of tender joints determined from the entries in table JointTableCell. • The datatype of the column would be tinyint as it can have 0 to 28 values. • By default, tender joint count will be zero.
swollenJointCount tinyint • This column stores the count of swollen joints determined from the entries in table JointTableCell. • The datatype of the column would be tinyint as it can have 0 to 28 values. • By default, swollen joint count will be zero.
patientGlobalAssessDiseaseActivityDas28 decimal Patient's global assessment of disease activity using the DAS28 scoring system. Example: 3.2, 4.5, 2.1, etc.
providerGlobalAssessDiseaseActivityDas28 decimal Provider's global assessment of disease activity using the DAS28 scoring system. Example: 3.5, 4.2, 2.3, etc.
esr decimal • This column stores the ESR value that can be captured on the UI. • The datatype of the column would be decimal as it can have 0 to 999.999 values. • This column is NULL by default to avoid showing 0.00 on the screen which will be incorrect.
crp decimal • This column stores the CRP value that can be captured on the UI. • The datatype of the column would be decimal as it can have 0 to 999.999 values. • This column is NULL by default to avoid showing 0.00 on the screen which will be incorrect.
das28Score decimal • This column stores the DAS28 score that can be calculated from the other values in This column and displayed on the UI. • The datatype of the column would be decimal as it can have 0 to 999.999 values. • This column is NULL by default to avoid showing 0.00 on the screen which will be incorrect.
cdaiScore decimal • This column stores the CDAI score that can be calculated from the other values in This column and displayed on the UI. • The datatype of the column would be decimal as it can have 0 to 999.999 values. • This column is NULL by default to avoid showing 0.00 on the screen which will be incorrect.
globalNotes varchar • This column stores the notes that can be added and updated for the visit from the Homunculus Examination screen. • The datatype of the column would be varchar(max) in case of MSSQL and text in case of MySQL • This column value is empty by default
patientGlobalAssessDiseaseActivityCdai decimal Patient's global assessment of disease activity using the CDAI scoring system. Example: 3.2, 4.5, 2.1, etc.
providerGlobalAssessDiseaseActivityCdai decimal Provider's global assessment of disease activity using the CDAI scoring system. Example: 3.5, 4.2, 2.3, etc.

Table prisma_pot_act_detail
Description This table stores action details of the plan of treatment from external documents.
Column DataType Description
id (Primary Key) bigint This is the primary key column, which uniquely identifies each record in the table. It is of type bigint(20) and is auto-incremented. Example: 1, 2, 3, etc.
sectionId bigint This column stores the ID of the section to which the activity belongs. It is of type bigint(20) and can have a default value of NULL. This is a foreign key column that refers to the `id` column in the `prisma_ccd_document_section` table. Example: 1, 2, 3, etc.
actId varchar This column stores the unique identifier for the activity. It is of type varchar(50) and can have a default value of NULL. Example: "ACT-001", "ACT-002", etc.
actTime varchar This column stores the time when the activity occurred. It is of type varchar(50) and can have a default value of NULL. Example: "2021-09-01T12:34:56", "2021-09-02T14:30:00", etc.
moodCodeValue varchar This column stores the mood code value associated with the activity. It is of type varchar(100) and can have a default value of NULL. Example: "inent", "request", etc.
code varchar This column stores a code representing the type of activity. It is of type varchar(100) and can have a default value of NULL. Example: "CODE-001", "CODE-002", etc.
statusCode varchar This column stores the status code of the activity. It is of type varchar(100) and can have a default value of NULL. Example: "STATUS-001", "STATUS-002", etc.
referenceText text This column stores the reference text associated with the activity. It is of type longtext. Example: "This is a reference text for the activity.", "Another reference text for a different activity.", etc.

EHI Export Schema
Table collectionmanagement
Description This table stores information about the collection management process for patients with outstanding payments.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the collection management table. Example: 1, 2, 3.
patientid int identifier for the patient associated with the collection management record. This could be a foreign key referring to the `patients` table, specifically the `id` column. Example: 101, 102, 103.
collectiontype varchar Type of collection process being used for the patient. Example: 'Standard', 'Urgent', 'Legal'.
LettersentFlag smallint Indicates whether a letter has been sent to the patient regarding their outstanding payment (1 for sent, 0 for not sent). Example: 0, 1.
InsertedDate datetime Date and time when the record was inserted into the table. Example: '2021-08-01 12:34:56'.
userid int identifier for the user who inserted the record. This could be a foreign key referring to the `users` table, specifically the `id` column. Example: 1001, 1002, 1003.
PrevCollectionType varchar Previous type of collection process used for the patient, if any. Example: '', 'Standard', 'Urgent'.
ActionTaken varchar Description of the action taken during the collection process. Example: 'Sent reminder email', 'Called patient', 'Initiated legal action'.
FullyPaidFlag smallint Indicates whether the patient has fully paid their outstanding balance (1 for fully paid, 0 for not fully paid). Example: 0, 1.
guarantorid int identifier for the guarantor associated with the patient's outstanding payment. This could be a foreign key referring to the `guarantors` table, specifically the `id` column. Example: 2001, 2002, 2003.

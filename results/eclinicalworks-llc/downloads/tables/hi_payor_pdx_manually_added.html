Table hi_payor_pdx_manually_added
Description This table stores information about manually added payor diagnosis records, including payor details, patient and encounter information, and ICD codes.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table (e.g., 1, 2, 3).
payorid varchar Unique identifier for the payor, can be NULL (e.g., 'P123', 'P456').
payorname varchar Name of the payor, cannot be NULL (e.g., 'Insurance A', 'Insurance B').
pdxtype varchar Type of the diagnosis record, default value is 'Diagnosis(es)'.
patientid int Unique identifier for the patient, cannot be NULL. Foreign key relation with the `users` table, `UID` column (e.g., 101, 102).
encounterid int Unique identifier for the encounter, cannot be NULL. Foreign key relation with the `enc` table, `encounterid` column (e.g., 1001, 1002).
encounterdate datetime Date of the encounter, cannot be NULL (e.g., '2021-01-01', '2021-01-02').
enclockstatus smallint Lock status of the encounter, represented as a tiny integer (e.g., 0, 1).
icdcode varchar ICD code for the diagnosis, cannot be NULL (e.g., 'J20.9', 'M54.5').
icddesc varchar Description of the ICD code, cannot be NULL (e.g., 'Acute bronchitis, unspecified', 'Low back pain').
itemid int Unique identifier for the item, cannot be NULL. Foreign key relation with the `items` table, `itemid` column (e.g., 2001, 2002).
status varchar Status of the record, default value is 'Manually Added'.
actiondetails varchar Additional details about the action performed, stored as text (e.g., 'Added by user X', 'Reviewed by user Y').
createddate datetime Date and time when the record was created, cannot be NULL (e.g., '2021-01-01 12:34:56', '2021-01-02 14:56:32').
deleteflag tinyint Flag to indicate if the record is deleted, represented as a tiny integer, default value is '0' (e.g., 0 for not deleted, 1 for deleted).

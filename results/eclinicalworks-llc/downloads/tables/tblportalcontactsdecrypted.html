Table tblportalcontactsdecrypted
Description This table stores decrypted contact information for individuals associated with a portal, including their personal details, address, and phone numbers.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
contactid int Unique identifier for the contact. Example: 101, 102, 103.
pid int Unique identifier for the portal the contact is associated with. Example: 1001, 1002, 1003.
name varchar Full name of the contact. Example: "John Doe", "Jane Smith".
relation varchar Relation of the contact to the portal. Example: "Manager", "Employee".
address varchar First line of the contact's address. Example: "123 Main St".
address2 varchar Second line of the contact's address (optional). Example: "Apt 4B".
city varchar City of the contact's address. Example: "New York", "Los Angeles".
state varchar State of the contact's address. Example: "NY", "CA".
zipcode varchar Zip code of the contact's address. Example: "10001", "90001".
homePhone varchar Home phone number of the contact. Example: "(555) 123-4567".
workPhone varchar Work phone number of the contact (optional). Example: "(555) 987-6543".
delflag int Indicates if the record has been deleted (1) or not (0). Example: 0, 1.
contactType int An integer representing the type of contact (e.g., 1 for emergency contact, 2 for contact 1,3 for contact 2). Example: 1, 2,3.
timeVal datetime time stamp of when the record was created or updated. Example: "2022-01-01 12:34:56".
isFamilyMember tinyint Indicates if the contact is a family member (1) or not (0). Example: 0, 1.
isEmergencyContact tinyint Indicates if the contact is an emergency contact (1) or not (0). Example: 0, 1.
dob date Date of birth of the contact (optional). Example: "1980-01-01".
doi tinyint Indicates if the date of birth is incomplete (1) or complete (0). Example: 0, 1.
cellPhone varchar Cell phone number of the contact. Example: "(555) 123-9876".

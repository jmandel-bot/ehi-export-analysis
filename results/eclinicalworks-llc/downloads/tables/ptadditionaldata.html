EHI Export Schema
Table ptadditionaldata
Description This table stores additional information about patients in relation to their reference facility and department, as well as their care status and statement hold alerts.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
patientid int identifier for the patient associated with the additional data. This could be a foreign key referring to the `id` column in the `patients` table. Example: 101, 102, 103.
RefFacid int identifier for the reference facility associated with the patient. This could be a foreign key referring to the `id` column in the `facilities` table. Example: 1, 2, 3.
RefDeptid int identifier for the reference department associated with the patient. This could be a foreign key referring to the `id` column in the `departments` table. Example: 10, 20, 30.
CareInactive tinyint A flag indicating whether the patient's care is inactive (1) or active (0). Example: 0, 1.
deleteFlag tinyint A flag indicating whether the record has been deleted (1) or not (0). Example: 0, 1.
stmtholdalert int An integer value representing the statement hold alert status for the patient. Example: 0, 1, 2.

Table rpm_enrollment_details
Description This table stores the enrollment details of patients in remote patient monitoring programs, including consent type, provider information, and disenrollment details.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT. This column stores integer value which could grow with number of users added to the care team and hence integer is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index.
ptacoid int Contains the Primary Key column id of Ptacoprograms table. It stores the PHM enrollment id. The value of this field can be NULL. There is a Foreign Key associated with this field - FK_rpm_enrollment_details_ptacoid referencing to id column of ptacoprograms table. The data type chosen is INT as this field is pointing to id column of ptacoprograms table which is also int.
patientid int Contains the Primary Key column id of user table. It stores the unique id of a Patient. The value of this field can not be NULL. There is a Foreign Key associated with this field - FK_rpm_enrollment_details_patientid referencing to UID column of user table. The data type chosen is INT as this field is pointing to UID column of user table which is also int.
consent_type varchar This column stores type of consent provided i.e. Verbal, Written, Pending, Decline, Not Required etc.  This is an enum because there are fixed types of consents available.  This field can have NULL value. Therefore, the default value of this field is NULL. The value of this field cannot have a name more than 20 characters long as this is an enum type , therefore the data type is VARCHAR(20). We cannot give a default value from one of the enum values as the user needs to make a selection and we do not show any value by default when we launch the screen.
provider_id int This column stores the Primary Care Provider id who is going to provide care to the patient  Contains the Primary Key column id of user table. It stores the unique id of a Provider. The value of this field can be NULL.  There is a Foreign Key associated with this field - FK_rpm_enrollment_details_provider_id referencing to UID column of user table. The data type chosen is INT as this field is pointing to UID column of user table which is also int.
disenrolled_by int This column stores the id of Provider who has disenrolled the patient from RPM. This field can have a NULL value.  The default value for this field is not provided as the patient will be disenrolled once the enrollment record is inserted in DB. There is a Foreign Key constraint associated with this field - FK_rpm_enrollment_details_disenrolled_by referencing to UID column of user table.  The data type chosen is INT as this field is pointing to UID column of user table which is also int.
disenrolled_datetime datetime This column stores the Date Time when the patient is disenrolled from RPM. This field cannot have a NULL value.  Therefore, the default value of this field is '1901-01-01 00:00:00' for MSSQL and '0000-00-00 00:00:00' for MySQL. This field does not have an index linked to it.
deleteflag tinyint Flag indicates the record added is deleted, datatype is tinyint - 0: not deleted 1: deleted. This field will always have a value hence defined as NOT NULL. By default, this column should not be in deleted status hence default 0 is chosen.
disenrollment_notes varchar This column stores the notes captured while disenrolling a patient from RPM program. The datatype is selected as Varchar(max). This field can have a NULL value.

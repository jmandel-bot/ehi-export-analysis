Table familyplanning_ny
Description This table stores information related to family planning visits for patients in New York, including demographic, medical, and insurance details.
Column DataType Description
id (Primary Key) int The unique identifier for each record in the table (e.g., 1, 2, 3).
Encid int The unique identifier for each patient encounter (e.g., 101, 102, 103).
Patientid int The unique identifier for each patient (e.g., 1001, 1002, 1003).
ClinicNo varchar The clinic number where the visit took place (e.g., C001, C002, and C003).
ClientNo varchar The unique identifier for the client (e.g., CL001, CL002, and CL003).
DateOfVisit datetime The date of the patient's visit (e.g., 2021-01-01, 2021-01-02, 2021-01-03).
PurposeOfVisit varchar This column stores information about purpose of visit. Possible values for this column are : 1-Initial Medical Exam, 2-Annual Medical Exam, 3-Method Check/Maintenance, 4-Counseling, 5-Pregnancy Test, 6-Medical Problem/Follow-up
DOB datetime The date of birth of the patient (e.g., 1990-01-01, 1990-01-02, 1990-01-03).
MedSvcPrvded varchar This column is used to represent Medical services provided during the visit. List of possible values is large, hence omitted. Some of them are 31-Hgb / Hct, 02-Pap Smear, 10-Bimanual Pelvic, 12-Male Exam, 32-Urine Dipstick etc
ProviderMedSvc varchar This column is used to represent Provider code for the Medical services provided. Possible values are 1 - Physician, 3 - Other Clinician, 2 - PA / NP / CNM , 4 - Non-Clinician
CounsSvcPrvded varchar This column is used to represent Counseling services provided during the visit. List of Possible values is large. Some of them are 01-Contraceptive, 03-Infertility, 05-Pregnancy, 09-STD, 10-Preconception etc
ProviderCounsSvc varchar This column is used to represent Provider code for the counseling services provided. Possible values are 1 - Physician, 3 - Other Clinician, 2 - PA / NP / CNM , 4 - Non-Clinician
BegnContrMethd varchar This column is used to represent Beginning contraceptive method used by the patient. List of possible values is large, hence omitted. Some of them are 21 - Oral - Extend. Cycle, 04 - Diaphragm, 06 - Spermicide, 15 - Female Condom, 01. Sterilization etc
EndContrMethd varchar This column is used to represent Ending contraceptive method used by the patient. List of possible values are large, hence omitted here. Some of them are 02 - Oral, 18 - Vaginal Ring, 19 - Sponge, 20 - Abstinence, 14 - Hormonal Inj. - 3 mo etc
RsnNoContraMethd varchar This column is used to represent reason code for not using contraceptive method on patient. Possible values are 1 - Pregnancy, 2 - Infertility, 3 - Seeking Pregnancy, 4 - Other Medical Reasons, 5 - Relying on Female Method, 6 - Other, 7 - Not Sexually Active, 0 - Same Sex Partner, 8 - Vasectomy, 9 - Condom
PregTest varchar This column is used to represent Pregnancy test result. 1 means Negative and 2 means Positive.
IfPregTestPostv varchar This column is used to represent whether pregnancy was desired or not. Possible values in this column are 1 - Desired Now, 4- Not Desired, 2 - Desired Sooner , 3 - Desired Later
ReffrdElsewhere varchar This column is used to represent if the patient is refferred somewhere else. This column holds reason for which patient is referred somewhere else. Possible values are 1 - Pregnancy, 2 - Sterilizatio, 3 - Infertility, 4 - Medical Problem/Follow-up, 5 - CBE F/U, 6 - Primary Care
IntprtrNeeded varchar This column is used to represent whether the bilinugal interpreter or stafff is required. Value 1 means it is required and 2 means it is not required.
STDSvc varchar This column is used to represent different STD test and treatment performed on Patient. Possible values are 01 - Gonorrhea Test, 03 - Syphilis Test, 05 - Herpes Diagnosis, 07 - HPV Diagnosis, 09- Chlamydia Test, 02 - Gonorrhea Treatment, 04 - Syphilis Treatment, 06 - Herpes Treatment, 08 - HPV Treatment, 10-Chlamydia Treatment
Race varchar This column is used to represent patient's race. Possible values are 1-White, 2-Black / Afr American, 3-American Indian, 4-Alaskan Native, 5-Asian, 7-Pacific Islanders/Hawaaiian Native, 6-Other
InsCovrg varchar This column is used to represent whether the Primary care is covered for private insurance. Value 1 means Yes and 2 mean No.
HIVPreTestCoun varchar This column is used to represent HIV pre test counselling. Value 1 means Yes and 2 means No.
HIVTest varchar This column is used to represent whether the HIV Test was performed or not. Value 1 means HIV test was performed and 2 means HIV test was not performed.
HIVTestResult varchar This column is used to represent HIV test result. Value 1 means Positive and 2 means Negative or Inconclusive.
HIVPostTestCoun varchar This column is used to represent HIV post test counselling. Value 1 means Yes and 2 means No.
NoTimesPreg varchar The number of times the patient has been pregnant, up to 5 characters (e.g., 0, 1, and 2).
NoTimesBirth varchar The number of times the patient has given birth, up to 5 characters (e.g., 0, 1, and 2).
MonFamilyIncome varchar The monthly family income, up to 20 characters (e.g., $2000, $3000, $4000).
FamilyDependts varchar The number of family dependents, up to 5 characters (e.g., 1, 2, 3).
ZipCode varchar The patient's ZIP Code, up to 10 characters (e.g., 10001, 10002, 10003, etc.).
Ethnicity varchar This column is used to represent ethnicity of patient. Value 1 means Hispanic and 2 means Other.
Sex varchar This column is used to represent gender of patient. Value 1 means Female and 2 means Male.
HighestGrade varchar The highest grade completed by the patient, up to 50 characters (e.g., High school, College, Graduate school, etc.).
StudStatus varchar This column is used to represent student status of patient. Possible values are 1-Full Time Student, 2-Part Time Student, 3- Not a Student
County varchar The patient's county of residence, up to 10 characters (e.g., Albany, Bronx, Kings, etc.).
OtherHealthCareSrcs varchar This column is used to represent whether the other healthcare sources are used by patient. Possible value 1- Yes, 2 - No
MedcadNo varchar The Medicaid number for the patient, up to 20 characters (e.g., M123456, M123457, M123458, etc.).
InsPaidAmt varchar By Default this coulmn does not have any value. It only stores blank value.
OtherIns varchar By Default this coulmn does not have any value. It only stores blank value.
LastPGEndMonth varchar The last pregnancy end month, up to 10 characters (e.g., January, February, and March).
LastPGEndYear varchar The last pregnancy end year, up to 10 characters (e.g., 2019, 2020, 2021, etc.).
AssignedChargeCat varchar This column stores the data about assigned charge category. For example 01 - No Charge, 05 - Full Fee (100% of Scale), 02 - Title XIX, 04 - Private Insurance etc
IsPrimaryCovered varchar This column stores data about coverage of primary care in case of private insurance. Value 1 means primary care is covered and Value 2 means otherwise.
PmtSrc varchar By Default this coulmn does not have any value. It only stores blank value.
PregLastDate varchar The last date of the pregnancy, up to 10 characters (e.g., 2021-01-01, 2021-01-02, 2021-01-03, etc.).
deleteFlag tinyint The flag to indicate if the record is deleted, represented by an integer (e.g., 0 for not deleted, 1 for deleted).
livingchildren varchar The number of living children, up to 5 characters (e.g., 0, 1, and 2).
ssn varchar The patient's Social Security number, up to 10 characters (e.g., 123-45-6789, 987-65-4321, etc.).
primarylanguage varchar The patient's primary language, up to 10 characters (e.g., English, Spanish, Chinese).
patientname varchar The patient's full name, up to 100 characters in length (e.g., John Doe, Jane Smith, Maria Garcia, etc.).

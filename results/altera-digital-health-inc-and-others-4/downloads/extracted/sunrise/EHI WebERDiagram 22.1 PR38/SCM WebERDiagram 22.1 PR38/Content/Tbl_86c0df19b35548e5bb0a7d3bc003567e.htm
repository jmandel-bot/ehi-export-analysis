dbo.SXACDReferencedObsEntryItem 
Default Table Name  SXACDReferencedObsEntryItem 
Definition  We need a way to save and retrieve information when a user “copy forwards by reference” information from a previous note. This could be a Section or Set (or potentially any Observation Entry item). Example: In the current Structured Notes, if someone has values from an earlier note that they wish to “copy forward into the current note”, then they selected the sections, and the data is saved, giving the impression that everything copy forwarded was performed at the date/time of the current note. In 4.0, we need to provide a “reference” to an older document, so, you can “copy forward by reference” (instead of by value!) the responses from the earlier note, and it would be displayed on the note “Referenced from (5/13/2003)” (or whatever date). This way, someone reading the note will know that a QA / PW / Observation / History section was done at an earlier date.

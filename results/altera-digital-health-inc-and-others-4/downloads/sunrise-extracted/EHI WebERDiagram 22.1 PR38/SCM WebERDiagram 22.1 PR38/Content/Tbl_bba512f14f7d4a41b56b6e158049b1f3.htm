dbo.SXASHMsgPriorityType 
Default Table Name  SXASHMsgPriorityType 
Definition  A requirement for SHM is to have the ability to set a message priority, much like setting high priority in Outlook. The initial priority values are High, Medium, Low with the ability to have the customer customize these values via the bulk load process or by a future administration tool within SHM. To create other priorities, choose an appropriate PriorityCode and give it an ImportanceLevel relative to the others in the table. For example, to make a Medium-High Priority code choose an ImportanceLevel between the values of High and Medium (ie 13), to make an Ultra-High Priority, choose an ImportanceLevel of 100. If you need to create another priority in-between 2 other priorities and there is no “room” to fit an ImportanceLevel between these two values, you can multiply all ImportanceLevel values in this table by some factor (ie 5) to get more available ImportanceLevels to choose from.

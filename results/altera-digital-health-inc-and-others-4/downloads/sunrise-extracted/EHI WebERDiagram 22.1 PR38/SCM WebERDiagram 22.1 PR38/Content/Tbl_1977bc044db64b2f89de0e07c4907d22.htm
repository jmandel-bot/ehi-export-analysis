dbo.SXAHMVaccineCatalogItem 
Default Table Name  SXAHMVaccineCatalogItem 
Definition  This Health Management configuration table defines the Vaccine Catalog, which is describes every Vaccine in the system that can be linked into the Health Management system and scheduled for a client. 

dbo.SXAHMVaccineCatalogItem Columns

Build    int  NO  Identifies the build number and patch level of the software that inserted/updated this row. The format is the Build number followed by 3 digits for the patch level which is typically 000. The build number is tied to specific versions of the software, e.g. Sunrise 15.3 is Build 5021, Sunrise 16.3 is Build 5503.

VaccineCatalogItemID    int  NO  Auto generated unique primary key value. 
CVXCode    varchar(30)  NO  This number identifies the vaccine we are administering. It is the vaccine administration code defined in the HL7 standards. There can be several CVX codes for each Vaccine, which would vary based on factors like adult, child, live, dialysis, or preservatives. Each CVX code for a Vaccine would be associated with a separate row and a unique name. 
Name    varchar(100)  NO  Specifies what Vaccine we are defining, along with any additional characteristics associated with the CVX code; like target age, preservatives, etc. This name should be unique. 
VaccineTypeCode    varchar(100)  YES  Specifies the type of vaccine defined by this row, which changes the safety, effectiveness, and side affects of receiving the vaccine. There are many types of Vaccines, which are determined by the nature or preparation of the bacteria or viruses in the vaccine, including: killed (Inactivated), Attenuated, Toxoid, Subunit, Conjugate, Experimental, Component etc. 
Description    varchar(2000)  YES  Provides a detailed description of this Vaccine, including all available information. This is an optional field.

SiteID    smallint  NO  Identifies the unique distributed database ID where the row was inserted. Distributed databases are no longer supported so this column is deprecated.

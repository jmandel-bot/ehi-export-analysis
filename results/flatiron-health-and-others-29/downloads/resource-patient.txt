FHIR® [Patient] Resource

This page provides technical documentation specific to Flatiron’s deployment of the FHIR® [Patient] Resource. Flatiron’s development of this API includes and builds upon the required components outlined within the Health Level 7® (HL7) FHIR® standard. Within the Profiles Supported row in the table below, you can find a link to the relevant HL7® FHIR® standard profile(s). All Flatiron supported API resources are based on FHIR® version 4.0.1 as required by the (g)(10) Standardized API for Patient and Population Services certification criteria. This page will be updated periodically as Flatiron expands our FHIR® API integration options. 

[Patient] Resource Summary

FHIR® Version Supported

	

4.0.1




Profiles Supported

	

Patient Profile (US Core Implementation Guide STU 6.1.0)




HL7® Definition of Patient

	

Link




API Syntax

	

GET and POST operations are supported. PUT, and DELETE operations are not supported.




USCDI Data Elements Supported by this Resource

	

First Name, Last Name, Previous Name, Middle Name, Suffix, Date of Birth, Preferred Language, Address, Phone Number




Primary Location(s) for This Data in OncoEMR®

	

Demographics




Static Resource ID for a Record?*

	

Yes

*Some types of records may have new resource IDs assigned when certain data changes occur in OncoEMR. Resources with static resource IDs will retain the same ID as changes are made.

[Patient] Profile - Supported Data Elements and Value Sets

Flatiron supports the data elements and search parameters outlined within the US Core Profiles (US Core Implementation Guide STU 6.1.0). US Core Profiles outline specific constraints applied to the Patient Resource for a given use case.

All data elements listed as MANDATORY or MUST SUPPORT within the US Core Implementation Guide are returned if valued when the Patient Resource is called. No additional data elements are included within Flatiron’s Patient Resource beyond MANDATORY and MUST SUPPORT data elements.

All value sets or terminology code systems listed as REQUIRED are applied within Flatiron’s Patient API. 

[Patient] Profile - Supported Search Parameters & Request/Response Example

Supported Interaction

	

SearchType




Supported Search Parameters [DataType]

	

_id [token] OR

identifier [token] OR

name [string] OR

birthdate [date] AND name [string] OR

gender [token] AND name [string] 




URL Syntax

	
GET [base]/Patient[id]

GET [base]/Patient?identifier={system|}[code]

GET [base]/Patient?name=[string]

GET [base]/Patient?birthdate=[date]&name=[string]

GET [base]/Patient?gender={system|}[code]&name=[string]




Return Type

	

JSON




Return Schema

	

See example here

 

Supported Extension: US Core Tribal Affiliation Extension v6.1.0

US Core Tribal Affiliation Extension profile represents a tribe or band with which a person associates and, optionally, whether they are enrolled.  

Return Schema
 

{

            "extension": [

              {

                "url": "isEnrolled",

                "valueBoolean": true

              },

              {

                "url": "tribalAffiliation",

                "valueCodeableConcept": {

                  "coding": [

                    {

                      "system": "https://terminology.hl7.org/5.0.0/CodeSystem-v3-TribalEntityUS.htm"",

                      "version": "2014-03-26",

                      "code": "2",

                      "display": "Agua Caliente Band of Cahuilla Indians of the Agua Caliente Indian Reservation, California"

                    }

                  ],

                  "text": "Agua Caliente Band of Cahuilla Indians of the Agua Caliente Indian Reservation, California"

                }

              }

            ],

            "url": "http://hl7.org/fhir/us/core/StructureDefinition/us-core-tribal-affiliation""

          }



Supported Extension: US Core Ethnicity Extension v6.1.0

The US Core Ethnicity Extension allows one or more ethnicity codes from OMB Ethnicity Categories

Return Schema

 {
            "extension": [
              {
                "url": "ombCategory",
                "valueCoding": {
                  "system": "urn:oid:2.16.840.1.113883.6.238",
                  "version": "4.0.0",
                  "code": "2135-2",
                  "display": "Hispanic or Latino"
                }
              },
              {
                "url": "text",
                "valueString": "Hispanic or Latino"
              }
            ],
            "url": "http://hl7.org/fhir/us/core/StructureDefinition/us-core-ethnicity""
          },


Supported Extension: US Core Gender Identity Extension v6.1.0

The US Core Gender Identity Extension provides concepts to describe the gender a person identifies as. This extension is not used to describe a person’s sexual orientation (who they are attracted to).

Return Schema

 {
            "extension": [
              {
            "url": "http://hl7.org/fhir/us/core/StructureDefinition/us-core-genderIdentity"",
            "valueCodeableConcept": {
              "coding": [
                {
                  "system": "urn:oid:2.16.840.1.113762.1.4.1021.101",
                  "version": "2023-10-01",
                  "code": "446151000124109",
                  "display": "Identifies as male gender"
                }
              ],
              "text": "Identifies as male gender"
            }
          }
        ],

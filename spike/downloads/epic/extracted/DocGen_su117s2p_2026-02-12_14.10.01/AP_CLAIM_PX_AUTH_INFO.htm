AP_CLAIM_PX_AUTH_INFO
Description:
This table contains information about the service-level authorizations used for the AP Claim's services.

Primary Key
Column Name Ordinal Position
TX_ID 1
LINE 2

Column Information
Name Type Discontinued?
1 TX_ID NUMERIC No
The unique identifier for the transaction record.
2 LINE INTEGER No
The line number for the information associated with this record. Multiple pieces of information can be associated with this record.
3 AUTH_ID NUMERIC No
The matching Authorization ID for the AP Claims procedure.
4 OPERATIONAL_MATCH_YN VARCHAR No
Whether the authorization is needed for the operational authorizations requirements. 'Y' indicates that this is needed to satisfy the operational authorizations requirements. 'N' or NULL indicate that it's not.
May contain organization-specific values: No
Category Entries:
No
Yes
5 BENEFIT_MATCH_YN VARCHAR No
Whether the authorization is needed for the benefits authorizations requirements. 'Y' indicates that this is needed to satisfy the benefits authorizations requirements. 'N' or NULL indicate that it's not.
The category values for this column were already listed for column: OPERATIONAL_MATCH_YN
6 VENDOR_CONTRACT_MATCH_YN VARCHAR No
Whether the authorization is needed for the vendor contract. 'Y' indicates that this is needed for the vendor contract. 'N' or NULL indicate that it's not.
The category values for this column were already listed for column: OPERATIONAL_MATCH_YN
7 COUNTS_USED INTEGER No
The count used from this authorization. Note that this remains set even if the claim gets voided, or the service line is deleted.
8 COUNTS_CALCULATED INTEGER No
The count used that is calculated during AP claim adjudication.
9 COUNTS_SHARED_C_NAME VARCHAR No
Whether the service-level authorization count used is shared with a previously-adjudicated claim or service.
May contain organization-specific values: No
Category Entries:
No
Yes, shared with another claim
Yes, shared with another service
10 AUTH_ATTACH_SOURCE_C_NAME VARCHAR No
Whether the linked authorization was added manually or by the system.
May contain organization-specific values: No
Category Entries:
Adjudication Referral Matching
Manual
EDI Claim File
11 SUGGESTED_AUTH_YN VARCHAR No
Whether the attached authorization was flagged and suggested as a potential authorization to a user.

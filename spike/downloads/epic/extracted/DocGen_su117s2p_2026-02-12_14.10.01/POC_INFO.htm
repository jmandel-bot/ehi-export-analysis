POC_INFO
Description:
This table contains no-add single item information for the Plan of Care (POC) master file.

Primary Key
Column Name Ordinal Position
RECORD_ID 1

Column Information
Name Type Discontinued?
1 RECORD_ID NUMERIC No
The unique identifier for the plan of care record.
2 RECORD_STATUS_C_NAME VARCHAR No
The record status, such as active or soft-deleted.
May contain organization-specific values: No
Category Entries:
Soft Deleted
Hidden
Hidden and Soft Deleted
3 POC_CREATED_IN_DTTM DATETIME (Local) No
This item contains the instant at which the POC was created.
4 POC_REFRESHED_DTTM DATETIME (Local) No
This item contains the instant at which the POC was last refreshed with up-to-date patient info.
5 POC_LAST_SAVED_DTTM DATETIME (Local) No
This item contains the instant at which the POC was last saved with user-entered information.
6 POC_COMPLETED_DTTM DATETIME (Local) No
This item contains the instant at which the POC was filed and/or sent.
7 PLAN_OF_CARE_C_NAME VARCHAR No
This item contains the POC type.
May contain organization-specific values: No
Category Entries:
Plan Of Care
Plan Of Care Update
8 UPDATE_POC_ID NUMERIC No
This item contains a pointer to the POC being updated.
9 EPISODE_SEQ_NUM INTEGER No
This item contains the parent POC sequence number.
10 UPDATE_SEQ_NUM INTEGER No
This item contains the child POC sequence number.
11 EPISODE_ID NUMERIC No
This item contains the link to the episode in which the POC was created.
12 CARE_PLAN_CSN NUMERIC No
The contact serial number of the care plan contact that contains the care plan information pulled into the POC.
13 CARE_PLAN_INFO_C_NAME VARCHAR No
If the POC contains care plan visit information, problems, or both.
May contain organization-specific values: No
Category Entries:
Visit Data
Problem Data
Both Visit and Problem Data
14 POC_MEDS_CSN NUMERIC No
The contact serial number of the patient contact that contains the medications list for the POC.
15 RECORD_CREATION_DT DATETIME No
This item stores the date the record was created.
16 EPSD_ELECTION_DATE DATETIME No
The date of hospice election.
17 EPSD_EFFECTIVE_DATE DATETIME No
The date the hospice agency began taking care of the patient.
18 TRIAGE_CODE_C_NAME VARCHAR No
The patient's triage code, as documented in the episode.
May contain organization-specific values: Yes
19 POC_START_DT DATETIME No
Start date for a hospice plan of care. The plan of care is effective until the next POC/POC update's start date.
20 POC_END_DT DATETIME No
Virtual item which returns the end date of a hospice plan of care. This is determined based on other plans of care (and, sometimes, the episode discharge date). Typically a hospice plan of care will end whenever a subsequent plan of care is created. A plan of care update will end when a subsequent plan of care update is created.
21 POC_CODE_STATUS VARCHAR No
The patient code status when the plan of care is completed.
22 DISPLAY_NM VARCHAR No
This item contains the display name for a plan of care.
23 STAT_C_NAME VARCHAR No
This item contains the current status of a plan of care.
May contain organization-specific values: No
Category Entries:
Draft
Completed
Signed

Revised
Addending
Addended
Deleted

26 PAT_ID VARCHAR No
This item contains the patient ID for whom the current plan has been created.
27 SPOC_SIGNED_DATE DATETIME No
The date the last required participant signed the plan.
28 TEMPLATE_YN VARCHAR No
This item is true if a plan is a template agreement. Template agreements are not agreements for any particular patient, but are used, temporarily, to contain information used to create multiple agreements which may be on different patients.
May contain organization-specific values: No
Category Entries:
No
Yes
29 TEMPLATE_POC_ID NUMERIC No
This item links an agreement to the template agreement from which it was copied.
30 POC_RATIFIED_INST_DTTM DATETIME (UTC) No
Contains the instant at which the POC was ratified.
31 POC_RATIFIED_YN VARCHAR No
This item indicates if the agreement is ratified or not. A blank value indicates that no decision has been made. A value of 1 (true) indicates that the agreement is ratified. A value of 2 (false) indicates that a decision was made to not ratify the agreement.
May contain organization-specific values: No
Category Entries:
Yes
No
32 PLAN_UNDER_REVIEW_YN VARCHAR No
Whether the hospice POC is under review. While under review the POC cannot be signed.
The category values for this column were already listed for column: TEMPLATE_YN
33 CODE_STATUS_CMT VARCHAR No
The patient code status comment when the plan of care is completed.
34 HH_ASMNT_CSN_ID NUMERIC No
The unique contact serial number of the assessment contact used to populate the home health Plan of Care.
35 HH_CERT_PER_CSN_ID NUMERIC No
The unique contact serial number of the encounter that created the certification period associated with the home health Plan of Care.
36 POC_AUTO_CREATED_YN VARCHAR No
Indicates whether the Plan of Care has been automatically created. 'Y' Indicates that this Plan of Care was automatically created. 'N' or NULL indicate that it was manually created.
The category values for this column were already listed for column: TEMPLATE_YN
37 HH_POC_CONVERTED_YN VARCHAR No
Indicates whether the home health Specialty Plan of Care record corresponds to an old plan and was created out of the conversion.

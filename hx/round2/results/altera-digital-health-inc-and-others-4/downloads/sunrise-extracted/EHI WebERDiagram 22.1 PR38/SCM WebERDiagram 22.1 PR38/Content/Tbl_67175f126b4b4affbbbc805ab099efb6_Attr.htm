SiteID 
Definition  Identifies the unique distributed database ID where the row was inserted. Distributed databases are no longer supported so this column is deprecated. 
Build 
Definition  Identifies the build number and patch level of the software that inserted/updated this row. The format is the Build number followed by 3 digits for the patch level which is typically 000. The build number is tied to specific versions of the software, e.g. Sunrise 15.3 is Build 5021, Sunrise 16.3 is Build 5503. 
TouchedBy 
Definition  Information about the user that last updated the row. This will typically be the UserID (IDCode from CV3User table) and may be prefixed with an environment identifier followed by an underscore, e.g. b96d2bf5_jongmore. 
TouchedWhen 
Definition  The date and time that the row was last updated and will be based on the Enterprise time zone. 
CreatedBy 
Definition  Information about the user that created the row. This will typically be the UserID (IDCode from CV3User table) and may be prefixed with an environment identifier followed by an underscore, e.g. b96d2bf5_jongmore. 
CreatedWhen 
Definition  The date and time that the row was created and will be based on the Enterprise time zone. 
Active 
Definition  Denotes whether the row is active or not, 0 means the row has been soft deleted or deactivated. 
OutpatientOrderDispenseID 
Definition  Auto generated unique primary key value. 
OutpatientOrderID 
Definition  A foreign key to the OutpatientOrderID column in the SXAMMOutpatientOrder table. 
StockItemProductID 
Definition  A foreign key to the StockItemProductID column in the SXAMMStockItemProduct table. 
PharmacyID 
Definition  A foreign key to the PharmacyID column in the SXAMMPharmacy table. 
RefillNum 
Definition  If this is the first time the item is being dispensed (not a refill) the Refill Num will be zero.

,Refills will start at the number 1 and increase by 1 for each subsequent refill. 
DispensedQty 
Definition  Identifies the Quantity being dispensed 
DispensedUOMCode 
Definition  Identifies the Unit of Measure associated with the Dispensed Qty. Links to CV3UnitOfMeasure table via Code. 
DaysSupplyNum 
Definition  Identifies the Days Supply being dispensed 
Note 
Definition  Notes for the Dispense (if required) 
ReasonCode 
Definition  Identifies the reason for making the dispense record inactive.

Links to the CV3CodedReason table via the code.

EnteredByUserGUID 
Definition  Identifies the pharmacy user who entered the dispense information. 
OutpatientTypeID 
Definition  A foreign key to the OutpatientTypeID column in the SXAMMOutpatientType table. 
DebitInventory 
Definition  Indicates if the dispense should debit the inventory or not (the inventory may already have been decremented if the order is for a cream or puffer that was received as an inpatient) 
CostAmt 
Definition  The calculated Cost of the dispensed order/refill 
PriceAmt 
Definition  The calculated price of the dispensed order/refill 
CalculationDetails 
Definition  Identifies the formatted text associated with the Price Popup window that shows the detailed calculation steps. 
MSrepl_tran_version 
Definition  Optimistic lock identifier. 
RxNote 
Definition  This column stores notes provided by pharmacists during processing an Rx. This is different than the current Note column, which is storing clinicians' notes/instruction to pharmacy. 
BillForRxType 
Definition  This column stores a value indicating what type of billing was applied to a particular dispense. This could be different from the the configured value because pharmacists can manually change the value before processing a prescription. 
ProdPkgID 
Definition  This column stores what package is being dispensed for an ordered item. 
MSReplrowguid 
Definition  Unique GUID used to support merge replication.

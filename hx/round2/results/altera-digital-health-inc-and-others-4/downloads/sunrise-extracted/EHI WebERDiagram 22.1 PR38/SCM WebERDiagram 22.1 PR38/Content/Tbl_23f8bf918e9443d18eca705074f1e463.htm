dbo.CV3OrderTemplateModificationHistory 
Default Table Name  CV3OrderTemplateModificationHistory 
Definition  This table stores modification history on order templates added in regular, generic or linked set. 

dbo.CV3OrderTemplateModificationHistory Columns 
ColumnName  Domain  Datatype  NULL  Definition 
Build    int  NO  Identifies the build number and patch level of the software that inserted/updated this row. The format is the Build number followed by 3 digits for the patch level which is typically 000. The build number is tied to specific versions of the software, e.g. Sunrise 15.3 is Build 5021, Sunrise 16.3 is Build 5503. 
CreatedBy    varchar(50)  NO  Information about the user that created the row. This will typically be the UserID (IDCode from CV3User table) and may be prefixed with an environment identifier followed by an underscore, e.g. b96d2bf5_jongmore. 
CreatedWhen    datetime  NO  The date and time that the row was created and will be based on the Enterprise time zone. 
MSRowVersion    timestamp  NO  Optimistic lock identifier using rowversion datatype. 
OrderTemplateModificationHistoryID    int  NO  Auto generated unique primary key value. 
OrderGUID    HVCIDdt  YES  Order GUID. Modification history is tracked for this order template. 
FieldName    varchar(40)  YES  Name of modified order entry field. 
Label    varchar(40)  YES  Label of the modified field 
FromValue    varchar(2000)  YES  Original value of the modified field 
ToValue    varchar(2000)  YES  Updated value of the modified field 
UserGUID    HVCIDdt  YES  GUID of user who modified the order template 
UserNameAndOccupation    varchar(255)  YES  PHI: Name and occupation of user who modified the order template: ' ()'. This field is used for historical purposes. 
MSReplrowguid    uniqueidentifier  NO  Unique GUID used to support merge replication.

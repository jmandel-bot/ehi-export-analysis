ORDER_DISP_INFO_3
Description:
This table contains dispense information for orders.

Primary Key
Column Name Ordinal Position
ORDER_MED_ID 1
CONTACT_DATE_REAL 2

Column Information
Name Type Discontinued?
1 ORDER_MED_ID NUMERIC No
The unique identifier (.1 item) for the order record.
2 CONTACT_DATE_REAL FLOAT No
A unique contact date in decimal format. The integer portion of the number indicates the date of contact. The digits after the decimal distinguish different contacts on the same date and are unique for each contact on that date. For example, .00 is the first/only contact, .01 is the second contact, etc.
3 RX_FILL_NOTE_ID VARCHAR No
Prescription fill level note from the requesting source.
4 ACQ_COST_CODE_C_NAME VARCHAR No
Cost code used when calculating acquisition cost for the order.
May contain organization-specific values: Yes
Category Entries:

CMS FFP
CMS FFP FOR UNIT DOSE ITEMS
WAC
Acquisition Cost
Suggested Wholesale Price
Alternative Benchmark Price
Manufacturer Suggested Retail Price
Pharmacy Retail Price
5 CHARGING_CSN NUMERIC No
Contains the patient contact serial number (CSN) of the pharmacy visit contact to use for charge and payment posting.
6 EMRG_FILL_YN VARCHAR No
Is the current fill an emergency fill. An emergency fill is a short term refill that is approved by a pharmacist so that a patient can stay on their therapy while waiting for a refill authorization request (RAR) to be approved.
May contain organization-specific values: No
Category Entries:
No
Yes
7 EMERG_FILL_RSN_C_NAME VARCHAR No
The discrete reason the user gave for creating an emergency fill.
May contain organization-specific values: Yes
8 EMERG_FILL_NOTE_ID VARCHAR No
Stores the ID for a note (HNO) containing the free text additional details given for why it is necessary to create an emergency fill.
9 TAX_DUE_TOTAL NUMERIC No
This column holds the rounded value for the total amount due in taxes when the medication was sold.
10 FILL_START_DATE DATETIME No
Indicates the date the patient is expected to begin taking a particular dispense of a prescription.
11 TAX_EXEMPT_YN VARCHAR No
The column holds whether the tax was exempted when the medication was sold.
The category values for this column were already listed for column: EMRG_FILL_YN
12 ADJUD_REVERSE_ONLY_YN VARCHAR No
This indicates that an adjudication with a failed reversal should not be resubmitted once the reversal failure is corrected. This will only ever be set for adjudications from Hyperspace. It is not used for Willow Ambulatory.
The category values for this column were already listed for column: EMRG_FILL_YN
13 RX_FILL_NAME_BRAND VARCHAR No
The brand name for the prescription fill. E.g. in the name "TYLENOL (acetaminophen) 325 mg tablets", this would be "TYLENOL".
14 RX_FILL_NAME_GENERIC VARCHAR No
The generic name for the prescription fill. E.g. in the name "TYLENOL (acetaminophen) 325 mg tablets", this would be "acetaminophen".
15 RX_FILL_IS_BRAND_YN VARCHAR No
Is the fill for a brand medication. This helps determine how the medication name for the fill should be displayed. E.g. if it's a fill for a brand medication, the name would be "TYLENOL (acetaminophen) 325 mg tablets". If it's a fill for a generic medication, the generic medication name would display first. The default is No.
The category values for this column were already listed for column: EMRG_FILL_YN
16 RX_FILL_NAME_DOSAGE VARCHAR No
The dosage for the prescription fill. E.g. in the name "TYLENOL (acetaminophen) 325 mg tablets", this would be "325 mg tablets".
17 RX_TAX_ROUNDING_METHOD_C_NAME VARCHAR No
This column contains the rounding method used to calculate the total taxes for a medication sold within a transaction in an outpatient pharmacy.
May contain organization-specific values: No
Category Entries:
Standard
Round Up
18 TAX_EXEMPTED_TOTAL NUMERIC No
This column contains the rounded value for total amount of taxes exempted when the medication was sold.
19 FILL_END_DATE DATETIME No
Indicates the date the patient is expected to end taking a particular dispense of a prescription.

The therapy type for this order's fill

May contain organization-specific values: Yes
24 FILL_DOSES_NUM INTEGER No
The number of doses included in the fill. Typically calculated from the fill start and end date, but can be manually changed
25 SCHEDULED_FILL_YN VARCHAR No
Specifies whether this is a scheduled fill.

This column stores the unique ID for the receiving provider's address on the refill request. It is used to identify an address using I SER 21000.

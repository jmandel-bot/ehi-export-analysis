QRY_RECORD_INFO
Description:
The QRY_RECORD_INFO table contains information about the no-add single response data for query records (records in the QRY master file).

Primary Key
Column Name Ordinal Position
QUERY_ID 1

Column Information
Name Type Discontinued?
1 QUERY_ID NUMERIC No
The unique identifier for the query record.
2 EXTERNAL_IDENTIFIER VARCHAR No
The ID received from a third party to uniquely identify the record in their system.
3 STATUS_C_NAME VARCHAR No
The overall status of the query.
May contain organization-specific values: No
Category Entries:
Active
Response Received
Completed
Deleted
Deactivated
4 PAT_ID VARCHAR No
The patient ID the query refers to.
5 PAT_ENC_CSN_ID NUMERIC No
The patient contact (CSN) the query refers to.
6 CREATION_UTC_DTTM DATETIME (UTC) No
The instant the query was created in UTC.
7 UPDATE_UTC_DTTM DATETIME (UTC) No
Stores the instant the query record was updated in UTC.
8 QUERY_WORKFLOW_C_NAME VARCHAR No
The workflow this query is a part of (Computer Assisted Physician Documentation (CAPD), Clinical Documentation Improvement, etc.). This value is used to drive functionality and reporting.
May contain organization-specific values: No
Category Entries:
HB Coding Query
CDI Query
Coding Documentation Deficiency
NoteReader CDI Nudge
PB Coding Query
Pre-Visit Suggestion
Potential Condition Nudge
9 QRY_HIDE_PHYS_YN VARCHAR No
If this is set to 1-Yes, it will prevent physicians from seeing the query in the NoteReader Clinical Documentation Improvement (CDI) activity. If NoteReader CDI is in silent mode, all incoming queries will have this flag set automatically. CDI users can reset this flag by activating the query manually.
May contain organization-specific values: No
Category Entries:
No
Yes
10 RECORD_CREATION_DT DATETIME No
Stores the date the record was created.
11 INSTANT_OF_UPDATE_DTTM DATETIME (Local) No
Stores the instant the record was last locked/unlocked.

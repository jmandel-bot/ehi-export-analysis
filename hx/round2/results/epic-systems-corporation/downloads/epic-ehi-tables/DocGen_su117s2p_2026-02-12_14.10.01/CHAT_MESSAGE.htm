CHAT_MESSAGE
Description:
Table containing Secure Chat message info.

Primary Key
Column Name Ordinal Position
CONVERSATION_ID 1
CONTACT_DATE_REAL 2
LINE 3

Column Information
Name Type Discontinued?
1 CONVERSATION_ID NUMERIC No
The unique identifier (.1 item) for the secure chat conversation record.
2 CONTACT_DATE_REAL FLOAT No
A unique contact date in decimal format. The integer portion of the number indicates the date of contact. The digits after the decimal distinguish different contacts on the same date and are unique for each contact on that date. For example, .00 is the first/only contact, .01 is the second contact, etc.
3 LINE INTEGER No
The line number for the information associated with this contact. Multiple pieces of information can be associated with this contact.
4 INST_SENT_UTC_DTTM DATETIME (UTC) No
The instant the message was sent.
5 MESSAGE_TYPE_C_NAME VARCHAR No
This defines the type of message (for example, a normal message, a user joining the conversation, or a user leaving)
May contain organization-specific values: No

7 SENDER_USER_ID VARCHAR No
The ID of the user that sent the message.
8 SENDER_USER_ID_NAME VARCHAR No
The name of the user record. This name may be hidden.
9 SENDER_MYPT_ID VARCHAR No
The ID of the MyChart user who sent a Secure Chat message.
10 PAT_MESSAGE_ID VARCHAR No
The ID of the MyChart message linked to this Secure Chat message.
11 DESCRIPTION_SS VARCHAR No
Description of the order or saved work linked to this message
12 MESSAGE_IS_EDITED_YN VARCHAR No
Whether the text of this message has been edited or deleted since it was originally sent.
May contain organization-specific values: No

14 SENDER_LOGIN_DEPARTMENT_ID_EXTERNAL_NAME VARCHAR No
The external name of the department record. This is often used in patient correspondence such as reminder letters.

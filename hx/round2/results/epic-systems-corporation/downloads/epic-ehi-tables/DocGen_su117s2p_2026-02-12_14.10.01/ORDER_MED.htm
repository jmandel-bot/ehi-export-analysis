ORDER_MED
Description:
The ORDER_MED table enables you to report on medications ordered in EpicCare (prescriptions). We have also included patient and contact identification information for each record.

Primary Key
Column Name Ordinal Position
ORDER_MED_ID 1

Column Information
Name Type Discontinued?
1 ORDER_MED_ID NUMERIC No
The unique ID of the order record associated with this medication order. This is an internal unique identifier for ORD master file records in this table and cannot be used to link to CLARITY_MEDICATION.
2 PAT_ID VARCHAR No
The unique ID of the patient record for this line. This column is frequently used to link to the PATIENT table.
3 PAT_ENC_DATE_REAL FLOAT No
A unique, internal contact date in decimal format. The integer portion of the number indicates the date of the contact. The digits after the decimal distinguish different contacts on the same date and are unique for each contact on that date. For example, .00 is the first/only contact, .01 is the second contact, etc.
4 PAT_ENC_CSN_ID NUMERIC No
The unique contact serial number (CSN) for the patient contact associated with this medication order. This number is unique across patients and encounters in your system. If you use IntraConnect, this is the Unique Contact Identifier (UCI).
5 ORDERING_DATE DATETIME No
The date when the medication order was placed.
6 ORDER_CLASS_C_NAME VARCHAR No
The category number for the order class. This value is used to define how clinical systems process the order.
May contain organization-specific values: Yes
Category Entries:
Normal
7 PHARMACY_ID NUMERIC No
The unique ID of the pharmacy record that is associated with this medication order. This column is frequently used to link to the RX_PHR table. This field is only populated if the clinical system user selects a specific pharmacy from the list, otherwise the field is null. This field is only populated by the ambulatory clinical system, not the pharmacy system.
8 PHARMACY_ID_PHARMACY_NAME VARCHAR No
The name of the pharmacy.
9 ORD_CREATR_USER_ID VARCHAR No
The EMP ID (.1) of the user who signed the order (for a non-signed and held order) or the last person who performed a sign and hold or release action for a signed and held order.
10 ORD_CREATR_USER_ID_NAME VARCHAR No
The name of the user record. This name may be hidden.
11 MEDICATION_ID_MEDICATION_NAME VARCHAR No
The name of this medication record.
12 DESCRIPTION VARCHAR No
The description of the order. This information is found in the Order field of clinical system’s Order Detail window.
13 DOSAGE VARCHAR No
The dispensation amount for the prescription entered by the user in the orders activity. This amount is stored as a string in the orders database.
14 QUANTITY VARCHAR No
The quantity of the prescription being dispensed as entered by the user.
15 REFILLS VARCHAR No
The number of refills allowed for this prescription as entered by the user.
16 START_DATE DATETIME No
The date when the medication order started. The date appears in calendar format.
17 END_DATE DATETIME No
The date when the medication order is to end.
18 DISP_AS_WRITTEN_YN VARCHAR No
Indicates whether or not the prescription should be dispensed as written for this medication.
May contain organization-specific values: No
Category Entries:
Yes
No
19 RSN_FOR_DISCON_C_NAME VARCHAR No
The category number for the reason a prescription has been discontinued. This column contains data only in prescription orders that have been discontinued.
May contain organization-specific values: Yes
20 MED_PRESC_PROV_ID_PROV_NAME VARCHAR No
The name of the service provider. This item may be hidden in a public view of the CLARITY_SER table.
21 NONFRM_XCPT_CD_C_NAME VARCHAR No
The category number for medication's exception code. This code explains the reason a non-formulary medication was ordered.
May contain organization-specific values: Yes
22 PANEL_MED_ID_MEDICATION_NAME VARCHAR No
The name of this medication record.
23 UPDATE_DATE DATETIME (Local) No
The date and time when this row was created or last updated in Clarity.
24 ORDER_INST DATETIME (Local) No
The date and time the order was placed. The date appears in calendar format.
25 DISPLAY_NAME VARCHAR No
The name of the medication as it appears on the medication record itself.
26 AS_MEDICATION_ID_MEDICATION_NAME VARCHAR No
The name of this medication record.
27 HV_HOSPITALIST_YN VARCHAR No
Indicates whether this is a hospitalist order. A Y indicates a hospitalist order.
The category values for this column were already listed for column: DISP_AS_WRITTEN_YN
28 ORDER_PRIORITY_C_NAME VARCHAR No
The category number for the priority assigned to an order.
May contain organization-specific values: Yes
29 MED_ROUTE_C_NAME VARCHAR No
The category number for the route of administration of a medication.
May contain organization-specific values: Yes
30 DISCON_USER_ID VARCHAR No
The unique ID of the user who discontinued the order.
31 DISCON_USER_ID_NAME VARCHAR No
The name of the user record. This name may be hidden.
32 DISCON_TIME DATETIME (UTC) No
The date and time when the medication order was discontinued. The date appears in calendar format.
33 CHNG_ORDER_MED_ID NUMERIC No
The unique ID of the changed or reordered medication order that this order replaced. This column is frequently used to link back to the ORDER_MED table.
34 PEND_APPR_USER_ID VARCHAR No
The unique ID of the user who approved a pended order.
35 PEND_APPR_USER_ID_NAME VARCHAR No
The name of the user record. This name may be hidden.
36 PEND_REF_REAS_C_NAME VARCHAR No
The category number for the reason a pended medication was refused.
May contain organization-specific values: Yes
37 HV_DISCR_FREQ_ID VARCHAR No
The unique ID of the discrete frequency record associated with this medication order. This column is frequently used to link to the IP_FREQUENCY table.
38 HV_DISCR_FREQ_ID_FREQ_NAME VARCHAR No
The name of the frequency record.
39 HV_DISCRETE_DOSE VARCHAR No
The discrete dose for a medication as entered by the user in the orders activity.
40 HV_DOSE_UNIT_C_NAME VARCHAR No
The category number for the dosage unit of a medication.
May contain organization-specific values: Yes

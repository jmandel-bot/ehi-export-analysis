PERSON_ID NUMBER N This is the value of the unique primary identifier of the person table. It is an internal system assigned number.
PRIORITY_CD NUMBER N The Priority_Cd identifies the priority of the order.
PROTOCOL_REQ_IND NUMBER Y Indicates a protocol is required for this exam. Null indicates the order existed prior to protocol functionality being available in Millennium.

RAD_PACS_ID NUMBER N This field is used as a unique identifier for PACS interfaces.
REASON_FOR_EXAM VC2(4000) Y The Reason_For_Exam field contains a textual reason of why the procedure was performed.
REFER_LOC_CD NUMBER N The Refer_Loc_Cd identifies where the order was referred from.
REMOVED_BY_ID NUMBER N If the order was removed after completion, the Removed_By_Id contains the Id of the person performing the remove.
REMOVED_CD NUMBER N *** OBSOLETE ***
REMOVED_DT_TM DATE Y If the order was removed after completion, the Removed_Cd Identifies why the exam was removed.
REPLACED_ORDER_ID NUMBER N This column, when valued, will reference the order that was cancelled as a part of a replace transaction.
REPORTING_PRIORITY_CD NUMBER N This column contains the code value to allow for either ROUTINE or STAT priority. The STAT priority alerts the radiology tech to focus on first.
REPORT_STATUS_CD NUMBER N The Report_Status_Cd identifies the current status of the report that a part of this order.
REQUESTED_TZ NUMBER Y Time zone associated with the corresponding DT_TM column.
REQUEST_DT_TM DATE Y The Request_Dt_Tm field contains the date and time that the order was requested to be performed.
RESTORED_IND NUMBER Y The Restored_Ind indicates whether or not this order was restored from the repository for the purpose of changing the report.
SEQ_EXAM_ID NUMBER N The Seq_Exam_Id is a foreign key to the Exam_Data table. It serves as a link to the exam info within the exam management area.
START_DT_TM DATE Y The Start_Dt_tm contains the date and time that the exam was started. This is used for turnaround times.
START_TZ NUMBER Y Time zone associated with the corresponding DT_TM column.
TRANS_WORKGROUP_CD NUMBER N The Trans_Workgroup_Cd identifies the transcription workgroup that the report will be loaded to.

UPDT_ID NUMBER N The person_id of the person from the personnel table (prsnl) that caused the last insert or update of the row in the table. The UPDT family of columns are typically used for housekeeping and external system process and should never be depended on for solution specific logic.
UPDT_TASK NUMBER N The registered (assigned) task number for the process that inserted or updated the row. The UPDT family of columns are typically used for housekeeping and external system process and should never be depended on for solution specific logic.
VETTING_DT_TM DATE Y This column will contain the date and time that the exam was vetted.
VETTING_PRSNL_ID NUMBER N This column will contain the id of the personnel that vetted the exam.
VETTING_STATUS_FLAG NUMBER N This is the vetting status of the exam.Valid values for this column are: 0 - Not Vetted; 1- Accepted; 2- Rejected; 3- Replaced; 4 - In Process

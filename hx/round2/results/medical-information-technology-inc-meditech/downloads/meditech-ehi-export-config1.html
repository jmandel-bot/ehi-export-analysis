Export Zip Overview

The export zip contains files to help a consumer understand and navigate the folder tree and files contained within.

File

Description


README.txt

 Contains a description of the export zip file, including
  • Table of Contents for documents within various sub folders
    • anything not in parentheses corresponds to a folder or file name found after extracting the zip file
    • anything in parentheses is additional descriptive data
EHIEXPORTSCHEMA.txt Contains Documentation URL (directs to this page) and Product Version

ACCOUNTS_INDEX.html

or

ACCOUNTS_INDEX.xml


Enables human readability of parts of the export package via a web browser


Table of Contents.ndjson

 Contains metadata for every data file in the export zip to aid in machine readability
  • Format is ndjson
  • DocumentReference resources conform to the EHI Export API Implementation Guide (draft).
  • More details below
Note: All MEDITECH product versions contain at a minimum the files listed above.  Some product versions may contain extra files inside of a ZCOMMON folder used to aid in human readability.

Section Information

Each section exists as a folder or file within the export zip.  This table provides details about what is contained within each section.

Section

Details

Electronic Chart

Folder: AANNNNNNNNNN_Patient_Name - the folder name represents the account number concatenated with the patient name. 

  • Folder: NNNN_Category - the folder name includes an internal sort value (NNNN) and is concatenated with the category name. It contains folders representing subcategories defined by the organization
    • Folder: NNNN_Subcategory - the folder name includes an internal sort (NNNN) and is concatenated with the subcategory name. It contains one or more files representing one or more documents
      • Within the folders are documents associated with the account within each category/subcategory. File formats may include images (png, jpg, tif, bmp) or pdf files

Folder: Record_Documents_Patient_Name - the folder name represents patient level (not account specific) concatenated with the patient name.

  • Folder: NNNN_Category
    • Folder: NNNN_Subcategory
      • Within the folders are documents associated with the account within each category/subcategory. File formats may include images (png, jpg, tif, bmp) or pdf files

Folder: Global_Documents_Patient_Name - the folder name indicates global documents (apply across all encounters) and is concatenated with the patient name

  • Folder: NNNN_Category
    • Folder: NNNN_Subcategory
      • Within the folders are documents associated with the account within each category/subcategory. File formats may include images (png, jpg, tif, bmp) or pdf files

Ambulatory Results

Folder: Ambulatory Results

  • Ambulatory (Manual Entry) Results.pdf

    • Contains Ambulatory results entered manually for micro and lab

  • Ambulatory OV Results.pdf

    • Contains Ambulatory results coming from an outside/other vendor for micro and lab

Authorization & Referral

Management Reports

Folder: ARMEHI

  • ARMEHI-ArmAuthData.pdf

    • Contains data of all authorizations and referrals for a particular patient

  • ARMEHI-InsEligCopayDeductData.pdf

    • Contains insurance eligibility and deduction data

Financial Reports

Folder: Financial EHI

  • File: FinancialEHI.txt

    • Contains a Patient Accounting report that includes financial transactions for the patient

  • File: ResidentTrustEHI.txt

    • Contains a Resident Trust report that contains financial transaction for the patient (Expanse, 6.1x only)

  • File: CostEstimation.txt

    • Contains cost estimates for the patient (Expanse only)

FHIR Resource Bundle

File: US Core FHIR Resources.json

  • Contains all available FHIR resources for the exported patient leveraging Patient $everything

    • FHIR R4: US Core STU 3.1.1 Implementation Guide

    • More information at fhir.meditech.com

Historical Ambulatory Data

 Folder: Historical Ambulatory Data
  • Within the folder are document files or folders associated with the account(s) and/or record named by document title concatenated with a unique record identifier. File formats may include images (png, jpg, tif, bmp), text, or pdf files
  • Contains historical ambulatory documentation migrated from a previous version of the software

Immunization History

 Folder: Patient Immunization History
  • File: Patient Immunization History.pdf
    • Contains a list the patient's immunization history

Immunizations

 Folder: Immunization Record
  • File: Immunization Record.txt
    • Contains patient immunization data

Implantable Devices

 Folder: Implantable Devices Data
  • File: Implantable Devices Data.txt
    • Contains patient medical/implantable devices

Patient Notices

 Folder: Patient Notices
  • File: Patient Notices.pdf
    • Contains patient and physician interactive message

Population Health

 Folder: Population Health
  • File: Population Health.pdf
    • Contains population health external data (aggregated data received from a vendor) 

Provider Messages

 Folder: Provider Messages
  • File: Provider Messages.txt
    • Contains patient and physician interactive message

Structured Clinical Documents

Folder: CCDA

  • Contains all structured Consolidated-CDA documents that have been created and possibly sent outside of the organization. The document format will match one of the following releases.

    • C-CDA R2.1: C-CDA Templates for Clinical Notes R2.1 Companion Guide, Release 2

    • C-CDA R1.1: HL7 Implementation Guide: S&I Framework Transitions of Care Companion Guide to Consolidated-CDA for Meaningful Use Stage 2, Release 1 – US Realm

Utilization Review

 Folder: Utilization Review
  • File: Utilization Review.pdf
    • Contains case management utilization reviews
Note: Some folders may contain a CHART.xml file which is leveraged when rendering parts of the chart for human readability.


Table of Contents.ndjson Schema

The ndjson file contains an array of file metadata (FHIR Document Reference resources) for all of the patient data files contained within the export zip file.  The pseudo-JSON below explains what each field represents.

{
   "resourceType":"DocumentReference",
   "id": <<Identifier of this DocumentReference>>,
   "meta": {
      "id": <<Identifier of this metadata>>,
      "versionId": <<Identifier of this version>>,
      "lastUpdated": <<When this was last updated>>,
      "source": <<Where the resource came from>>,
      "profile": [
         "http://fhir.org/argonaut/ehi-api/StructureDefinition/ehi-document-reference"
      ],
      "tag": [
         {
            "code": "ehi-export",
            "display": "generated as part of an ehi-export request"
         }
      ]
   },
   "implicitRules": <<A set of rules under which this content was created>>,
   "language": <<Language of the resource content>>,
   "status": "current",
   "docStatus": <<Status options - preliminary | final | amended | entered-in-error>>,
   "type": <<CodeableConcept indicating kind of document>>,
   "subject": {
      "reference": <<Patient Reference, resource is available in the US Core FHIR Resources.json>>
   },
   "date": <<Date the file was created>>,
   "description": <<File description>>,
   "content": [
      {
         "attachment": {
            "id": <<Identifier of this attachment>>,
            "contentType": <<File type>>,
            "url": <<Pathway to the location of the file>>,
            "size": <<File size>>,
            "title": <<Title>>,
            "creation": <<Date the file was created>>
         }
      }
   ],
   "context": {
      "id": <<Identifier of this context>>,
      "encounter": [
         {
            "reference": <<Encounter Reference, resource is available in the US Core FHIR Resources.json>>
         }
      ],
      "period": {
         "start": <<Beginning of the context period>>,
         "end": <<End of the context period>>
      }
   }
}

ESCALATION_EVENTS
Description:
Audit events for the escalation.

Primary Key
Column Name Ordinal Position
ESCALATION_ID 1
LINE 2

Column Information
Name Type Discontinued?
1 ESCALATION_ID NUMERIC No
The unique identifier (.1 item) for the subject name record.
2 LINE INTEGER No
The line number for the information associated with this record. Multiple pieces of information can be associated with this record.
3 CHANGE_LOCAL_DTTM DATETIME (Attached) No
Instant there was a change event for the escalation.
4 CHANGE_LEVEL INTEGER No
Level in the path the change event occurred for the escalation.
5 ESC_CHANGE_TYPE_C_NAME VARCHAR No
Change type for the escalation.
May contain organization-specific values: No
Category Entries:
Automatically Advanced
Manually Advanced
Request Failed
Completed
Start Request
Take Responsibility
Remove Responsibility
Undo Completed
Cancel Request
Reminder Sent
User Declined
Added Participants
Removed Participants
No Available Users Advancement
6 CHANGE_USER_ID VARCHAR No
EMP ID of recipient who changed escalation.
7 CHANGE_USER_ID_NAME VARCHAR No
The name of the user record. This name may be hidden.
8 AFFECTED_PROVIDERTEAM_ID NUMERIC No
PCT ID of a provider team affected by an escalation change.
9 AFFECTED_PROVIDERTEAM_ID_RECORD_NAME VARCHAR No
The name of the record.
10 AFFECTED_CALL_ORDER INTEGER No
The call order of a PCT affected by an escalation event.
11 AFF_ROLE_TRTMNT_TEAM_REL_C_NAME VARCHAR No
The role in the role hierarchy of a PCT affected by an escalation event.
May contain organization-specific values: Yes
12 CHANGE_UTC_DTTM DATETIME (UTC) No
UTC instant there was a change event for the escalation.
13 USER_LOGIN_DEPT_ID_EXTERNAL_NAME VARCHAR No
The external name of the department record. This is often used in patient correspondence such as reminder letters.
14 USER_WORKSTATION_ID_WORKSTATION_NAME VARCHAR No
This is the internal workstation name.

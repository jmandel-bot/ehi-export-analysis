AUTH_REQUEST_HX_2
Description:
This table contains a snapshot of information about an individual authorization request as of a point in time.

Primary Key
Column Name Ordinal Position
AUTH_REQUEST_ID 1
CONTACT_DATE_REAL 2

Column Information
Name Type Discontinued?
1 AUTH_REQUEST_ID NUMERIC No
The unique identifier (.1 item) for the authorization request record.
2 CONTACT_DATE_REAL FLOAT No
A unique contact date in decimal format. The integer portion of the number indicates the date of contact. The digits after the decimal distinguish different contacts on the same date and are unique for each contact on that date. For example, .00 is the first/only contact, .01 is the second contact, etc.
3 START_DATE DATETIME No
The date from which this authorization request is effective.
4 EXPIRATION_DATE DATETIME No
The date after which the authorization request is no longer valid. The system will re-calculate this date as changes are made to the authorization unless it was entered by the user.
5 EXPIRATION_USER_SET_DATE DATETIME No
The user-entered date after which the authorization request is no longer valid.
6 SUBMIT_CLM_BY_DATE DATETIME No
The date by which claims must be submitted to be paid. The system will re-calculate this date as changes are made to the authorization unless it was entered by the user.
7 SUBMIT_CLM_BY_USER_SET_DATE DATETIME No
The user-entered date by which claims must be submitted to be paid.
8 AUTH_SERVICE_TYPE_C_NAME VARCHAR No
The service type of the authorization request. This is used after the UM service type is ready for use and the authorization workspace is enabled.
May contain organization-specific values: Yes
No Entries Defined
9 AUTH_CARE_SETTING_C_NAME VARCHAR No
The care setting of the authorization request. This item groups service types and drives the authorization workflow.
May contain organization-specific values: No
Category Entries:
Inpatient
Outpatient
Home Health
Hospice
10 FINALIZED_EDIT_REASON_C_NAME VARCHAR No
The finalized edit reason category ID for the authorization request
May contain organization-specific values: Yes
Category Entries:
Correction of Compliance Documentation
Correction of Request Documentation
Correction of Decision Documentation
11 IS_FINALIZED_EDIT_YN VARCHAR No
Indicates whether this contact was created during finalized edit for this authorization request. 'Y' indicates that this contact was created during finalized edit. 'N' or NULL indicate that this contact was not created during a finalized edit.
May contain organization-specific values: No
Category Entries:
Yes
No
12 EXTCVG_PAYER_NAME VARCHAR No
The external coverage's payer name.
13 EXTCVG_PLAN_NAME VARCHAR No
The external coverage's plan name.
14 EXTCVG_MEM_NUM VARCHAR No
The external coverage's member number.
15 EXTCVG_STAT_C_NAME VARCHAR No
The external coverage's status
May contain organization-specific values: No
Category Entries:
Active
Cancelled
Draft
Entered-in-error
16 EXTCVG_FROM_DATE DATETIME No
The external coverage's effective from date.
17 EXTCVG_TO_DATE DATETIME No
The external coverage's effective to date.

Centriq Overview

The Centriq Electronic Health Information (EHI) Export feature allows users to generate a data extract containing all Protected Health Information (PHI) stored in Centriq for a single patient or a population of patients. The export generates three main categories of data:

  • Consolidated CDA – The set of clinical data for a specified encounter or visit.
  • Fast Healthcare Interoperability Resource (FHIR) bundle – A larger set of clinical data collected over the history of the patient.
  • Financial data – Patient billing and claim information, in pipe-delimited format.

The export creates a.zip file containing the following files. These files house the data listed above, as well as some supporting data.

File Description
Read_Me.TXT A text file containing basic information about the .zip file, as well as demographic information about the patient, the healthcare organization responsible for the PHI, and the software information responsible for generating the extract.
xxxxx.XML The Consolidated-CDA document, in XML format. This structured document contains the patient’s demographic details and any clinical information collected during the patient’s visit.
xxxxx.JSON The Fast Healthcare Interoperability Resource (FHIR) bundle in JavaScript Object Notation (JSON) format. This structured document also contains demographic and clinical data on the patient, but it contains additional clinical data not supported by the Consolidated-CDA document.
xxxxx.csv The Fast Healthcare Interoperability Resource (FHIR) bundle in JavaScript Object Notation (JSON) format. This structured document also contains demographic and clinical data on the patient, but it contains additional clinical data not supported by the Consolidated-CDA document.
xxxxx.csv A set of pipe delimited files containing financial data. Several CSV files are included, with file names in this format: MR# Encounter#_ DataType.csv (for example, 184654-0001_BillHistory.csv).
USCDI_v1.CSS The Cascading Style Sheet (CSS) used to help display information from the Consolidated-CDA document.
USCDI_V1.XSLT An Extensible Stylesheet Language Transformations (XSLT) file that, along with the CSS file and Public folder, is used to help display information from the Consolidated-CDA document.
Public (folder) A folder containing images to help with displaying information from the Consolidated-CDA document.

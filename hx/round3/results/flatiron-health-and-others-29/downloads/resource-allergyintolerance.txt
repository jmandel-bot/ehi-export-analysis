Resource: AllergyIntolerance
FHIR® [AllergyIntolerance] Resource

This page provides technical documentation specific to Flatiron’s deployment of the FHIR® [AllergyIntolerance] Resource. Flatiron’s development of this API includes and builds upon the required components outlined within the Health Level 7® (HL7) FHIR® standard. Within the Profiles Supported row in the table below, you can find a link to the relevant HL7® FHIR® standard profile(s). All Flatiron supported API resources are based on FHIR® version 4.0.1 as required by the (g)(10) Standardized API for Patient and Population Services certification criteria. This page will be updated periodically as Flatiron expands our FHIR® API integration options.

[AllergyIntolerance] Profile - Supported Data Elements and Value Sets

Flatiron supports the data elements and search parameters outlined within the US Core Profiles (US Core Implementation Guide STU 6.1.0). US Core Profiles outline specific constraints applied to the AllergyIntolerance Resource for a given use case. 

All data elements listed as MANDATORY or MUST SUPPORT within the US Core Implementation Guide are returned if valued when the AllergyIntolerance Resource is called. No additional data elements are included within Flatiron’s AllergyIntolerance Resource beyond MANDATORY and MUST SUPPORT data elements.

All value sets or terminology code systems listed as REQUIRED are applied within Flatiron’s AllergyIntolerance API.

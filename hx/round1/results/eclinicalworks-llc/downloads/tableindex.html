EHI Export Schema
eClinicalWorks is certified to the Electronic Health Information (EHI) Export criterion at 45 CFR ยง170.315(b)(10). This means that eClinicalWorks enables a defined set of users to create an export of all EHI stored in the eClinicalWorks EHR for a single patient, or to initiate a request for export of all EHI stored in the eClinicalWorks EHR for an entire patient population. For more details, refer to eClinicalWorks' Mandatory Disclosures available at www.eclinicalworks.com/resources/certified-ehr-technology.

The documentation below lists the database tables that fall within the scope of electronic health information (EHI). Each page contains a description of the table and its purpose, the names of the columns within each table, the description/purpose of the column, its data type, and any primary key (if applicable). Scanned documents are included in a separate folder apart from the database tables.

Any datetime value listed as '1900-01-01' or '0000-00-00' should be interpreted as a default value for the database when the field is blank within the database table and not indicative of any clinical meaning.

The EHI export may contain proprietary codes not owned by eClinicalWorks. This EHI extract of data includes 3rd party data, such as CPT codes. By downloading this data, you agree that the dictionaries housed within this extract will not be used for commercial or other unauthorized purposes. Due to the presence of sensitive data, such as SSN, the data extract must be carefully reviewed and excluded before sharing it with individuals outside of the practice.

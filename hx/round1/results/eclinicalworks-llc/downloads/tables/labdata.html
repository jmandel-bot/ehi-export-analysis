EHI Export Schema
Table labdata
Description This table stores information about laboratory test reports, including test results, statuses, and other related data.
Column DataType Description
Reportid (Primary Key) int This column stores the auto-incremented primary key.
Encounterid int This column stores the ID of the encounter for which a lab is added. This column refers to the encounterid column from the enc table. If the lab added is for a future date, then this column holds the newly created encounterid (i.e., a new encounter is created for the future dated lab and the existing encounterid on which the future dated lab is added is documented in the ordencounterid column in the same table).
Itemid int This column stores the ID of the item/lab added for the patient. This column refers to the itemid column from the items table.
reason varchar This column stores the reason for which a lab test is ordered or received, selected from the drop-down list by the end user. This column refers to the itemName column from the items table.
received int This column stores an integer that indicates whether the lab result has been received (0 for not received, 1 for received).
result varchar This column stores the result of the lab test received, selected from the drop-down list by the end user. These column refers to the itemName column from the items table.
ResultDate datetime This column stores the date on which the lab results were received.
Notes varchar This column stores the free-text or selected pre-configured keywords as notes added by the end user for that particular lab of the patient.
status int This column stores the values 0 and 1, where 0 indicates an Open Order Status and 1 indicates a Reviewed Order Status.
Addendum text This column stores notes documented by the end user after lab/DI orders are marked as reviewed.
priority int This column stores values 0 and 1, where 1 indicates the lab order was a high-priority test and 0 indicates the lab order was not high-priority.
assignedToid int This column stores the identifier for the staff member assigned to the lab report (e.g., 10001, 10002, 10003).
deleteFlag int This column stores the value 0 or 1, which indicates whether the lab report has been deleted (0 for not deleted, 1 for deleted).
vmid varchar Column not in use
Type smallint This column stores the value 0 to indicate the test is a lab, 1 to indicate the test is diagnostic imaging, and 3 to indicate the test is a procedure.
Billable smallint This column stores the value 0 or 1, where 1 indicates the lab is in-house and 0 indicates the lab is not in-house.
displayIndex int This column includes the display order of the lab test in the user interface (e.g., 1, 2, 3).
fromLabid int This column stores an ID of the lab interface from which the lab results are received. This column refers to the id column from the lablist table. The value 0 is stored when a result is not received from any electronic interface.
collDate datetime This column stores the date when the sample was collected (e.g., 2021-08-31).
collTime datetime This column stores the time when the sample was collected (e.g., 14:30:00).
tolabid int This column stores an ID of the lab interface to which the lab order is transmitted. This column refers to the id column from the lablist table. The value 0 displays when the lab order is not transmitted through an electronic interface.
billClient smallint This column stores a value that indicates whether the client should be billed for the lab test (0 for no, 1 for yes).
collSource varchar This column stores the source of the test specimen collected, selected from the drop-down list by the end user. This column refers to the itemName column from the items table.
collVolume varchar This column stores the volume of the test specimen collected.
collUnits varchar This column stores the measurement unit of the volume of the test specimen collected (e.g., mL, L, Bottles).
clinicalinfo nvarchar This column stores clinical information about the test ordered.
fasting smallint This column stores values 0 and 1, where 1 indicates that the order required the patient to fast and 0 indicates that the patient was not required to fast.
billType char This column stores the flags C and N, where C indicates that an order has to be billed to the client and N indicates that the order does not have to be billed to the client.
Pregid int This column stores the pregnancy ID of the patient for whom the lab order is placed. This column refers to the Pregid column from the obf_pregnancy table.
OBFLabType smallint This column stores the type of lab test for obstetrics and gynecology (e.g., 0 for routine, 1 for specialized).
publishToPortal int This column stores a value that indicates whether the lab report should be published to the eClinicalWorks Patient Portal (0 for no, 1 for yes).
commonNotesFlag char This column stores a value that indicates whether common notes are used for the lab report (e.g., N for no, Y for yes).
futureflag int This column stores a values that indicates whether the lab test is scheduled for a future date (0 for no, 1 for yes).
prevencounterid int This column stores the encounterid as the previous encounterid for which the future dated order is moved to today's date. This column refers to the encounterid column from the enc table.
ordencounterid int If the order is future dated order, this column stores the ID of the encounter for which the order is created. This column refers to the encounterid column from the enc table.
FutureOrderDate datetime This column stores the date when the lab test is scheduled for a future order (e.g., 2021-10-01).
ReviewedBy int This column stores the ID of the end user who reviews the received result for the ordered test. This column refers to the UID column from the user table.
ReviewedDate datetime This column stores the date when the lab report was reviewed (e.g., 2021-09-02).
ReviewedTime datetime This column stores the time when the lab report was reviewed (e.g., 15:30:00).
transDate datetime This column stores the date when the lab report was transferred (e.g., 2021-09-03).
transTime datetime This column stores the time when the lab report was transferred (e.g., 16:30:00).
profileid varchar This column stores the vendor sent identifier string for samples in the result received.
IntNotes text This column stores the free-text or selected pre-configured keywords as internal notes which could be used to communicate internally, e.g., with the nurse, MA or providers.
HMFlag smallint This column stores the flag 0 or 1, where 1 indicates when the alert is suppressed when a lab has already been performed for a patient outside of this practice, and 0 indicates otherwise.
ccresulttolist varchar This column stores a list of the last and first names of licensed providers and referring providers selected while the lab is ordered.
ccResultToListids varchar This column stores a list of the provider IDs from the ccResultToList column.
HasACopy int This column stores the flag 0 or 1, where 1 indicates the result is copied to other providers apart from the provider who has placed the order, and 0 indicates otherwise.
ResultStatus varchar This column stores the status of the lab result received from the vendor, where P indicates preliminary result, F indicates final result, and C indicates corrected result.
collDescription varchar This column stores the description about the specimen to be collected for the placed order.
ordPhyName varchar This column stores the last and first name of the person who ordered the lab.
ordPhyid int This column stores the identifier for the ordering physician (e.g., 8001, 8002, 8003).
refPhyFlag int This column stores the flag 0 or 1, where 0 indicates when the lab is ordered by the licensed provider and 1 indicates when the lab is ordered by referring provider/physician.
labrefid varchar This column stores the reference identifier for the lab test (e.g., LR12345).
resultime datetime This column stores the time when the lab result was obtained (e.g., 17:30:00).
PSEHold int This column stores a flag 0 or 1, where 1 indicates when interface transmission is allowed for futue orders and the user selects to hold the order at the LIS/vendor end and 0 indicates otherwise.
emailSent tinyint This column stores a flag that indicates whether an email was sent for the lab report (0 for no, 1 for yes).
dtEmailSent datetime This column stores the date when the email was sent for the lab report (e.g., 2021-09-04).
ActualFasting tinyint This column stores fasting information of the patient, where 0 inidcates fasting, 1 inidcates not fasting, and 2 inidcates not recorded.
OrderInstructions text This column stores the instructions for the lab test order.
performedby varchar This column stores the name of the person who performed the lab test (e.g., Jane Doe).
hl7id1 varchar This column stores the first HL7 identifier for the lab test (e.g., HL70123).
hl7id2 varchar This column stores the second HL7 identifier for the lab test (e.g., HL70234).
hl7id3 varchar This column stores the third HL7 identifier for the lab test (e.g., HL70345).
newcol int Column not in use
cancelled tinyint This column stores a flag that indicates whether the lab test was canceled (0 for not canceled, 1 for canceled).
resultSentStatus tinyint This column stores a flag that indicates whether the lab result was sent (0 for not sent, 1 for sent).
approvalstatus tinyint This column stores a flag indicating the approval status of an order, where 0 indicates no approval required, 1 inidcates approval required, 2 inidcates approved, and 3 inidcates rejected.
SendVmsg tinyint This column stores a flag 0 or 1, where 1 inidcates when a message is sent for a lab and 0 inidcates otherwise.
websync tinyint This column stores a flag that indicates whether the lab report was synchronized with the web portal (0 for no, 1 for yes).
patientidentifier varchar This column stores patient details that are received in the HL7 message.
portalViewTime datetime This column stores the date and time when the lab report was viewed on the eClinicalWorks Patient Portal (e.g., 2021-09-05 18:30:00).

EHI Export Schema
Table speciality_form_type
Description This table stores the JSP entries for Specially Forms and Smart Forma along with the configuration information regarding both types of forms.
Column DataType Description
id (Primary Key) int This is a primary key which stores an id of a JSP entry with the help of which Specialty Forms/Smart forms are configured.
name varchar This column stores the name of the Specialty Form/Smart Form.
form_url varchar This column stores the actual URL of the JSP . This URL gets called when User opens up Specialty Form/Smart form during Patient's encounter.
form_type varchar This column stores the category name under which Specialty Form name should be visible. For Smart Forms, the value would always be Smart Forms.
isPracticeForm tinyint This column stores a value based upon which Smart Forms are visible on the UI.
code varchar A short code representing the speciality form type (e.g., "A", "B").
isSmartForm tinyint This column stores a value which indicates if a form entry belongs to Smart Form.
delflag tinyint This column stores a value which indicates if a Specialty Form/Smart form is Deleted or Not Deleted. 1: Deleted;0 : Not Deleted.
parentid int This column stores an id of a parent Specialty Form who has child forms. For example, Medical Intake, Detox etc.
isTabGroup tinyint This column stores a value which indicates if a specific Specialty Form is a Parent form. 1 : Parent Form 0:Child Form
displayIndex int This column stores a value according to which Child Form should be arranged or visible
sfid int The identifier of the related smart form, if any (e.g., 1, 2). If there is no related smart form, the value will be NULL.
zoomval int This column stores the custom zoom value fro Speciality Form/Smart Form. It ensures proper format of a form while the form is being sent , i.e., it is used by a framework to ensure that the forms are not getting cut while faxing.
includeIncFile varchar A flag indicating to include patientinfo.inc file in the form , if the file is supposed to be included then the value is "yes" otherwise "no".

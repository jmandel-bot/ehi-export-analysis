Table derm_biopsy
Description This table stores information about dermatological biopsies, i.e., differential diagnosis, collection method and description, including details about the final diagnosis, action and user.
Column DataType Description
id (Primary Key) int This is auto-incremented primary key in This column increase with each pathology log i.e. biopsy procedure with pathology lab. (Example: 1, 2, 3)
treatmentdetailid int This column stores id for each biopsy procedure treatment order placed by user. This refers to column treatmentdetailid from table derm_treatmentdetails.
userid int This stores user id for provider who has created or modified biopsy. It refers to UID field in user table.
differentialdiagnosis varchar This column stores differential diagnosis of biopsy taken for pathology lab. It gets its value from value column from table derm_structprocedures with respect to item value configured in itemkey "DermBiopsyFields".
methodofcollection varchar This column stores method of collection of biopsy taken for pathology lab. It gets its value from value column from table derm_structprocedures with respect to item value configured in itemkey "DermBiopsyFields".
description text This column stores description of biopsy taken for pathology lab. It gets its value from value column from table derm_structprocedures with respect to item value configured in itemkey "DermBiopsyFields".
assessmentid int This column stores assessment id associated to procedure ordered for this biopsy. It refers to column itemid from table items.
assessmentcode varchar This column stores assessment code associated with procedure ordered for this biopsy.
notes text A text field containing any additional notes or comments about the biopsy. (Example: "Patient experienced minimal bleeding during the procedure.")
isaddressed int An integer indicating whether the biopsy has been addressed (1) or not (0). (Example: 0, 1)
actionid int This column stores action id added for this biopsy. This column refers column id from table lsm_actions.

Table bh_signature
Description This table stores information about signatures, including the user who signed, the section they signed, and the time the signature was taken.
Column DataType Description
id (Primary Key) int Unique identifier for each signature entry. (Example: 1, 2, 3)
uniqueid int Unique identifier for each unique signature. (Example: 101, 102, 103)
sectionid int identifier for the section that the signature is associated with. Foreign key relation: `log_section.id`. (Example: 5, 6, 7)
signUserid int identifier for the user who signed the section. (Example: 1001, 1002, 1003)
signUserName varchar Name of the user who signed the section. (Example: "John Doe", "Jane Smith", "Alice Johnson")
degreeCredential varchar Degree or credential of the user who signed the section. (Example: "PhD", "MBA", "MD")
ftpDirPath varchar File path to the directory where the signature file is stored. (Example: "/path/to/signature/files/")
signTakenTime datetime Date and time when the signature was taken. (Example: "2022-01-01 12:34:56")
isAdhoc tinyint Flag indicating whether the signature is ad-hoc (1) or not (0). (Example: 0, 1)
capturedBy int identifier for the user who captured the signature. Foreign key relation: `users.UID`. (Example: 2001, 2002, 2003)
deleteFlag tinyint Flag indicating whether the signature is deleted (1) or not (0). (Example: 0, 1)
fileName varchar Name of the signature file. (Example: "signature1.png", "signature2.jpg", "signature3.svg")

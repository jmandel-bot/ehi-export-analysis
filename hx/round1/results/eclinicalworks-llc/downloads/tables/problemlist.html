EHI Export Schema
Table problemlist
Description This table stores information about patients' problems, including details about the problem, its status, and related medical codes.
Column DataType Description
patientid int This column stores patient identifier for a patient, whom the problem list item is associated to. It refers to column UID from user table.
encounterid int Stores encounter id which the problem list item is associated to. Column refers to encounterid column in enc table.
asmtid int This column stores assessment id of the added problem list item. This column refers the column itemid from items table.
specify varchar This column stores text about the concise specification of the problem list item added. Pre defined keywords can also be selected as a specification of the problem list item in the user interface.
notes text This column stores the notes entered by the end user pertaining to the problem list item added. Pre defined keywords can also be selected as notes in the User Interface.
deleteFlag tinyint This column stores value 0 / 1 indicating if the added problem list item is marked deleted using the User Interface, this is soft delete. 0 - Not Deleted , 1 -Deleted
onsetdate varchar This column stores the onset date of the problem added. The format used is MM/DD/YYYY.
userid int This stores the id of the end user who adds/ modifies the the problem list item. This id could be Provider's or a Staff. This column refers to column UID from users.
logdate varchar This column is to store the date on which the problem list is modified. The format used is MM/DD/YYYY.
inactiveFlag smallint This column stores flag 0 or 1. 1 - when clinical status is set to inactive , 0 - Otherwise.
WUStatus varchar This column is to store one of the work-up status of the problem. confirmed, w/u pending, w/u negative, erroneous Dx, unknown or differential.
condition varchar This column stores one of the mentioned clinical statuses for the added problem list item: active, well-controlled, stable, poorly-controlled, resolved, inactive, recurrence, remission.
SlNo (Primary Key) int This column is an auto incremented Primary Key of This column .
displayIndex int This column stores the sequence number of the problem list item displayed in the User Interface.
Risk varchar This column is to store the risk level of the added problem: high, medium or low.
AddedDate varchar This column stores the date the problem list item was added.
problemtype varchar This column stores one of the below values as the type of the problem: Finding of Functional Performance Activity, Cognitive Function Finding, and Complaint.
Resolved tinyint This column stores a flag 0 or 1, 1 - when the clinical status is set to resolved. 0 - Otherwise.
resolvedon varchar This column stores the date on which the clinical status is set to resolved for the problem added.
SNOMED varchar This column stores the SNOMED code of the item (problem) added.
SNOMEDDesc varchar This column stores the description of the SNOMED code from snomed column.
SNOMEDDescid varchar This column stores an id of the SNOMED description. This column refers itemid column from items table.
synonymid int An integer representing the unique identifier of the ICD synonym related to the problem. This column has a foreign key relation with the `id` column in the `icd_synonyms` table. Example: 9876

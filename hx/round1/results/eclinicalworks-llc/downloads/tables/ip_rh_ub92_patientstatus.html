EHI Export Schema
Table ip_rh_ub92_patientstatus
Description This table stores the details of patient status.
Column DataType Description
ID (Primary Key) int Auto increment primary key of the table. Data type defined as: INT with size 11
Code varchar This column stores code for the patient's status.
Name varchar This column stores name for the patient's status
delflag tinyint This field stores delete flag of the column.
status tinyint This column stores status for the patient dictionary shipped by ecw.
modiBy int This column stores the id of a user by whom disposition dict data is being modified.modiBy is reference column storing uid field value of user's table.
modiTime datetime This field stores the timestamp of block being created or edited.
ID (Primary Key) int This is an auto-incremented primary key column in this table. It is used to uniquely identify the patient status.The data type is int as per business requirement and the column cannot be NULL.
Code varchar This column stores the code for the patient status.
Name varchar This column stores the name of the patient status code.
Delflag tinyint A flag to indicate if the patient status code is deleted(1) or not(0). The data type is tinyint and the default value is 0. The column cannot be NULL.
Status tinyint This column is a legacy column used in the code which belonged to Acute care project in the past. So, although there is no concrete usage available for this column, we have still added it to avoid any functionality break due to removal from code.
modiBy int Stores the user Id of the person who last modified the patient status. It refers to uid column in users table. Datatype is int. The default value for this column is NULL.
modiTime datetime Stores the date and time the patient status was last modified. The datetime data type allows for precise storage of date and time values. The default value is NULL.

EHI Export Schema
Table vitalslog
Description This table stores a log of vital sign measurements taken during patient encounters, including the values, actions taken, and exceptions.
Column DataType Description
id (Primary Key) int Auto-incremented primary key in This column .
encounterid int This column stores id of an encounter for which vitals related log is being stored. This column refers to encounterid column in enc table.
vitalid int This column stores id of a vital for which log is being recorded. This column refers to itemid column in items table.
propid int This column stores id of a property for a vital. Column refers to propid column in properties table. This values is always set to 62 which indicates property Unit Of Measurement.
Value varchar This column stores value of the vital which is being logged.
actiontaken varchar This column stores the action being taken by the end user on the vital. i.e Update, Delete
UpdatedBy int This column stores id of the user who performed update or delete action for the vital on an encounterid. This column refers to id column in user table.
UpdatedTime datetime This column stores date and time when vital was updated or deleted by the end user. Time is stored in '2011-01-14 08:48:41.000' format.
value2 text This column stores value of the vital which is being logged.
nottaken tinyint This column stores the vitals not taken flag for the vital being logged
exceptionid int This column stores the exception reason id for the vital being logged. It has a foreign key relation with the `vital_exceptionlist` table's `id` column

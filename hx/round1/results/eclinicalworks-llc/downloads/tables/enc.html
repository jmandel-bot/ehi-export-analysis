EHI Export Schema
Table enc
Description This table stores encounter and related information which acts as primary data for that encounter to all the other tables storing various information related to the encounter.
Column DataType Description
encounterid (Primary Key) int The column stores an id of the encounter of a patient. This column is an auto incremental primary key.
patientid int This column stores the identifier number of a patient, for whom the appointment is created. It refers to column "UID" from the "user" table.
doctorid int This column stores the appointment provider id of the encounter. Foreign key relationship: "UID" column of the "user" table.
date datetime This column stores the date when the appointment was created. The format used is YYYY-MM-DD
time varchar This column is not used anymore.
startTime datetime This column stores the start time of the appointment created. The format used is: HH:MM:SS.
endTime datetime This column stores the end time of the appointment created. The format used is: HH:MM:SS.
reason text This column stores a combination of pre-defined text as well as free-text for the Reason section of an appointment for a patient from the user interface.
VisitType varchar This column stores the selected visit type of the patient for an appointment from the User Interface. It refers to column name from the visitcodes table.
roomno varchar This column stores the room number at the facility where the encounter took place.
STATUS varchar This column stores the current visit status of an appointment created. This column refers to column code from visitstscodes table.
deleteFlag int This column stores values "0 or "1" indicating whether this appointment is deleted. This is soft delete. A "0" means not deleted, while a "1" means deleted.
encType smallint This column stores one of the values listed below which denotes the type of the encounter: 1 - Office Visit, 2 - Tel Encounter, Virtual Visits, 3 - Out of Office, 4 - Claim, 5 - Lab, 6 - Web Encounter, 7 - ePrescription refills, 8 - PTDASH, 9 - Orderset with patient id as 8663.
encLock smallint This column stores values "0" or "1" which indicate if an encounter is locked or not. A "1" means locked, and a "0" means not locked.
Notes text This columns stores additional notes documented for the encounter.
generalNotes text This column stores the free text general notes documented related to an appointment created.
vmid varchar This column stores a string as unique identifiers for each row. Currently, this column is not actively used in the system.
facilityid int This column stores an id of the facility selected for the created encounter.This columns refers to the column id from edi_facilities.
POS int This column stores the place of service code where an encounter took place. This column refers to column code from edi_pos table.
arrivedTime datetime This column stores the current time when the patient is checked in for the appointment created. The format used is YYYY-MM-DD HH:MM:SS.
timeIn datetime This column stores the checked in time of the patient for an appointment. The format used is: YYYY-MM-DD HH:MM:SS.
timeOut datetime This column stores the check out time of the patient for an appointment. The format used is: YYYY-MM-DD HH:MM:SS.
stsAfterArr varchar This column stores values IN or OUT which indicate the arrival status of a patient. "IN" means when a patient is checked in, while "OUT" means when patient has checked out.
depTime datetime This column stores the current time when the patient is checked out for the appointment created. The format used is: YYYY-MM_DD HH:MM:SS.
stsAfterChkIn varchar The status after the patient checked in (e.g., 'Waiting', 'In consultation', 'Completed').
HL7id int The visit number sent by the vendor for an interface created appointment.
newPt smallint This column stores values 0 or 1 which indicate a new patient. A "1" means a new patient, and a "0"means not a new patient.
Dx varchar This column stores diagnosis or reason of the patient for the visit (appointment). This column refers to column itemname from the items table.
reminded int This column stores flags "0" or "1". A "1" is when the appointment is created using Nimbus and patient opts for the Remind Me option. A "0" means otherwise.
remindedDate datetime This column is not used anymore.
VisitCopay decimal This column stores the co-pay amount set for the appointment from the encounterid column of the patient.
CopayChanged int This column stores values "0" or "1" which indicate Co-Pay/Claim changes for the visit of the patient. A "1" means changed, and a "0" means not changed.
ClaimRequired int This column is no longer used.
ClaimReq int This column stores flags "0" or "1". A "1" means the claim is required, and a "0" means the claim is not required.
VisitTypeOverriden varchar This column stores code id for the selected visit type for the encounter. This column refers to column codeid from the visitcodes table.
refPrName varchar This column stores the last, first name of the referring provider at the facility.
refPrid int Integer identifier for the referring provider, this column refers to UID column of user table.
PSACGroup int This column stores the PSAC Group id of the user when a user tries to access the Encounters window of a patient whose records are marked as confidential.
deptid int This column stores an id of the department to which the encounter is associated. This column refers to the column deptid from the departments table.
Resourceid int This column stores the id of a resource such as anyone on the staff (except a physician), devices, or even rooms.
EncEligibilityStatus char This column stores one of the below flags when eligibility for primary insurance is checked at encounter level. Flags include: E = Eligible V = Not Verified, N = InEligible
EligibilityMessage varchar This column stores messages received from the insurance company for the eligibility request sent for the primary insurance.
Invoiceid int This column stores claim (invoice) number corresponding to the encounter. This column refers to the id column of the edi_invoice table.
EligSubid int This column stores an id for the eligibility submitted request. This column refers to the column submissionid from the x271respDetails table.
EncEligibilityStatus2 char This column stores one of the below flags when eligibility for secondary insurance is checked at encounter level. Flags include: E = Eligible V = Not Verified, N = InEligible
EligibilityMessage2 varchar This column stores message received from the Insurance company for the eligibility request sent for the secondary insurance.
EncEligibilityStatus3 char This column stores one of the below flags when eligibility for tertiary insurance is checked at the encounter level. Flags include: E = Eligible V = Not Verified, N = InEligible
EligibilityMessage3 varchar This column stores messages received from the insurance company for the eligibility request sent for the tertiary insurance.
trans_dictation int This column stores the flags 0 or 1. A "1" means when the physician documents the notes for the encounter, and a "0" means otherwise. This column is not being used anymore. Confirm with developer.
waitTime datetime This column stores the estimated wait time for a patient in the waiting room. The format used is YYYY-MM-DD HH:MM:SS.
websync tinyint This column stores the flags 0 or 1. A "1" means the encounter is uploaded to portal, and a "0" means the encounter is not uploaded to portal.
dateIn datetime This column stores the date when the patient is checked in by the end user. The format used is YYYY-MM-DD HH:MM:SS.
dateOut datetime This column stores the time stamp when the encounter from the encounterid column is marked as done. The format used is: YYYY-MM-DD HH:MM:SS.
notesDoneTime datetime This column is to store time when the encounter from the encounterid column is marked as done. The format used is: HH:MM:SS.
notesLoginid int This column stores an id of the logged in user who documents the encounter. This column refers the UID column from the user table.
notesLogOutid int This column stores an id of the provider who marks the encounter from the encounterid column as done. This column refers to the column UID from the user table.
visittypeid int This column stores an id of the selected visit type while creating an appointment. This column refers to the column codeid from the visitcodes table.
UTCStartTime datetime This column stores the start time in UTC format. This column is not used anymore.
UTCEndTime datetime This column stores end time in UTC format. This column is not used anymore.
visitstscodeid int This column stores the visit status code id of the appointment. This column refers to the column codeid from the visitstscodes table.
practiceid int This column stores the practice id for the created apppointment. This column refers to the column id from the practice table.
immnotessendstatus int This column stores flags 0 or 1. A "1" means when the HL7 outbound message is sent, and a "0" means otherwise.
transitionofcare tinyint This column stores flags 1, 2, or 0 indicating whether or not transition of care data was received. A "1" means the electronic summary of care record was not received, a "2" means the electronic summary of care record was received, and a "0" means otherwise.
prNotesSendStatus int This column stores flags 0, 1, 2, 5, 6, 7, and 8 which indicate the sent status of outbound Progress Notes of a patient to a vendor. A “0” means when they were not transmitted, a “1” means the transmission was successful, a “2” means the transmission failed, a “5” means a transmission is in progress, a “6” means it needs to be retransmitted, a “7” means the Progress Note XML is not available, and an “8” means the Progress Notes are part of a PSAC group.
readytobill tinyint This column stores flags 0 or 1. A "1" means when the encounter is ready to be sent to EMR SPOLE to CBO HUB, and a "0" means otheriwse.
reviewbeforebill tinyint This column stores flags 0 or 1. A "1" means if CBO is supposed to review a spoke encounter before creating claim, and a "0" means otherwise.
EncEligibilityStatus4 varchar This column stores one of the below flags when eligibility for other insurances is checked at the encounter level. Flags include: E = Eligible V = Not Verified, N = InEligible
EligibilityMessage4 varchar This column stores messages received from the insurance company for the eligibility request sent for any other insurance (vision, dental).
gUID varchar This is a unique row identifier and the data is not used by the system.
afterCareStatus varchar This column stores the status code for the encounter which is marked for after care. This column refers column code in aftercarestatuscodes table.
callbackHistory text This column stores the documented call back history notes to a specific sub status of the after care encounter.
waittime_minutes int This column stores the estimated wait time in minutes for a patient in the waiting room.
medicalmodifieddate varchar This column stores the time stamp when medical history is last modified in the encounter of the patient. The format used is YYYY-MM-DD HH:MM:SS.
surgicalmodifieddate varchar This column stores the time stamp when surgical history is last modified in the encounter of the patient. The format used is YYYY-MM-DD HH:MM:SS.
hospmodifieddate varchar This column stores the time stamp when hospitalization data is last modified in the encounter of the patient. The format used is YYYY-MM-DD HH:MM:SS.
familymodifieddate varchar This column stores the time stamp when family history is last modified in the encounter of the patient. The format used is YYYY-MM-DD HH:MM:SS.
socialmodifieddate varchar This column stores the time stamp when social history is last modified in the encounter of the patient. The format used is YYYY-MM-DD HH:MM:SS.
allergymodifieddate varchar This column stores the time stamp when an allergy is last modified in the encounter of the patient. The format used is YYYY-MM-DD HH:MM:SS.
supproviderid int This column stores the supervising provider assigned to the appointment. Refers to the UID column of the user table.
aftercareSubStatus varchar This column stores the sub status code for the encounter which is marked for after care. This column refers to the column code in the aftercaresubstatuscodes table.
ApptStatusReasonRequired varchar This column stores the reason documented by the end user on change of the appointment status in corrections module.
implantmodifieddate varchar This column stores the current time stamp when an implant is modified in the encounter of the patient. The format used is YYYY-MM-DD HH:MM:SS.
dentalmodifieddate varchar This column stores the current time stamp when dental history is modified in the encounter of the patient. The format used is: YYYY-MM-DD HH:MM:SS.
ocularhistmodifieddt varchar This varchar value represents the date when the ocular history was last modified. Format is: yyyy-mm-dd hh:mm:ss
ocularsurgmodifieddt varchar This is a varchar value representing the date when the ocular surgery history was last modified. Format is: yyyy-mm-dd hh:mm:ss
isShared int This column stores numeric values to indicate the status of a shared Progress Notes template. A "1" means - , a "2" means in progress, and a "3" means the column is used when the item key EnableShareTemplate is enabled.
gynmodifieddate varchar This is a varchar value representing the date when the gyn history was last modified. The format is: yyyy-mm-dd hh:mm:ss
obmodifieddate varchar This is a varchar value representing the date when the obstetrical history was last modified. Format is: yyyy-mm-dd hh:mm:ss
qmbflag tinyint This column stores "0"or "1" to indicate whether the encounter is QMB enabled or not.
bhhistorymodifieddate datetime This is a datetime value representing the date and time when the behavioral health history was last modified (e.g., '2021-08-31 14:30:00', '2021-09-01 10:15:00', '2021-09-02 16:45:00').

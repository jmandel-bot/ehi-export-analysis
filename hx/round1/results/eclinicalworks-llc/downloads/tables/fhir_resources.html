EHI Export Schema
Table fhir_resources
Description This table stores information about FHIR (Fast Healthcare Interoperability Resources) resources, which are standardized specifications for exchanging healthcare information electronically.
Column DataType Description
resource_id (Primary Key) int This is an integer column that serves as the primary key for the table. It uniquely identifies each FHIR resource in the table. Example values could be 1, 2, 3, etc.
resource_name varchar This is a varchar(100) column that stores the name of the FHIR resource. It has a unique constraint, meaning that each resource name must be unique within the table. Example values could be "Patient", "Observation", "Condition", etc.

EHI Export Schema
Table vision_allocationdiscount
Description This table stores information about discounts and adjustments applied to vision allocation encounters.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
encounterid int identifier for the encounter to which the discount is applied. This is a foreign key referring to the `id` column in the `encounters` table. Example: 101, 102, 103.
adjustmentcode varchar A code representing the type of adjustment applied to the encounter. Example: 'A01', 'B02', 'C03'.
discounttype tinyint A numeric value representing the type of discount applied (e.g., 0 for percentage, 1 for fixed amount). Example: 0, 1.
discountrate decimal The rate of discount applied to the encounter, stored as a decimal value. Example: 10.00, 15.50, 20.25.
totaladjustment decimal The total amount of adjustment applied to the encounter, stored as a decimal value. Example: 100.00, 150.50, 200.25.
deleteflag tinyint A flag indicating whether the record has been deleted (1) or not (0). Example: 0, 1.
addedby int identifier for the user who added the record. This could be a foreign key referring to the `id` column in the `users` table. Example: 1, 2, 3.
updatedby int identifier for the user who last updated the record. This could be a foreign key referring to the `id` column in the `users` table. Example: 1, 2, 3.
deletedby int identifier for the user who deleted the record. This could be a foreign key referring to the `id` column in the `users` table. Example: 1, 2, 3.
addedtime datetime time stamp for when the record was added. Example: '2021-10-01 12:34:56'.
updatedtime datetime time stamp for when the record was last updated. Example: '2021-10-02 14:56:32'.
deletedtime datetime time stamp for when the record was deleted. Example: '2021-10-03 16:12:45'.

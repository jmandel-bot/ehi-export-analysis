Table ip_rh_patient_precaution
Description Table is used to store patient precaution details.
Column DataType Description
id (Primary Key) int This column stores auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer (Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
encounterId int This column is FOREIGN KEY which refers to encounterId column of enc table. This column stores high range numeric value hence integer is selected and This column will always have value hence we keep NOT NULL.
patientId int This column stores patient id of patient against which user has documented data. This column is FOREIGN KEY which refers to uid column of users table. This column stores high range numeric value hence integer is selected, and this column will always have a value hence defined as NOT NULL.
precautionId int This column is used to identify which precaution item is added for patient encounter. The value of this column get added when a record get added by user for the patient encounter. This column belongs to id column of ip_rh_dict_isolation or ip_rh_dict_precaution_details or ip_rh_dict_patient_attr table . This column stores high range numeric value hence integer is selected, and this column will always have a value hence defined as NOT NULL
Prtype tinyint This column stores type precaution is added for user. This column store 1,2,3 values so we choose tinyint as datatype.1-Isolation Precaution 2-Precaution details 3.Precaution attribute
modiBy int This column stores user id of user. This column is used to identify which user has last modified record (precaution item) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store high range of numeric value hence integer is selected. This column will not always have value hence we keep NULL DEFAULT NULL.
modiTime datetime This Column stores datetime as we want to store a datetime when the item is modified, so we are using DATETIME as data type of column. This column will not always have a value hence defined as NULL DEFAULT NULL
deleteFlag tinyint This will store 0 and 1 value. 0-not deleted and 1-deleted. This column is used to identify whether row is in use or not. We are using TINYINT as datatype as we just want to store 0 or 1 value. By default, row will be in use, so we keep default value is 0.

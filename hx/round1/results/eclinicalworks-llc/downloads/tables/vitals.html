EHI Export Schema
Table vitals
Description This table stores the vitals recorded during patient encounters.
Column DataType Description
encounterid int This column stores enocunterid for which vitals are being recorded. This column refers to encounterid column in enc table.
vitalid int This column stores itemid associated with name of the vitals being documented during the encounter. This column refers to itemid column in items table. Mapping of vital names and itemids are stored in vitaltypes table.
propid int This columns stores itemid to get unit of measurement (UOM). This column refers to itemid column in items table. Mapping of itemid and propid is stored in itemdetail table.
value varchar This column stores values for different vitals provided by end users during the encounter.
id (Primary Key) int This column is auto-incremented primary key in This column .
UpdatedBy int This column stores user id of the provider/staff who updated/added vitals information for the encounter. This column refers to UID column in user table.
UpdatedTime datetime This column stores date and time when provider/staff added/updated vitals information. Column does not store individual time stamp for each vital for an encounter, time stamp is updated for all vitals even if one of the values is changed. time stamp is stored in 'YYYY-MM-DD HH:mm:ss.sss' format.
defaultValue varchar A varchar(1024) field storing the default value for the vital property. Example: "97.0"
valueHtm varchar A text field that may store an HTML representation of the vital property value. Example: "98.6"
value2 varchar This column stores value being documented by end user for 'vision' vital otherwise this column is left blank for other vitals.
nottaken tinyint A tinyint(1) field indicating whether the vital was not taken (1) or taken (0). Example: 0
exceptionid int An integer representing the unique identifier for an exception related to the vital, if any. This column has a foreign key relation with the `vital_exceptionlist` table's `id` column. Example: 3

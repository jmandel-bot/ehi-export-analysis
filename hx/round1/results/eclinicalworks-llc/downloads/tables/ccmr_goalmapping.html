EHI Export Schema
Table ccmr_goalmapping
Description This table contains mapping of goals, objectives and interventions for care plan.This table stores the mapping of goals, objectives, and interventions for specific encounters and problems.
Column DataType Description
encounterId int This column stores careplan and Patient Problems encounterId. Column refers to encounterid column in enc table.
goalId int This column store id of a goal which is being mapped. Column refers to id column in structdatadetail table.
objId int This column store id of an objective which is being mapped. Column refers to id column in structdatadetail table.
catId int This column stores problem Id used in careplan / patient's problems. This column refers to items.itemID.
interventionid int This column store id of an intervention which is being mapped. Column refers to id column in structdatadetail table.
problemEncId int This column stores encounterId from enc table which is use to hold careplan & problems encounterid. Value 0 means BH data added is for patient's problem, value greater than 0 means BH data added is for Care plan and this column holds problem EncounterId added in care plan from ccmr_atrisk table's encounter column.
Id (Primary Key) int This column stores the unique identifier for each record in the `ccmr_goalmapping` table and is the primary key. It auto-increments with each new record. Example: 1
deleteflag tinyint This column stores a boolean value (tinyint) to indicate whether the record is deleted or not. A value of 0 means not deleted, and a value of 1 means deleted. The default value is 0. Example: 0
goal varchar This column stores the text description of the goal. Example: "Improve patient's blood pressure."
objective varchar This column stores the text description of the objective related to the goal. Example: "Monitor blood pressure regularly."
intervention varchar This column stores the text description of the intervention related to the objective. Example: "Prescribe medication for blood pressure control."

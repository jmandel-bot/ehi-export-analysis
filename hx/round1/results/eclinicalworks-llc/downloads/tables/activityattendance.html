Table activityattendance
Description This table stores information about the player participated in the events.
Column DataType Description
id (Primary Key) int Unique identifier for each attendance record. Example: 1, 2, 3.
Activityid int identifier for the activity that the user attended. This is a foreign key that refers to the id column in the activities table. Example: 101, 102, 103.
userid int identifier for the user who attended the activity. This is a foreign key that refers to the id column in the user table. Example: 1001, 1002, 1003.
status varchar The status of the user's attendance, such as Active(Full-Go) or Limited/Restricted. Example: Active(Full-Go), Limited/Restricted, Probable, Questionable, Doubtful.
attended tinyint A binary flag indicating whether the user attended the activity (1) or not (0). Example: 0, 1.
deleteflag tinyint A binary flag indicating whether the attendance record has been deleted (1) or not (0). Example: 0, 1.
injurylink varchar This column stores ids for injury selected. The column has been declared as VARCHAR because this column may store multiple ids for diagnosis separated by ',' (i.e., 250909,252643).
OtherReason varchar A brief description of any other reason for the user's attendance status, if applicable. Example: family emergency, car trouble).

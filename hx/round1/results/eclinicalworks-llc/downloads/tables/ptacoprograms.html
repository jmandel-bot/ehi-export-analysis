EHI Export Schema
Table ptacoprograms
Description This table stores information about patient programs, their enrollment status, and related details.
Column DataType Description
id (Primary Key) int This column is auto-incremented primary key in This column . Unique id is assigned to each enrolled program for a patient when they are created using user interfaceby the end user.
patientId int This column is used to store the patientid for whom the program is enrolled by the end user. Column refers to uid column in user table. This column stores numeric data which could grow with number of visits over the years hence integer(Range: 0-2,147,483,647) is selected.
programId int Stores the program id of the program enrolled for a patient. Column refers to id column in acoprograms table. This column stores numeric data which could grow with number of visits over the years hence integer(Range: 0-2,147,483,647) is selected.
STATUS int Stores 0 ,1 based on the enrollment status of the program for a patient 0 - the program is still not ended 1 - once the program ends after duration entered by the end user, the status is updated to 1
startdate datetime Stores the date when the program was enrolled for a patient.
programterm varchar Stores the duration entered by the end user for the enrolled program for a patient. It can have values in months,days,weeks,years. 
enddt datetime time stamp for the end date of the patient's enrollment in the program. Example: '2021-06-30 00:00:00'
reason text Column to save the reason for enrolling the patient to the program.
enrollmentstatus varchar This column used to store integer value which indicates that the entry is for enrolled or enrollment request 0 - Enrollment Request 1 - Enrolled
source varchar This column stores the facility name from which enrollment request created or patient is enrolled
documentid int Unique identifier for a document associated with the patient's enrollment. Example: 98765
addeddatetime datetime time stamp for when the record was added to the table. Example: '2021-01-01 10:00:00'
declineReason varchar A short description or code representing the reason for declining the program. Example: 'Not interested', 'Insurance issues'
delFlag int This column stores value 0 / 1 indicating this enrollment is deleted. This is soft delete. 0 - Not Deleted , 1 -Deleted
ConsentStatus int Status code for the patient's consent to the program. Example: 0 (not consented), 1 (consented)
timerStatus int Status code for the timer associated with the program. Example: 0 (inactive), 1 (active)
consentDate datetime Date when the patient provided consent for the program. Example: '2021-01-01'
payorId int Unique identifier for the payor associated with the patient's enrollment. Example: 45678
phm_disenroll_reason_id int This column stores disenrollment reason Id which the user selects while disenrolling a patient from a phm program. This column refers to id column of phm_disenroll_reasons table. The data type of this field is INAT.
noOfCC tinyint Number of chronic conditions selected during enrollment for CCM/PCM program Example: 1, 2
declineReasonId int Unique identifier for the decline reason, foreign key referring to `phm_decline_reasons` table's `id` column.
outreachStatus tinyint Status code for the outreach efforts related to the patient's enrollment, foreign key referring to `phm_outreach_status` table's `id` column.
assignedToId int Unique identifier for the user assigned to the patient's enrollment, foreign key referring to `users` table's `uid` column.
nextContactDate datetime Date for the next scheduled contact with the patient. Example: '2021-02-01'
cohortId int Unique identifier for the cohort the patient belongs to, foreign key referring to `phm_cohort` table's `id` column.
expiryStatus tinyint Status code for the column indicated that whether user has been selected Duration or Expiry Date from Enrollment screen while enrolling CCMR program 1 - Expiry Date 0 - Duration
mdm_value tinyint This field stores MDM value for a patient in CCM through which the user can identify the risk and take appropriate action. This field stores 0, 1, 2 and 3 values. 0 - Undefined. 1 - Low, 2 - Moderate, 3 - High. The data tpe of this field is tinyint.

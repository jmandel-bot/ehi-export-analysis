EHI Export Schema
Table fhirappvendors
Description This table stores the information of external app vendors. The app can be provider apps such as SMART on FHIR, CDS Hooks OR bulk/backend apps
Column DataType Description
id (Primary Key) int This is a primary key
name varchar This column stores the name of the app.
deleteflag tinyint This column specifies whether given app is deprecated/deleted. 0 - the app is active 1- the app is deprecated/deleted
hubrefId int This column stores the primary key of the app at Contenthub server
appimgurl varchar This column stores the app image which is shown on the configured app list pop up on patient hub or progress notes
datecreated datetime This column stores the date time when the app is created first time
applaunchurl varchar This column stores the app launch url. The app is launched on this url when the user clicks on the app image. The app image is shown on the configured app list pop up on patient hub or progress notes
details varchar This column stores the description of the app. This description is provided by the app vendor when it registers the app on developer's portal.
isodaactivated tinyint This column indicated whether the app is activated by the user on EMR.
isappgranttoapu tinyint This column indicated whether the app is activated by the user on EMR.
intereststatus tinyint This column is used to show the app activation progress. The app can be in one of these progress steps: 1) Request Pending 2) Activate 2) Configuration etc
cost varchar This column stores the cost information of the app.
costdescription varchar This column stores the cost description of the app.
apptype varchar This column stores the type of the app such as provider, bulk etc
productname varchar This column stores the productname used for the app activation on product activation screen.
subAppType varchar This column stores the subtype of the apps such as CDS Hooks, Backend single patient app etc
feedbackneeded tinyint This column is used by CDS Hooks apps. It denotes whether the app requests/needs feedback from the user when the user makes decision on the cards provided by the app.

EHI Export Schema
Table careplan_dates
Description This table stores information about care plan dates, including notes, signing details, and the date when the care plan was signed.
Column DataType Description
id (Primary Key) int This is the primary key column and contains a unique integer value for each record in the table. For example: 1, 2, 3, etc.
CarePlanid int This column stores an integer value representing the care plan associated with the record. It can be a foreign key referring to the `id` column in another table careplan id.
Notes text This column stores text data containing any notes or comments related to the care plan date. Example values: "Patient showed improvement", "Follow-up needed", etc.
SignedByid int This column stores an integer value representing the user who signed the care plan. It can be a foreign key referring to the `id` column in another table containing user details. Example values: 5, 15, 30, etc.
SignedType int This column stores an integer value (either 0 or 1) indicating the type of signing for the care plan. Example values: 0 (unsigned), 1 (signed).
SignedDate datetime This column stores a datetime value representing the date and time when the care plan was signed. Example values: '2021-08-01 14:30:00', '2022-01-15 10:15:00', etc.

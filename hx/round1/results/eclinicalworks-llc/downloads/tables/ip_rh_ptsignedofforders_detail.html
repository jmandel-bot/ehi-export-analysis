Table ip_rh_ptsignedofforders_detail
Description This table used for tracking the details of orders that have been signed off, including who signed off and when
Column DataType Description
signedOffDetailId (Primary Key) int Unique identifier for each signed-off order detail. Auto-increments with each new record, ensuring a unique ID for each entry.
stagingOrderId int Identifier linking to ip_rh_ptstagingorders. A default of 0 indicates a placeholder or uninitialized state.
orderId int Identifier for the order that has been signed off. A default of 0 indicates a placeholder or uninitialized state.
signedOffById int Identifier for the user who signed off the order. Allows for tracking who approved the order, though it's optional.
signedDateTime datetime Timestamp of when the order was signed off. The default value is likely a non-null placeholder indicating an unknown or uninitialized date
orderAncestorId int Identifier for the original order from which this order may have been derived.
deleteFlag tinyint Flag indicating whether the order detail has been marked for deletion. Typically, 0 for false (not deleted) and 1 for true (deleted).
pnEncId int Used for linking to patient data and patient encounter details.

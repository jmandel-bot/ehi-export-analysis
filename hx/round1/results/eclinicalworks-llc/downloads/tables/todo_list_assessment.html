EHI Export Schema
Table todo_list_assessment
Description This table stores information about assessments related to items in a To Do List, including treatment and clinical notes, codes, coding systems, onset dates, and HCC flags.
Column DataType Description
id (Primary Key) int This is the primary key for the table and is an auto-incrementing integer value. It uniquely identifies each assessment in the table. Example: 1, 2, 3, etc.
todo_list_id int This is a foreign key that refers to the id column in the todo_list table. It represents the id of the todo list that the assessment is associated with. Example: 1, 2, 3, etc.
item_id int This is a foreign key that refers to the itemid column in the items table. It represents the id of the item in the todo list that the assessment is related to. Example: 1, 2, 3, etc.
treatment_notes varchar This is a text column that stores any treatment notes related to the assessment. Example: Patient responded well to the treatment.
clinical_notes varchar This is a text column that stores any clinical notes related to the assessment. Example: Patient showed signs of improvement after the treatment.
code varchar This is a varchar(1000) column that stores a code related to the assessment, which can be up to 1000 characters long. Example: ICD10:A01.1.
coding_sys varchar This is a varchar(50) column that stores the coding system used for the code column. It can be up to 50 characters long. Example: ICD-10, SNOMED CT.
onset_date datetime This is a date column that stores the onset date of the condition or issue related to the assessment. Example: 2022-01-01.
hcc_flag tinyint This is a tinyint(1) column that stores a flag indicating whether the assessment is related to an Hierarchical Condition Category (HCC) or not. It can have a value of 0 (not related) or 1 (related). Example: 0, 1

EHI Export Schema
Table patientasaplist
Description This table stores ASAP List information which shows in Resource Schedule's bump list screen
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
patientId int Unique identifier for the patient. It can be a foreign key referring to the `id` column in the `patients` table. Example: 101, 102, 103.
encounterId int Unique identifier for the patient's encounter. It can be a foreign key referring to the `id` column in the `encounters` table. Example: 1001, 1002, 1003.
notes varchar Notes or comments related to the patient's ASAP status, with a maximum length of 1000 characters. Example: "Patient needs immediate attention due to severe pain."
trUserId int Unique identifier for the user who added the patient to the ASAP list. It can be a foreign key referring to the `id` column in the `users` table. Example: 1, 2, 3.
orderValue tinyint A tiny integer value representing the priority order of the patient in the ASAP list. Example: 1, 2, 3.
currDate datetime The current date when the patient was added to the ASAP list. Example: '2022-01-01', '2022-01-02'.
currStartTime datetime The start time of the current ASAP period. Example: '08:00:00', '14:00:00'.
currEndTime datetime The end time of the current ASAP period. Example: '12:00:00', '18:00:00'.
renewDate datetime The date when the ASAP status was last renewed. Example: '2022-01-03', '2022-01-04'.
renewStartTime datetime The start time of the renewed ASAP period. Example: '09:00:00', '15:00:00'.
renewEndTime datetime The end time of the renewed ASAP period. Example: '13:00:00', '19:00:00'.
inAsapList tinyint A tiny integer value indicating whether the patient is still in the ASAP list (1) or not (0). Example: 1, 0.
timestamp datetime The time stamp when the record was last updated. Example: '2022-01-01 12:34:56', '2022-01-02 14:56:00'.

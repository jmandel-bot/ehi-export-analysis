EHI Export Schema
Table todo_list_cdscard
Description This new table stores suggestion cards received as a response from CDS Hooks apps. Provider then uses these cards to render better patient care.
Column DataType Description
id (Primary Key) int This column stores an auto-incremented primary key field
todo_list_Id int This column stores reference to To-Do item in providers' queue. It is a foreign key pointing to id column of table todo_list
encounterId int This column stores encounter information for current To-do item is added. Refrence to to encounterid column of enc table
triggerId smallint This column stores value of id column of table cdsappstriggers i.e. this column is a foreign key pointing to id column of table cdsappstriggers
details varchar This field is part of card JSON received as a response from CDS Hooks app. It stores the details of the card
indicator varchar This field is part of card JSON received as a response from CDS Hooks app. It stores the indicator information.
card_uuid varchar This columns stores individual instance types. It is unique per card. It is used to identify the card within EMR as well as CDS Hooks app.This identifier is also used to provide user's feedback to the app. It stores randomly generated UUID
source_label varchar This field is part of card JSON received as a response from CDS Hooks app. It stores the source label.
source_url varchar This field is part of card JSON received as a response from CDS Hooks app. It stores the source url.
link_label varchar This field is part of card JSON received as a response from CDS Hooks app. It stores the label for the link.
link_url varchar This field is part of card JSON received as a response from CDS Hooks app. It stores the link url.
link_type varchar This field is part of card JSON received as a response from CDS Hooks app. It stores the link type.

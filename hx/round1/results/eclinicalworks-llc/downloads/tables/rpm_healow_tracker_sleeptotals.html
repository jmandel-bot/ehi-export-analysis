EHI Export Schema
Table rpm_healow_tracker_sleeptotals
Description When a Provider prescribes a Sleep device for a patient, the patient will take readings from that device and will upload it to Healow. From healow, these readings are transferred to EMR side. We have created this new table rpm_healow_tracker_sleeptotals to store the Sleep readings for the patients.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT. This column stores integer value which could grow with number of sleeptotals records added and hence integer is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index.
UID int This column stores unique id of a Patient to identify the readings belonging to the patient which references the UID column of the user table. The data type chosen is INT. This field will always have a value hence defined as NOT NULL.
device_action int This column stores the Vendor id from whom the device has been purchased. Contains device_action that refers to Primary Key device_action column of rpm_vendors_list table and data type is INT. The value of this field can be NULL.
measure_datetime datetime This column stores the date time when the measurement was done and data type is DATETIME. This field will always have a value hence defined as NOT NULL. 
sleep_id bigint This column is used by Healow to store unique sleep id and data type is INT. This field will always have a value hence defined as NOT NULL.
sleep_start datetime This column stores the time when sleep started and data type is DATETIME. This field will always have a value hence defined as NOT NULL.
sleep_end datetime This column stores the time when sleep ended and data type is DATETIME. This field will always have a value hence defined as NOT NULL.
sleep_total varchar This column stores total hours of sleep and data type is VARCHAR(100). This column stores characters that does not exceed 100 characters and hence VARCHAR(100) is selected. This field will always have a value hence defined as NOT NULL. The default value of this field is 0.
sleepsound_total varchar This column stores total hours of sound sleep and data type is VARCHAR(100). This column stores characters that does not exceed 100 characters and hence VARCHAR(100) is selected. This field will always have a value hence defined as NOT NULL. The default value of this field is 0
sleeplight_total varchar This column stores total hours of light sleep and data type is VARCHAR(100). This column stores characters that does not exceed 100 characters and hence VARCHAR(100) is selected. This field will always have a value hence defined as NOT NULL. The default value of this field is 0
sleepawake_total varchar This column stores total hours of sleep when patient was awake and data type is VARCHAR(100). This column stores characters that does not exceed 100 characters and hence VARCHAR(100) is selected. This field will always have a value hence defined as NOT NULL. The default value of this field is 0.
has_timeline tinyint This column is used by Healow to store if the sleep has time line and data type is TINYINT. This field can have a NULL value. The default value of this field is 0 since sleep record by default has timeline. 0 - No Timeline  1 - Has Timeline
has_distribution tinyint This column is used by Healow to store if the sleep has light, sound distribution and data type is TINYINT. This field will always have a value hence defined as NOT NULL. The default value of this field is 0 as the sleep has light sound distribution. 0 - Does not have Distribution  1 - Has Distribution
enteredbed_previousday tinyint This column is used by Healow to store if the patient entered the bed on previous day and data type is TINYINT. This field can have a NULL value and default value is NULL. 0 - Entered Bed On Previous Day  1 - Did not Enter Bed On Previous Day
delflag tinyint Flag indicates the record added is deleted, datatype is tinyint- 0 - Not Deleted, 1 - Deleted. This field will always have a value hence defined as NOT NULL. By default, this column should not be in Deleted status hence default 0 is chosen.
reviewed tinyint This column stores 0, 1 to determine whether the reading is reviewed or not and data type is TINYINT. This field will always have a value hence defined as NOT NULL.  The default value of this field is 0. Values 0 : Reading has not been reviewed by the Provider 1 : Reading has been reviewed by the Provider
reviewedByid int This column stores the Provider id who has reviewed the reading and data type is INT. Contains reviewedByid which refers to UID column of user table. This field will always have a value hence defined as NOT NULL.
reviewedDateTime datetime This column stores the date time when the reading was reviewed and data type is DATETIME. The default value of this field can be NULL. 
isManuallyAdded tinyint This column stores 0, 1 whether the reading is manually added by patient or not and data type is TINYINT. This field will always have a value hence defined as NOT NULL.The default value of this field is 0. Values  0 :Not added manually by the patient. 1 : Added manually by the patient.
device_id int This column stores the Unique id of the Device and data type is INT. Contains device_id that refers to device_id column of rpm_supported_devices_list table.The value of this field can be NULL.
modifiedDateTime datetime This column stores the date time when the reading was inserted and data type is DATETIME. This field will always have a value hence defined as NOT NULL.
source_id int Contains " source_id" value referenced from id column of the rpm_healow_tracker_source table, datatype is INT. This column could be store high range numeric value hence integer is selected. This field will always have a value hence defined as NOT NULL.
isRPMData tinyint Flag indicates the readings received from healow end are valid or not, datatype is tinyint - 0 - Not Valid, 1 - Valid. This field will always have a value hence defined as NOT NULL. By default, this column should not be in valid status hence default 0 is chosen.

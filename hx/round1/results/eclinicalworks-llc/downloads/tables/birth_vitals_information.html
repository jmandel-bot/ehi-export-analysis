Table birth_vitals_information
Description This table stores information about the various vitals of a patient taken during birth.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the birth_vitals_information table. Example: 1, 2, 3, etc.
birthvitalsid int identifier for the birth event, which is a foreign key that refers to the `id` column in the `birth_vitals` table. Example: 101, 102, 103, etc.
vitaltype varchar This column stores the standard type of vital which is documented. It can contain values like 'Ht', 'Wt', 'Wt-kg', 'WtPercentile' etc. Hence, varchar(20) is selected. This field will always have a value hence defined as NOT NULL.
VALUE decimal This column stores the vital value. As it can contain Wt in gms eg. 5000 gm or Ht as 30.25 inches, the datatype DECIMAL(6,2) is selected. This field will always have a value hence defined as NOT NULL.
unit varchar This column stores the unit associated with the vital value. As it can contain unit like gm, cm, inches, kg, the datatype varchar(10) is selected. This field will always have a value hence defined as NOT NULL.
state tinyint This column stores if the vital value is in abnormal range. By default, state is 0 which means normal range. value if 1 indicates its abnormal range. Hence the datatype tinyint is selected. This field will always have a value hence defined as NOT NULL.

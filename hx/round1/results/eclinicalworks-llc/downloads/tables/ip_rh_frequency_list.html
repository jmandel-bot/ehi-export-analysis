Table ip_rh_frequency_list
Description This table is a master table for frequency details. It stores different frequency details.
Column DataType Description
frequencyId (Primary Key) int This is an auto-incrementing integer column, serving as the primary key. It uniquely identifies each record in the table. The identity keyword indicates that the SQL Server will automatically increment this value when new rows are inserted. This column's purpose is to provide a unique identifier for each record as new frequency in the table.
freqCode varchar This column stores string type value is called frequency code which can be 60-character long string. It cannot be NULL and has a default value of an empty string.
freqDescription varchar This is a string (varchar) column with a maximum length of 255 characters. It cannot be NULL and has a default value of an empty string. This column could be used to store a longer description of the frequency description.
isActive tinyint This is an integer column with a default value of 1 and it cannot be NULL. This column might be used to indicate whether the frequency status is active (possibly 0) or inactive (possibly 1).
scheduleType tinyint This column stores integer type value. It maps schedule type to frequency. This is foreign key, It belongs to frequencyTypeId column from ip_rh_frequency_types_master
deleteFlag tinyint This is a tinyint column with a default value of 0 and it cannot be NULL. This could be a logical delete flag, where a value of 1 indicates the record is deleted and 0 indicates it's active.

Table apptbuffer
Description This table stores information about appointment buffers, which are temporary holding areas for appointment data during operations such as cut, copy, or paste appointments.
Column DataType Description
id (Primary Key) int This is the primary key and a unique identifier for each record in the apptbuffer table. It is an integer value that is automatically incremented with each new record. Example: 1, 2, 3, etc.
encounterid int This is a foreign key that refers to the encounter table's primary key. It is an integer value representing the unique identifier for an encounter. Example: 101, 102, 103, etc.
InBuffer int This is an integer value that indicates whether the appointment is currently in the buffer or not. A value of 1 means the appointment is in the buffer, while a value of 0 means it is not. Example: 1 or 0.
User varchar This is a varchar(255) column that stores the username of the user who performed the operation on the appointment. Example: 'johndoe', 'janedoe', etc.
OperationType varchar This is a varchar(10) column that stores the type of operation performed on the appointment, such as 'CUT' or 'COPY'. Example: 'CUT' or 'COPY'.
trusername varchar This is a varchar(255) column that stores the username of the user who is responsible for the transaction. It can be NULL if no user is responsible. Example: 'johndoe', 'janedoe', or NULL.
patientid int This is a foreign key that refers to the patient table's primary key. It is an integer value representing the unique identifier for a patient. Example: 1001, 1002, 1003, etc.

EHI Export Schema
Table gender_identity_options
Description This table stores the different gender identity options along with their descriptions and SNOMED codes.
Column DataType Description
id (Primary Key) int This is the primary key column that stores a unique integer value for each gender identity option. Example values: 1, 2, 3.
NAME varchar This column stores the name of the gender identity option as a varchar(150) value. Example values include: "Male", "Female", "Non-binary".
Description varchar This column stores a brief description of the gender identity option as a varchar(200) value. Example values: "A person who identifies as male", "A person who identifies as female", "A person who does not identify as exclusively male or female".
Snomed_code varchar This column stores the SNOMED code associated with the gender identity option as a varchar(50) value. SNOMED codes are a standardized system for identifying medical concepts. Example values: "S-12345", "S-67890", "S-09876".
deleteflag tinyint This column stores a tinyint(2) value to indicate whether the gender identity option has been deleted or not. A value of 0 means the option is not deleted, while a value of 1 means the option is deleted. Example values: 0, 1.

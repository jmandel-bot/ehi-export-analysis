Table prisma_pot_observation_entityact_detail
Description This table stores detailed information about entity act observations.
Column DataType Description
id (Primary Key) bigint This is the primary key column, which stores a unique identifier for each record in the table. It is of type bigint(20) and is auto-incremented. Example: 1, 2, 3, etc.
observationDetailId bigint This column stores the identifier of the related observation detail. It is of type bigint(20) and can have a default value of NULL. This column has a foreign key relation with the `prisma_pot_observation_detail` table's `id` column. Example: 101, 102, 103, etc.
actId varchar This column stores the identifier of the act. It is of type varchar(50) and can have a default value of NULL. Example: "ACT001", "ACT002", "ACT003", etc.
actTime varchar This column stores the time of the act. It is of type varchar(50) and can have a default value of NULL. Example: "2021-09-01T12:34:56", "2021-09-02T14:30:00", "2021-09-03T10:15:30", etc.
moodCodeValue varchar This column stores the mood code value associated with the activity. It is of type varchar(100) and can have a default value of NULL.
code varchar This column stores the code of the act. It is of type varchar(100) and can have a default value of NULL.
statusCode varchar This column stores the status code of the act. It is of type varchar(100) and can have a default value of NULL. Example: "COMPLETED", "PENDING", "CANCELLED", etc.
referenceText text This column stores the reference text related to the act. It is of type longtext and can store large amounts of text. Example: "This act was performed as part of a routine checkup.", "The act was cancelled due to unforeseen circumstances.", etc.

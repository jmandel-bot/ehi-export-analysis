Table patient_gender_identity
Description This table stores information about patients' gender identity, including the reason for their gender identity and any related actions or modalities.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
patientid int Unique identifier for each patient, referring to the patient's id in the `patients` table (foreign key relation). Example: 101, 102, 103.
modified_by int Unique identifier for the user who last modified the record, referring to the user's id in the `users` table (foreign key relation). Example: 1001, 1002, 1003.
gi_id int Unique identifier for the patient's gender identity, referring to the gender identity id in the `gender_identities` table (foreign key relation). Example: 1, 2, 3.
gi_reason varchar Reason for the patient's gender identity, stored as a string. Example: "Personal preference", "Medical condition", "Cultural background".
isactive tinyint A flag indicating whether the record is active (1) or inactive (0). Example: 0, 1.
deleteflag tinyint A flag indicating whether the record has been deleted (1) or not (0). Example: 0, 1.
added_on datetime Date and time when the record was added to the table. Example: '2021-08-01 12:34:56'.
inactive_date datetime Date and time when the record was marked as inactive. Example: '2021-09-01 12:34:56'.
action varchar Action taken related to the patient's gender identity, stored as a string. Example: "Counseling", "Hormone therapy", "Surgery".
modality varchar Modality used for the action taken, stored as a string. Example: "In-person", "Telemedicine", "Group therapy".

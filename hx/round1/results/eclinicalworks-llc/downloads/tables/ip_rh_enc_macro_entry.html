EHI Export Schema
Table ip_rh_enc_macro_entry
Description Table is used to store encounters and documented template mapping.
Column DataType Description
id (Primary Key) int This column stores auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer (Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
encounterID int This column is used to identify for which patient encounter user has documented data. This column is FOREIGN KEY which refers to encounterid column of enc table. This column stores a high range numeric value hence integer is selected, and this column will not always have a value hence defined as NULL DEFAULT NULL.
templateName varchar This column stores template name value added by user. We are using VARCHAR (128) as data type because this column's value will not be more than 128 characters. This column will not always have a value hence defined as NULL DEFAULT NULL.
deleteFlag tinyint This will store 0 and 1 value. 0-not deleted and 1-deleted. This column is used to identify whether row is in use or not. We are using TINYINT as datatype as we just want to store 0 or 1 value. By default, row will be in use, so we keep NOT NULL DEFAULT 0.
createdBy int This column stores user id of user. This column is used to identify which user has added record (module wise template) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store a high range of numeric value hence integer is selected. This column will always have a value hence defined as NOT NULL.
createdDateTime datetime This Column stores datetime as we want to store a datetime when the item is added, so we are using DATETIME as data type of column. This column will always have a value hence defined as NOT NULL.
modifiedBy int This column stores user id of user. This column is used to identify which user has last modified record (module wise template) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store a high range of numeric value hence integer is selected. This column will not always have a value hence defined as NULL DEFAULT NULL.
modifiedDateTime datetime This Column stores datetime as we want to store a datetime when the item is added, so we are using DATETIME as data type of column. This column will not always have a value hence defined as NULL DEFAULT NULL.
templateId int This column stores templateID which is used to identify which template is added for patient encounter. The value of this column gets inserted when the record gets added. This column is FOREIGN KEY which refers to id column of ip_rh_clinical_sd_template table. This column will store a high range of numeric value hence integer is selected. This column will always have a value hence defined as NOT NULL.
masterEncounterId int This column is used to store patient episodic level encounter id which is used to identify inserted row/record is belongs to which episode encounter. The value of this column gets inserted when the record get added. This column is FOREIGN KEY which refers to encounterid column of enc table. This column stores a high range numeric value hence integer is selected, and this column will not always have value hence we keep NULL DEFAULT NULL.
departmentId int This column stores departmentId. The value of this column gets inserted when the record get added to identify record belongs to which department. This column is FOREIGN KEY which refers to id column of ip_rh_loc_department table. This column stores a high range numeric value hence integer is selected, and this column will not always have value hence we keep NULL DEFAULT NULL.

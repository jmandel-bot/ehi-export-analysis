EHI Export Schema
Table ip_rh_enc_mappings
Description Table is used to mapping of master encounter id and associated encounterID.
Column DataType Description
id (Primary Key) int This column stores auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer (Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
masterEncId int This column stores maser encounter id against which there could be multiple associated encounter id present. This column is FOREIGN KEY which refers to encounterId column of enc table. This column will store a high range of numeric value hence integer is selected. This column will always have value hence we keep NOT NULL.
assocEncId int This column stores associated encounter id of master encounter id. This column is FOREIGN KEY which refers to encounterId column of enc table. This column will store a high range of numeric value hence integer is selected. This column will always have value hence we keep NOT NULL.
deleteFlag tinyint This will store 0 and 1 value. 0-not deleted and 1-deleted. This column is used to identify whether row is in use or not. We are using INT because we are storing integer value here. By default, row will be in use, so we keep NOT NULL DEFAULT 0.
modifybyid int This column is used to identify which user has last modified record (precaution item) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store a high range of numeric values hence integer is selected. This column will not always have a value hence defined as NULL DEFAULT NULL.
modifiedDate datetime This Column stores datetime as we want to store a datetime when the item is added, so we are using DATETIME as data type of column. This column will not always have a value hence defined as NULL.
moduleEncId int This column is FOREIGN KEY which refers to encounterId column of enc table. This column will store a high range of numeric values hence integer is selected. This column will not always have value hence we keep NULL DEFAULT NULL

EHI Export Schema
Table edi_inveob_cas
Description This table stores information related to the EOB (Explanation of Benefits) CAS (Claim Adjustment Reason Codes) for electronic data interchange (EDI) in healthcare for manual posting of payment
Column DataType Description
InvEobCASid (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
InvEobid int identifier for the related EOB record. This is a foreign key referring to the primary key in the `InvEob` table. Example: 101, 102, 103.
GroupCode varchar Group code for the claim adjustment reason. Example: 'CO', 'PR', 'OA'.
ReasonCode varchar Reason code for the claim adjustment. Example: '45', '50', '125'.
amount decimal Amount associated with the claim adjustment. Example: 100.00, 50.50, 25.25.
deleteFlag tinyint Flag to indicate if the record is deleted (1) or not (0). Example: 0, 1.
CreatedBy int identifier for the user who created the record. Example: 1, 2, 3.
CreatedDate datetime Date when the record was created. Example: '2021-01-01', '2021-02-01', '2021-03-01'.
CreatedTime datetime Time when the record was created. Example: '12:00:00', '14:30:00', '16:45:00'.
ModifiedBy int identifier for the user who last modified the record. Example: 1, 2, 3.
MOdifiedDate datetime Date when the record was last modified. Example: '2021-01-01', '2021-02-01', '2021-03-01'.
ModifiedTime datetime Time when the record was last modified. Example: '12:00:00', '14:30:00', '16:45:00'.
timestamp datetime time stamp of the last modification or creation of the record. Automatically updates on modification. Example: '2021-01-01 12:00:00', '2021-02-01 14:30:00', '2021-03-01 16:45:00'.
IsERAPostedCASCode tinyint Flag to indicate if the CAS code is posted to ERA (Electronic Remittance Advice) (1) or not (0). Example: 0, 1.
PostedAs tinyint identifier for the type of posting. Example: 1, 2, 3.

EHI Export Schema
Table rpm_patient_healowlinked_data
Description This table would store the every day list of the patients which have any healow linked devices.Everyday when user visits first time the Eligible queue it would make a request call to Healow server and receive the unique list of patients having healow linked devices.The previous day data is cleared and new data is inserted into the table.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT. This column stores integer value which could grow with number of patients healow linked device data added. It is AUTO_INCREMENT column and has PRIMARY KEY index.
patientid int It stores the unique id of a Patient. The value of this field cannot be NULL. There is a Foreign Key associated with this field -FK_rpm_patient_healowlinked_data_patientid referencing to UID column of user table. The data type chosen is INT as this field is pointing to UID column of user table which is also int.Since the patient will have unique entry for the day we have defined unique index UX_rpm_patient_healowlinked_data_patientid_modifiedDateTime on the column.
modifiedDateTime datetime Contains date time when the record was last modified, and datatype is DATETIME. This column stores datetime hence time stamp is selected and it will be inserted when data is received via call to healow server and hence it is kept as NOT NULL.

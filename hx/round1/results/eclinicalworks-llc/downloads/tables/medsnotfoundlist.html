EHI Export Schema
Table medsnotfoundlist
Description This table stores missing medications in the EMR database which are being imported from CCR. Also, it saves the mapped drug details with the missing medications data.
Column DataType Description
id (Primary Key) int This column is auto increment id value for not found drug entry
encounterid int This column contains encounterid value for which drug mapping added for not found drug during import attempted from eEHX or DRTLA tab for medication. It refers to enc.encounterid column.
trUserid int This is user id of logged in user who attempted medication import. It refers to user.UID column.
DrugName varchar This column contains name of CCR's drug which not found in EMR database.
Direction varchar column contains directions (dose , route and frequency) of drug received from CCR
MappedDrugName varchar This column contains local emr's drug name which is getting mapped with not found CCR drug.
MappedDrugDirection varchar column contains directions (dose , route and frequency) of drug getting mapped
hasMatch tinyint this is flag to identify if EMR database has matching drugs this will set to 0 if mapping drug detail not saved
mtimestamp datetime This column contains last updated time stamp value for data row.
delflag tinyint delete flag set to 1 if row deleted
MappedDrugItemid int This column refers Item id of drugname mapped with not found drug. It refers Itemid column of items table
CCRDataObjectid int This column will have CCR xml object id
refReqid int This column has referral request id.
DoctorsFlag int This flag has meds status value taking -1 not taking -2 and unknown -4

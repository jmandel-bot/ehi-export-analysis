Table prisma_pot_observation_detail
Description This table stores information about observation made during plan of treatment which includes observation time, mood, code status and other information.
Column DataType Description
id (Primary Key) bigint A unique identifier for each observation detail record. It is of type bigint(20) and is the primary key. Example: 1, 2, 3.
sectionId bigint A foreign key reference to the `prisma_ccd_document_section` table's `id` column. It is of type bigint(20) and stores the identifier of the section the observation belongs to. Example: 101, 102, 103.
observationId varchar A unique identifier for the observation. It is of type varchar(100). Example: "obs123", "obs456", "obs789".
observationTime varchar The time when the observation was made. It is of type varchar(50). Example: "2021-09-01T10:30:00", "2021-09-02T14:45:00".
moodCodeValue varchar This column stores the mood code value associated with the activity. It is of type varchar(100) and can have a default value of NULL. Example: "inent", "request", etc.
code varchar A code representing the type of observation. It is of type varchar(100).
statusCode varchar The status code of the observation. It is of type varchar(100). Example: "completed", "in-progress", "cancelled".
referenceText text A long text field containing reference information for the observation. It is of type longtext. Example: "Patient was feeling dizzy.", "Patient reported chest pain."
observationValue text A long text field containing the actual value of the observation. It is of type longtext. Example: "120/80", "98.6°F", "72 bpm".
authorId varchar A unique identifier for the author of the observation. It is of type varchar(50). Example: "auth001", "auth002", "auth003".
authorName varchar The name of the author of the observation. It is of type varchar(100). Example: "Dr. John Smith", "Nurse Jane Doe".

EHI Export Schema
Table rpm_healow_tracker_list
Description This table stores information about different types of health trackers used in the healow application.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT. This column stores integer value which could grow and hence integer is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index.
tracker_type varchar Contains the Tracker Type to store the Device Types such as glucose, bmi, bp, heartRate, spo2 etc. This field will always have a value hence defined as NOT NULL. There is a unique key constraint - UX_rpm_healow_tracker_list_tracker_type is added  on this column. The Tracker Type name cannot have a name more than 60 characters long, therefore the data type is VARCHAR(60).
display_name varchar Contains the Tracker Name to be displayed on the UI. This column contains a meaningful name of Tracker Type. There is no index associated with this field. The Tracker Display Name cannot have a name more than 255 characters long, therefore the data type is VARCHAR(255).
description varchar Contains the Tracker Description with more details of specific Tracker Type. This field can have a NULL value hence this is a NULLable field. As the column has no limit and we have free text, therefore we have defined it as VARCHAR(MAX) / TEXT.
tracker_order int Contains the Tracker Order which is used at Healow side. This field is not used at EMR side.  This field can have a NULL value. However, the default value of this field is 0. This field does not have any index associated with it. The data type is INT as the data structure of This column is designed in reference to the healow tables.
vitalType varchar Contains the EMR Vital type which can be used for measurement of this Tracker Type. This field can have a NULL value. This field does not have any index associated with it. The expected values for this field can be glucose, BMI, sleep, distance, steps, BP and so on. The value of this field cannot have a name more than 255 characters long, therefore the data type is VARCHAR(255).
delflag tinyint Flag indicates the record added is deleted, datatype is tinyint - 0: not deleted 1: deleted. This field will always have a value hence defined as NOT NULL. By default, this column should not be in Deleted status hence default 0 is chosen.

Table prisma_encounter_service_location
Description This table stores information about service locations related to specific encounters.
Column DataType Description
id (Primary Key) bigint Unique identifier for each record in the table. Example: 1, 2, 3.
encounterDetailId bigint Foreign key that refers to the `id` column in the `prisma_encounter_details` table. It represents the encounter to which the service location is related. Example: 101, 102, 103.
serviceLocationId varchar Unique identifier for the service location. Example: "SL001", "SL002", "SL003".
locationName varchar Name of the service location. Example: "Main Hospital", "Urgent Care Center", "Primary Care Clinic".
locationTypeCode varchar Code representing the type of service location. Example: "HOSP", "UCC", "PCC".
locationTypeName varchar Name of the service location type. Example: "Hospital", "Urgent Care", "Primary Care".
addressLine1 varchar First line of the service location's address. Example: "123 Main St".
addressLine2 varchar Second line of the service location's address (optional). Example: "Suite 200".
city varchar City where the service location is situated. Example: "New York", "Los Angeles", "Chicago".
state varchar State where the service location is situated. Example: "NY", "CA", "IL".
postalCode varchar Postal code of the service location. Example: "10001", "90001", "60601".
country varchar Country where the service location is situated. Example: "USA", "Canada", "UK".
workPhone varchar Work phone number of the service location. Example: "(555) 123-4567".
workFax varchar Work fax number of the service location (optional). Example: "(555) 987-6543".
startDate datetime Date and time when the service location started being related to the encounter. Example: "2021-01-01 10:00:00".
endDate datetime Date and time when the service location stopped being related to the encounter (optional). Example: "2021-01-10 18:00:00".

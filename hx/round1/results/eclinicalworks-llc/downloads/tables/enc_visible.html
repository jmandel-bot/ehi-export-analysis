EHI Export Schema
Table enc_visible
Description This table stores information blocking details of the given encounterID.
Column DataType Description
id (Primary Key) int Primary key of enc_visible table. This column stores numeric data which could grow. As such, the integer range: 0-2,147,483,647 is selected.
encid int This column is used to store the encounter id which the block encounter details are associated to. The column refers to the encounterid column in enc table. This column stores numeric data which could grow with the number of visits over the years. As such the integer range 0-2,147,483,647 is selected.
notshow tinyint This column is used to store block encounter status. Possible values of this column are 0 or 1 hence tinyint is selected. The default value of this column will be 0. If a value is set to 1 then the associate encounter is blocked for the Patient Portal.
UID int This column is used to store the user id of the user who has blocked the encounter to publishing. This column is used to store the user id of the user who has blocked the encounter to publishing.This column refers to the UID column in the user table.
exceptionreasonid int This column is used to store the exception reason id. This column refers to the exceptionReasonid column in the information_blocking_reasons table. The default value of this column will be NULL. If an encounter is not blocked for the Patient Portal this column is set to NULL.
exceptionreasonComment varchar This column is used to store the reason of why enc is restricted or blocked for the Patient Portal. The default value of this column will be BLANK. As per functionality, a user can insert a reason up to 255 characters.
inserttime datetime This column is used to store updated timestamp whenever a record is inserted or updated. The default value of this column will be NULL.

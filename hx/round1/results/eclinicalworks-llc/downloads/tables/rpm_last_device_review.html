Table rpm_last_device_review
Description This table maintains the last reading reviewed by the user
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT. This column stores integer value which could grow with number of users added to the care team and hence integer is selected . It is AUTO_INCREMENT column and has PRIMARY KEY index
patientid int Contains the Primary Key column UID of user table. It stores the unique id of a Patient and its data type is INT. The default value of this field is not provided. This field will always have a value hence defined as NOT NULL. There is a Foreign Key constraint - FK_rpm_last_device_review_patientid linked to this field referencing to UID column of user table.
deviceTypeid int Contains deviceTypeid value referenced from id column of the rpm_healow_tracker_list table, datatype is INT. This field will always have a value hence defined as NOT NULL. The data type of id column of the rpm_healow_tracker_list table is INT. Therefore, the data type of this field is INT.  There is a Foreign key constraint linked to this field - FK_rpm_last_device_review_deviceTypeid referencing to id column of rpm_healow_tracker_list table.
readingDate datetime Contains the actual reading date time of the patient and datatype is DATETIME. By default, current system datetime needs to be inserted. This field will always have a value hence defined as NOT NULL.
modifiedBy int This column stores the User id who has modified the recoed last and its data type is INT. The value of this field cannot be NULL. There is a Foreign Key constraint - FK_rpm_last_device_review_modifiedBy referencing to UID column of users table.
modifiedDatetime datetime Contains "modifiedDatetime " value when a user is added/updated from into a care team and datatype is DATETIME.
deleteFlag tinyint Flag indicates the last reading added is deleted, datatype is tinyint - 0 - Not Deleted, 1 - Deleted. This field will always have a value hence defined as NOT NULL. By default, this column should not be in Deleted status hence default 0 is chosen.

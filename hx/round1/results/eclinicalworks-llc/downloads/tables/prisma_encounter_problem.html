Table prisma_encounter_problem
Description This table stores information about problems encountered during a patient's medical encounter, such as diagnoses or conditions.
Column DataType Description
id (Primary Key) bigint A unique identifier for each record in the table. Example: 1, 2, 3.
encounterDetailId bigint A foreign key that refers to the `id` column in the `prisma_encounter_details` table. It represents the specific encounter detail associated with the problem. Example: 101, 102, 103.
problemTypeCode varchar A short code representing the type of problem encountered. Example: "PTC1", "PTC2", "PTC3".
problemType varchar A more descriptive name for the problem type. Example: "Chronic Condition", "Acute Illness", "Injury".
problemCode varchar A code representing the specific problem or diagnosis. Example: "P001", "P002", "P003".
problemname varchar The name of the specific problem or diagnosis. Example: "Diabetes", "Hypertension", "Fracture".
problemText varchar A textual description of the problem or diagnosis. Example: "Type 2 Diabetes Mellitus", "Stage 2 Hypertension", "Closed fracture of the left radius".
authorId varchar An identifier for the healthcare professional who recorded the problem. Example: "A001", "A002", "A003".
authorName varchar The name of the healthcare professional who recorded the problem. Example: "Dr. John Smith", "Nurse Jane Doe", "Dr. Alice Johnson".
diagnosisConditionReference varchar A reference to an external system or standard for the diagnosis or condition. Example: "ICD10:E11.9", "SNOMED:38341003", "ICD10:I10".
diagnosisConditionDisplay varchar A display-friendly version of the diagnosis or condition reference. Example: "ICD-10: E11.9 - Type 2 diabetes mellitus without complications", "SNOMED CT: 38341003 - Hypertension", "ICD-10: I10 - Essential (primary) hypertension".

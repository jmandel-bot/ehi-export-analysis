EHI Export Schema
Table asc_devicetestdata
Description This table stores the device test data for patients, including raw and parsed data, along with other related information.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3, etc.
patientid int identifier for the patient associated with the device test data. Example: 101, 102, 103, etc.
encounterid int identifier for the encounter associated with the device test data. Example: 1001, 1002, 1003, etc.
ascencounterid int identifier for the ASC (Ambulatory Surgery Center) encounter associated with the device test data. Example: 2001, 2002, 2003, etc.
deviceid int identifier for the device that generated the test data. Example: 301, 302, 303, etc.
devicerawdata text Raw data collected from the device in text format. Example: "123,456,789", "987,654,321", etc.
deviceparseddata text Parsed data from the device in text format. Example: "Heart Rate: 80 bpm, Blood Pressure: 120/80", "Temperature: 98.6 F", etc.
receiveddatetime datetime Date and time when the device test data was received. Example: '2021-09-01 12:34:56', '2021-09-02 14:30:00', etc.
processeddatetime datetime Date and time when the device test data was processed. Example: '2021-09-01 13:00:00', '2021-09-02 15:00:00', etc.
reconcile int Integer value indicating whether the data has been reconciled or not. Example: 0 (not reconciled), 1 (reconciled).
STATUS varchar Status of the device test data record. Example: 'Pending', 'Processed', 'Error', etc.
usrid int identifier for the user who processed the device test data. Example: 401, 402, 403, etc.
delflag tinyint Tiny integer value indicating whether the record has been deleted or not. Example: 0 (not deleted), 1 (deleted).
devicemonitorid int identifier for the device monitor associated with the device test data. Example: 501, 502, 503, etc.

EHI Export Schema
Table ip_rh_homemedication
Description This table is used to store all documented home medication data.
Column DataType Description
homemedid (Primary Key) int This is auto-incremented primary key in this table.
Encounterid int This column is used to store the encounterid and is a foreign key referencing the id column in the enc table.
itemid int This column is used to store the item id of the drug which is added as home medication for the patient. This is required to identify the exact drug name for the patient's home medication.
doctorsflag smallint This column is used to store doctor flag value which represents the home med drug status. It stores values like: 1 - Taking 2 - Not Taking 4 - Unknown
displayindex tinyint This column is used to store display index.
strength varchar This column is used to store strength value.
formulation varchar This column is used to store formulation value.
route varchar This column is used to store route values.
frequency varchar This column is used to store frequency value.
Take varchar This column is used to store take value.
Refill smallint This column is used to store refill value that represents the home med refill value.
islongterm tinyint This column is used to store the flag value of long term. Possible Values: 0 - not long term 1 - long term
startdate date This column is used to store start date of home medication drug.
stopdate date This column is used to store stop date of home medication drug.
Ndc varchar This column is used to store NDC code of drug.
rxorderno varchar This column is used to store RX order no code of drug.
prescribedby varchar This column is used to store prescriber's first name and last name.
prescribedbyid int This column is used to store prepared by user Id. This column is the foreign key of users table uid column which used to store userId.
loggedinuserid int This column is used to store logged in user Id.This column is the foreign key of users table uid column which used to store userId.
modifiedbyid int This column is used to store modified by user Id.This column is the foreign key of users table uid column which used to store userId.
modifieddatetime datetime This column is used to store modified date and time in yyyy-mm-dd hh:mm:ss format.
lasttaken datetime This column is used to store last taken date and time in yyyy-mm-dd hh:mm:ss format.
Notes varchar This column is used to store notes data.
discontinuestopnotes varchar This column is used to store discontinue notes data.
received tinyint This column is used to store the flag value for received. The default value is 0. Possible values: 0 - not received 1 - received
receivedByUserId int This column is used to store home med received by user Id.This column is the foreign key of users table uid column which used to store userId.
receivedDualVerify tinyint This column is used to store the flag value for received dual verify. The default value is 0. Possible values: 0 - not selected 1 - selected
returned tinyint This column is used to store the flag value for returned. Possible values: 0 - not returned 1 - returned
returnedByUserId int This column is used to store home med returned by user Id. This column is the foreign key of users table uid column which used to store userId.
returnedDualVerify tinyint This column is used to store the flag value for returned dual verify. The default value is 0. Possible values: 0 - not selected 1 - selected
disMedStatus smallint This column is used to store discontinued home medication status. The default value is NULL as this field is not mandatory. It stores value like -1 - No action, 0 - Stop, 1 - Start, 2 - Continue, 100 - Hold, -2 - Stopped, -3 - Started, -4 - Continued, -5 - Holded,
ipOrderStatus smallint This column is used to store patient order status. The default value is NULL as this field is not mandatory. Possible values: -1 - Do not order, 1 - Order, 2 - Do not discontinue, 3 - Discontinue, 4 - Modify, 5 - Keep, 6 - Auto Discarded.
deleteFlag tinyint This column is used to maintain whether record is removed by user or not. The default is 0 as the record is not deleted initially. Desired / expected value is as per below, 0 - this order record is not deleted 1 - this order record is deleted.

Table report_writer_sentence
Description This table stores information about sentences used in report writing, including their order, relationships to other sentences, and whether they are triggered or child sentences.
Column DataType Description
sentenceid (Primary Key) int This is the primary key and an auto-incrementing integer that uniquely identifies each sentence in the table.
notes varchar This column is used to store the sentence. As per requirement, user can not enter NULL value. Hence, not NULL constraint is suitable here. As per requirements, user can add up to 40,000 characters for each sentence. Hence, Text/varchar*max) column type is suitable here.
sectionid int This column is used to store id of section table. Each sentence should associated with particular section. Hence, not NULL constraint is suitable here. This column is a foreign key that references to report_writer_section (sectionid). Sectionid column of report_writer_section is INT type. Hence, INT datatype is suitable here.
sentenceOrder int This column is used to store sequence number of sentence. Each sentence should have associated order index. Hence, not NULL constraint is suitable here. Each section can have multiple sentences and usesr can change order of each sentence. Hence, this column has INT datatype.
isTriggeredSentence tinyint This column is used to store triggered sentence details. As per requirement by default sentence should not be triggered sentence. Hence, default value of this column is 0. Each sentence should have 0 or 1 value. Hence, not NULL constraint is suitable here. 0: Sentence is not a triggered sentence. 1: Sentence is triggered sentence. This column can have only 0 or 1 value. Hence, TINYINT is suitable for this column.
isChildSentence tinyint This column is used to store details about child sentence. As per requirement, by default, sentence should not be child sentence. Hence, default value of this column is 0. Each sentence should have 0 or 1 value. Hence, not NULL constraint is suitable here. 0: Sentence is not a child sentence  1: Sentence is child sentence This column can have only 0 or 1 value. Hence, TINYINT is suitable for this column.
parentSentenceid int This column is used to store parent sentence id if this sentence is child sentence. The default value of this column is NULL. If a child sentence has a column value as 1, then only this column will have a parent sentence id. This column will have sentenceid. Hence, INT datatype is suitable here.
deleteFlag tinyint This column is used for soft deleting records, the value should be 1 or 0. By default all sentence should not be deleted. Hence, default value of this column is 0. 1 : Deleted 0 : Not Deleted This column can have only 0 or 1 value. Hence, TINYINT is suitable for this column.

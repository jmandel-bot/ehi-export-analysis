Table crc_alerttypes
Description This table stores information about different types of alerts, their descriptions, and various flags to indicate their properties and visibility.
Column DataType Description
id (Primary Key) int This is the primary key column, which stores a unique integer value for each alert type. For example, 1, 2, 3, etc.
alerttype varchar This column stores a short string (varchar) representing the type of alert. For example, "TFLEX", "AFLEX", "TATEX", etc.
description varchar This column stores a longer string (varchar) that provides a brief description of the alert type. For example, "System warning", "Data error", "Informational message", etc.
deleteflag tinyint This is a tinyint column that indicates whether the alert type has been deleted or not. A value of '0' means the alert type is not deleted, and a value of '1' means it is deleted.
alertCategory varchar This column stores a short string (varchar) representing the category of the alert. For example, "system", "data", "user", etc.
isActivePractiveView tinyint This is a tinyint column that indicates whether the alert type is active in the Practice View or not. A value of '0' means the alert type is not active, and a value of '1' means it is active.
isActiveRPView tinyint This is a tinyint column that indicates whether the alert type is active in the RP View or not. A value of '0' means the alert type is not active, and a value of '1' means it is active.
isAdvanced tinyint This is a tinyint column that indicates whether the alert type is considered advanced or not. A value of '0' means the alert type is not advanced, and a value of '1' means it is advanced.

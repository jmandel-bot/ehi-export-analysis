EHI Export Schema
Table telenc
Description This table stores information about Telephone and Web Encounters, including details about the caller, message, action taken, and assignment.
Column DataType Description
Ticketid (Primary Key) int Unique identifier for each telephonic encounter record. Example: 1, 2, 3.
encounterid int identifier for the encounter associated with the telephonic encounter. Example: 101, 102, 103.
UID int identifier for the user who created the telephonic encounter. Example: 1001, 1002, 1003.
Caller varchar Name or description of the caller. Example: "John Doe", "Jane Smith".
message text Text of the message received during the telephonic encounter. Example: "Patient is experiencing severe pain."
actiontaken text Text describing the action taken in response to the telephonic encounter. Example: "Prescribed pain medication."
priority tinyint Priority level of the telephonic encounter, with 0 being the default. Example: 0, 1.
assignedTo varchar Name of the person the telephonic encounter is assigned to. Example: "Dr. Johnson", "Nurse Alice".
assignedToid int identifier for the person the telephonic encounter is assigned to. Example: 2001, 2002, 2003.
pmcid int Stores id for pharmacy, patient has selected during telephone encounter. Column refers to pmcid column from pharmacy table.
virtualflag tinyint Flag indicating if the telephonic encounter is virtual, with 0 being the default. Example: 0, 1.
WebStatus text This column is not used by application anymore hence default value NULL is stored.
TimeTaken varchar This column is not used by application anymore hence it stores the assigned default value.
telencLock tinyint This column stores addressed or open status of telephone or web encounter. 0 - Open, 1 - Addressed / Addressed and Docs Reviewed
teldeleteflag tinyint This column stores Telephone encounter delete status. 0 - Not Deleted 1 - Deleted encounter.
telencType tinyint This column stores encounter type of the current visit. 2 - Telephone Encounter and Virtual Visit, 6 - Web Encounter.
inpatientflag tinyint This column stores flag 0 or 1. 1 - if the visit is inPatient , 0 - Otherwise.
refReqid int This column stores an id of the incoming referral request for the encounter from encounterid column. This column refers to column Reqid from nhxtelenc table.
source tinyint Source of the telephonic encounter, with 0 being the default. Example: 0, 1, 2.

EHI Export Schema
Table enc_multiappt
Description This table contains the relationship between encounterId and guId column with enc table
Column DataType Description
id (Primary Key) int This is the primary key for the table, which is an integer value that is automatically incremented for each new record. It uniquely identifies each record in the table. Example: 1, 2, 3, etc.
encounterID int This column stores the integer value of the encounter ID, which is a unique identifier for an encounter. It can be used to establish a foreign key relationship with another table that contains encounter details. Example: 101, 102, 103, etc.
guId varchar This column stores a varchar(255) value representing the globally unique identifier (GUID) for the multiple appointments. This is a unique alphanumeric string that can be used to identify the appointments across different systems. Example: '3F2504E0-4F89-11D3-9A0C-0305E82C3301', '6BDD62F2-6B4F-4EB7-A2F7-7E6C684D4439', etc.

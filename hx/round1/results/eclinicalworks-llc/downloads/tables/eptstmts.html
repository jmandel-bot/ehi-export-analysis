EHI Export Schema
Table eptstmts
Description This table stores electronic patient statements, including statement details, patient information, and other related data.
Column DataType Description
statementid int An integer representing the unique identifier for the statement. Example: 12345.
statementdate datetime A date representing the date the statement was created. Example: '2021-09-01'
patientid int An integer representing the unique identifier for the patient associated with the statement. Example: 67890
statementXml text A longtext column containing the XML data of the statement. Example: '...'
totalBalance decimal A decimal (9.2) representing the total balance of the statement. Example: 250.75
reminded int An integer(1) indicating whether a reminder has been sent for the statement (1 for yes, 0 for no). Example: 0
remindedDate datetime A date representing the date the reminder was sent, if applicable. Example: '2021-09-15'.
uploadid varchar Not Available
createdtime datetime A time stamp representing the date and time the record was created or last updated. Example: '2021-09-01 12:34:56'.
Practiceid int An integer(16) representing the unique identifier for the medical practice associated with the statement. Example: 98765.
Relid int An integer (11) representing a relationship identifier, possibly related to another table. Example: 54321.
StatementType int An integer(2) representing the type of statement (e.g., 0 for regular, 1 for overdue). Example: 0
ePtStmtSlNo (Primary Key) int An integer(11) representing the auto-incremented primary key for the table. Example: 1
websync tinyint A tinyint(4) indicating whether the statement has been synced with a web service (1 for yes, 0 for no). Example: 0
OptOutPtStmt tinyint A tinyint(2) indicating whether the patient has opted out of receiving statements (1 for yes, 0 for no). Example: 0
PtStmtBillingFacilityid int An integer(11) unsigned representing the unique identifier for the billing facility associated with the statement. Example: 23456

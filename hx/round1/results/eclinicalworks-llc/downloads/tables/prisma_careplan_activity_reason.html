Table prisma_careplan_activity_reason
Description This table stores the reasons for care plan activities, which can be either a condition or a diagnostic.
Column DataType Description
ID (Primary Key) int This is the primary key for the table, and it is an auto-incrementing integer value. It uniquely identifies each record in the table. Example: 1, 2, 3, etc.
CAREPLAN_ACTIVITY_ID int This is a foreign key column that refers to the `ID` column in the `prisma_careplan_activity_details` table. It represents the care plan activity for which the reason is being stored. Example: 5, 6, 7, etc.
CONDITION_ID int This is an optional integer column that stores the ID of the condition that serves as the reason for the care plan activity. If the reason is a diagnostic, this column will be NULL. Example: 10, 11, 12, etc.
DIAGNOSTIC_ID int This is an optional integer column that stores the ID of the diagnostic that serves as the reason for the care plan activity. If the reason is a condition, this column will be NULL. Example: 15, 16, 17, etc.

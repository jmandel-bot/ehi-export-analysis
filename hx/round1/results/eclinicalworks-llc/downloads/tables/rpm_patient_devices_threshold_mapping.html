Table rpm_patient_devices_threshold_mapping
Description This table stores the device threshold mapping values for trackers for the patient.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT. This column stores integer value which could grow with number of RPM patient devices threshold mapping values added and hence integer is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
patientid int This column stores unique id of a Patient which references the UID column of the user table. The data type chosen is INT. This field will always have a value hence defined as NOT NULL.
deviceTypeid int Contains " device_Typeid" value referenced from id column of the rpm_healow_tracker_list table, datatype is INT. This field will always have a value hence defined as NOT NULL
abnormal_low1 varchar Contains the abnormal low value for the bmi,tempreture,spo2,heart rate, blood pressure and blood glucose device and data type is VARCHAR(10). This column stores characters which cannot exceed 10 characters hence VARCHAR(10) is selected and default value is NULL
abnormal_high1 varchar Contains the abnormal high value for the bmi,tempreture,spo2,heart rate, blood pressure and blood glucose device and data type is VARCHAR(10). This column stores characters which cannot exceed 10 characters hence VARCHAR(10) is selected and default value is NULL
baseline_low1 varchar Contains the baseline low value for blood pressure, blood glucose, bmi, tempreture, spo2 and heartrate device and data type is VARCHAR(10). This column stores characters which cannot exceed 10 characters hence VARCHAR(10) is selected and default value is NULL
baseline_high1 varchar Contains the baseline high value 2  for the device and data type is VARCHAR(10). This column stores characters which cannot exceed 10 characters hence VARCHAR(10) is selected and default value is NULL
abnormal_low2 varchar Contains the abnormal low value for blood glucose,bmi,spo2 device and data type is VARCHAR(10). This column stores characters which cannot exceed 10 characters hence VARCHAR(10) is selected and default value is NULL
abnormal_high2 varchar Contains the abnormal high value for blood glucose,spo2,bmi device and data type is VARCHAR(10). This column stores characters which cannot exceed 10 characters hence VARCHAR(10) is selected and default value is NULL
baseline_low2 varchar Contains the baseline low value for blood glucose,spo2,bmi device and data type is VARCHAR(10). This column stores characters which cannot exceed 10 characters hence VARCHAR(10) is selected and default value is NULL
baseline_high2 varchar Contains the baseline high value 2  for the device and data type is VARCHAR(10). This column stores characters which cannot exceed 10 characters hence VARCHAR(10) is selected and default value is NULL
targeted_calories int Contains the value of the calories count targeted for the patient and data type is INT and default value is NULL
targeted_distance int Contains the value of the walk distance targeted for the patient and data type is INT and default value is NULL
targeted_floors int Contains the value of the floor climbing count targeted for the patient and data type is INT and default value is NULL
targeted_steps int Contains the value of the steps count targeted for the patient and data type is INT and default value is NULL
targeted_sleep int Contains the value of the sleep count targeted for the patient and data type is INT and default value is NULL
measure_id int Contains "measure_id" value referenced from id column of the `rpm_device_measure ` table, datatype is INT. This field will always have a value hence defined as NOT NULL
measure_id1 int Contains the rpm device measure id and data type is INT. This field will always have a value hence defined as NOT NULL
modifiedBy int Contains the UID value referenced from UID column of user table and data type is INT. This field will always have a value hence defined as NOT NULL.
modifiedDatetime datetime Contains the date time when the record was last modified and data type is DATETIME. This column stores datetime hence Datetime is selected. This field will always have a value hence defined as NOT NULL.
deleteFlag tinyint Flag indicates patient device threshold mapping data is Deleted or not, datatype is tinyint - 0 - Not Deleted, 1 - Deleted. This field will always have a value hence defined as NOT NULL. By default, patient device threashold data should not be in Not Deleted status hence default 0 is chosen.
bmi_height decimal Contains a patient height value for BMI device.

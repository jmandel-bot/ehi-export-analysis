EHI Export Schema
Table labdataex
Description This table stores information about laboratory data, including report details, processing status, and sending options.
Column DataType Description
reportid (Primary Key) int This column stores a primary key. This column stores the ID of the lab and refers to the reportid column in the labdata table.
HL7ResultDate datetime This column stores the result date in the Inbound Lab HL7 result message received from the vendor through the interface. The format used is YYYY-MM-DD HH:MM:SS.
HL7ResultTime datetime This column stores the result time in the Inbound Lab HL7 result message received from the vendor through the interface. The format used is YYYY-MM-DD HH:MM:SS.
urllink text This column stores the PACS link that is received in the Inbound Lab/DI HL7 result messaage from the vendor through the interface or that is pre-defined in the labitemkey for the interface implementation along with extra parameters (like MrnNumer, Accession id, etc.) that get added when the result is processed.
interfacestatus varchar This column stores the textual status of the lab/DI order sent or result received through the interface.
processedresult text This column stores the XML format data for displaying the pink paper clip.
VisitType varchar This column stores the visit type that may be received in the Inbound Lab HL7 result message from the vendor through the interface or can be inserted manually from the user interface.
DateReceived datetime This column stores the date and time when the lab/DI HL7 result message is received from the vendor through the interface. The format used is YYYY-MM-DD HH:MM:SS.
resultrcvddatetime datetime This column stores the date when the result is received. The format used is YYYY-MM-DD HH:MM:SS.
MigrateFlag tinyint Column not in use
SentOutReqNo varchar This column stores the Requisition Number sent in the Outbound Lab/DI Orders HL7 message to the vendor through the Orders Generic Interface.
facilityid int This column stores the facility ID. This value can come in the HL7 message and may be inserted/updated through the interface or manually. This column refers to the id column from from the edi_facilities table.
specimenReceiveTime datetime This column stores the time when the specimen (e.g., blood sample or urine sample) was received. This value can come in the HL7 message and may be inserted/updated through the interface or manually.
specimenReceiveDate datetime This column stores the date when the specimen (e.g., blood sample or urine sample) was received. This value can come in the HL7 message and may be inserted/updated through the interface or manually. The format used is YYYY-MM-DD HH:MM:SS.
isPACSLinkEnabled tinyint This column stores a flag 0 or 1, where 0 indicates the PACS button on the DI Result window is disabled and 1 inidcates it is enabled. This column is updated based on a lab item key setting when a Lab/DI Inbound HL7 Order Update (ORM) message is received from the vendor through the interface.
observationDate datetime This column stores the observation date received in the Inbound Lab Results HL7 message from the vendor through the interface. The format used is YYYY-MM-DD HH:MM:SS.
observationTime datetime This column stores the observation time received in the Inbound Lab Results HL7 message from the vendor through the interface. The format used is YYYY-MM-DD HH:MM:SS.
SpecimenActionCode varchar This column stores the Specimen Action Code received in the Inbound Lab Results HL7 message from the vendor through the interface.
PlacerGroupNumberid varchar This column stores the Placer Group Number ID (unique ID) received in the Inbound Lab Results HL7 message from the vendor through the interface.
PatientAltRace varchar This column stores the race of the patient received in the Inbound Lab Results HL7 message from the vendor through the interface.
bodysitecode varchar This column stores the value of the Body Site field on the result window for DI and procedure orders only. This refers to the bodysitecode column from the bodysites table.
ABNPrompted tinyint This column stores values 0 or 1 to indicate whether the Advanced Beneficiary Notice (ABN) has been prompted or not while ordering Lab or Diagnostic Imaging. 0 indicates that ABN was not prompted and 1 indicates ABN was prompted.
ABNOption tinyint This column stores a value 1, 2, or 3 to indicate which option has been selected by the end user when ABN was prompted. 1 indicates I agree, bill my insurance, 2 inidcates I agree, DO NOT bill my insurance, and 3 indicates I DO NOT want these tests. These mappings are stored in the abnoptions table.
AQRejectionReason varchar This column stores the reason documented while rejecting pending orders.
exceptionReasonid int This column is used to store the Exception Reason ID. The default value of this column is NULL. If the encounter is not blocked for the eClinicalWorks Patient Portal, this column is set to NULL. This column refers to the exceptionReasonid column in the information_blocking_reasons table. This column stores numeric data which may increase, so it is an integer data type.
exceptionReasonComment varchar This column is used to store the reason why an encounter is restricted/blocked for the eClinicalWorks Patient Portal. As per the functionality, users can document up to 255 characters for the reason. The default value of this column is empty.
movedOrderFlag tinyint This column is used to store the status of a moved order. By default, this column value is 0. The value 1 indicates the order was moved from a Current Order to a Future Order, and the value 2 indicates the order was moved from a Future Order to a different future date.
SentOutReqNoOrdNum int This column stores the generated requisition number. By default, this column value is 0.
printlabid int This column stores the same requisition number if two future orders on the same date were placed through different interfaces. The default value of this column is 0.
SendToMode varchar This column stores the mode for sending the lab report (e.g., Print, Fax, eTransmit, eTransmitOnly).
SendToName varchar This column stores the name of the recipient for the lab report (e.g., John Doe).
SendToFaxNumber varchar This column stores the fax number of the recipient for the lab report (e.g., 555-123-4567).
FaxLogid int This column stores the unique identifier for the fax log (e.g., 1, 2, 3) with a foreign key relation to faxlogs.Logid.
TransmitLabid int This column stores the unique identifier for the transmitedt lab (e.g., 1, 2, 3) with a foreign key relation to lablist.id.
SendToStatus tinyint This column stores the status of sending the lab report (e.g., 1 for sent, 2 for pending, 3"for failed, and 4 for not sent).
sendToAddressDetails varchar This column stores the address details of the recipient for the lab report (e.g., 123 Main St, City, State, ZIP code).

EHI Export Schema
Table eicr_transaction_logs
Description This table stores the Electronic Case Reporting transaction logs for electronic health record (EHR) encounters, including information about the user, transaction type, status, and other related details.
Column DataType Description
id (Primary Key) int Unique identifier for each transaction log entry. Example: 1, 2, 3.
encounter_id int Identifier for the encounter associated with the transaction log. Foreign key relation to `enc` table's `encounterID` column. Example: 101, 102, 103.
user_id int Identifier for the user who performed the transaction. Foreign key relation to `users` table's `uid` column. Example: 1001, 1002, 1003.
transaction_type tinyint Numeric code representing the type of transaction. Example: 1 (Outgoing), 2 (Incoming)
transaction_id varchar Unique identifier for the transaction, represented as a string. Example: "TX123", "TX124", "TX125".
priority tinyint Numeric code representing the priority level of the transaction.
STATUS tinyint Numeric code representing the status of the transaction.
error_reason tinyint Numeric code representing the reason for a transaction failure, if applicable.
transaction_time datetime time stamp of when the transaction occurred. Example: "2021-09-01 12:34:56", "2021-09-02 14:23:45".
isResent tinyint Flag indicating whether the transaction has been resent (1) or not (0). Example: 0, 1.
SUBJECT varchar Subject of the transaction, represented as a string.
summary_preview_text varchar Preview text of the transaction summary, represented as a string.
transmission_mode tinyint Numeric code representing the mode of transmission for the transaction. Example: 1 (Automatic),2 (Manual).

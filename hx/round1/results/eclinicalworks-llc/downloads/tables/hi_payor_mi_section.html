EHI Export Schema
Table hi_payor_mi_section
Description This table is used to record information about member insights care gaps item is reviewed by user or not.
Column DataType Description
id int A unique identifier for each record in the table. Example: 1, 2, 3.
patientid int The identifier of the patient associated with the record. This is a foreign key that refers to the `users` table's `UID` column. Example: 101, 102, 103.
emrguid varchar Its an emr unique guid for every member insights pull from EMR.
sectionname varchar It is section name which is received from member insights.
sectiondata varchar It is section data which is received from member insights.
dataguid varchar Its an unique guid for data row exists in every member insights pull from EMR.
completed tinyint A flag indicating whether the record has been completed (1) or not (0). Example: 0, 1.
payorname varchar It is payor name from where member insights data recieved of patient
externalalerts_id int The identifier of the external alerts associated with the record. This is a foreign key that refers to the `externalalerts` table's `id` column. Example: 101, 102, 103.
createdby int The identifier of the user who has taken action on member insights care gaps items from todo > all gaps. This is a foreign key that refers to the `users` table's `UID` column. Example: 101, 102, 103.
createdbyname varchar Username who has took action on member insights care gaps items from todo > all gaps.
createddate datetime The date and time when the record was created. Example: "2021-01-01 12:00:00", "2021-02-15 14:30:00", "2021-03-20 10:45:00".
updatedby int The identifier of the user who has updated action on member insights care gaps items from todo > all gaps. This is a foreign key that refers to the `users` table's `UID` column. Example: 101, 102, 103.
updatedbyname varchar Username who has updated action on member insights care gaps items from todo > all gaps.
updateddate datetime The date and time when the record was last updated. Example: "2021-01-02 12:00:00", "2021-02-16 14:30:00", "2021-03-21 10:45:00".
deleteflag tinyint A flag indicating whether the record has been deleted (1) or not (0). Example: 0, 1.

Table prisma_immunization_performer
Description This table stores information about the performers involved in the immunization process, including their practitioner ID, role, and display name.
Column DataType Description
ID (Primary Key) int This is the primary key for the table, and it is an integer value that is automatically incremented for each new record. For example, the first record will have an ID of 1, the second record will have an ID of 2, and so on.
IMMUNIZATION_ID bigint This is a foreign key column that refers to the `id` column in the `prisma_immunization_details` table. It is a bigint value that represents the unique identifier of the immunization record associated with the performer. For example, if the immunization record has an ID of 100, the `IMMUNIZATION_ID` will also be 100.
PRACTITIONER_ID int This is an optional integer column that stores the unique identifier of the practitioner who performed the immunization. For example, if the practitioner has an ID of 50, the `PRACTITIONER_ID` will be 50.
PRACTITIONER_ROLE_ID int This is an optional integer column that stores the unique identifier of the practitioner's role during the immunization process. For example, if the practitioner's role has an ID of 10, the `PRACTITIONER_ROLE_ID` will be 10.
PERFORMER_DISPLAY varchar This is an optional varchar column with a maximum length of 255 characters that stores the display name of the performer. For example, if the performer's display name is "Dr. John Doe", the `PERFORMER_DISPLAY` will be "Dr. John Doe".

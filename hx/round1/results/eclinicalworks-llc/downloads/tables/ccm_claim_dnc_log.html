Table ccm_claim_dnc_log
Description This table stores information about patient who opted for Do Not Create Claim in PHM Billing batch process. It stores CCM batch Id, CCM Batchdetails Id and modified information.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer(Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
batchid int Contains auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer(Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index.
batchDetailid int This column stores unique id of a ccm batch details which references the id column of the ccm_batchdetails table. It refers to the batch data for patients of specific billing month. The data type chosen is INT. This column does not allow a NULL value.
modiBy int This column is used to store user id to identify which user modify the record last. It refers to primary key UID of user table. Data type is int(11).
modiTime datetime Contains the date and time when the record was last modified by a user. The datatype of this field is datetime. This field does not allow a NULL value.

EHI Export Schema
Table encounterdata
Description This table stores additional information about patient encounters, including medical history, notes, and appointment details.
Column DataType Description
encounterid (Primary Key) int This column stores id of an encounter for which additional information related to Patient's visit is documented. This column refers to the column encounterid from the enc table.
chiefComplaint text This column stores chief complaints seperated by new line character for the encounter.
currentMedication varchar This column is not being used.
HPINotes text Notes related to the patient's history, physical examination, and illness, stored as text.
ExamNotes text This column is not being used as of now. Column only has a null value.
AsmtNotes text This free text column stores Assessment Notes for a given encounter.
TreatNotes text This column is not being used as of now. Column only stores null values and no insert or update statement is found in application code base.
NextAppt text This column stores free text notes entered by a provider or staff regarding follow-up appointments of the patient.
NextApptOpt int This column is not being used.
LockEnc int Not in use.
AddtlNotes text This column is not being used.
pastHistory text This column stores medical history seperated by a new line character for the encounter.
AllergiesFlag char This column stores values "Y" or "N" to indicate the provider verified the allergies with the patient. A "Y" means the allergies verified, a "N" means allergies are not verified.
complaintsFlag char This column is not being used as of now. Columns always have NULL or ''" values.
historyFlag char This column stores values Y or N to indicate whether a provider or staff has verified the medical history of a patient or not. A "Y" means the medical history is verified, and an "N" means the medical history is not verified.
vitalsFlag char This column stores value Y or N to indicate whether vitals are recorded by staff or not. A "Y" means recorded and an "N" means not recorded.
medicationFlag char This column stores values Y or N to indicate the provider has confirmed the patient’s medication. A "Y" means confirmed and an "N" means not confirmed.
copay varchar This column stores payment amount received from the patient for an encounter.
paymentType varchar This column is not being used.
checkNo varchar This column is not being used.
notesstatus int This column stores values 0, 1, and 2. A “0” means the default value when record is inserted, a “2” means when an end user marked the Progress Notes as done.
displayIndex int This column stores a digit to indicate the order in which records should be listed in the user interface. For this column, the displayindex column is always set to 1.
HospitalizationFlag char This column stores values Y or N to indicate that the provider verified the patient’s hospitalization history. A "Y" means the history was verified, and an "N" means it was not verified.
SurgicalFlag char This column stores values Y or N to indicate that the provider verified the patient’s surgical history. A "Y" means verified and an "N" means not verified.
PregnantFlag char This column stores values "Y" or "N" to indicate that the patient is pregnant. A "Y" means the patient is pregnant, and an "N" means the patient is not pregnant.
BreastFeedingFlag char This column stores values Y or N to indicate whether the patient is breastfeeding. A “Y” means the patient is breastfeeding, and an “N” means the patient is not breastfeeding.
NonContributoryFlag char This column stores values Y or N to indicate that the patient's family conditions listed are not contributing to the patient's current condition in the case of an adopted child. A “Y” means contributing, and an “N” means not contributing.
SurgicalHistoryFlag char This column stores values Y or N to indicate whether patient has denied having surgical history. A "Y" means the patient denied having surgical history, an "N" means the patient has surgical history.
HospitalizationHistoryFlag char This column stores values Y or N to indicate if the patient denied prior hospitalization. A "Y" means they were denied prior hospitalization, and an"N" means the patient has hospitalization history.
chieficds text Column is not being used.
historyicds text This column stores the ICD code of a diagnosis a provider/staff has added as part of a patient's medical history.
AsmtNotesHeader text When 'enable_assessment_headernotes' itemkey is enabled, this column stores free text header notes from the Assessment window.
NextApptReason varchar This column stores the reason for next (follow-up) appointment as documented by a provider/staff.
FollowUpNA tinyint This column stores flags 0 or 1 to indicate whether a follow-up appointment is required a patient. A "0" means a follow-up appointment is required, and a "1" means a follow-up appointment is not required.
LockOrDoneDate datetime This column stores the date when a progress note of an encounter is locked or marked as done by the end user. The date is stored in 'YYYY-MM-DD 00:00:00.000' format.
SocialHistoryFlag char This column stores values Y or N to indicate the provider verified the patient’s social history. A "Y" means verified, and an "N" means not verified.
FamilyHistoryFlag char This column stores values Y or N to indicate the provider verified the patient’s family history. A "Y" means verified, and an "N" means not verified.
doneBy int This column is not being used.
doneDate datetime This column is not being used.
ImmunizationsVerified char This column stores values Y or N to indicate the provider verified the patient’s immunization records. A "Y" means verified and an "N" means records are not yet verified.
medreconciliationinternalflag char This column stores flags Y or N to indicate whether medication reconciliation has been performed or not. A “Y” means medication reconciliation was performed, and an “N” means medication reconciliation was not performed.
historyicdids text This column stores item id for a diagnosis added by a provider or staff as part of a patient's medical history. This column refers to the itemid column in the items table.
mddatelastseen datetime This column stores the date when the podiatric patient was last seen. This information is documented by a provider or staff on the Additional Billing Data window from the Billing window on the Progress Notes. The date format is: 'YYYY-MM-DD 00:00:00.000'.
cancellationReason varchar This column stores the free text reason documented by a staff member for the cancellation of an appointment.
transportationRequired tinyint This column stores values 0 or 1 to indicate if transportation is required for the patient or not. A "0" means transportation is not required, and a "1" means transportation is required.
transportationStatus varchar This column stores the status of transportation in case transportation is provided to the patient.
occHealthType tinyint This column stores values 0 or 1 to indicate whether the appointment is an occupational health appointment or not. A "0" means it is not an occupational health appointment, a “1” means the appointment is an occupational health appointment. This flag is used by the Occupational Health module of eCW.
NoReportedMedHx char This column stores values Y or N to indicate whether the patient denies reporting their medical history. A “Y” means the patient denies past medical history, a “N” means the past medical history of the patient is documented. This option is only used when item key: showDeniesPastMedicalHxOption is enabled.
NKDAFlag char This column stores values Y or N to indicate no known drug allergies (N.K.D.A) in the allergies/intolerance. A "Y" means there here are no known drug allergies, and an "N" means allergies have been documented.
trackingBoardNotes_status tinyint This column denotes notes status for each respective encounter. Possible values for this column are 0, 1, 2. Default is "0" which indicates that there are no notes for encounter. A "1" indicates there are notes which exist but one or many notes are not acknowledged. A "2" indicates all notes are acknowledged.
InitialTreatmentDate datetime This column stores the date when a patient received treatment for the first time which is used for billing purposes.
ImplantsFlag char This column stores values Y or N to indicate if information related to implants has been verified with the patient or not. A “Y” means implant related information is verified, and an “N” means implant related information is not verified.
InvRespPartyid int This column stores the id of a guarantor for an encounter. When overwriting a guarantor at the encounter or claim level this id takes precedence over the valueset in the patients table. This column refers to the UID in the user table.
InvRespPartyRelid tinyint This column stores the id for a relationship of a guarantor mentioned in InvRespPartyid with the patient whose encounter is being recorded.
InvIsGrPt tinyint This column stores values 0 or 1 to indicate whether the guarantor mentioned in InvRespPartyid column is also a registered patient with the practice or not. A "0" means that they are a registered patient, and a "1" means they are not a patient.
PLVerifiedBy int This column stores the id of the user who marked the patient's problem list as verified. This column refers to the UID column in the user table.
PLVerified tinyint This column stores values 0 or 1 to indicate whether the Problem List of the patient has been verified or not. A "0" means the problem list is not verified, and a "1" means the Problem List is verified by user.
PLVerifiedOn varchar This column stores the date and time when the problem list was marked as verified by the end user. Date and time is stored in: 'YYYY-MM-DD hh:mm:ss' format.
behavioralHistoryVerified tinyint Not Available.

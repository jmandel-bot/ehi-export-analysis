EHI Export Schema
Table aftercarestatuscodes
Description This table stores information about aftercare status codes, including their unique code, status, color, and whether they are billable or not.
Column DataType Description
codeid (Primary Key) int This is the primary key column, which is an integer that uniquely identifies each aftercare status code. It is auto-incremented, meaning that it will automatically increase in value as new records are added. Example values: 1, 2, 3, etc.
code varchar This column stores abbreviated codes for different statuses used in the AfterCare module.
status varchar This is a unique varchar column that stores the description of the aftercare status. Example values: Completed, In Progress, Cancelled, etc.
color varchar This is a unique varchar column that stores the color associated with each aftercare status for display purposes. Example values: red, blue, green, etc.
NonBillable tinyint This column stores values 0/1 to indicate whether the status is Billable or Non Billable.

EHI Export Schema
Table rpm_healow_tracker_bloodpressure
Description When a Provider prescribes a BP device for a patient, the patient will take readings from that device and will upload it to Healow. From healow, these readings are transferred to EMR side. We have created this new table rpm_healow_tracker_bloodpressure to store the Blood Pressure readings for the patients.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT. This column stores integer value which could grow with number of bloodpressure records added and hence integer is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index.
UID int This column stores unique id of a Patient which references the UID column of the user table. The data type chosen is INT. This field will always have a value hence defined as NOT NULL.
device_action int This column stores the Vendor id from whom the device has been purchased. Contains "deviceAction" value referenced from primary key id of rpm_vendors_list table and data type is INT.This column could be store high range numeric value hence integer is selected.The value of this field can be NULL.
systolic varchar This column stores the High Blood Pressure (systolic) and the data type is VARCHAR(255).This column stores characters that does not exceed 255 characters and hence VARCHAR(255) is selected. The value of this field can be NULL. The expected values for this field can be 80, 100, 120 etc. 
diastolic varchar This column stores the Low Blood Pressure (diastolic) and the data type is VARCHAR(255).This column stores characters that does not exceed 255 characters and hence VARCHAR(255) is selected. The value of this field can be NULL. The expected values for this field can be 80, 100, 120 etc. There is no index attached to this field.
heart_rate varchar This column stores the heart rate of patient and the data type is VARCHAR(255). This column stores characters that does not exceed 255 characters and hence VARCHAR(255) is selected. The value of this field can be NULL.The expected value for this field can be 60, 70, 80, 100 BPM.
measure_datetime datetime This column stores the date time when the measurement was done and the data type is DATETIME. This field will always have a value hence defined as NOT NULL.
delflag tinyint Flag indicates the record added is deleted, datatype is tinyint - 0 - Not Deleted, 1 - Deleted. This field will always have a value hence defined as NOT NULL. By default, this column should not be in Deleted status hence default 0 is chosen.
reviewed tinyint This column stores 0, 1 to determine whether the reading is reviewed or not and data type is TINYINT. This field will always have a value hence defined as NOT NULL and default value of this field is 0. Values 0 : Reading has not been reviewed by the Provider 1 : Reading has been reviewed by the Provider
reviewedByid int This column stores the Provider id who has reviewed the reading and data type is INT. Contains the reviewedByid that refers to UID column of users column.This field will always have a value hence defined as NOT NULL. 
reviewedDateTime datetime This column stores the date time when the reading was reviewed and the data type is DATETIME. The default value of this field can be NULL. This column stores datetime hence Datetime is selected.
isManuallyAdded tinyint This column stores 0, 1 whether the reading is manually added by patient or not and the data type is TINYINT. This field will always have a value hence defined as NOT NULL and the default value is 0. Values 0 : Not added manually by the patient. 1 : Added manually by the patient.
device_id int This column stores the Unique id of the Device and data type is INT. Contains device_id which refers to device_id column of rpm_supported_devices_list. The value of this field can be NULL.
modifiedDateTime datetime Contains " modifiedDateTime" value at time when blood pressure data is shipped and datatype is DATETIME. This column stores datetime hence Datetime is selected. This field will always have a value hence defined as NOT NULL.
source_id int This column stores the type of source from where the readings are retrieved and data type is INT. Contains source_id that refers to id column of rpm_healow_tracker_source table.This field can have a NULL value. The default value of this field is not provided.
isRPMData tinyint Flag indicates the readings received from healow end are valid or not, datatype is tinyint - 0 - Not Valid, 1 - Valid. This field will always have a value hence defined as NOT NULL. By default, this column should not be in valid status hence default 0 is chosen.

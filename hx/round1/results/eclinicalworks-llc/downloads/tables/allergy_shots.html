Table allergy_shots
Description This table stores information about allergy shots given to patients, including details about the formulation, dose, and any reactions.
Column DataType Description
id (Primary Key) int Unique identifier for each allergy shot record.
PATIENTid int identifier for the patient receiving the allergy shot.
QUEUEid int identifier for the queue the allergy shot is in.
FORMULATIONid int identifier for the specific formulation of the allergy shot.
BOTTLEid int identifier for the bottle containing the allergy shot.
CONid int identifier for the concentration of the allergy shot.
ENCid int identifier for the encounter associated with the allergy shot, can be NULL.
STATUS int Integer representing the status of the allergy shot (e.g., 0 for not given, 1 for given).
DOSE varchar The dose of the allergy shot, stored as a string (e.g., "0.5ml").
DUEDATE datetime The date the allergy shot is due to be given.
LOTNO varchar The lot number of the allergy shot, can be NULL.
EXPDATE datetime The expiration date of the allergy shot, can be NULL.
LOCATION varchar The location where the allergy shot is given, can be NULL.
ROUTE varchar The route of administration for the allergy shot (e.g., "intramuscular"), can be NULL.
REACTION varchar Any reaction the patient had to the allergy shot, can be NULL.
FREQUENCY varchar The frequency of the allergy shot (e.g., "weekly"), can be NULL.
GIVENDATE datetime The date and time the allergy shot was given, can be NULL.
GIVENBY varchar The name of the person who gave the allergy shot, can be NULL.
GIVENBYUSERid int The user id of the person who gave the allergy shot, can be NULL.
IS_PV_REACTION tinyint A flag indicating if there was a post-vaccination reaction (1 for yes, 0 for no), can be NULL.
PV_REACTION varchar The post-vaccination reaction, if any, can be NULL.
PV_REACTION_TIME datetime The date and time of the post-vaccination reaction, can be NULL.
PV_NOTES varchar Any notes related to the post-vaccination reaction, can be NULL.
PV_USERid int The user id of the person who recorded the post-vaccination reaction, can be NULL.
PV_USERNAME varchar The username of the person who recorded the post-vaccination reaction, can be NULL.
LOCKSTATUS tinyint A flag indicating the lock status of the record (1 for locked, 0 for unlocked), default is 1.
IS_SHOT_VERIFIED tinyint A flag indicating if the shot has been verified (1 for yes, 0 for no), default is 0.
ISCHECKEDIN tinyint A flag indicating if the patient is checked in for the shot (1 for yes, 0 for no), default is 0.
DOSEid int identifier for the dose, can be NULL. Foreign key relation to `shot_schedule_dtl` table, `id` column.
VIALNOVERIFIED varchar The verified vial number for the allergy shot, can be NULL.
VERIFYMETHODUSED varchar The method used to verify the allergy shot (e.g., "barcode"), can be NULL.
BARCODESETTING tinyint A flag indicating the barcode setting (1 for enabled, 0 for disabled), can be NULL.

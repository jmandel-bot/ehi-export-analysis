Table prisma_careplan_details
Description This table stores the details of care plans, including information about the care plan, its status, intent, category, author, and related dates.
Column DataType Description
ID (Primary Key) int Unique identifier for each care plan detail record. Example: 1, 2, 3
CAREPLAN_ID varchar Unique identifier for the care plan. Example: "CP001", "CP002", "CP003"
SECTION_ID bigint Identifier for the section the care plan belongs to. Foreign key relation to `prisma_ccd_document_section.id`. Example: 1001, 1002, 1003
SOURCE_ID int Identifier for the source of the care plan. Example: 101, 102, 103
TITLE varchar Title of the care plan. Example: "Diabetes Management", "Post-surgery Rehabilitation"
CAREPLAN_STATUS varchar Status of the care plan. Example: "active", "completed", "on hold"
INTENT varchar Intent of the care plan. Example: "plan", "order", "proposal"
CATEGORY_CODE varchar Code for the care plan category. Example: "CAT001", "CAT002", "CAT003"
CATEGORY_DISPLAY varchar Display name for the care plan category. Example: "Chronic Condition Management", "Acute Care"
AUTHOR_PRACTITIONER_ID int Identifier for the authoring practitioner of the care plan. Example: 201, 202, 203
AUTHOR_CARETEAM_ID int Identifier for the authoring care team of the care plan. Example: 301, 302, 303
CREATED datetime Date and time when the care plan was created. Example: "2021-09-01 12:34:56"
NOTE varchar Notes related to the care plan. Example: "Patient needs regular follow-ups", "Monitor blood sugar levels closely"
CAREPLAN_START datetime Start date and time for the care plan. Example: "2021-09-01 00:00:00"
CAREPLAN_END datetime End date and time for the care plan. Example: "2021-12-31 23:59:59"
ENCOUNTER_ID int Identifier for the encounter related to the care plan. Example: 401, 402, 403
isException tinyint Flag to indicate if there is an exception in the care plan (1 for true, 0 for false). Example: 0, 1
exceptionMessage varchar Message describing the exception, if any. Empty if there is no exception.

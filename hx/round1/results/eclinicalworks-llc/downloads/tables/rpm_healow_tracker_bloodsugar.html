EHI Export Schema
Table rpm_healow_tracker_bloodsugar
Description This table is use when a Provider prescribes a Glucose device for a patient, the patient will take readings from that device and will upload it to Healow. From healow, these readings are transferred to EMR side. We have created this new table to store the Blood Glucose readings along with related information such as meal type, measurement time, and device used for the patients.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT(11). This column stores integer value which could grow with number of bloodsugar records over the years hence integer(Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
UID int This column stores unique id of a Patient to identify the readings belonging to the patient which references the UID column of the user table. The data type chosen is INT. This field will always have a value hence defined as NOT NULL.
glucose varchar This column stores glucose data of patient its datatype is VARCHAR(255). This column stores characters which cannot exceed 255 characters hence Varchar(255) is selected. This field will support NULL value hence its default value is NULL.
meal_type int This column is use for store type of meal e.g Pre meal Post meal its datatype is int.  This field will always have a value hence defined as NOT NULL and its DEFAULT value is 0. The possible values for this column include the below  meal_type -Meal type: 1 for Breakfast, 2 for lunch, 3 for dinner, 4 for other
meal_prepost_flag tinyint This column is for storing premeal or postmeal flag for determine the type of meal its datatype is TINYINT.  This field will always have a value hence defined as NOT NULL and it DEFAULT value is 0. The possible values for this column include the below  meal_prepost_flag 1 for Meal taken before taking food  2 for after taking food
measure_datetime datetime This column stores the date time when the measurement was done its datatype is DATETIME. This column stores datetime hence Datetime is selected. This field will always have a value hence defined as NOT NULL.
delflag tinyint Flag indicates the record added is deleted, datatype is tinyint - 0 - Not Deleted, 1 - Deleted. This field will always have a value hence defined as NOT NULL. By default, this column should not be in Deleted status hence default 0 is chosen.
reviewed tinyint This column stores 0, 1 to determine whether the reading is reviewed or not and data type is TINYINT. This field will always have a value hence defined as NOT NULL and default value of this field is 0. Values 0 : Reading has not been reviewed by the Provider 1 : Reading has been reviewed by the Provider
reviewedByid int This column stores the Provider id who has reviewed the reading its datatype is int. Contains reviewedByid which is referenced from UID column of user table. This column could be store high range numeric value hence integer is selected, The default value for this column is NULL.
reviewedDateTime datetime This column stores the date time when the reading was reviewed its datatype is DATETIME. This column stores datetime hence Datetime is selected. This field could be NULL and its default value is NULL.
isManuallyAdded tinyint This column stores 0, 1 whether the reading is manually added by patient or not and the data type is TINYINT. This field will always have a value hence defined as NOT NULL and the default value is 0. Values 0 : Not added manually by the patient. 1 : Added manually by the patient.
device_id int This column stores the Unique id of the Device its datatype is int. Contains device_id which is referenced from device_id column of rpm_supported_devices_list table . This column could be store high range numeric value hence integer is selected, This column can have a NULL value.
modifiedDateTime datetime It is a date and time column for store current time when readings added its datatype is DATETIME. This column stores datetime hence Datetime is selected. This field will always have a value hence defined as NOT NULL.
device_action int The device action is the store vendor id for mapping purpose and its datatype is INT. Contains device_action which is referenced from id column of rpm_vendors_list table. This column could be store high range numeric value hence integer is selected, This column value could be supporting a NULL value.
source_id int This column stores the type of source from where the readings are retrieved its datatype is INT. This column is foreign key and it refer to rpm_healow_tracker_source table id column. This column value could be supporting a NULL value.
isRPMData tinyint Flag indicates the readings received from healow end are valid or not, datatype is tinyint - 0 - Not Valid, 1 - Valid. This field will always have a value hence defined as NOT NULL. By default, this column should not be in valid status hence default 0 is chosen.

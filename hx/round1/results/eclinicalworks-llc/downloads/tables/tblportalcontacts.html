Table tblportalcontacts
Description This table stores contact information for individuals related to a specific portal, including their personal details, address, and phone numbers.
Column DataType Description
id (Primary Key) int Unique identifier for each contact entry in the table. Example: 1, 2, 3.
contactid int Unique identifier for the contact. Example: 101, 102, 103.
pid int Unique identifier for the portal to which the contact is related. This is a foreign key referring to the id column in another table (e.g., `tblportals`). Example: 1, 2, 3.
name varchar Full name of the contact. Example: "John Doe", "Jane Smith".
relation varchar Relation of the contact to the portal. Example: "Manager", "Employee".
address varchar First line of the contact's address. Example: "123 Main St".
address2 varchar Second line of the contact's address (if applicable). Example: "Apt 4B".
city varchar City where the contact lives. Example: "New York", "Los Angeles".
state varchar State where the contact lives. Example: "NY", "CA".
zipcode varchar Zip code of the contact's address. Example: "10001", "90001".
homePhone varchar Home phone number of the contact. Example: "(555) 123-4567".
workPhone varchar Work phone number of the contact. Example: "(555) 987-6543".
delflag int A flag indicating if the contact has been deleted (1) or not (0). Example: 0, 1.
contactType int An integer representing the type of contact (e.g., 1 for emergency contact, 2 for contact 1,3 for contact 2). Example: 1, 2,3.
isFamilyMember tinyint A flag indicating if the contact is a family member (1) or not (0). Example: 0, 1.
isEmergencyContact tinyint A flag indicating if the contact is an emergency contact (1) or not (0). Example: 0, 1.
dob varchar Date of birth of the contact in string format. Example: "1990-01-01", "1985-12-31".
doi tinyint A flag indicating if the date of birth is incomplete (1) or complete (0). Example: 0, 1.
cellPhone varchar Cell phone number of the contact. Example: "(555) 111-2222".

Table prisma_careplan_activity_details
Description This table stores the detailed information about care plan activities, including their status, timing, and location.
Column DataType Description
ID (Primary Key) int Unique identifier for each care plan activity detail. Example: 1, 2, 3.
CAREPLAN_ID int The identifier of the care plan to which this activity detail belongs. This is a foreign key that refers to the `prisma_careplan_details` table's `id` column. Example: 101, 102, 103.
DETAILS_CODE varchar A code representing the specific care plan activity detail. Example: "CPAD001", "CPAD002", "CPAD003".
DETAILS_DISPLAY varchar A human-readable description of the care plan activity detail. Example: "Physical therapy", "Medication administration", "Dietary consultation".
DETAILS_SYSTEM varchar The system used to define the care plan activity detail codes. Example: "http://example.com/careplan/details", "http://anotherexample.com/details".
DO_NOT_PERFORM tinyint A boolean flag indicating whether the activity should not be performed (1) or should be performed (0). Example: 0, 1.
DETAILS_KIND varchar The kind or category of the care plan activity detail. Example: "Procedure", "Observation", "Communication".
ACTIVITY_START_DATE datetime The start date and time of the care plan activity. Example: "2021-08-01 09:00:00", "2021-08-02 14:30:00".
ACTIVITY_END_DATE datetime The end date and time of the care plan activity. Example: "2021-08-01 10:00:00", "2021-08-02 15:30:00".
TIMING_CODE varchar A code representing the timing of the care plan activity. Example: "TC001", "TC002", "TC003".
TIMING_DISPLAY varchar A human-readable description of the timing of the care plan activity. Example: "Every 4 hours", "Twice a day", "Once a week".
ACTIVITY_STATUS varchar The current status of the care plan activity. Example: "Not started", "In progress", "Completed", "Cancelled".
LOCATION_ID int The identifier of the location where the care plan activity takes place. Example: 1, 2, 3.
STATUS_REASON_CODE varchar A code representing the reason for the current activity status. Example: "SRC001", "SRC002", "SRC003".
STATUS_REASON_DISPLAY varchar A human-readable description of the reason for the current activity status. Example: "Patient unavailable", "Lack of resources", "Change in patient condition".

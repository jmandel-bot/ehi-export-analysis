EHI Export Schema
Table ocularsurgicalhistory
Description This table stores information about ocular surgical history for each encounter.
Column DataType Description
encounterid int This column is used to store the encounter id for which the Ocular Surgical History is being stored. This column is referenced from enc table and hence cannot be NULL.
reason varchar This column is used to store the Ocular Surgical history description. No entry will be made if description is empty hence this column is defined as not NULL. Users can either enter free text or select from keywords and usually will not exceed 255 characters.
DATE varchar This is used to store the date related to the reason being recorded. This date column may contain two or more concatenated dates eg - '10/14 & 12/13'. This column is not used in 'where' clause. Date may not present for all the reasons hence NULLable and default value is an empty string.
displayIndex smallint Determines the order in which the reasons will be displayed on the Progress Notes and Ocular Surgical History window. In case no display order is not present, a default of 1 is stored.

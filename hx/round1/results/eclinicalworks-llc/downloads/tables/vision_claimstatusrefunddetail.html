Table vision_claimstatusrefunddetail
Description This table stores information about the claim status refund details for each vision claim.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3, etc.
claimstatusrefundid int identifier for the claim status refund, which is a foreign key referring to the primary key of another table (not specified in the provided SQL). Example: 101, 102, 103, etc.
claimid varchar identifier for the claim ids associated with the refund.
claimstatus varchar idenfy claim status which belong to claims. e.g. PRFR (patient Ready for refund), IRFR (Insurance ready for refund)
exception varchar Description of any exception or error that occurred during the refund process. Example: "Invalid claim id", "Refund amount exceeds limit", "", etc. (empty string if no exception)
STATUS tinyint A tiny integer representing the status of the refund detail record. Example: 0 (inactive), 1 (active)

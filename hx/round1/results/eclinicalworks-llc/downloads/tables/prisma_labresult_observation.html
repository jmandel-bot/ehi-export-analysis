Table prisma_labresult_observation
Description This table stores lab result observations, including information about the observation, the result value, and the performer of the observation.
Column DataType Description
id (Primary Key) bigint Unique identifier for each lab result observation (e.g., 1, 2, 3).
labresultId bigint Identifier for the associated lab result in the `prisma_labresult_details` table. Foreign key relation: `prisma_labresult_details.id` (e.g., 101, 102, 103).
observationObtainedOn varchar Date and time when the observation was obtained, stored as a string (e.g., "2021-08-01 12:34:56").
resultValue varchar The result value of the observation, stored as a long text (e.g., "5.6", "Positive").
resultValueUnit varchar The unit of measurement for the result value, stored as a string (e.g., "mg/dL", "mmol/L").
orderCode varchar A code representing the type of lab test ordered, stored as a string (e.g., "12345", "67890").
orderName varchar The name of the lab test ordered, stored as a string (e.g., "Blood Glucose", "Hemoglobin A1c").
interpretationCode varchar A code representing the interpretation of the result, stored as a string (e.g., "H", "L").
interpretationName varchar The name of the interpretation of the result, stored as a string (e.g., "High", "Low").
lowRegerenceValue varchar The lower reference value for the result, stored as a string (e.g., "4.0", "5.0").
lowRegerenceValueUnit varchar The unit of measurement for the lower reference value, stored as a string (e.g., "mg/dL", "mmol/L").
highRegerenceValue varchar The higher reference value for the result, stored as a string (e.g., "6.0", "7.0").
highRegerenceValueUnit varchar The unit of measurement for the higher reference value, stored as a string (e.g., "mg/dL", "mmol/L").
obsperformerid varchar Identifier for the performer of the observation, stored as a string (e.g., "P001", "P002").
obsPerformerFirstname varchar The first name of the performer of the observation, stored as a string (e.g., "John", "Jane").
obsperformerLastName varchar The last name of the performer of the observation, stored as a string (e.g., "Doe", "Smith").
obsperformerMiddleName varchar The middle name of the performer of the observation, stored as a string (e.g., "A", "B").
obsperformerSuffix varchar The suffix of the performer's name, stored as a string (e.g., "Jr.", "III").
obsPerformerOrganizationName varchar The name of the organization the performer belongs to, stored as a string (e.g., "LabCorp", "Quest Diagnostics").
isException tinyint A flag indicating if there is an exception, stored as an unsigned tiny integer (e.g., 0 for false, 1 for true).
exceptionMessage varchar Message describing the exception, if any. Empty if there is no exception.
sourcePriority int An integer representing the priority of the source of the observation (e.g., 1, 2, 3).
codeSystemPriority int An integer representing the priority of the code system used for the observation (e.g., 1, 2, 3).
duplicatedBy int An integer representing the identifier of the observation that duplicates this observation (e.g., -1 for not duplicated, 2 for duplicated by observation with ID 2).
currentStatus varchar The current status of the observation, stored as a string (e.g., "Active", "Inactive").
stReferenceValue varchar The standard reference value for the result, stored as a string (e.g., "4.5", "6.5").

Table todo_list_lab
Description This table stores the relationship between To Do Lists and their items, including any additional notes for each item in the list.
Column DataType Description
id (Primary Key) int This is the primary key column that uniquely identifies each record in the todo_list_lab table. It is an integer with a maximum length of 11 digits and is automatically incremented. Example: 1, 2, 3, etc.
todo_list_id int This column stores the id of the todo list to which the item belongs. It is a foreign key that refers to the id column in the todo_list table. It is an integer with a maximum length of 11 digits and cannot be NULL. Example: 1, 5, 10, etc.
item_id int This column stores the id of the item in the todo list. It is a foreign key that refers to the itemid column in the items table. It is an integer with a maximum length of 11 digits and can be NULL. Example: 1, 4, 7, etc.
notes varchar This column stores any additional text notes related to the item in the todo list. It is a text data type and can store large amounts of text. Example: Buy milk from the grocery store, Call John to discuss the project, etc.

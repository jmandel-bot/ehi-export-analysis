EHI Export Schema
Table vision_near_testing
Description This table stores information related to near vision testing performed for patients during their encounters.
Column DataType Description
id (Primary Key) int This is the primary key for the table, which uniquely identifies each record. It is an integer value that is automatically incremented by the database. Example: 1, 2, 3, etc.
encounterid int This is a foreign key that refers to the `enc` table's `encounterid` column. It represents the unique identifier for a patient's encounter during which the near vision testing was performed. Example: 101, 102, 103, etc.
pu_acc_amp varchar This column stores the accommodative amplitude value for the patient's near vision testing. It is a varchar(6) field. Example: '2.50D', '3.00D', '1.75D', etc.
pbu_near_add varchar This column stores the near addition value for the patient's near vision testing. It is a varchar(5) field. Example: '+1.0', '+1.5', '+2.0', etc.
nra varchar This column stores the negative relative accommodation value for the patient's near vision testing. It is a varchar(5) field. Example: '-2.0', '-1.5', '-1.0', etc.
pra varchar This column stores the positive relative accommodation value for the patient's near vision testing. It is a varchar(5) field. Example: '+1.0', '+1.5', '+2.0', etc.
bin_xcyl varchar This column stores the binocular cross-cylinder value for the patient's near vision testing. It is a varchar(5) field. Example: '0.50', '0.75', '1.00', etc.

EHI Export Schema
Table emcoder_historyhpi
Description This table stores the history of present illness (HPI) information for a patient encounter for E&M Computation purposes when E&M Coder V2 is used for calculations. This information will be entered by the user.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3
encid int encid refers to encounter id in encounters table. Foreign key relation to the `enc` table (`encounterID` column)
location int Integer representing the location is present or not. Example: 1 (Present), 0 (Not-Present)
locationdesc text Text description of the location of the illness or symptom. Example: "Head", "Chest", "Abdomen"
duration int Integer representing the duration of the illness or symptom is present or not. Example: 1 (Present), 0 (Not-Present)
durationdesc text Text description of the duration of the illness or symptom. Example: "1 week", "3 days", "2 months"
modifactor int Integer representing the modifying factor of the illness or symptom is present or not. Example: 1 (Present), 0 (Not-Present)
modifactordesc text Text description of the modifying factor of the illness or symptom. Example: "Improved with rest", "Worsened with activity", "No change with medication"
severity int Integer representing the severity of the illness or symptom is present or not. Example: 1 (Present), 0 (Not-Present)
severitydesc text Text description of the severity of the illness or symptom. Example: "Mild", "Moderate", "Severe"
timing int Integer representing the timing of the illness or symptom is present or not. Example: 1 (Present), 0 (Not-Present)
timingdesc text Text description of the timing of the illness or symptom. Example: "Intermittent", "Constant", "Waxing and waning"
associated int Integer representing the associated symptoms or signs is present or not. Example: 1 (Present), 0 (Not-Present)
associateddesc text Text description of the associated symptoms or signs. Example: "Nausea", "Fever", "Shortness of breath"
quality int Integer representing the quality of the illness or symptom is present or not. Example: 1 (Present), 0 (Not-Present)
qualitydesc text Text description of the quality of the illness or symptom. Example: "Sharp", "Dull", "Aching"
context int Integer representing the context of the illness or symptom is present or not. Example: 1 (Present), 0 (Not-Present)
contextdesc text Text description of the context of the illness or symptom. Example: "At work", "During exercise", "While sleeping"
points int Integer representing the total assigned to the HPI element (location, duration, modifactor, severity, timing, associated, quality, context). Example: 1, 2, 3
addedby int Identifier for the user who added the record. Foreign key relation to the `users` table (`uid` column). Example: 1001, 1002, 1003
addeddate datetime Date and time when the record was added. Example: '2021-09-01 10:30:00', '2021-09-02 14:45:00'
updatedby int Identifier for the user who last updated the record. Foreign key relation to the `users` table (`uid` column). Example: 1001, 1002, 1003
updateddate datetime Date and time when the record was last updated. Example: '2021-09-01 12:30:00', '2021-09-02 16:45:00'
deleteflag tinyint Tiny integer indicating if the record has been deleted (1) or not (0). Example: 0, 1

Table prisma_encounters
Description This table stores information about patient records meta data, including details about the service, facility, providers, and diagnosis.
Column DataType Description
id (Primary Key) int Unique identifier for each encounter (e.g., 1, 2, 3).
recordId int It is a FK to prisma_records. Identifier for the associated record
serviceDate datetime Date and time of the service, can be NULL (e.g., '2021-09-01 14:30:00').
serviceDateType tinyint 0-Service Date, 1-Admission Date, 2-Last Retrieval Date
admissionDate datetime Date and time of admission, can be NULL (e.g., '2021-09-01 12:00:00').
dischargeDate datetime Date and time of discharge, can be NULL (e.g., '2021-09-05 10:00:00').
titleId tinyint Identifier for the title as a tiny integer, default value is 0 (e.g., 1, 2, 3).
source varchar Source of the encounter information as a string, can be NULL (e.g., 'Hospital A', 'Clinic B').
sourceAddress varchar Address of the source as text, can be NULL (e.g., '123 Main St, City, State, Zip').
facilityName varchar Name of the facility as a string, can be NULL (e.g., 'Facility A', 'Facility B').
facilityAddress varchar Address of the facility as text, can be NULL (e.g., '456 Elm St, City, State, Zip').
subTitle varchar Subtitle of the encounter as a string, can be NULL (e.g., 'SubTitle A', 'SubTitle B').
principalDiagnosis varchar Principal diagnosis of the encounter as text, can be NULL (e.g., 'Diabetes', 'Hypertension').
deleteFlag tinyint Flag indicating if the encounter is deleted, default value is 0 (e.g., 0 for not deleted, 1 for deleted).
xslId int Identifier for the associated XSL record, can be NULL (e.g., 5, 10, 15). Foreign key relation to `prisma_records_xsl.id`.
sectionOrder varchar Order of sections as a string, can be NULL (e.g., '1,2,3', '2,3,1').
reasonForAppt varchar Reason for the appointment as text, can be NULL (e.g., 'Follow-up', 'Initial Consultation').
recordType tinyint Type of record as a tiny integer, default value is 0 (e.g., 1, 2, 3).
sourceRecordType tinyint Type of source record as a tiny integer, default value is 0 (e.g., 1, 2, 3).
encType smallint Type of encounter as a small integer, default value is 0 (e.g., 1, 2, 3).
admittingProvider varchar Name of the admitting provider as a string, can be NULL (e.g., 'Dr. John Doe', 'Dr. Jane Smith').
appointmentProvider varchar Name of the appointment provider as a string, can be NULL (e.g., 'Dr. John Doe', 'Dr. Jane Smith').
isException tinyint Flag indicating if the encounter is an exception, default value is 0 (e.g., 0 for not an exception, 1 for an exception).
exceptionMessage varchar Exception message as a string, default value is an empty string (e.g., '', 'Error: Invalid Data').
duplicatedBy int Identifier for the duplicated encounter, default value is 0 (e.g., 0 for not duplicated, 1 for duplicated).
pid int Identifier for the associated patient, can be NULL (e.g., 100, 200, 300). Foreign key relation to `users.uid`.

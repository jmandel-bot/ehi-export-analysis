EHI Export Schema
Table fhir_write_back_audit_request_detail
Description This table stores the details of audit requests related to FHIR (Fast Healthcare Interoperability Resources) write-back operations.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
audit_request_id int Identifier for the associated audit request. Foreign key relation with `fhir_write_back_audit_request(id)`. Example: 101, 102, 103.
patient_id int Identifier for the associated patient. Foreign key relation with `users (UID)`. Example: 1001, 1002, 1003. This column can have NULL values.
processing_time datetime time stamp of when the request was processed. Example: '2021-09-01 12:34:56'.
resource_type_id int Identifier for the type of FHIR resource involved in the write-back operation. Foreign key relation with `fhir_write_back_resource_type(id)`. Example: 1, 2, 3.
resource_location_id varchar Location identifier of the FHIR resource. Example: 'Location/123', 'Location/456'. This column can have NULL values.
todo_record_id int Identifier for the associated todo record. Foreign key relation with `todo_list(id)`. Example: 2001, 2002, 2003. This column can have NULL values.
request_status smallint Status code of the request. Default value is '1'.

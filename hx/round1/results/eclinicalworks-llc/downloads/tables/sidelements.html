EHI Export Schema
Table sidelements
Description This table stores information about various elements related to subscribers, employees, and their relationships within a company.
Column DataType Description
id (Primary Key) int This is the primary key and an auto-incrementing integer that uniquely identifies each record in the table. Example: 1, 2, 3, etc.
pid int This is a foreign key column that refers to the `Pid` column in the `patients` table.
subscriberNo varchar This is a varchar(25) column that stores the unique subscriber number for each individual. Example: 'SUB12345', 'SUB67890', etc.
employeeid varchar This is a varchar(25) column that stores the unique employee id for each individual. Example: 'EMP12345', 'EMP67890', etc.
groupid varchar This is a varchar(11) column that stores the unique group id to which the individual belongs. Example: 'GRP123', 'GRP456', etc.
companyid varchar This is a varchar(10) column that stores the unique company id to which the individual belongs. Example: 'COMP123', 'COMP456', etc.
relationshipCode char This is a char(2) column that stores a code representing the relationship between the subscriber and the employee. Example: '01' for self, '02' for spouse, '03' for child, etc.
dependentCode char This is a char(3) column that stores a code representing the dependent status of the individual. Example: '001' for primary, '002' for secondary, '003' for tertiary, etc.
Alternateid varchar This is a varchar(255) column that stores an alternate id for the individual, which can be used in cases where the primary id is not available or applicable. Example: 'ALT12345', 'ALT67890', etc.

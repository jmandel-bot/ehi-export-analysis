EHI Export Schema
Table vision_schirmer
Description This table stores the Schirmer test results for patients' right and left eyes during an encounter.
Column DataType Description
id (Primary Key) int This is the primary key for the table, and it is an integer that uniquely identifies each vision sensory test. It is an auto-incrementing value. Example: 1, 2, 3, etc.
encounterid int This column represents the encounter id associated with the vision sensory test. It is an integer and a foreign key that refers to the `enc` table's `encounterid` column. Example: 101, 102, 103, etc.
r_eye smallint This is a tiny integer column that represents the type of vision sensory exam performed. It can be NULL. Example: 1 (for exam type A), 2 (for exam type B), etc.
l_eye smallint This is a tiny integer column that represents the type of prescription given after the vision sensory test. It can be NULL. Example: 1 (for prescription type A), 2 (for prescription type B), etc.

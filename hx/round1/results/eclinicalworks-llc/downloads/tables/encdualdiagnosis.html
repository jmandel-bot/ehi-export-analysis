EHI Export Schema
Table encdualdiagnosis
Description This table contains the information regarding the ICD/CDX details based on encounter. (When the user wants to save ICD-9 and ICD-10 codes, data will be saved in this table.)
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3
deleteflag smallint Indicates if the record has been deleted (1) or not (0). Example: 0, 1
encounterid int Identifier for the medical encounter associated with the dual diagnosis. This is a foreign key that refers to the `id` column in the `encounters` table. Example: 1001, 1002, 1003
encdxitemid int Identifier for the dual diagnosis item associated with the encounter. Example: 2001, 2002, 2003
icd9itemid int Identifier for the ICD-9 diagnosis code associated with the dual diagnosis. Example: 3001, 3002, 3003
icd9code varchar ICD-9 diagnosis code for the dual diagnosis. Example: '250.00', '401.9', '414.01'
icd10itemid int Identifier for the ICD-10 diagnosis code associated with the dual diagnosis. Example: 4001, 4002, 4003
icd10code varchar ICD-10 diagnosis code for the dual diagnosis. Example: 'E10.9', 'I10', 'I25.10'

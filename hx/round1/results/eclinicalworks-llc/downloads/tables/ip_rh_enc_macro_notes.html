Table ip_rh_enc_macro_notes
Description Table is used to store notes of templates documented for encounters.
Column DataType Description
id (Primary Key) int This column stores auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer (Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
encounterId int This column is used to identify for which patient encounter user has documented data.This column is FOREIGN KEY which refers to encounterid column of enc table. This column stores a high range numeric value hence integer is selected, and this column will not always have a value hence defined as NULL.
entryId int This column stores mapping id of template and patient encounter. This column is FOREIGN KEY which refers to id column of ip_rh_enc_macro_entry table. This column stores a high range numeric value hence integer is selected, and this column will always have a value hence defined as NOT NULL.
templateId int This column store template which is added for patient encounter and against that template user has documented notes. This column is FOREIGN KEY which refers to id column of ip_rh_clinical_sd_template table. This column stores a high range numeric value hence integer is selected, and this column will always have a value hence defined as NOT NULL.
templateNotes varchar This column store's notes added by user for item. We are using datatype as text because users can enter large text as notes. This column will not always have value hence defined NULL DEFAULT NULL.
answerString varchar This column store's answer html string added by user for item. We are using datatype as text because users can enter large text as answer. This column will not always have value hence defined NULL DEFAULT NULL.
deleteFlag tinyint This will store 0 and 1 value. 0-not deleted and 1-deleted. This column is used to identify whether row is in use or not. We are using TINYINT as datatype as we just want to store 0 or 1 value. By default, row will be in use, so we keep NOT NULL DEFAULT 0.
createdBy int This column stores user id of user. This column is used to identify which user has added record (module wise template notes) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store high range of numeric value hence an integer is selected. This column will always have a value hence defined as NOT NULL.
createdDateTime datetime This Column stores datetime as we want to store a datetime when the item is added, so we are using DATETIME as data type of column. This column will always have a value hence defined as NOT NULL.
modifiedBy int This column stores user id of user. This column is used to identify which user has last modified record (module wise template notes) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store high range of numeric value hence integer is selected. This column will not always have a value hence defined as NULL DEFAULT NULL.
modifiedDateTime datetime This Column stores datetime as we want to store a datetime when the item is added, so we are using DATETIME as data type of column. This column will not always have a value hence defined as NULL DEFAULT NULL.

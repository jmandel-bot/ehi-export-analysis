Table prisma_careplan_activity_reason_code
Description This table stores the reason codes associated with care plan activities, which are used to provide more context and information about why a specific activity is included in a care plan.
Column DataType Description
ID (Primary Key) int This is the primary key for the table, which uniquely identifies each record. It is an integer value that is automatically incremented when a new record is added. Example: 1, 2, 3, etc.
CAREPLAN_ACTIVITY_ID int This is a foreign key that refers to the `ID` column in the `prisma_careplan_activity_details` table. It is an integer value that represents the care plan activity for which the reason code is associated. Example: 101, 102, 103, etc.
REASON_CODE varchar This is a varchar column that stores the reason code for the care plan activity. It can be a maximum of 100 characters long and can be NULL. Example: "R01", "R02", "R03", etc.
REASON_DISPLAY varchar This is a varchar column that stores a human-readable description of the reason code. It can be a maximum of 255 characters long and can be NULL. Example: "Patient request", "Doctor recommendation", "Insurance requirement", etc.

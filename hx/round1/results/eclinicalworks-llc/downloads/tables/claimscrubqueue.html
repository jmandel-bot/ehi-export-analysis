EHI Export Schema
Table claimscrubqueue
Description This table stores information about claims in a queue that need to be processed or scrubbed.
Column DataType Description
id (Primary Key) int This is the primary key, an auto-incrementing integer that uniquely identifies each record in the table. Example: 1, 2, 3, etc.
loadsharingtaskid int This is a varchar(50) column that stores the batch number of the drug used in the vision treatment. Example: 'B12345', 'B67890', etc.
claimid int This is a varchar(100) column that stores the name of the drug used in the vision treatment. Example: 'DrugA', 'DrugB', etc.
claimqueuestatus int This is a varchar(15) column that stores the date when the drug information was recorded, likely in the format 'YYYY-MM-DD'. Example: '2021-09-01', '2021-09-02', etc.
claimupdatedstatuscode varchar This is a varchar(10) column that stores the time when the drug information was recorded, likely in the format 'HH:mm:ss'. Example: '14:30:00', '15:45:00', etc.
consumerid int Unique identifier for each record in the vision_stereopsis table. It is an integer value that auto-increments. Example: 1, 2, 3.
consumedtime datetime Unique identifier for the patient encounter during which the vision stereopsis test was performed. It is an integer value. Example: 101, 102, 103. This column has a foreign key relation with the `enc` table, referencing the `encounterid` column.
completedtime datetime Indicates the type of prescription used during the test. It is an unsigned tinyint value. Example: 0, 1, 2.
deleteflag tinyint Stores the result of the "fly" test as a string. Example: 'Pass', 'Fail'.

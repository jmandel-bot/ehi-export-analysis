EHI Export Schema
Table emar_medications
Description This table is used to store the medication administration schedule details. It is used in the Medication Administration module.
Column DataType Description
Id (Primary Key) int This is Primary key for the table.
ItemId int This column stores itemid of an Rx scheduled for administration. This column refers to itemid column in items table.
EncounterId int This column stores EncounterId in which Rx is scheduled for administration. This column refers to encounterid column in enc table.
NDC_Code varchar This column stores NDC_Code of the Rx which is scheduled for administration.
PatientId int This column stores PatientId for which Rx is scheduled for administration. It refers to uid column of user table.
DirectAdmin int This column stores if Rx is administered directly by user. It store NULL or '0' or '1' value
FrequencyId int This column stores frequency of Rx administration schedule. It can be NULL. It refers to the id column from emar_frequency.
Route varchar This column store Route of Rx administration schedule. It cannot be NULL.
StartDate datetime This column stores start date of Rx administration schedule.
Calculated_StopDate datetime This column stores calculated stop date of Rx administration schedule.
Actual_StopDate datetime This column stores actual stop date of Rx administration schedule.
SpeacialInstruction text This column store provider's instructions for Rx administration schedule.
PreparedBy varchar This column store User Id of Rx administration schedule. This column refers to uid from user table.
DeleteFlag int This column store status of Rx administration schedule. Value can be 0 or 1. 0 is Not deleted and 1 is deleted.
Createdtimestamp datetime This column store the time of creation of Rx administration schedule.
HasStatDose int This column store the information that Rx is given as Stat when schedule is created. Value can be 0 or 1.
HasFirstDose int This column store the information that Rx first dose is given when schedule is created. Value can be 0 or 1.
IsVerified tinyint This column store the information that Rx is verified when schedule is created. Value can be 0 or 1.
VerifiedBy varchar This column store the information that which user verified Rx when schedule is created. Value can be 0 or 1.
VerifiedDateTime datetime This column store the information Rx verified date and time.
PillPassLocationId int This column stores id of a pharmacy from where medication was given to inmate. This column is for Corrections module. Column refers to pharmacy.pmcid
MedicationType tinyint This column store the information Medication Type of Rx schedule. It can be DOT or KOP type of medication. DOT - 0 and KOP - 1
PRNFlag tinyint This column store the information PRN Type of Rx schedule. It can be PRN or not PRN type of medication. 1 - PRN and 0 - not PRN.
StoppedFlag tinyint This column store the flag for stopping Rx schedule. 0- stopped and 1 - not stopped.
DrugSource varchar This column store the drug source of Rx schedule.
KOPDueDays int This column store the days in between twp administration of Rx when medication if of type KOP.
Dispense varchar This column store the dispense of medication scheduled.
alert varchar This column store the alert for medication scheduled.
assignedto int This column store the usedid of assignedTo of medication scheduled. This column refers to uid from user table.
Inactive tinyint This column store the status of medication scheduled. 0 is active and 1 is inactive.
OnHold tinyint Column stores flag to indicate whether the medication is on hold or not. 0 - Medication is not on hold, 1 - Medication is on hold.
RxOrderNo varchar This column store the RxOrderNo of medication scheduled. It refers to RxOrderNo from oldrxmain table.
FilledMedname varchar This column store the medication name filled by Pharmacy.
FilledDose varchar This column store the dose filled by Pharmacy.
FilledAmount varchar This column store the amount of medication filled by Pharmacy.
fillerid varchar It store a unique id fillerid sent from Pharmacy.

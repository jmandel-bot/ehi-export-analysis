EHI Export Schema
Table credtransaction
Description This table stores information about credit transactions, including details about the transaction, payment status, and related entities such as facilities and encounters.
Column DataType Description
id (Primary Key) int Unique identifier for each credit transaction record. Example: 1, 2, 3.
ptid int Patient identifier associated with the credit transaction. Example: 101, 102, 103.
Amount varchar The amount of the credit transaction. Example: "100.00", "50.00", "25.00".
Transid varchar Unique identifier for the transaction. Example: "T12345", "T67890", "T111213".
Message varchar encrypted text
AuthCode varchar encrypted text
PostedOn datetime Date and time when the transaction was posted. Example: "2021-08-01 12:34:56", "2021-08-02 14:56:32".
SettledOn datetime Date and time when the transaction was settled. Example: "2021-08-03 16:12:34", "2021-08-04 18:24:36".
TranStatus varchar Status of the transaction. Example: "success", "failed", "pending".
paymentStatus varchar Status of the payment. Example: "pending", "completed", "failed".
credResponse text encrypted text
tranidentifier varchar Unique UUID
Merchantid varchar Merchant details of payment vendor
stmtfacid int Statement facility identifier. Example: 201, 202, 203.
facilityName varchar Name of the facility associated with the transaction. Example: "Facility A", "Facility B", "Facility C".
paymentFacid int Payment facility identifier. Example: 301, 302, 303.
paymentId int This stores the encounter id on which referral was created. This value is same as value from refEncid from referral table
encounterid int identifier for the encounter related to the transaction. This is a foreign key that refers to the `encounterid` column in the `enc` table. Example: 501, 502, 503.

Table ccm_batch
Description This table stores information about PHM Billing batch process including batch date, last run batch date, success, failure, do not create, total and alerts. It stores information about created and modified date time. Each month / year pair will have only one entry in ccm_batch table.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer(Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index.
success int Contains total number of successful patients for PHM Billing batch process of a billing month. The data type of this column is INT. The value of this column cannot be NULL, hence it has a default value of 0.
failure int Contains total number of failed patients for PHM Billing batch process of a billing month. The data type of this column is INT. The value of this column cannot be NULL, hence it has a default value of 0.
doNotCreateClaim int Contains total number of patients for which the user has selected Do Not Create Claim option for PHM Billing batch process of a billing month. The data type of this column is INT. The value of this column cannot be NULL, hence it has a default value of 0.
total int Contains total number of patients for PHM Billing batch process of a billing month. The data type of this column is INT. The value of this column cannot be NULL, hence it has a default value of 0.
createClaims int Contains total number of patients for which the user has selected Do Not Create Claim option earlier and now the user has added them back to reprocess queue for PHM Billing batch process of a billing month. The data type of this column is INT. The value of this column cannot be NULL, hence it has a default value of 0.
delFlag tinyint Flag indicates if this data is deleted or not, datatype is TINYINT. This column stores binary flag i.e 0/1. 0: not deleted 1: deleted - By default the record must not be in deleted status hence default 0 is chosen.
status int Contains current status of PHM Billing batch process. The allowed values are 1, 2 and 3. 1 indicates Run, 2 for Generate and 3 is for Complete. The data type of this column is tinyint.
lastUpdatedUID int This column stores unique id of a user which references the UID column of the user table. It refers to the user who modified the record last. The data type chosen is INT. This column does not allow a NULL value.
alert int Contains total number of patients who have alerts for PHM Billing batch process of a billing month. The data type of this column is INT. The value of this column cannot be NULL, hence it has a default value of 0.
batchDate datetime Contains the month for which the batch is run. The datatype of this field is datetime. This field does not allow NULL values.
lastBatchRunDt datetime Contains the date when the batch was last run by a user for specific billing month. The datatype of this field is datetime. This field does not allow NULL values.
notes text Contains text notes added by the user when phm billing batch is processed. The data type of this field is TEXT. This field allows a NULL value.
createdDtTime datetime Contains the date and time when the batch was first run by a user for specific billing month. The datatype of this field is datetime. This field does allows a NULL value.
modifiedDtTime datetime Contains the date and time when the batch record was last modified by a user for specific billing month. The datatype of this field is datetime. This field allows a NULL value.

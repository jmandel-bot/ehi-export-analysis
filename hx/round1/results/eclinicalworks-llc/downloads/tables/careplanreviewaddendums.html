EHI Export Schema
Table careplanreviewaddendums
Description This table stores the addendums or additional information related to care plan reviews, along with the user who added the addendum and the date it was added.
Column DataType Description
id (Primary Key) int This is the primary key for the table, which uniquely identifies each addendum entry. It is an integer value that auto-increments with each new entry. For example: 1, 2, 3, etc.
reviewid int This column stores the id of the care plan review to which the addendum is related. It is a foreign key that refers to the `id` column in the `careplanreviews` table. For example: 101, 102, 103, etc.
addendum text This column stores the text content of the addendum. It is a mandatory field and can store large amounts of text. For example: "Additional notes on patient's progress", "Updated medication list", etc.
userid int This column stores the id of the user who added the addendum. It is a foreign key that refers to the `id` column in the `users` table. For example: 1001, 1002, 1003, etc.
DATE datetime This column stores the date and time when the addendum was added. It is a datetime value. For example: '2021-09-01 12:34:56', '2021-09-02 14:30:00', etc.

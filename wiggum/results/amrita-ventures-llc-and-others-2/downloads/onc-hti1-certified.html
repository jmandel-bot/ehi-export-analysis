ONC Certified Solution

Darena Health Version 2

Darena Health (formerly MeldRx, BlueButtonPRO) is the nation’s first ONC Certified FHIR API which is offered as a multi-tenant cloud-based architecture positioned as a Software as a Service (SaaS) solution for all stakeholders within the Healthcare continuum. The Darena Health API provides EHRs, payers, hospitals, and providers with an application that satisfies all the mandated requirements for patient data access and more. The API provides compliance with the mandated FHIR v4 standard, the United States Core Data for Interoperability (USCDI) v3.1 , as well as all requirements defined by CMS and ONC in the 21st Century Cures Act interoperability rules.

Darena Health addresses the non-information-blocking attestation by enabling near real-time information exchange between providers, patients, payers, large health systems, and EHRs. Along with the CMS-approved patient-facing app, the application puts patients in control of their personal and current medical information.

The ONC-Certified Darena Health API allows commercial and homegrown EHRs to expedite the development process to provide a Certified FHIR v4 API to their end users by interfacing with this pre-certified solution.

ONC-certified Darena Health API includes the following features:

  • Secured using HIPAA-compliant OAuth user authorization

  • Customizable login and unique patient token creation

  • Native FHIR v4 data models. No translation is needed to a proprietary API

  • Delivers XML or JSON via API

  • Fully scalable to accommodate your growth with no diminished performance rates.

  • SMART on FHIR compatible for patient access applications, both current and future.

  • Data Analytics to allow the data to provide actionable outcomes improvement.

  • User-friendly adapters to allow communication of patient data between multiple EHRs and payers into FHIR data models.

  • Compatible with the Health IT structured data (ICD-10, RxNorm, SNOMED, etc.) and USCDI data set.

  • Converts CCDA R2.1 to FHIR resources on demand

  • Compliant to - 170.213(b) United States Core Data for Interoperability (USCDI), October 2022 Errata, Version 3 (v3.1)

  • Compliant to – 170.205(a)(6) HL7® CDA® R2 Implementation Guide: C-CDA Templates for Clinical Notes STU Companion Guide, Release 4.1 – US Realm

  • Retrieves all or specific classes of patient data (e.g. problems or medications), filtered date, and or a date range

170.315 (b)(10) EHI Export Documentation

Darena Health supports the export of EHI as follows:

1.      FHIR Bulk Export Format: The user can export all available structured data through FHIR APIs as outlined in our swagger documentation. The output format is based on the FHIR Bulk data standard  

 2.      For non-FHIR resources, Darena Health supports the export as DocumentReference resource types. This resource can be used with any file, including CCDA.

Table dialysis_enrollment
Description This table stores information about patients enrolled in dialysis treatment at various facilities, including their plan type, shift time, and other relevant details.
Column DataType Description
id (Primary Key) int This is the primary key, an auto-incrementing integer that uniquely identifies each enrollment record. Example: 1, 2, 3, etc.
patientid int This is a foreign key, an integer that refers to the unique identifier of a patient in the `patients` table. Example: 101, 102, 103, etc.
facilityid int This is a foreign key, an integer that refers to the unique identifier of a facility in the `facilities` table. Example: 201, 202, 203, etc.
plantype varchar This is an enumeration column that stores the dialysis treatment plan type. It can have two possible values: 'M/W/F' (Monday, Wednesday, Friday) or 'T/T/S' (Tuesday, Thursday, Saturday).
shifttime varchar This is an enumeration column that stores the shift time for the dialysis treatment. It can have three possible values: 'Morning', 'Afternoon', or 'Evening'.
deleteFlag int This is an integer column that indicates whether the record has been deleted or not. A value of '0' means the record is active, while a non-zero value means the record has been deleted.
userid int This is a foreign key, an integer that refers to the unique identifier of a user in the `users` table who created or modified the enrollment record. Example: 301, 302, 303, etc.
mdate datetime This is a time stamp column that stores the date and time when the enrollment record was last modified. Example: '2021-09-01 12:34:56'.

EHI Export Schema
Table relation_codes
Description This table stores the contacts' relationship codes and their corresponding relation names, along with a flag indicating whether the relation is a family member or not.
Column DataType Description
Code char this column store relation code
Relation varchar This column stores relation name.
isFamilyMember tinyint This column mark whether the current relation entry in the table is Family Member for the respective patient. Possible values for this column are 0 and 1 hence it is declared as tiny int and size as 1.Default is 0 which indicates that the relation entry is not Family Member. With story # implementation, relation codes specified in Table 1 are updated with isFamilyMember=1.

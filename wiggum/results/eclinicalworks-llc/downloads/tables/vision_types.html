EHI Export Schema
Table vision_types
Description This table stores information about different vision types, including names and associated codes.
Column DataType Description
id (Primary Key) int This is the primary key of the table, which uniquely identifies each vision type. It is an integer with a maximum of 3 digits and is auto-incremented. Example values: 1, 2, 3, etc.
NAME varchar It defines vision type name like Lens, Frame, Addons, Contact Lens, Exam
TYPE int Defines vision type. e.g. 1 for Lens, 2 for Contact Lens, 3 for Frame, 4 for Addon, 5 for Exam
cptid int This column stores an integer value representing the CPT (Current Procedural Terminology) id associated with the vision type. It has a maximum of 11 digits and has a default value of '0'. Example values: 12345, 67890, etc. There is no foreign key relation specified for this column.
cptcode varchar This column stores a variable character field representing the CPT code associated with the vision type. It has a maximum length of 10 characters and has a default value of an empty string (''). Example values: '1234A', '5678B', etc. There is no foreign key relation specified for this column.

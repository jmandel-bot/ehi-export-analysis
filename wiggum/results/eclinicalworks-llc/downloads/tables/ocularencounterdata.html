EHI Export Schema
Table ocularencounterdata
Description This table stores information related to ocular encounters, including the verification of ocular history and ocular surgical history, as well as the users who verified these histories.
Column DataType Description
encounterid (Primary Key) int This is the primary key for the table and represents the unique identifier for each ocular encounter. Example: 1024.
isOcularHistoryVerified bit This is a bit column that indicates whether the ocular history has been verified or not. A value of 1 means verified, and 0 means not verified. Example: 1.
ocularHistoryVerifiedBy int This is an integer column that stores the user id of the person who verified the ocular history. It has a foreign key relation with the `users` table and the `UID` column. Example: 1234.
isOcularSurgicalVerified bit This is a bit column that indicates whether the ocular surgical history has been verified or not. A value of 1 means verified, and 0 means not verified. Example: 0.
ocularSurgicalVerifiedBy int This is an integer column that stores the user id of the person who verified the ocular surgical history. It has a foreign key relation with the `users` table and the `UID` column. Example: 5678.

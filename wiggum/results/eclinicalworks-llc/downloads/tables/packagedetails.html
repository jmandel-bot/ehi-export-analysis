EHI Export Schema
Table packagedetails
Description This table stores information about different packages and their associated details, such as visit codes, facility IDs, wait times, and doctor IDs.
Column DataType Description
Id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
PackageId int Identifier for the package associated with the record. This is a foreign key referring to the ID column of the package table. Example: 101, 102, 103.
VisitCode int Identifier for the visit code associated with the packge. This is a foreign key referring to the CodeId column of the visitcodes table. Example: 101, 102, 103.
deleteflag tinyint A flag indicating whether the record is deleted or not. A value of 0 means the record is not deleted, and a value of 1 means the record is deleted. Example: 0, 1.
facilityId int Identifier for the facility associated with the package. This could be a foreign key referring to another table containing facility information. Example: 2001, 2002, 2003.
waitTime varchar The estimated wait time for the package, stored as a string. Example: "30 mins", "1 hour", "2 hours".
sameTimeAppt tinyint A flag indicating whether the package allows for same-time appointments. A value of 0 means same-time appointments are not allowed, and a value of 1 means same-time appointments are allowed. Example: 0, 1.
reason varchar The reason for the package, stored as a string. Example: "Routine checkup", "Follow-up visit", "Emergency".
doctorId int Identifier for the doctor associated with the package. This is a foreign key referring to the uid column of users table. Example: 3001, 3002, 3003.

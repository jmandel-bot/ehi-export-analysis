Table correctional_specialneeds
Description This table stores special needs for a specific patient. It stores the duration, status, and other details regarding special needs for the patient.
Column DataType Description
id (Primary Key) int This column is used only in data computation tool side to store some prid.
neednameid int This is primary key and auto increment integer value.
patientid int This column store measure_reportdata_59023 tables id value. This is one kind of foreign-key which will contain multi records for measure_reportdata_59023 table.
startdate datetime This column store patient id for respective measures.
enddate datetime This column store encounter id for respective measures.
needstatus int This column store encounter date for respective measures.
modbyuser varchar This column store measure signature for respective measures. With use of this column we can decide this patient record exist in IPP, Deniminator, Exclusion, Numerator, Exception, Data Missing values. x>=1 : IPP, x>=2 : Deniminator, x==3 : Exclusion, x==4 : Numerator, x==5 : Exception, x==6 : Data Missing.
modtimestamp datetime This column is used only in data computation tool side to store some prid.
delflag tinyint This is primary key and auto increment integer value.
needmanualindex int This column store measure_reportdata_510 tables id value. This is one kind of foreign-key which will contain multi records for measure_reportdata_510 table.

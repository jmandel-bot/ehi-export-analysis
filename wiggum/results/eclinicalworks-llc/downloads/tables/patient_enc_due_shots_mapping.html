EHI Export Schema
Table patient_enc_due_shots_mapping
Description This table stores the mappings of patients' encounters and their due shots, including information about how many days late the shot is and whether the shot is scheduled for the future.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
ENCOUNTERid int identifier for the encounter associated with the due shot. This is a foreign key that refers to the `encounter` table's `encounterid` column. Example: 101, 102, 103.
SHOTid int identifier for the specific shot that is due. This is a foreign key that refers to the `allergy_shots` table's `id` column. Example: 201, 202, 203.
PATIENTid int identifier for the patient who is due for the shot. This is a foreign key that refers to the `users` table's `UID` column. Example: 301, 302, 303.
DAYSLATE int Number of days late for the due shot. A positive value indicates the shot is overdue, while a negative value indicates the shot is not yet due. Example: 0, 5, -3.
ISFUTURESHOT int A flag indicating whether the shot is scheduled for the future. A value of 1 means the shot is scheduled for the future, while a value of 0 means it is not. Example: 0, 1.

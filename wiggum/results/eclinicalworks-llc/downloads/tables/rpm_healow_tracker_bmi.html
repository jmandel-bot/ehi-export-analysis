EHI Export Schema
Table rpm_healow_tracker_bmi
Description When a provider prescribes a BMI device for the patient,the patient will take the readings from the device and upload it to Healow.From healow these readings are transferred to EMR side.This table stores all the BMI readings ncluding weight, height, and measurement timestamps. It also contains information about the data source, device used, and whether the data has been reviewed or manually added for the patients.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT. This column stores integer value which could grow with number of RPM healow tracker bmi data added and hence integer(Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
UID int This column stores unique id of a Patient to identify the readings belonging to the patient which references the UID column of the user table. The data type chosen is INT. This field will always have a value hence defined as NOT NULL.
weight varchar Contains the weight of the patient and data type is VARCHAR(255) and default value is NULL. This column stores characters which cannot exceed 255 characters hence VARCHAR(255) is selected.
height varchar Contains the height of the patient and data type is VARCHAR(255) and default value is NULL. This column stores characters which cannot exceed 255 characters hence VARCHAR(255) is selected.
height_ts datetime Contains time stamp value at time when height data is shipped, datatype is DATETIME.  This column stores datetime hence Datetime is selected. This field will always have a value hence defined as NOT NULL. 
bmi varchar Contains the body mass index value measured for the patient and data type is VARCHAR(255) and default value is NULL. This column stores characters which cannot exceed 255 characters hence VARCHAR(255) is selected.
measure_datetime datetime Contains " measure_datetime " value at time when fitness data is measured, datatype is DATETIME.  This column stores datetime hence Datetime is selected. This field will always have a value hence defined as NOT NULL.
delflag tinyint Flag indicates bmi data is Deleted or not, datatype is tinyint - 0 - Not Deleted, 1 - Deleted. This field will always have a value hence defined as NOT NULL. By default, bmi data should not be in Deleted status hence default 0 is chosen.
reviewed tinyint Contains value that specifies if the bmidata is reviewed or not and data type is TINYINT. This field will always have a value hence defined as NOT NULL and default value is 0.  Values 0 :Readings has not been reviewed by Provider 1-Readings has been reviewed by provider 2-Reading has been reviewed and ignored by the provider
reviewedByid int Contains the UID value referenced from UID column of user table and data type is INT and default value is NULL.  This column is a foreign key and it refer to user table UID column. This column could be store high range numeric value hence integer is selected.
reviewedDateTime datetime Contains the date and time when fitness data is reviewed by the reviewer, datatype is DATETIME and default value is NULL.  This column stores datetime hence Datetime is selected.
isManuallyAdded tinyint Contains the flag that specifies if the bmi data was measured by tracker or manually added and data type is TINYINT. Here 0: indicate not manually added and 1: indicate manually added. This field will always have a value hence defined as NOT NULL.  By default, fitness data should not be manually added hence default 0 is chosen
device_id int Contains " device_id" value referenced from device_id column of the rpm_supported_devices_list table, datatype is INT and default value is NULL. This column could be store high range numeric value hence integer is selected.
modifiedDateTime datetime Contains " modifiedDateTime" value at time when bmi data is shipped, datatype is DATETIME.  This column stores datetime hence Datetime is selected. This field will always have a value hence defined as NOT NULL.
device_action int This column stores the Vendorid from whom the device has been purchased.  This column could be store high range numeric value hence integer is selected. Contains " device_action " value referenced from id column of the rpm_vendors_list table, datatype is INT.
source_id int Contains " source_id" value referenced from id column of the rpm_healow_tracker_source table, datatype is INT. This column could be store high range numeric value hence integer is selected. This field will always have a value hence defined as NOT NULL.
isRPMData tinyint Flag indicates the readings received from healow end are valid or not, datatype is tinyint - 0 - Not Valid, 1 - Valid. This field will always have a value hence defined as NOT NULL. By default, this column should not be in valid status hence default 0 is chosen.

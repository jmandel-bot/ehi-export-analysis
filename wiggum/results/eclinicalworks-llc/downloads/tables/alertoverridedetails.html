EHI Export Schema
Table alertoverridedetails
Description This table stores additional information related to the suppression status of CDSS Alerts for patients.
Column DataType Description
id (Primary Key) int This column is an auto incremented primary key for this column.
Patientid int This column stores an id of the patient for whom alerts are suppressed. This column refers to the column UID from the user table.
Measureid varchar This column stores id of the measure/alert which is suppressed for a patient. This column refers to the column Measureid from the measureconfiguration table.
encounterid int This column stores 0 if the alert is suppressed from Patient Hub or the encounter id if it is suppressed from the Progress Note.
TrUserid int This column stores the id of the user who suppressed the alert for a patient. This column refers to the column UID from the user table.
NeverRemind tinyint This column stores flag 0 or 1. 1 - permanent suppression of alerts, and 0 - temporary suppression of alerts.
status varchar This column stores one of the below pre-defined values as status. 1P, 2P, 3P and ERRONEOUS. These values are mapped with the values in the statusLabel column respectively. 1P - MEDICAL REASON, 2P - PATIENT REASON, 3P - SYSTEM REASON & ERRONEOUS - ERRONEOUSDIAGNOSES.
statusLabel varchar This column stores one of the below pre-defined values as status label. MEDICAL REASON, PATIENT REASON, SYSTEM REASON & ERRONEOUS DIAGNOSES. These values are mapped with the values in the status column respectively. 1P - MEDICAL REASON, 2P - PATIENT REASON, 3P - SYSTEM REASON & ERRONEOUS - ERRONEOUSDIAGNOSES.
reset tinyint This column stores flags 0 or 1.
modifiedtime datetime This column stores a time stamp when the alert was suppressed. The format used is YYYY-MM-DD HH:MM:SS.

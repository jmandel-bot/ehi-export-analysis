Table birth_apgar_screening
Description This table stores information about the Apgar screening test results for a patient during birth.
Column DataType Description
id (Primary Key) int Unique identifier for each Apgar screening record. Example: 1, 2, 3.
birthvitalsid int identifier for the birth vitals record associated with this Apgar screening. This is a foreign key that refers to the `id` column in the `birth_vitals` table. Example: 101, 102, 103.
NAME varchar Name of the Apgar screening test. Example: "Apgar Test".
score smallint This column stores the apgar score. Default value is -1 which means no score is selected. It can contain values from -1 to 9.
timevalue tinyint This column stores the time after birth when the screening was performed. It can have value as 1, 5, 10. Hence, the datatype tinyint is selected. This field will always have a value hence defined as NOT NULL.
timeunit varchar This column stores the unit in which timvalue is selected and can have values eg. 'min' and hence, datatype varchar(10) is selected. This field will always have a value hence defined as NOT NULL.
comments varchar This contains comments for each apgar screening test and cannot have more than 100 chaaracters, hence, datatype varchar(100) is selected

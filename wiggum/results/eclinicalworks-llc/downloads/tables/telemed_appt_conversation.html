EHI Export Schema
Table telemed_appt_conversation
Description This table is used for the call summary feature. Provider/staff and patient conversation and relative summary can be retracted for chart documentation also used for updating import summary and translation of original conversation
Column DataType Description
Id (Primary Key) int This column is used to store unique id for the table which is auto increment.
enc_id int this column is used to store encounter id so each encounter have their own call summary FOREIGN KEY (enc_id)REFERENCES enc(encounterid)
org_conversation nvarchar This column is used for storing conversation. Conversation can contain Unicode characters used for language i.e., Hindi, Spanish
feedback tinyint On conversation summary, user feedback as like/dislike is stored for analytics. it will store 0 or 1, 0 for dislike and 1 for like, this column is based on user input so null value is possible.
conversation varchar This column is used to conversation between provider/staff/contacts/patient.
summary varchar AI generated TV Call summary will display in call summary page, this is store summary of call.
updated_summary varchar This column is used to store summary which gets imported in the charts, this is kept to showcase what was imported against the summary generated. This will help in case s regarding the wrong data imports.
updated_timestamp datetime This column is used for storing timestamp for any updates made to chart against the summary received.
duration int This column is used for storing Call/Conversation duration for which transcription was enabled in seconds.

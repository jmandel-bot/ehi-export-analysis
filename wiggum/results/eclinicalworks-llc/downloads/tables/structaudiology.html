EHI Export Schema
Table structaudiology
Description This table stores structured audiology data related to patient encounters, including information about the category, item, and value of the data, as well as any comments and timestamps for record creation and modification.
Column DataType Description
structureDataid (Primary Key) int Unique identifier for each structured audiology data entry. (Example: 1, 2, 3)
encounterid int identifier for the patient encounter to which the structured audiology data is related. Foreign key relation to `enc` table and `encounterid` column. (Example: 101, 102, 103)
categoryid int identifier for the category of the structured audiology data. Foreign key relation to `items` table and `itemid` column. (Example: 10, 11, 12)
itemid int identifier for the specific item within the category of the structured audiology data. (Example: 1001, 1002, 1003)
structureDataDetailid int identifier for the detailed information of the structured audiology data. Foreign key relation to `structdatadetail` table and `id` column. (Example: 2001, 2002, 2003)
VALUE varchar Value of the structured audiology data in a varchar format. (Example: "Normal", "Mild", "Severe")
valueid int identifier for the value of the structured audiology data. Foreign key relation to `structdataoptions` table and `optionid` column. (Example: 3001, 3002, 3003)
comments varchar Additional comments related to the structured audiology data, stored as a varchar with a maximum length of 500 characters. (Example: "Patient reported difficulty hearing in noisy environments.")
recordUpdated tinyint A tinyint flag indicating whether the record has been updated (1) or not (0). (Example: 0, 1)
recordCreatedDate datetime time stamp for when the structured audiology data record was created. (Example: "2022-01-01 12:34:56")
recordModifiedDate datetime time stamp for when the structured audiology data record was last modified. (Example: "2022-01-02 14:23:45")

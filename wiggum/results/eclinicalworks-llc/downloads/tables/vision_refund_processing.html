EHI Export Schema
Table vision_refund_processing
Description This table stores information about refund checks, including the associated file, user details, and the time the record was created.
Column DataType Description
ID (Primary Key) int Unique identifier for each refund processing record. Example: 1, 2, 3.
StoreNum varchar Store number where the refund is being processed. Example: '1001', '1002', '1003'.
OrderNo varchar Unique order number associated with the refund. Example: 'ORD123', 'ORD124', 'ORD125'.
EncounterId int Foreign key referring to the encounter record associated with the refund. Example: 101, 102, 103. Foreign key relation: `enc.EncounterId`
RefundReason text Text description of the reason for the refund. Example: 'Damaged item', 'Wrong item delivered', 'Customer changed mind'.
DateCreated datetime Date and time when the refund processing record was created. Example: '2021-08-01 12:34:56', '2021-08-02 14:23:45'.
POSRefundDate datetime Date and time when the refund was processed at the Point of Sale (POS). Example: '2021-08-03 10:15:30', '2021-08-04 16:45:20'.
status int Status code of the refund processing. Example: 0 (Pending), 1 (Approved), 2 (Rejected).
deleteflag int Flag to indicate if the record is deleted or not. Example: 0 (Not deleted), 1 (Deleted).
ReinstatementInd int Indicator for reinstatement of the refund. Example: 0 (Not reinstated), 1 (Reinstated).
ProcessedDate varchar Date when the refund was processed, stored as a string. Example: '2021-08-05', '2021-08-06'.
Notes varchar Additional notes or comments about the refund processing. Example: 'Refund approved by manager', 'Customer requested store credit'.
RefundStatus int Column refers Refund status that defines refund is processed or not

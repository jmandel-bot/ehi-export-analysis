Table cardiosofttests
Description This table stores information about various cardio tests performed on patients.
Column DataType Description
id (Primary Key) int Unique identifier for each test record. (Example: 1, 2, 3)
RecordedAt varchar time stamp when the test record was created, stored as a string. (Example: '2021-09-15 12:34:56')
ModifiedAt varchar time stamp when the test record was last modified, stored as a string. (Example: '2021-09-15 12:34:56')
Encid int Unique identifier for the encounter associated with the test. (Example: 101, 102, 103)
Patientid int Unique identifier for the patient who underwent the test. (Example: 1001, 1002, 1003)
labid int Unique identifier for the lab where the test was performed. (Example: 1, 2, 3)
labType int Integer representing the type of lab. (Example: 1, 2, 3)
labName varchar Name of the lab where the test was performed. (Example: 'Lab A', 'Lab B', 'Lab C')
TestType varchar Enum representing the type of cardio test performed. Possible values are 'RECG', 'STRS', 'SPRO', 'HLTR', 'ABPR'. (Example: 'RECG')
DeleteFlag tinyint A tiny integer indicating if the record has been deleted (1) or not (0). (Example: 0, 1)
modifiedflag int An integer indicating if the record has been modified (1) or not (0). (Example: 0, 1)
filename varchar Name of the file associated with the test. (Example: 'testfile1.txt', 'testfile2.txt')
ftpdirpath varchar Directory path for the file on the FTP server. (Example: '/path/to/test/files/')

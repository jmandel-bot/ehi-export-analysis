EHI Export Schema
Table encex
Description This table stores information about specific encounters and their related cases, as well as the status and user responsible for the encounter.
Column DataType Description
id (Primary Key) int This is the primary key for the table, which is an auto-incrementing integer value that uniquely identifies each row in the table. Example: 1, 2, 3, etc.
Encid int This is a foreign key column that refers to the encounter id from enc table . It is an integer value that represents the id of the encounter. Example: 101, 102, 103, etc.
Caseid int This column stores case id asosciated with this encounter. This column refers column caseid from case_details table.
rpstatus tinyint This is a tinyint(1) column that represents the status of the encounter. It can have a value of 0 or 1, where 0 might represent inactive and 1 might represent active.
RPUserid int This is an integer value that represents the id of the user responsible for the encounter. Example: 10001, 10002, 10003, etc.
HospNewEstFlag char This is a char(2) column that stores a flag related to the hospital or establishment. Example: A1, B2, C3, etc.

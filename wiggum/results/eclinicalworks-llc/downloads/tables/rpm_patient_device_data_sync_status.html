EHI Export Schema
Table rpm_patient_device_data_sync_status
Description This table stores information about device readings sync status for RPM prescribed devices for a patient.
Column DataType Description
id (Primary Key) int This column is used to identify unique records. It is an auto incremented primary key of This column . Data type is int. This field doesn't allow NULL value. 
rpm_patient_devices_mapping_id int This is a foreign key column. It refers to primary key column id of rpm_patient_devices_mapping table. The data type of this column is INT and it does not allow a NULL value.
syncDateTime datetime This column contains date time when the sync happened at healow side. This column stores datetime hence the data type is DATETIME. This column cannot have a NULL value.
sync_status varchar This column stores the status of the last sync for selected patient. The values can be textual information such as Error of Successful and so on, therefore the data tyoe of this column is VARCHAR(20). This column can have a NULL value. This column has a check constraint because it can store either Error or Successful values.
error_code_id int This is a foreign key column. It refers to primary key column id of healow_data_sync_error_codes table. The data type of this column is INT and it allows a NULL value.
addedDateTime datetime This column contains date time when the record was created on emr side. This column stores datetime hence the data type is DATETIME. This column can have a NULL value.
modifiedDateTime datetime This column contains date time when the record was modified last. This column stores datetime hence the data type is DATETIME. This column can have a NULL value.
deleteFlag tinyint This flag indicates that this record has been deleted or not, datatype is TINYINT. This column stores an integer value hence tinyint is selected.

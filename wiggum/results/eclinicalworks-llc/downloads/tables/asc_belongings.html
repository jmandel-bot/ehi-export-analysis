EHI Export Schema
Table asc_belongings
Description This table is used to store a patient's belonging details.
Column DataType Description
id (Primary Key) int This column is an auto generated unique primary key.
encounterid int The identifier of the encounter to which the belonging is related. Foreign key relation to the encounters table (id column). Example: 101, 102, 103.
itemid int The identifier of the item that is considered a belonging. Foreign key relation to the items table (id column). Example: 1001, 1002, 1003.
statusid int The identifier of the status of the belonging. Foreign key relation to the statuses table (id column). Example: 1 (Available), 2 (In Use), 3 (Returned).
notes text Additional text notes about the belonging. Example: Item in good condition, Item damaged during use.
displayIndex smallint The display order of the belonging in a list or user interface. Example: 1, 2, 3.
receivedDate datetime The date and time when the belonging was received. Example: 2021-08-01 14:30:00.
returnDate datetime The date and time when the belonging was returned. Example: 2021-08-15 16:45:00.
userid int The identifier of the user who is responsible for the belonging. Foreign key relation to the user table (id column). Example: 10001, 10002, 10003.
returnFlag tinyint A flag indicating whether the belonging has been returned or not. Example: 0 (Not Returned), 1 (Returned).
deleteFlag smallint A flag indicating whether the record has been deleted or not. Example: 0 (Not Deleted), 1 (Deleted).
returnEncounter int The identifier of the encounter during which the belonging was returned. Foreign key relation to the encounters table (id column). Example: 104, 105, 106.

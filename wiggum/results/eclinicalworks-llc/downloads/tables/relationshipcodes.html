EHI Export Schema
Table relationshipcodes
Description This table stores relationship codes and their corresponding names, print names, and custom names. It also indicates whether a code has been deleted or not.
Column DataType Description
codeid (Primary Key) int This is the primary key and an auto-incrementing integer column that uniquely identifies each relationship code in the table. Example: 1, 2, 3, etc.
codename varchar This is a varchar(25) column that stores the name of the relationship code. It has a NOT NULL constraint and a default value of an empty string. Example: 'code1', 'code2', 'code3', etc.
printname varchar This is a varchar(255) column that stores the print name of the relationship code. It has a NOT NULL constraint and a default value of an empty string. Example: 'Relationship Code 1', 'Relationship Code 2', 'Relationship Code 3', etc.
isdeleted int This is an int(1) column that indicates whether the relationship code has been deleted or not. It has a default value of '0'. Example: 0 (not deleted), 1 (deleted).
customname varchar This is a varchar(30) column that stores a custom name for the relationship code, if any. It has a default value of an empty string. Example: 'Custom Code 1', 'Custom Code 2', 'Custom Code 3', etc.

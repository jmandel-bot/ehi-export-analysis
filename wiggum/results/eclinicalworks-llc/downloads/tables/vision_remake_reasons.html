Table vision_remake_reasons
Description This table stores information about reasons for remaking a vision, including details about the category, primary reason, and whether or not the reason is deleted.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3
cat_id int Foreign key mapping of id of vision_master table
Description varchar A brief description of the reason for remaking the vision. Example: "Lens defect", "Incorrect prescription", "Damaged frame"
primary_reason_id int Foreign key mapping of id of vision_master table
isdeleted tinyint A tiny integer (0 or 1) indicating whether the record is deleted (1) or not (0). Example: 0, 1
createdby int Integer representing the user id who created the record. Example: 101, 102, 103
createdon datetime Date and time when the record was created. Example: '2021-09-01 12:34:56', '2021-09-02 14:30:00'
updatedby int This column stores the unique identifier for the encounter associated with the Schirmer test results. It is an integer value and has a foreign key relation with the `enc` table's `encounterid` column. For example, if the Schirmer test results are recorded during encounter number 5, the `encounterid` value will be 5.
updatedon datetime This column stores the Schirmer test result for the patient's right eye. It is a small integer value that can be NULL if the test result is not available. For example, if the Schirmer test result for the right eye is 15 mm, the `r_eye` value will be 15.
displayorder int This column stores the Schirmer test result for the patient's left eye. It is a small integer value that can be NULL if the test result is not available. For example, if the Schirmer test result for the left eye is 12 mm, the `l_eye` value will be 12.

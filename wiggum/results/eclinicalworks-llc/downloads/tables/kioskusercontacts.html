Table kioskusercontacts
Description This table stores information about the contacts of kiosk users, including their personal details, address, phone numbers, and their relationship to the user.
Column DataType Description
id (Primary Key) int This is an auto incremented primary key of This column .
RelatedPtid int This is an integer that represents the patient id related with the patient.This column refers to pid column from patients table.
pid int This is an integer that represents the patient id associated with the contact.This column refers to pid column from patients table.
fName varchar First name of the contact. (e.g., "John", "Jane", "Alice")
lName varchar Last name of the contact. (e.g., "Doe", "Smith", "Johnson")
relation varchar Relationship of the contact to the kiosk user. (e.g., "Mother", "Father", "Spouse")
address1 varchar First line of the contact's address. (e.g., "123 Main St", "456 Elm St")
address2 varchar Second line of the contact's address, if applicable. (e.g., "Apt 4B", "Suite 200")
city varchar City where the contact lives. (e.g., "New York", "Los Angeles", "Chicago")
state varchar State where the contact lives. (e.g., "NY", "CA", "IL")
zipCode varchar Zip code of the contact's address. (e.g., "10001", "90001", "60601")
homePhone varchar Home phone number of the contact. (e.g., "555-123-4567", "555-987-6543")
workPhone varchar Work phone number of the contact, if applicable. (e.g., "555-321-7654", "555-654-9872")
CellPhone varchar Cell phone number of the contact. (e.g., "555-111-2222", "555-333-4444")
Guardian char Indicates if the contact is a legal guardian of the kiosk user. (e.g., "Y" for Yes, "N" for No)
CountryCode varchar Country code of the contact's address. (e.g., "US", "CA", "GB")
displayorder tinyint Display order of the contact in the user interface. (e.g., 1, 2, 3)
contactid int This column stores an contact id of contact. This column refers to id column from contacts table.
isEmergencyContact char Indicates if the contact is an emergency contact for the kiosk user. (e.g., "1" for Yes, "0" for No)
kioskDemographicid int This column stores an demographic id of contact where demographics changes request was made from kiosk. This column refers to id column from kioskuser table.
contactDeleted char Indicates if the contact record has been deleted. (e.g., "1" for Yes, "0" for No)
HIPAA char Indicates if the contact has HIPAA authorization. (e.g., "Y" for Yes, "N" for No)
isFamilyMember tinyint Indicates if the contact is a family member of the kiosk user. (e.g., 1 for Yes, 0 for No)

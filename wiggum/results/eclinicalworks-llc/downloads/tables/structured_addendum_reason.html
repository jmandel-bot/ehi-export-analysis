Table structured_addendum_reason
Description This table stores the reasons for structured addendums, along with information about the user who created the reason and timestamps for when the reason was created.
Column DataType Description
reasonId (Primary Key) int Unique identifier for each reason, with an integer value. For example: 1, 2, 3.
reason varchar The text description of the reason for the structured addendum. For example: "Typographical error", "Missing information".
userId int The unique identifier (integer) of the user who created the reason. This is a foreign key that refers to the `users` table and the `uid` column. For example: 101, 102, 103.
createdDate datetime The date when the reason was created, in the format YYYY-MM-DD. For example: 2021-08-01, 2021-08-02.
createdTime datetime The time when the reason was created, in the format HH:MM:SS. For example: 14:30:00, 15:45:30.
deleteflag tinyint A tiny integer (0 or 1) indicating whether the reason has been deleted (1) or not (0). For example: 0, 1.
nonStructuredReason tinyint A tiny integer (0 or 1) indicating whether the reason is non-structured (1) or structured (0). For example: 0, 1.

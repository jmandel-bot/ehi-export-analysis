EHI Export Schema
Table kioskusers
Description This table stores information about kiosk users, including their personal details, contact information, and demographic data.
Column DataType Description
id (Primary Key) int This is an auto incremented primary key of This column .
UID int This column stores an user id associated with demographic changes. This column refers to UID column from user table.
email varchar Email address of the kiosk user, can be NULL (e.g., john@example.com, jane@example.com).
dob varchar Date of birth of the kiosk user in varchar format, can be NULL (e.g., '1990-01-01', '1985-12-31').
gender varchar Gender of the kiosk user, can be NULL (e.g., 'Male', 'Female', 'Non-binary').
ethnicity_name varchar Name of the user's ethnicity, can be NULL (e.g., 'Hispanic', 'Non-Hispanic').
race_name varchar Name of the user's race, can be NULL (e.g., 'White', 'Black', 'Asian').
language_id varchar id of the user's preferred language, can be NULL (e.g., '1', '2', '3').
city varchar City where the user lives, can be NULL (e.g., 'New York', 'Los Angeles').
state varchar State where the user lives, can be NULL (e.g., 'NY', 'CA').
address1 text First line of the user's address, can be NULL (e.g., '123 Main St').
phoneNo varchar User's phone number, can be NULL (e.g., '555-123-4567').
del_flag char Deletion flag, '0' for active and '1' for deleted (e.g., '0', '1').
web_sync char Web synchronization flag, '0' for not synced and '1' for synced (e.g., '0', '1').
cellNo varchar User's cell phone number, can be NULL (e.g., '555-987-6543').
workNo varchar User's work phone number, can be NULL (e.g., '555-321-7890').
address2 text Second line of the user's address, can be NULL (e.g., 'Apt 4B').
zipCode varchar User's zip code, can be NULL (e.g., '10001', '90001').
picFileName varchar File name of the user's profile picture, default is an empty string (e.g., 'john_profile.jpg').
otherRace varchar Other race information if not listed, default is an empty string (e.g., 'Multiracial').
sorowid int This column stores an sexual orientation options id of that user. This column refers to id column from sexual_orientation_options table.
socodeOtherReason varchar This column stores an another sexual orientation option which is not avaliable so user can add
girowids varchar This column stores an gender identity options id of that user. This column refers to id column from gender_identity_options table.
gicodeOtherReason varchar This column stores an another gender identity option which is not avaliable so user can add
birthSex varchar Birth sex of the user, can be NULL (e.g., 'Male', 'Female').
ethnicityids varchar This column stores an ethinicty codes of that user. This column refers to code column from ethinicity table.
raceids varchar This column stores an race codes of that user. This column refers to code column from race_codes table.
soReceived tinyint Flag indicating whether some data is received, '0' for not received and '1' for received (e.g., '0', '1').
birthSexReceived tinyint Flag indicating whether birth sex data is received, '0' for not received and '1' for received (e.g., '0', '1').
giReceived tinyint Flag indicating whether some other data is received, '0' for not received and '1' for received (e.g., '0', '1').
ethnicityidsReceived tinyint Flag indicating whether ethnicity ids are received, '0' for not received and '1' for received (e.g., '0', '1').
raceidsReceived tinyint Flag indicating whether race ids are received, '0' for not received and '1' for received (e.g., '0', '1').
dataImported char Flag indicating whether data is imported, '0' for not imported and '1' for imported (e.g., '0', '1').
ts datetime time stamp of the record creation or update (e.g., '2021-01-01 12:34:56').
facilityid int This column stores an id of facility where the kiosk login was failed. This column refers to id column from edi_facilities table.
kioskDemographicSettingLogid int This column stores an log id of kiosk demographic setting associated with that demographic changes. This column refers to id column from kioskDemographicSettingLogDetails table.
source tinyint Flag to indicate the source of the record. Example: '0' (eCheck-in), '1' (Kiosk).
insuranceJson varchar Meta data information of insurances uploded by user in json format
driverLicenseJson varchar Meta data information of driving license uploded by user in json format
preferred_name varchar User's preferred name, can be NULL (e.g., 'Johnny', 'Janey').
encid int This is an integer that represents the encounter id associated with the demographic changes.This column refers to encounterid column from enc table.
webEncid int This is an integer that represents the web encounter id associated with the demographic changes.This column refers to encounterid column from enc table.

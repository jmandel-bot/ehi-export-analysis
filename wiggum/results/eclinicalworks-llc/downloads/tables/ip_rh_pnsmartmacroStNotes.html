Table ip_rh_pnsmartmacroStNotes
Description This table is used to store the narrative or sentences for the template.
Column DataType Description
id (Primary Key) int This is an auto-incremented primary key column in this table. The data type is int because the notes created could be huge in number. The column cannot be NULL as it will always have an id.
templateId int This column refers to the id column in ip_rh_clinical_sd_template table. It stores the templateId for which the narrative sentences are created. The data type is int. This column will always have a value and hence it cannot be NULL.
parentId int This column refers to id column in the same table i.e ip_rh_pnsmartmacroStNotes table i.e when the current sentence is a child sentence of another sentence. It gets populated only for rows who has childSentence column value as 1.
sentenceNote varchar This column stores the sentences/narratives for the corresponding template id. The data type is varchar(max). As the column could store huge data as value and we have free text, therefore we have defined it as varchar(max) and the default value is NULL.
sentenceOrder smallint This column stores the order in which the sentences are created for the corresponding template. The data type is smallint as compared to int is because the expected sentences displayed will not exceed the limitation provided by the smallInt datatype.. This column is NOT NULL and the default value is 0.
childSentence tinyint This column stores the values 0 or 1. If the value is 1 the sentence is a child sentence else not. The data type is tinyint and the default value is 0. The column cannot be NULL.
triggerQuestionId int This column refers to the id column in ip_rh_structdatadetail table. It stores the question id that is triggered for the corresponding sentence id. The data type is INT and the default value is NULL.
triggerOptionId int This column refers to the optionId column in ip_rh_structdataoptions table. It stores the option id that is triggered for the corresponding sentence id. The data type is INT and the default value is NULL.
mandatory tinyint This column stores flags 0/1 to indicate whether the sentence is mandatory to fill or not for the template on the user facing screen.0 means the sentence is not mandatory and 1 means the sentence is mandatory
deleteFlag tinyint A flag indicating if the sentence is deleted (1) or not (0). The tinyint data type allows for efficient storage of binary values.
createdBy int Stores the user id of the person who created the sentence. It refers to uid column in users table.
modiBy int Stores the user id of the person who last modified the sentence. It refers to uid column in users table.
modiTime datetime Stores the date and time the sentence was last modified. The datetime data type allows for precise storage of date and time values. The default value is NULL.
createdDateTime datetime Stores the date and time the sentence was created. The datetime data type allows for precise storage of date and time values. This field will always have a value hence defined as NOT NULL.

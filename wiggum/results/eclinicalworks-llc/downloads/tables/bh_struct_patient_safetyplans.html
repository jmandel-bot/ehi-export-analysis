Table bh_struct_patient_safetyplans
Description This table stores information about patient safety plans, including their associated encounter, category, item, and detail IDs, as well as values, notes, and display indices.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
encounterid int The identifier of the encounter associated with the patient safety plan. Foreign key relation: `enc` table, `encounterid` column. Example: 101, 102, 103.
catid int The identifier of the category associated with the patient safety plan. Foreign key relation: `items` table, `itemid` column. Example: 1, 2, 3.
itemid int The identifier of the item associated with the patient safety plan. Foreign key relation: `items` table, `itemid` column. Example: 10, 11, 12.
detailid int The identifier of the detail associated with the patient safety plan. Foreign key relation: `structdatadetail` table, `id` column. Example: 100, 101, 102.
VALUE varchar The value of the patient safety plan, stored as a string with a maximum length of 2000 characters. Example: "High risk", "Low risk", "Moderate risk".
valueid varchar The identifier of the value associated with the patient safety plan, stored as a string with a maximum length of 2000 characters. Example: "1", "2", "3".
notes varchar Additional notes related to the patient safety plan, stored as text. Example: "Patient has a history of self-harm", "Patient has a strong support system".
displayIndex int The display index of the patient safety plan, stored as an integer. Example: 0, 1, 2.
deleteflag tinyint A flag indicating whether the record has been deleted, stored as a tiny integer (1 for deleted, 0 for not deleted). Example: 0, 1.
enc_templateid int The identifier of the encounter template associated with the patient safety plan. Foreign key relation: `bh_patient_safetyplans` table, `id` column. Example: 1001, 1002, 1003.

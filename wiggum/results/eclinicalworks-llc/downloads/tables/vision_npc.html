Table vision_npc
Description This table stores information about non-player characters (NPCs) in a vision system, including encounter ID, type of vision prescription, and various vision-related attributes.
Column DataType Description
id (Primary Key) int This is the primary key and an auto-incrementing integer that uniquely identifies each record in the vision_npc table. Example: 1, 2, 3, etc.
encounterid int This is a foreign key that refers to the `encounterid` column in the `enc` table. It represents the unique identifier for an encounter with an NPC. Example: 101, 102, 103, etc.
rx_type tinyint This is a tiny integer column that represents the type of vision prescription for the NPC. It can be NULL or an unsigned integer value. Example: 0, 1, 2, etc.
npc_blur smallint This is a small integer column that represents the blur value for the NPC's vision. It can be null or an integer value. Example: 10, 20, 30, etc.
npc_break smallint This is a small integer column that represents the break value for the NPC's vision. It can be null or an integer value. Example: 5, 10, 15, etc.
npc_recovery smallint This is a small integer column that represents the recovery value for the NPC's vision. It can be null or an integer value. Example: 3, 6, 9, etc.
comments varchar This is a varchar column with a maximum length of 255 characters that stores any additional comments or notes about the NPC's vision. It has a default value of an empty string. Example: "NPC has astigmatism.", "NPC wears glasses.", etc.

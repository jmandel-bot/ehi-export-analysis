Table prisma_procedure_service_location
Description This table stores information about service locations related to specific medical procedures.
Column DataType Description
id (Primary Key) bigint A unique identifier for each record in the table. Example: 1, 2, 3.
procedureDetailId bigint A foreign key that refers to the `id` column in the `prisma_procedure_details` table. It represents the relationship between the service location and the specific medical procedure. Example: 101, 102, 103.
serviceLocationId varchar A unique identifier for the service location. Example: "SL001", "SL002", "SL003".
locationName varchar The name of the service location. Example: "Main Hospital", "Downtown Clinic", "Uptown Health Center".
locationTypeCode varchar A code representing the type of service location.
locationTypeName varchar A descriptive name for the type of service location. Example: "Hospital", "Clinic", "Health and Learning Center".
addressLine1 varchar The first line of the service location's address. Example: "123 Main St", "456 Elm St", "789 Oak St".
addressLine2 varchar The second line of the service location's address, if applicable. Example: "Suite 100", "Building 2", "Apt 3B".
city varchar The city where the service location is situated. Example: "New York", "Los Angeles", "Chicago".
state varchar The state where the service location is situated. Example: "NY", "CA", "IL".
postalCode varchar The postal code for the service location's address. Example: "10001", "90001", "60601".
country varchar The country where the service location is situated. Example: "United States", "Canada", "United Kingdom".
workPhone varchar The work phone number for the service location. Example: "(555) 123-4567", "(555) 987-6543", "(555) 111-2222".
workFax varchar The work fax number for the service location, if applicable. Example: "(555) 123-4568", "(555) 987-6544", "(555) 111-2223".

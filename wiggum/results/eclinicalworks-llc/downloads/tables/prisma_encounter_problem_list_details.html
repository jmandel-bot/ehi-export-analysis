Table prisma_encounter_problem_list_details
Description This table stores details about problems encountered during a patient's visit, including information about the problem, its status, and related observations.
Column DataType Description
id (Primary Key) bigint Unique identifier for each record in the table (e.g., 1, 2, 3).
sectionId bigint Identifier for the section to which the problem belongs; foreign key relation to `prisma_ccd_document_section.id` (e.g., 101, 102, 103).
entryId varchar Unique identifier for each entry within a section (e.g., "E1", "E2", "E3").
fhirId int Stores id of fhir_source_table.
addedOn varchar Date when the problem was added to the list (e.g., "2021-08-01", "2021-08-02", "2021-08-03").
statusCode varchar Code representing the status of the problem (e.g., "active", "inactive", "resolved").
statusName varchar Name of the status corresponding to the `statusCode` (e.g., "Active", "Inactive", "Resolved").
problemName varchar Name of the problem (e.g., "Diabetes", "Hypertension", "Asthma").
problemTypeCode varchar Code representing the type of problem (e.g., "PT1", "PT2", "PT3").
problemType varchar Description of the problem type corresponding to the `problemTypeCode` (e.g., "Chronic", "Acute", "Recurring").
onset varchar Date when the problem started (e.g., "2021-07-01", "2021-07-02", "2021-07-03").
resolvedOn varchar Date when the problem was resolved (e.g., "2021-08-15", "2021-08-16", "2021-08-17").
observationCode varchar Code representing the observation related to the problem (e.g., "OC1", "OC2", "OC3").
observationName varchar Name of the observation corresponding to the `observationCode` (e.g., "Blood Pressure", "Heart Rate", "Respiratory Rate").
isException tinyint Flag indicating if the problem is an exception (0 for false, 1 for true).
problemCondition varchar Condition of the problem (e.g., "Stable", "Worsening", "Improving").
problemWuCondition varchar Condition of the problem during the follow-up (e.g., "Stable", "Worsening", "Improving").
exceptionMessage varchar Message describing the exception, if any. Empty if there is no exception.
observationCodeSystem varchar Code system used for the `observationCode` (e.g., "LOINC", "SNOMED-CT", "ICD-10").
verificationStatusCode varchar Code representing the verification status of the problem (e.g., "confirmed", "refuted", "unknown").
encounterReference bigint Identifier for the encounter to which the problem is related (e.g., 201, 202, 203).
recorderReference int Identifier for the person who recorded the problem (e.g., 301, 302, 303).
asserterReference int Identifier for the person who asserted the problem (e.g., 401, 402, 403).
severityCode varchar Code representing the severity of the problem (e.g., "mild", "moderate", "severe").
severityDisplay varchar Description of the severity corresponding to the `severityCode` (e.g., "Mild", "Moderate", "Severe").
conditionCategoryDisplay varchar Description of the category of the condition (e.g., "Diagnosis", "Symptom", "Finding").
deathDiagnosis tinyint Flag indicating if the problem is related to the cause of death (0 for false, 1 for true).

Table patient_device_daily_review_status
Description This table stores the daily reading review status of rpm enrolled patient device. This includes information related to data status, review status, billable flag and review date.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT. This column stores integer value which could grow with number of transactions added and hence integer is selected . It is AUTO_INCREMENT column and has PRIMARY KEY constraint defined on this column
patientid int This column stores unique id of a Patient which references the UID column of the user table. The data type chosen is INT. This field will always have a value hence defined as NOT NULL.
rpm_enrollment_id int Contains enrollmentid value referenced from id column of the rpm_enrollment_details table, datatype is INT. This field will always have a value hence defined as NOT NULL..
reading_date datetime Reading date is the datetime for which we are receiving the data for any of the devices. This column stores date hence Date is selected. This field will always have a value hence defined as NOT NULL. By default '1901-01-01'/ '0000-00-00' for MSSQL/MYSQL accordingly is chosen.
review_status tinyint This column is introduced to maintain if the review for all devices for a patient is completed on a daily basis. It contains review status with values 0 and 1,(0-Pending,1-Complete). The datatype is TININT. By default we maintain a value as Pending. But once the review is complete for all devices we maintain it as 1.This field will always have a value hence defined as NOT NULL.
data_status tinyint This column is introduced to maintain if any one of the devices has either data not received/Contains data status of reading for all devices for a daily basis. If anyone of the device has below criteria, the corresponding value is maintained in this column. This uses TINYINT-- 0 - data_not_received, 1 - abnormal,2 abnormal/data_not_received, 3 complete.By default we would maintain 0 which means the data is not received. We have kept the data type of this field TINYINT as currently there are 4 possible values - 0, 1, 2 and 3.
isNonbillableDevice tinyint This column is introduced to check if any one of the devices for a patient has readings coming from a non prescribed vendor which could make it ineligible for billing. The data type chosen is TINYINT.This column could potentially contains contains two value 0 and 1. If anyone of the device has a reading that is non prescribed/non billable then we would maintain a value 1,else 0. By default we would maintain 0 and as it always has a value we defined it as NOT NULL
deleteFlag tinyint Flag indicates the last reading received for a day is deleted, datatype is tinyint - 0 - Not Deleted, 1 - Deleted. This field will always have a value hence defined as NOT NULL. By default, this column should not be in Deleted status hence default 0 is chosen.
modifiedBy int Contains the user id who has modified the record last. The data type chosen is INT. This field is expected to have a value always and has a value referencing the id column of the user table
modifiedDatetime datetime Contains "modifiedDatetime " value when a user is adds/updates reading for any of the devices and datatype is DATETIME. By default, current system datetime needs to be inserted. This field will always have a value hence defined as NOT NULL

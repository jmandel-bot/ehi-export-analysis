EHI Export Schema
Table document
Description This table stores the document relationship to patient identifier, staff, provider, and as well as the physical path of the file that is being mapped.
Column DataType Description
docID (Primary Key) int Unique identifier for each document (e.g., 1, 2, 3).
encId int Stores encounter Id which the document is associated to, if it is not associated to any encounter, id stored in this column is 0. Column refers to encounterid column in enc table.
filename varchar The original file name of the document (e.g., "report.pdf", "image.jpg").
customName varchar A custom name given to the document by the user (e.g., "Medical Report", "X-Ray Image").
doc_Type int Stores type of a document. so far application supports 96 different types of documents. This column refers to column id from documentfolders table.
scanDate datetime The date the document was scanned (e.g., '2021-09-01', '2021-09-02').
refId int This column stores an id of the referral, case manager, lab, patient.
ScannedBy varchar This column stores the full name of the person who scans the document. This could be a Provider or a Staff name.
Description varchar A text description of the document (e.g., "This is a medical report for patient X.").
PatientId int This column stores patient identifier for a patient, whom the document is associated to. It refers to column uid from users and pid from patients.
Review int Integer representing the review status of the document (e.g., 0 for not reviewed, 1 for reviewed).
ReviewerId int This column stores the id of the person who reviews the document associated to a patient. This id could be Provider's or a Staff selected in the User Interface. This column refers to column uid from users.
reviewername varchar This column stores the name of the person who reviews the document associated to a patient. This name could be Provider's or a Staff selected in the User Interface.
Priority int Integer representing the priority level of the document (e.g., 0 for low, 1 for high).
attachto varchar This column stores name of the chart or lab/DI/Procedure or x-ray or referral selected along with the date.
delFlag int This column stores value 0 / 1 indicating if the document is marked deleted using the User Interface, this is soft delete. 0 - Not Deleted , 1 -Deleted
folderName varchar This column is not being used by application anymore.
oldfilename varchar This column is not being used by application anymore.
migrateflag int This column stores flag 0 or 1. 1 - when a document is a migrated document from interface. 0 - otherwise.
uniqueDocId varchar This column stores a unique string assigned when each document retrieved from the external registry/ repository.
publishToEHX tinyint Integer representing whether the document is published to EHX (e.g., 0 for not published, 1 for published).
eHXSentStatus tinyint Integer representing the sent status of the document to EHX (e.g., 0 for not sent, 1 for sent).
FacilityId int This column holds the id of the Facility selected in the user interfaceby the end user. This columns refers to column id from edi_facilities.
refReqId int This column stores referral id of the incoming referral, when the attachments to the referral are moved to patient documents.
dirpath varchar This column stores the path of the directory from ftp server where the document is saved.
ftpserver varchar need to remove from source code.dead code
TempDocId varchar This column stores an unique id received from vendor in hl7message for the document of a patient.
pnCatId int This column stores an id of the category selected while documenting Examination for the patient. This column refers column itemid from items table.
pnItemId int This column stores an id of the item selected for the corresponding category while documenting the examination for a patient on Progress Note. This column refers to column itemid in items table.
injuryId varchar A string representing the injury ID associated with the document (e.g., "INJ-001", "INJ-002").
consentDocEHX tinyint This column stores flag 0 or 1. 1 - when document is uploaded with patient's consent from CCMR’s eEHX . 0 - otherwise.
insuranceid int This column stores an id of an insurance associated to Patient. This column refers column insId from insurance table.
modifieddatetime datetime Dead code in patientdocs.java file,need to remove it.
refType int This column indicates the reference type if the document was imported from Third party vendor for insurance lockbox or Patient lockbox. This column is not being used anymore.
claimRef int Foreign key referring to the claim reference ID in another table (e.g., 10000, 10001, 10002).
expirydate datetime The date the document expires (e.g., '2022-09-01', '2022-09-02').
servicedate datetime The date the document was serviced (e.g., '2021-08-01', '2021-08-02').
expiresuppress int This column stores a flag 0 or 1. 1 - when the document is suppressed. 0 - otherwise.
suggestedpatients varchar Dead code in TradeCopyTables file,need to remove it.
pagecount int future plan to use it
sizeinbytes bigint future plan to use it
fileformat varchar future plan to use it
drawingid int This column is not being used anymore. This was part of the old Dermatology module on exe side.
documentheight int future plan to use it
documentwidth int future plan to use it
resolution varchar future plan to use it
websync int This column is not used anymore but for international clients.This column stores a flag 0 or 1. 1 - indicates that the row data has been copied to table dmportal and is available to be uploaded to portal, 0 - indicates that the row has not been copied yet.
portalboxUpload int This column is not used anymore. This used to store flags like 0 or 1. 1- indicates that the value has been uploaded to portal, 0 - indicates that it has not been yet.
scanbyid int This column stores an id of the person who scans the document associated to a patient. This id could be Provider's or a Staff selected in the User Interface. This column refers to column uid from user table.
source tinyint future plan to use it
wcReportTypeCode char A two-character code representing the workers' compensation report type (e.g., "A1", "B2").
proxyuid int This column stores id of an user when proxy is enabled and questionnaires are filled by proxy user in the Portal end and same data syncs from portal to EMR. This column refers to uid column in user table.
tocId int Foreign key referring to the table of contents ID in another table (e.g., 1, 2, 3).
isCorrupt tinyint Integer representing whether the document is corrupt (e.g., 0 for not corrupt, 1 for corrupt).
publishToEmployerPortal tinyint Integer representing whether the document is published to the employer portal (e.g., 0 for not published, 1 for published).

Table asc_vitals_device
Description This table stores vital signs data collected from devices during patient encounters.
Column DataType Description
encounterid int The unique identifier for a patient encounter.NOT NULL DEFAULT '0'. Example: 12345
vitalid int The unique identifier for a specific vital sign.NOT NULL DEFAULT '0'. Example: 6789
VALUE varchar The recorded value of the vital sign.NOT NULL DEFAULT ' ' Example: "98.6"
id (Primary Key) int The primary key and unique identifier for each record in the table. Example: 1
UpdatedBy int The unique identifier of the user who last updated the record.NOT NULL DEFAULT '0' Example: 42
unitid int The unique identifier for the unit of measurement for the vital sign.NOT NULL DEFAULT '0'. Example: 3
batchid int The unique identifier for the batch of vital signs recorded during the encounter.NOT NULL DEFAULT '0'. Example: 101
value2 varchar An additional recorded value for the vital sign, if applicable.NOT NULL DEFAULT '0'. Example: "120/80"
qualifierSite int The unique identifier for the site where the vital sign was measured. NOT NULL DEFAULT '0'. Example: 5
qualifierPosition int The unique identifier for the position of the patient when the vital sign was measured.NOT NULL DEFAULT '0'. Example: 2
qualifierType int The unique identifier for the type of qualifier used for the vital sign measurement. NOT NULL DEFAULT '0'.Example: 4
UpdatedTime datetime The date and time when the record was last updated.DATETIME NOT NULL DEFAULT '0000-00-00 00:00:00'. Example: '2021-08-01 12:34:56'
deleteFlag int A flag indicating whether the record has been deleted (1) or not (0). Example: 0
notes varchar Additional notes or comments related to the vital sign measurement. Example: "Patient was feeling anxious during measurement."

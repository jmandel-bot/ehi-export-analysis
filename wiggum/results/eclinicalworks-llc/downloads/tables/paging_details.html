EHI Export Schema
Table paging_details
Description This table stores information about paging events, including details about the paging device, the time the pager was given, the status of the paging, and the associated user and patient information.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
CapCode varchar Unique code for the paging device. Example: '1234567890', '9876543210'.
Facilityid int identifier for the facility where the paging event occurred. Example: 101, 102, 103.
encid int identifier for the encounter associated with the paging event. Example: 1001, 1002, 1003.
PagerGivenAt datetime Time when the pager was given to the user. Example: '08:30:00', '14:45:00'.
PagingTime datetime Time when the paging event occurred. Example: '09:15:00', '15:30:00'.
SeqOfPaging int Sequence number of the paging event. Example: 1, 2, 3.
StatusDesc varchar Status of the paging event, can be 'Sent', 'Pending', or 'Cancelled'.
LastPagedAt datetime Time when the last paging event occurred for the same encounter. Example: '08:00:00', '14:00:00'.
Userid varchar Unique identifier for the user associated with the paging event. Example: 'user123', 'user456'.
ErrorLog varchar Error message, if any, related to the paging event. Example: 'Invalid CapCode', 'Network Error'.
PagerTextMessage text Text message content sent through the pager. Example: 'Patient in Room 101 needs assistance', 'Code Blue in ICU'.
SentStatus varchar Status of the sent message, can be 'Sent', 'Failed', or 'Pending'. Example: 'Sent', 'Failed'.
Patientid int identifier for the patient associated with the paging event. Example: 10001, 10002, 10003.
ScheduledDate datetime Date when the paging event was scheduled. Example: '2021-09-01', '2021-09-02'.

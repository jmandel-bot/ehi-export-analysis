EHI Export Schema
Table contacts
Description This table stores contact information for individuals related to patients, including emergency contacts and family members.
Column DataType Description
contactid (Primary Key) int Unique identifier for each contact entry. (e.g., 1, 2, 3).
pid int This column stores the id of the patient for whom contact information is added.This column refers to column UID from user table.
name varchar This column stores the name of the patient for whom contact information is added.
relation varchar This column stores the relation of the contact to the patient from the predefined value from the user interface.
address varchar First line of the contact's address. (e.g., "123 Main St").
address2 varchar Second line of the contact's address. (e.g., "Apt 4B").
city varchar City where the contact lives. (e.g., "New York", "Los Angeles").
state varchar State where the contact lives. (e.g., "NY", "CA").
zipcode varchar Zip code of the contact's address. (e.g., "10001", "90001").
homePhone varchar Home phone number of the contact. (e.g., "555-123-4567").
workPhone varchar Work phone number of the contact. (e.g., "555-987-6543").
vmid varchar An auto-generated unique string for each record is inserted in to this column .
delflag int This column stores values 0 or 1 indicating if the contact is deleted, this is a soft delete. 0 - Not Deleted , 1 -Deleted.
Lname varchar This column stores the free text last name of the contact.
Fname varchar This column stores the free text first name of the contact.
MI char This column stores the free text middle name of the contact.
Guardian char This column marks whether the current contact entry in the table is an emergency contact for the respective patient. Possible values for this column are 0 and 1 hence it is declared as tiny int and size as 1. Default value is 0 which indicates that the contact is not an emergency contact.
RelatedPtid int This column marks the preferred phone number of a contact. The default is 0 which indicates that the preferred phone is the cell phone.
HExt varchar This column stores the patient’s home phone extension number.
WExt varchar This column stores the patient’s work phone extension number.
Type1 varchar This column stores id of condition type information to get associated with 'activitydetailinfo' table.
Type2 varchar This column stores id of stadium type information to get associated with 'activitydetailinfo' table.
Fax varchar This column stores the patient’s fax number.
Email varchar This column stores the patient’s email address.
Notes varchar This column stores the patient’s attachment note.
Hippa char This column stores values 0, NULL, or 1 to indicate the patient authorized this contact to receive his/her Protected Health Information (PHI). 0/NULL -Not authorized , 1 -Authorized.
displayorder smallint This column stores a digit to indicate order in which contacts should be listed in user interface.
maidenname varchar This column stores the the original surname of a married woman patient.
CountryCode varchar This column stores the country code of the contact's address.
CellPhone varchar This column stores the contact's cell phone number.
isFamilyMember tinyint This column marks whether the current contact entry in the table is Family Member for the respective patient. Possible values for this column are 0 and 1 hence it is declared as tiny int and size as 1. Default is 0 which indicates that the contact is not a family member.
ECOrder tinyint This column marks the emergency contact precedence for the respective patient. In a real environment, the patient can have up to 0-2 digits of contacts hence this column is declared as tiny int and size as 4 (considering future growth). Default is 0. A lower value indicates higher precedence.
isEmergencyContact tinyint This column marks whether the current contact entry in the table is the emergency contact for the respective patient. Possible values for this column are 0 and 1 hence it is declared as tiny int and size as 1. Default is 0 which indicates that the contact is not an emergency contact.
preferredPhone tinyint This column marks the preferred phone number of a contact. The default value is 0 which indicates that the preferred phone is the cell phone.

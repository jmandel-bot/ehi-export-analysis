Table eligibilitypcecopayindicator
Description This table stores information about eligibility, co-pay, co-insurance, and deductible indicators for patient encounters and their associated cost estimates.
Column DataType Description
id (Primary Key) int This is the primary key column, which stores a unique integer identifier for each record in the table. Example: 1, 2, 3, etc.
encounterid int This column stores an integer value representing the encounter id, which is a foreign key that refers to the `encounterid` column in the `enc` table. Example: 101, 102, 103, etc.
estid int This column stores an integer value representing the cost estimate id, which is a foreign key that refers to the `estid` column in the `patient_costestimate_header` table. Example: 1001, 1002, 1003, etc.
isMulitpleCopay tinyint This is a tinyint(1) column that stores a boolean value (0 or 1) indicating whether there are multiple copays for the encounter. Example: 0 (False), 1 (True).
isMulitpleCoInsurance tinyint This is a tinyint(1) column that stores a boolean value (0 or 1) indicating whether there are multiple coinsurances for the encounter. Example: 0 (False), 1 (True).
isMulitpleDeductible tinyint This is a tinyint(1) column that stores a boolean value (0 or 1) indicating whether there are multiple deductibles for the encounter. Example: 0 (False), 1 (True).

Table prisma_procedure_observation
Description This table stores information about observations made during medical procedures, including details about the procedure, observation values, and status.
Column DataType Description
id (Primary Key) bigint This is the primary key column that uniquely identifies each record in the table. It is an auto-incrementing bigint(20) value. Example: 1, 2, 3, etc.
procedureDetailId bigint This column stores the ID of the procedure detail that the observation is associated with. It is a bigint(20) value and has a foreign key relation with the `prisma_procedure_details` table. Example: 10, 25, 30, etc. Foreign Key Relation: `prisma_procedure_details(id)`
procedureObsId varchar This column stores the unique identifier for the procedure observation. It is a varchar(200) value. Example: "OBS123", "OBS456", etc.
procedureValue varchar This column stores the value of the observation made during the procedure. It is a varchar(255) value. Example: "98.6", "120/80", etc.
status varchar This column stores the status of the observation, such as whether it is active, inactive, or completed. It is a varchar(50) value. Example: "active", "completed", etc.
valueCode varchar This column stores a code representing the type of value recorded in the observation. It is a varchar(50) value.
valueName varchar This column stores a descriptive name for the type of value recorded in the observation. It is a varchar(255) value. Example: "Temperature", "Blood Pressure", etc.

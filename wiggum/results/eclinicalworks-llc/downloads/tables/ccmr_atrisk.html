EHI Export Schema
Table ccmr_atrisk
Description This table stores the problem entered in Care Pathways for a patient along with its ccmr id, item id , encounter id etc. This table also used in NFL injury module to store player's injury related details like what is injury name, who has created, for which player it is created, when injury created etc.
Column DataType Description
AtRiskid (Primary Key) int This is a primary key which increments itself automatically.
ccmrid int This column stores the ccmr id associated with the problem entered in Care Plan. This column refers to ccmr_careplan.ccmrid.
Itemid int This column stores the item id of associated problem . This column refers to column item id of items table.
Problem varchar This column stores the Injury/Problem name for player
Lock_Ind int This column is used to save injury locked or not. It can have 0 and 1 value. 0 value means injury is not locked, 1 means injury is locked
DelStat int This column stores soft delete flag. 1: Soft Deleted;0: Not Deleted.
Uname varchar This column stores username of the user who is entering problem of a patient in a Care Plan.
upd_date datetime This column stores the date on which record has been updated recently.
Note text This column is not used in injury module
encounterid int This column stores the encounter id of patient encounter with the user. This column refers to enc.encounterid.
UID int This column stores UID of the user who have created injury. This column refers to users.UID.
probEndDate datetime This column is not used in injury module
reportedDate datetime This column stores a date on which problem of a patient was reported.

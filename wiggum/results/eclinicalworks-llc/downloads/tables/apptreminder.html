Table apptreminder
Description This table stores appointment reminders for patients, including details about the facility, provider, patient, and appointment information.
Column DataType Description
id (Primary Key) int Unique identifier for each appointment reminder. Example: 1, 2, 3.
facilityid int identifier for the facility where the appointment takes place. Example: 101, 102, 103.
providerid int identifier for the healthcare provider associated with the appointment. Example: 1001, 1002, 1003.
patientid int identifier for the patient who has the appointment. Example: 2001, 2002, 2003.
followup varchar A short description of the follow-up action required for the appointment. Example: "Lab test", "Physical therapy", "Consultation".
templateid varchar identifier for the template used to generate the appointment reminder message. Example: "T1", "T2", "T3".
date varchar Date of the appointment in the format YYYY-MM-DD. Example: "2022-01-15", "2022-02-20", "2022-03-25".
visittype varchar Type of visit for the appointment. Example: "Routine checkup", "Emergency", "Specialist consultation".
reason text Reason for the appointment, stored as a text. Example: "Annual physical examination", "Follow-up after surgery", "Discuss test results".
comments text Additional comments or notes related to the appointment, stored as a text. Example: "Bring medication list", "Arrive 15 minutes early", "Fasting required".
deleteflag int Indicates whether the appointment reminder has been deleted (1) or not (0). Example: 0, 1.
address varchar Specifies whether the reminder is sent to the patient ('P') or guardian ('G'). Example: 'P', 'G'.
sentflag int Indicates whether the reminder has been sent (1) or not (0). Example: 0, 1.
reminderMsg text The actual reminder message text sent to the patient or guardian. Example: "Your appointment is scheduled for tomorrow at 2 PM. Please arrive 15 minutes early.", "Don't forget your appointment on Monday at 10 AM. Bring your medication list."
encounterid int identifier for the specific encounter related to the appointment. Example: 3001, 3002, 3003.

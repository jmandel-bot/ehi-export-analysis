EHI Export Schema
Table epa_transaction
Description This table stores information about electronic prior authorization (EPA) transactions, including their status, type, and related details.
Column DataType Description
id (Primary Key) int Unique identifier for each EPA transaction (integer). Example: 1, 2, 3.
pa_reference_id varchar Unique reference ID for the prior authorization (string). Example: "PA12345", "PA67890".
txn_type varchar Type of the transaction (string). Example: "New", "Renewal", "Cancel".
txn_status varchar Status of the transaction (string). Example: "Pending", "Approved", "Denied".
grid_status varchar Status of the transaction in the grid view (string, NULLable). Example: "Active", "Inactive", NULL.
pn_note varchar Note related to the prior authorization (string, NULLable, up to 2000 characters). Example: "Patient needs this medication due to...", NULL.
case_id varchar Unique identifier for the case associated with the transaction (string, NULLable). Example: "CASE123", "CASE456", NULL.
authorization_number varchar Unique authorization number for the transaction (string, NULLable, up to 140 characters). Example: "AUTH12345", "AUTH67890", NULL.
effective_date datetime Date when the transaction becomes effective (date, NULLable). Example: "2022-01-01", "2022-02-01", NULL.
expiration_date datetime Date when the transaction expires (date, NULLable). Example: "2022-12-31", "2023-01-31", NULL.
appeal_available varchar Indicates if an appeal is available for the transaction (string, NULLable). Example: "Yes", "No", NULL.
appeal_expiration_date datetime Date when the appeal option expires (date, NULLable). Example: "2022-03-01", "2022-04-01", NULL.
cancel_reason_id smallint Unique identifier for the cancellation reason (small integer, NULLable). Example: 1, 2, 3, NULL.
cancel_note varchar Note related to the cancellation (string, NULLable, up to 200 characters). Example: "Patient no longer needs medication", NULL.
erx_request_id int Unique identifier for the e-prescription request associated with the transaction (integer, NULLable). Example: 1001, 1002, 1003, NULL.
payor_attachment varbinary Binary data for the payor attachment (varbinary, NULLable, up to 8000 bytes). Example: BLOB data, NULL.
created_by int Unique identifier for the user who created the transaction (integer). Foreign key relation to `users` table, `uid` column. Example: 101, 102, 103.
created_datetime datetime Date and time when the transaction was created (datetime, NULLable). Example: "2022-01-01 12:34:56", "2022-02-01 10:30:00", NULL.
updated_by int Unique identifier for the user who last updated the transaction (integer). Foreign key relation to `users` table, `uid` column. Example: 101, 102, 103.
updated_datetime datetime Date and time when the transaction was last updated (datetime, NULLable). Example: "2022-01-02 14:30:00", "2022-02-02 16:45:00", NULL.
scheduler_flag tinyint Flag to indicate if the transaction is scheduled (tiny integer, default '0'). Example: 0 (not scheduled), 1 (scheduled).
delete_flag tinyint Flag to indicate if the transaction is deleted (tiny integer, default '0'). Example: 0 (not deleted), 1 (deleted).
auth_update_datetime datetime Date and time when the authorization was last updated (datetime, NULLable). Example: "2022-01-03 09:00:00", "2022-02-03 11:15:00", NULL.

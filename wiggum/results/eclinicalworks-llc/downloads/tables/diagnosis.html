EHI Export Schema
Table diagnosis
Description This table stores the patient's diagnosis documented during the patient's encounters.
Column DataType Description
EncounterId int An integer representing the unique identifier of the medical encounter. Example: 12345
ItemId int An integer representing the unique identifier of the diagnosis item. Example: 6789
PrimaryAsmt int An integer representing whether the diagnosis is primary (1) or not (0). Example: 1
Specify varchar A varchar(1024) field containing additional information about the diagnosis. Example: "Type 2 Diabetes"
Notes text A text field containing any notes related to the diagnosis. Example: "Patient has a family history of diabetes."
displayIndex int An integer representing the display order of the diagnosis item. Example: 1
axis smallint A smallint(3) representing the axis of the diagnosis. Example: 0
AsmtOnsetDate varchar A varchar(10) field representing the date of the diagnosis onset. Example: "2021-09-01"
SlNo (Primary Key) int An integer representing the unique identifier (primary key) of the diagnosis record. Example: 1
websync tinyint A tinyint(3) indicating whether the record has been synchronized with the web (1) or not (0). Example: 0
Risk varchar An enum field representing the risk level of the diagnosis, with possible values of 'Low', 'Med', or 'High'. Example: 'High'
SNOMED varchar A varchar(50) field representing the SNOMED code for the diagnosis. Example: "44054006"
synonymid int An integer representing the unique identifier of the diagnosis synonym. Example: 123
SNOMEDDesc varchar A varchar(255) field containing the description of the SNOMED code. Example: "Diabetes mellitus type 2"
specifyHtml varchar A varchar(1024) field containing the HTML version of the `Specify` field. Example: "

Type 2 Diabetes

"
notesHtml varchar A text field containing the HTML version of the `Notes` field. Example: "

Patient has a family history of diabetes.

"

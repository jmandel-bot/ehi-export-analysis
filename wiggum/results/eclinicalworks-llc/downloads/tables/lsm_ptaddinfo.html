Table lsm_ptaddinfo
Description This table stores additional information about patients, such as their contact details, occupation, and how they found out about the service.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
Patientid int Unique identifier for the patient associated with the additional information. This could be a foreign key referring to the primary key of a `patients` table. Example: 101, 102, 103.
Alias tinyint A boolean flag indicating if the patient has an alias (1) or not (0). Example: 0, 1.
NickName varchar The patient's nickname, if any. Example: "Johnny", "Maddie".
Occupation varchar The patient's occupation. Example: "Software Engineer", "Nurse".
Programs varchar Programs the patient is enrolled in. Example: "Weight Loss", "Physical Therapy".
Subscription varchar The patient's subscription details. Example: "Monthly", "Annual".
ContactMethod varchar The preferred method of contact for the patient. Example: "Phone", "Email".
KnowsUsVia varchar How the patient found out about the service. Example: "Friend", "Online Advertisement".
Reorder varchar Information about the patient's reorder preferences. Example: "Auto-reorder", "Manual".
PrimaryLocation varchar The primary location of the patient. Example: "New York", "Los Angeles".
MobileNo varchar The patient's mobile phone number. Example: "+1-555-123-4567".
PagerNo varchar The patient's pager number, if any. Example: "555-987-6543".
OtherNo varchar Any other contact number for the patient. Example: "+1-555-321-7890".
Type varchar The type of patient, such as "New" or "Returning". Example: "New", "Returning".
Email varchar The patient's primary email address. Example: "john.doe@example.com".
AltEmail varchar The patient's alternative email address, if any. Example: "johndoe.alt@example.com".

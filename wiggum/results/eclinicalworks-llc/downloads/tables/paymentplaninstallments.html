Table paymentplaninstallments
Description This table stores information about the installments of various payment plans, including due dates, amounts, and payment status.
Column DataType Description
id (Primary Key) int Unique identifier for each installment record. Example: 1, 2, 3, etc.
planid int identifier for the payment plan that the installment belongs to. This can be a foreign key referring to the `id` column in the `paymentplans` table (assuming it exists). Example: 101, 102, 103, etc.
installmentno int The installment number within the payment plan. Example: 1, 2, 3, etc.
duedate datetime The date when the installment is due. Example: '2022-01-01', '2022-02-01', '2022-03-01', etc.
installmentamount decimal The amount to be paid for this installment. Example: 100.00, 200.00, 300.00, etc.
dueamount decimal The remaining amount to be paid for this installment. Example: 50.00, 150.00, 250.00, etc.
paid tinyint A flag indicating whether the installment has been paid or not. Example: 0 (not paid), 1 (paid).
paidamount decimal The amount that has been paid for this installment. Example: 0.00, 50.00, 100.00, etc.

Table patient_sexual_orientation
Description This table stores information about the sexual orientation of patients, including the reason for the orientation, the person who modified the information, and the status of the record.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
patientid int Unique identifier for the patient associated with the sexual orientation record. This is a foreign key that refers to the `id` column in the `patients` table. Example: 101, 102, 103.
modified_by int Unique identifier for the user who last modified the record. This is a foreign key that refers to the `id` column in the `users` table. Example: 1001, 1002, 1003.
so_id int Unique identifier for the sexual orientation type. This is a foreign key that refers to the `id` column in the `sexual_orientation_types` table. Example: 1 (Heterosexual), 2 (Homosexual), 3 (Bisexual).
so_reason varchar A brief description of the reason for the patient's sexual orientation. Example: "Personal preference", "Genetic factors", "Cultural influence".
isactive tinyint Indicates whether the record is active (1) or inactive (0). Example: 1, 0.
deleteflag tinyint Indicates whether the record has been deleted (1) or not (0). Example: 1, 0.
added_on datetime The date and time when the record was added to the table. Example: '2021-09-01 12:34:56'.
inactive_date datetime The date and time when the record was marked as inactive, if applicable. Example: '2021-10-01 12:34:56'.
action varchar A description of the action taken on the record, such as "Added", "Modified", or "Deleted". Example: "Added", "Updated", "Deleted".
modality varchar The method or system used to collect the sexual orientation information, such as "Self-report", "Interview", or "Medical record". Example: "Self-report", "Interview", "Medical record".

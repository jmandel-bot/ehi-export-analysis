EHI Export Schema
Table scribenotes
Description This table stores notes created by scribes during medical encounters.
Column DataType Description
id (Primary Key) int This is the primary key and a unique identifier for each note in the table. It is an integer that is automatically incremented. Example: 1, 2, 3, etc.
encounterid int This column stores the unique identifier of the medical encounter that the note is associated with. It is a foreign key that refers to the `id` column in the `encounters` table. Example: 101, 102, 103, etc.
scribenotes varchar This column stores the actual text of the scribe's notes. It is a text data type that can store a large amount of text. Example: "Patient complained of chest pain and shortness of breath."
websync tinyint This column indicates whether the note has been synchronized with a web-based system. It is a tiny integer with a default value of '0'. A value of '0' means not synchronized, and a value of '1' means synchronized. Example: 0, 1.
createdtime datetime This column stores the date and time when the note was created. It is a datetime data type. Example: '2021-09-15 14:30:00'.

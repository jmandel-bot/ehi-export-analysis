EHI Export Schema
Table dental_insurancebenefits
Description This table stores information about dental insurance benefits, including annual maximums, deductibles, and remaining amounts for both individuals and families.
Column DataType Description
id (Primary Key) int • This column will store the id of the dental_insurancebenefits. • The datatype of the column would be INT to accommodate many different insurances plan benefits data that can be added. • This would be unique and auto incremented and so PRIMARYKEY constraint is added
treatPlanNum int • This column stores the id of the saved dental treatment plan. • This is a reference to the column treatplannum of table dentaltreatmentplan.
annual_maximum decimal • This column stores annual maximum for the treatment plan. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field. • This is not used since V12 version, and it is used in older version of the product.
deductible decimal • This column stores the deductible for the treatment plan. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field. • This is not used since V12 version, and it is used in older version of the product.
TPDiscount decimal • This column stores discounts applied on treatment plans. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field
deleteFlag tinyint • This column determines whether the insurance benefits are deleted for the treatment plan. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value means the insurance benefits are not deleted. • •1 value means the insurance benefits are deleted.
individual_annualMax_remaining decimal • This column stores the individual annual maximum remaining for the treatment plan. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field
individual_deductible_remaining decimal • This column stores individual deductibles remaining for the treatment plan. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field
individual_annualMax decimal • This column stores individual annual maximum for the treatment plan. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field
individual_deductible decimal • This column stores individual deductibles for the treatment plan. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field
family_annualMax decimal • This column stores family annual maximum for the treatment plan • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field
family_deductible decimal • This column stores family deductibles for the treatment plan • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field
family_annualMax_remaining decimal • This column stores the family annual maximum remaining for the treatment plan. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field
family_deductible_remaining decimal • This column stores family deductibles remaining for the treatment plan. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field
benefits_enddate datetime • This column stores the benefits end date. This date is calculated based on anniversary month selected for the insurance plan.
age_limitation_check tinyint • This column stores the age limit checkbox captured on UI. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value means age limit is not checked for applied benefits on treatment plan. • 1 value means age limit is checked for applied benefits on treatment plan.
benefits_updateddate datetime • This column stores the server date and time stamp of the instant the operation being performed is saved to the database.
insid int • This column stores the id of the insurance. • This is a reference to the column insid of table insurance.
annualMax_remaining_used decimal • This column stores the annual maximum remaining used in the treatment plan. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field
deductible_remaining_used decimal • This column stores the deductible remaining used in the treatment plan. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field

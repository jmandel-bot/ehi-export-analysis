EHI Export Schema
Table procedurebillingmap
Description This table stores information about the billing of medical procedures performed during patient encounters, including details about the procedure, its complexity, and location.
Column DataType Description
encounterid int This column stores the encounterid. It refers the encounterid column present in enc table. It has size of int(11).
catid int This column stores the id of Laceration Repair category created for Procedures section when IsLacerationEnabled itemkey is true and migration script is executed. It refers the itemid of items table. It has size of int(11).
itemid int This stores the itemid of the organs created inside Laceration Repair category(Skin, Head etc). It refers the itemid column of items table. It has size of int(11)
woundno int This column stores the woundno created for a particular item. It refers the woundno column from structprocedurespl table. It has size of int(11)
lenVal float This column stores the wound size and hence, the data type is float. Default value is 0. If the wound length is not specified , this column will have a value as 0.
complexity varchar The data type of this column is varchar(20). It can have 3 values ie 'Simple' or 'Intermediate' or 'Complex'. By default, complexity will always be 'Simple'.
locationid int This column stores the id of the location. It refers the locationid column of laceration_location table. It has size of int(11).
groupno tinyint This column stores the group no. It refers the groupno column present in laceration_cptcode. Group no is identified by matching the complexity, locationid and length value in laceration_cptcode table.

Table casemanager
Description This table stores information about case managers, including their personal details, contact information, and preferences.
Column DataType Description
Id (Primary Key) int This column stores id of case manager. It is auto increment primary key. It defines identifier of case manager.
lname varchar Last name of the case manager (e.g., Smith, Johnson, Williams).
fname varchar First name of the case manager (e.g., John, Jane, Michael).
address1 varchar First line of the case manager's address (e.g., 123 Main St, 456 Oak St).
address2 varchar Second line of the case manager's address, if applicable (e.g., Apt 4B, Suite 200).
city varchar City where the case manager is located (e.g., New York, Los Angeles, Chicago).
State char Two-letter abbreviation of the state where the case manager is located (e.g., NY, CA, IL).
Zip varchar Zip code of the case manager's address (e.g., 10001, 90001, 60601).
Email varchar Email address of the case manager (e.g., john.smith@example.com, jane.johnson@example.com).
Phone varchar Phone number of the case manager (e.g., (555) 123-4567, (555) 987-6543).
Fax varchar Fax number of the case manager, if applicable (e.g., (555) 123-4568, (555) 987-6544).
ContactPreference varchar Preferred method of contact for the case manager (e.g., Email, Phone, Fax).
Notes varchar Additional notes or comments about the case manager (e.g., "Prefers morning calls", "On vacation until 10/15").
Dob datetime Date of birth of the case manager (e.g., 1980-01-01, 1990-12-31).
deleteflag tinyint A flag indicating if the case manager's record has been deleted (0 for not deleted, 1 for deleted).
EnterpriseId int Unique identifier for the enterprise the case manager belongs to. This can be a foreign key referring to the `Id` column in another table, such as an `enterprise` table.

EHI Export Schema
Table injorderencounters
Description This table stores information about encounters related to injections for patients, including details about the encounter, patient, and doctor involved.
Column DataType Description
id (Primary Key) int This column stores id which is primary key of type integer and length 4. This column is auto incremented and cannot be NULL.
injEncid int This column stores id of item given to patient on given encounter. This column is of type integer and length 4. Default value of this column is 0 and it cannot be NULL.
encid int This column stores encounter id of a given patient. This column is of type integer and length 4. Default value of this column is 0 and it cannot be NULL.
patientid int This column stores patient id. This column is of type integer and length 4. Default value of this column is 0 and it cannot be NULL.
injEncDate datetime This column stores encounter date. This column is of type datetime and length 8. Default value of this column is NULL.
deleteflag smallint This column is flag for delete entry. This column is of type smallint and length 2. Default value is 0 and cannot be NULL.
doctorid int This column stores provider id who has provided item(imm/inj) to patient. This column is of type integer and length 4. Default value of this column is 0 and it cannot be NULL.

EHI Export Schema
Table tblportalusers
Description This table stores information about portal users, including their personal details, contact information, and account settings.
Column DataType Description
UID int Unique identifier for the user,the column stores an UID of user table
uname varchar This column stores user defined username which can be used with password to log into the application.
upwd varchar This column stores user's password in encrypted format.
ufname varchar This column stores user's first name.
uminitial varchar This column stores user's middle initial.
ulname varchar This column stores user's last name.
uemail text This column stores user's email address.
uwebPage text This column stores user's web page address.
upaddress varchar Primary address of the user (string). Example: "123 Main St"
upcity varchar City of the primary address (string). Example: "New York"
upstate varchar State of the primary address (string). Example: "NY"
upPhone varchar Phone number of the user (string). Example: "555-123-4567"
zipcode varchar Zip code of the primary address (string). Example: "10001"
dob varchar Date of birth of the user (string). Example: "01/01/1990"
ssn varchar Social Security Number of the user (string). Example: "123-45-6789"
sex varchar Gender of the user (string). Example: "M"
upaddress2 varchar Secondary address of the user (string). Example: "Apt 4B"
suffix varchar Suffix of the user's name (string). Example: "Jr"
prefix varchar Prefix of the user's name (string). Example: "Mr"
initials varchar Initials of the user (string). Example: "JAD"
delFlag int This column stores values 0/1 to indicate whether the patient has been marked as deleted or not. 0 - If patient is NOT marked as deleted, 1 - If patient is marked deleted.
pmcid int Unique identifier for the user's portal management company (integer). No foreign key relation.
locked int This column stores flags 0/1 to indicate whether user has been locked out of the application account or not. 0 - User is NOT locked out, 1 - User is locked out of the account.
FormsSigned tinyint This column stores flags 0/1 to indicate whether user has signed the consent form or not. 0 - User has NOT signed the consent form, 1 - User has signed the consent form.
resetPassword tinyint This column stores flag 0/1 to indicate whether users password was reset or not. 0 - Password was NOT reset, 1 - Password was reset.
vmid varchar This is auto generated unique id for the record in This column from EMR.
invalid tinyint This column stores flags 0/1 to indicate whether the user is marked as active or inactive. 0 - Active, 1 - Inactive.
convDone tinyint Indicates if the user has completed the required conversion process (tinyint). Example: 0 (not done), 1 (done)
facilityid int Unique identifier for the user's associated facility (integer). No foreign key relation.

EHI Export Schema
Table information_blocking_reasons
Description This table stores exception reason for information blocking
Column DataType Description
id (Primary Key) int Primary key of information_blocking_reasons table. This column stores numeric data which could grow hence the integer range 0-2,147,483,647 is selected.
exceptionReasonid int This column is used to store Exception Reason id. This column does not expect NULL values as it is mandatory field in the user interface. This column required to stores unique id. Hence unique constraint is selected. This column stores numeric data which could grow hence integer range: 0-2,147,483,647 is selected.
exceptionReason varchar This column is used to store reason of why encounter is restricted/block for Patient Portal. As per functionality, user can insert a reason up to 255 characters.
inactive tinyint This column is used to make Exception Reason active/inactive. Possible values of this column are 0 or 1 hence tinyint is selected. The default value of this column will be 0 (Active). This column will store 0 or 1 hence tinyint is selected.

EHI Export Schema
Table enc_vital
Description This table stores vital stage information related to specific encounters.
Column DataType Description
id (Primary Key) int This is the primary key column, which stores a unique identifier for each record in the table. It is an integer with a maximum length of 11 digits and is automatically incremented as new records are added. Example values: 1, 2, 3, etc.
encounterid int This column stores the identifier of the encounter that the vital stage is associated with. It is a foreign key column that refers to the `encounterid` column in the `enc` table. It is an integer with a maximum length of 11 digits. Example values: 101, 102, 103, etc.
vitalstage varchar This column stores the stage of BP based on the BP and HT of the patient recorded per encounter. Example: ELEVATED, STAGE1, and STAGE2.

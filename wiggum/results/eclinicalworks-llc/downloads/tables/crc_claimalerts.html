Table crc_claimalerts
Description This table stores information related to claim alerts, including the type of alert, payer ID, and other relevant details.
Column DataType Description
id (Primary Key) int Unique identifier for each claim alert record. Example: 1, 2, 3.
invoiceid int identifier for the invoice associated with the claim alert. This could be a foreign key referring to the `id` column in an `invoices` table. Example: 101, 102, 103.
alerttype int identifier for the type of claim alert. This could be a foreign key referring to the `id` column in an `alert_types` table. Example: 1 (Missing Information), 2 (Incorrect Information), 3 (Rejected Claim).
payerid varchar Unique identifier for the payer associated with the claim alert. Example: 'PAYER123', 'PAYER456', 'PAYER789'.
actiondays int Number of days to take action on the claim alert. Example: 5, 10, 15.
fileid int identifier for the file associated with the claim alert. This could be a foreign key referring to the `id` column in a `files` table. Example: 1001, 1002, 1003.
alertresult text Text description of the result of the claim alert. Example: 'Resolved', 'Pending', 'Rejected'.
issubmitted tinyint Flag indicating whether the claim alert has been submitted (1) or not (0). Example: 0 (Not Submitted), 1 (Submitted).
alerttext varchar Brief text description of the claim alert. Example: 'Missing patient information', 'Incorrect billing code', 'Rejected due to duplicate claim'.
ignoreComment varchar Comment explaining why the claim alert is being ignored, if applicable. Example: 'Resolved internally', 'Duplicate alert', ''.

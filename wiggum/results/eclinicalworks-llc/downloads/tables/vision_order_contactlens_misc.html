Table vision_order_contactlens_misc
Description This table stores patients' contact lens order information.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table.
orderid int identifier for the order, referring to the primary key in the orders table.
productid int identifier for the product, referring to the primary key in the products table.
barcode varchar Barcode of the product, a string of up to 255 characters (e.g., "123456789012").
STATUS int Status of the order, represented as an integer (e.g., 0 for pending, 1 for processed).
quantity int Quantity of the product ordered, represented as an integer (e.g., 2).
side tinyint Side of the contact lens, represented as a tiny integer (e.g., 1 for left, 2 for right).
visionrxid int identifier for the vision prescription, referring to the primary key in the vision prescriptions table.
pair varchar Pair of contact lenses, represented as a two character string (e.g., "L1").
drno varchar This column indicates the reference for contact lens order which is a combination of encounterid and pair.
price decimal Price of the product that is represented as a decimal with 12 digits and four decimal places (e.g., 99.9900).
isdeleted tinyint Flag to indicate if the record is deleted, represented as a tiny integer (e.g., 0 for not deleted, 1 for deleted).
bc varchar Base curve of the contact lens, a string of up to 11 characters (e.g., "8.6").
dia varchar Diameter of the contact lens, a string of up to 11 characters (e.g., "14.0").
color varchar Color of the contact lens, a string of up to 30 characters (e.g., "Blue").
isOtherLens bit Flag to indicate if the lens is not a standard lens, represented as a bit (e.g., 0 for standard, 1 for non-standard).
productName varchar Name of the product, a string of up to 255 characters (e.g., "Acuvue Oasys").
category varchar Category of the product, a string of up to 30 characters (e.g., "Daily Disposable").
material varchar Material of the contact lens, a string of up to 30 characters (e.g., "Silicone Hydrogel").
oad varchar Overall diameter of the contact lens, a string of up to 11 characters (e.g., "14.5").
ozd varchar Optical zone diameter of the contact lens, a string of up to 11 characters (e.g., "9.0").
pc1 varchar First parameter code of the contact lens, a string of up to 15 characters (e.g., "PC123").
pc2 varchar Second parameter code of the contact lens, a string of up to 15 characters (e.g., "PC456").
pc3 varchar Third parameter code of the contact lens, a string of up to 15 characters (e.g., "PC789").
pc4 varchar Fourth parameter code of the contact lens, a string of up to 15 characters (e.g., "PC012").
ct varchar Center thickness of the contact lens, a string of up to 11 characters (e.g., "0.08").
sc varchar Sagittal depth of the contact lens, a string of up to 11 characters (e.g., "2.5").
tc varchar Total diameter of the contact lens, a string of up to 11 characters (e.g., "14.2").
SPH varchar Spherical power of the contact lens, a string of up to 11 characters (e.g., "-2.00").
CYL varchar Cylindrical power of the contact lens, a string of up to 11 characters (e.g., "-1.25").
AXIS varchar Axis of the contact lens, a string of up to 11 characters (e.g., "180").
NADD varchar Near addition power of the contact lens, a string of up to 11 characters (e.g., "+1.50").
DVA varchar Distance visual acuity, a string of up to 20 characters (e.g., "20/20").
NVA varchar Near visual acuity, a string of up to 20 characters (e.g., "20/30").
rxtype tinyint Prescription type, represented as a tiny integer (e.g., 0 for single vision, 1 for multifocal).
orderedby int identifier for the user who placed the order, referring to the primary key in the user table.
orderedon datetime Date and time when the order was placed, represented as a datetime value (e.g., "2021-01-01 12:34:56").
receivedby int identifier for the user who received the order, referring to the primary key in the user table.
receiveddate datetime Date and time when the order was received, represented as a datetime value (e.g., "2021-01-02 12:34:56").
inspectedby int identifier for the user who inspected the order, referring to the primary key in the user table.
inspectedon datetime Date and time when the order was inspected, represented as a datetime value (e.g., "2021-01-03 12:34:56").
pickup_updatedby int Identifier for the user who updated the pickup information, referring to the primary key in the user table.
collectedon datetime Date and time when the order was collected, represented as a datetime value (e.g., "2021-01-04 12:34:56").
received_qty tinyint Quantity of the product received, represented as a tiny unsigned integer (e.g., 2).
trackingNumber varchar Tracking number for the order shipment, a string of up to 30 characters (e.g., "1Z12345E").
est_deliverydate datetime Estimated delivery date of the order, represented as a date value (e.g., "2021-01-05").
passed_qty tinyint Quantity of the product that passed inspection, represented as a tiny unsigned integer (e.g., 2).
rejected_qty tinyint Quantity of the product that was rejected during inspection, represented as a tiny unsigned integer (e.g., 0).
cust_date_contacted datetime Date when the customer was contacted, represented as a date value (e.g., "2021-01-06").
est_collection_date datetime Estimated date for the customer to collect the order, represented as a date value (e.g., "2021-01-07").
iscustomerpickup tinyint Flag to indicate if the order is for customer pickup, represented as a tiny integer (e.g., 1 for pickup, 0 for delivery).
collector_name varchar Name of the person who collected the order, a string of up to 50 characters (e.g., "John Doe").
collected_qty tinyint Quantity of the product collected by the customer, represented as a tiny unsigned integer (e.g., 2).
uncollected_qty tinyint Quantity of the product not collected by the customer, represented as a tiny unsigned integer (e.g., 0).
collectionnotes varchar Notes about the order collection, a string of up to 3000 characters (e.g., "Customer will pick up on Friday.").
ordernotes varchar Notes about the order, a string of up to 3000 characters (e.g., "Customer prefers blue lenses.").
cancellationnotes varchar Notes about the order cancellation, a string of up to 3000 characters (e.g., "Customer changed their mind.").
reorder_qty tinyint Quantity of the product to be reordered, represented as a tiny unsigned integer (e.g., 1).
pickup_updatedon datetime Date and time when the pickup information was updated, represented as a datetime value (e.g., "2021-01-08 12:34:56").
createdby int Identifier for the user who created the record, referring to the primary key in the user table.
createdon datetime Date and time when the record was created, represented as a datetime value (e.g., "2021-01-01 10:00:00").
updatedby int identifier for the user who updated the record, referring to the primary key in the user table.
updatedon datetime Date and time when the record was updated, represented as a datetime value (e.g., "2021-01-01 11:00:00").
cancel_reasonid int identifier for the cancellation reason, referring to the primary key in the cancellation reasons table.
processor int identifier for the processor, referring to the primary key in the processors table.
contactlensid int identifier for the contact lens, referring to the primary key in the contact lenses table.
colorid int identifier for the color, referring to the primary key in the colors table.
facilityid int identifier for the facility, referring to the primary key in the facilities table.
location int identifier for the location, referring to the primary key in the locations table.

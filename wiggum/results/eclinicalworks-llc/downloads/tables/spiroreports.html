EHI Export Schema
Table spiroreports
Description This table stores information about spirometry reports for patients, including file details, report dates, and technician information.
Column DataType Description
reportid (Primary Key) int Unique identifier for each spirometry report. Example: 1, 2, 3.
patientid int identifier for the patient associated with the report. It has a foreign key relation with the user table, referring to the UID column. Example: 101, 102, 103.
encid int identifier for the encounter associated with the report. It has a foreign key relation with the enc table, referring to the encounterid column. Example: 1001, 1002, 1003.
labid int identifier for the lab where the report was generated. Example: 10, 11, 12.
labType int Type of lab where the report was generated. Example: 1 (hospital lab), 2 (private lab), 3 (research lab).
filePath varchar File path for the report file. Example: /reports/spiro/12345.pdf.
fileName1 varchar First file name associated with the report. Example: spiro_report_12345.pdf.
fileName2 varchar Second file name associated with the report. Example: spiro_report_12345_summary.pdf.
reportDate datetime Date when the report was generated. Example: 2021-08-01.
reviewDate datetime Date when the report was reviewed. Example: 2021-08-02.
reviewedBy varchar Name of the person who reviewed the report. Example: Dr. John Smith.
technician varchar Name of the technician who performed the spirometry test. Example: Jane Doe.
race int Race identifier for the patient. Example: 1 (Caucasian), 2 (African American), 3 (Asian).
delFlag int Deletion flag for the report. Example: 0 (not deleted), 1 (deleted).
FaxReportFileName1 varchar File name of the faxed report. Example: faxed_spiro_report_12345.pdf.
comments text Additional comments about the report. Example: Patient had difficulty performing the test.
ftpdirpath varchar FTP directory path for the report file. Example: /ftp/reports/spiro/12345.pdf.
pdfuploadsuccess int Indicator for successful PDF upload. Example: 0 (not successful), 1 (successful).
sdkversion varchar Version of the SDK used for generating the report. Example: 1.0.0, 1.1.0.

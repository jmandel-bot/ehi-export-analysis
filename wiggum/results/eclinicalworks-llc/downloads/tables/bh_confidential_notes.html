Table bh_confidential_notes
Description This table stores confidential notes related to patients, created by specific users.
Column DataType Description
id (Primary Key) int This is the primary key, an auto-incrementing integer that uniquely identifies each confidential note in the table. Example: 1, 2, 3, etc.
patientid int This is a foreign key relation (references `users` table's `UID` column) and an integer that represents the unique identifier of the patient associated with the confidential note. Example: 101, 102, 103, etc.
userid int This is a foreign key relation (references `users` table's `UID` column) and an integer that represents the unique identifier of the user who created the confidential note. Example: 201, 202, 203, etc.
name varchar This is a varchar(50) column that stores the name or title of the confidential note. Example: "Medical History", "Allergies", "Surgery Details", etc.
note varchar This is a text column that stores the content of the confidential note. Example: "Patient has a history of asthma and needs to avoid certain medications."
createdDateTime datetime This is a datetime column that stores the date and time when the confidential note was created. Example: "2021-09-01 12:34:56"
modifiedDatetime datetime This is a datetime column that stores the date and time when the confidential note was last modified. Example: "2021-09-02 14:30:00"
delflag tinyint This is a tinyint(1) column that stores a boolean value (0 or 1) to indicate whether the confidential note has been deleted (soft delete). A value of 0 means the note is not deleted, and a value of 1 means the note is deleted. Example: 0, 1

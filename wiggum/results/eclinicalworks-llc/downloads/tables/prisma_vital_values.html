EHI Export Schema
Table prisma_vital_values
Description This table stores vital sign values and related information for patients.
Column DataType Description
id (Primary Key) bigint Unique identifier for each record in the table (bigint).
vitalBatchId bigint Foreign key referencing the `prisma_vital_batch` table's `id` column (bigint). It represents the batch of vital signs this value belongs to.
vitalId varchar Identifier for the specific vital sign (varchar, 100). Example: "HR" for heart rate.
vitalSignCode varchar Code representing the vital sign (varchar, 255). Example: "8867-4" for heart rate.
vitalSign varchar Name of the vital sign (varchar, 100). Example: "Heart Rate".
vitalvalue varchar Value of the vital sign (varchar, 600). Example: "72".
vitalUnit varchar Unit of measurement for the vital sign (varchar, 25). Example: "bpm" for beats per minute.
valuetext varchar Text description of the vital sign value (varchar, 1000). Example: "Normal heart rate".
measurementTime varchar Time when the vital sign was measured (varchar, 50). Example: "2021-09-01 10:30:00".
interpretationCode varchar Code representing the interpretation of the vital sign value (varchar, 15). Example: "N" for normal.
interpretation varchar Text description of the interpretation of the vital sign value (varchar, 50). Example: "Normal".
authorId varchar Identifier for the author who recorded the vital sign (varchar, 50).
authorName varchar Name of the author who recorded the vital sign (varchar, 100). Example: "Dr. John Doe".
authorAddressLine1 varchar First line of the author's address (varchar, 100). Example: "123 Main St".
authorAddressLine2 varchar Second line of the author's address (varchar, 100). Example: "Apt 4B".
authorCity varchar City of the author's address (varchar, 25). Example: "New York".
authorState varchar State of the author's address (varchar, 10). Example: "NY".
authorPostalCode varchar Postal code of the author's address (varchar, 15). Example: "10001".
authorCountry varchar Country of the author's address (varchar, 10). Example: "US".
isException tinyint Flag indicating if the row has incomplete data to be eligible for showing to the user
exceptionMessage varchar list of ids of the exception messages indicationg missing data points for the row to be eligible for showing to the user
sourcePriority int Source priority to be considered while deduplicating entries
codeSystemPriority int Source code priority for deduplication for the vital values. Example: 1, 2, 3.
duplicatedBy int Identifier for the record that duplicates this one, if any (int). Example: -1 for no duplication.
status varchar Status of the vital sign value (varchar, 50). Example: "active".
noteTxt varchar Additional notes related to the vital sign value (varchar, 8000). Example: "Patient was calm during measurement".
issuedDateTime datetime Date and time when the vital sign value was issued (datetime). Example: "2021-09-01 10:35:00".
endDateTime datetime End Date and time for the vital observation (datetime). Example: "2021-09-01 11:00:00".
encounterId bigint Foreign key referencing the `prisma_encounter_details` table's `id` column (bigint). It represents the encounter associated with the vital sign value.
practitionerId int Foreign key referencing the `prisma_practitioner_details` table's `ID` column (int). It represents the practitioner who recorded the vital sign value.
standardVitalSign varchar Standard name of the vital sign (varchar, 100). Example: "Heart Rate".

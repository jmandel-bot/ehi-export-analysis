Table prisma_procedure_report
Description This table stores information about procedure reports, including their diagnostic reports and related documents.
Column DataType Description
ID (Primary Key) int This is the primary key column, which uniquely identifies each record in the table. It is an integer value that is automatically incremented for each new record. Example: 1, 2, 3, etc.
PROCEDURE_DETAIL_ID bigint This column stores the ID of the procedure detail associated with the report. It is a foreign key that refers to the `id` column in the `prisma_procedure_details` table. Example: 101, 102, 103, etc.
DIAGNOSTIC_REPORT_ID varchar This column stores the ID of the diagnostic report related to the procedure. It is a varchar(100) field that can be NULL if there is no diagnostic report associated with the procedure. Example: 'DR123', 'DR124', 'DR125', etc.
DOCUMENT_ID varchar This column stores the ID of the document related to the procedure report. It is a varchar(100) field that can be NULL if there is no document associated with the procedure report. Example: 'DOC567', 'DOC568', 'DOC569', etc.
RESOURCE_DISPLAY varchar This column stores a display string for the resource, which can be used for human-readable purposes. It is a varchar(255) field that can be NULL if there is no display value associated with the resource. Example: 'Procedure Report 1', 'Procedure Report 2', 'Procedure Report 3', etc.

Table pathologyreportdetails
Description This table stores the details of pathology reports generated for patients, including laboratory information, rule-out conditions, comments, status, and results.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
laboratory varchar Name of the laboratory where the pathology test was conducted. Example: LabCorp, Quest Diagnostics.
ruleout text Text containing the conditions that can be ruled out based on the pathology report. Example: Infection, Inflammation.
comments text Additional comments or notes related to the pathology report. Example: Sample quality was low, Re-test recommended.
reportid int This column stores lab report id of labdata table.
STATUS varchar The current status of the pathology report, either Open or Addressed. Example: Open, Addressed.
result varchar The result of the pathology test, which can be empty, Benign, or Malignant. Example: Benign, Malignant.
txDate datetime The date when the pathology test was conducted. Example: 2021-08-01, 2021-08-02.
itemname varchar The name of the item or sample tested in the pathology report. Example: Biopsy, Blood Sample.
itemid int Unique identifier for the item or sample tested. Example: 1, 2, 3.
patientid int This column stores identifier number of a patient. It refers to column UID from user table.
providerid int This column stores provider id, refers to doctorid from doctors table.
deleteflag tinyint A flag indicating whether the record has been deleted (1) or not (0). Example: 0, 1.

EHI Export Schema
Table vision_order
Description This table stores patients' vision orders. It is a parent table for the Dispense module.
Column DataType Description
id (Primary Key) int This column is an auto incremented primary key.
status int This column indicates the status of the order placed. This column refers to column id from vision_master table.
assignedto int Foreign key referring to the user who is assigned to the vision order. Refers to the `id` column in the `users` table.
dispensequeue_id int Foreign key referring to the dispense queue the vision order belongs to. Refers to the `id` column in the `dispensequeue` table.
customer_orderid int This column is not in use.
display_order tinyint This column is not in use.
order_comm_method varchar This column is not in use.
amendment_reason int This column is not in use.
collection_notes text This column stores additional notes (free text) added by the dispenser for this order.
isdelayed tinyint This column is not in use.
shape_file varchar This column is not in use.
estimated_days decimal This column is not in use.
isitem_validate tinyint This column is not in use.
isframe_follow tinyint Tiny integer indicating if the frame in the vision order is followed (1) or not (0).
lab_notes text This column is not in use.
lab_partyid int This column is not in use.
isdeleted tinyint Tiny integer indicating if the vision order is deleted (1) or not (0).
createdby int Foreign key referring to the user who created the vision order. Refers to the `id` column in the `users` table.
createdon datetime Date and time when the vision order was created.
updatedby int Foreign key referring to the user who last updated the vision order. Refers to the `id` column in the `users` table.
updatedon datetime Date and time when the vision order was last updated.
promotionamount decimal Decimal value representing the promotion amount applied to the vision order.
taxamount decimal Decimal value representing the tax amount applied to the vision order.
unitprice decimal Decimal value representing the unit price of the vision order.
totalprice decimal Decimal value representing the total price of the vision order.
facility_id int Foreign key referring to the facility where the vision order is processed. Refers to the `id` column in the `facility` table.
patient_id int Foreign key referring to the patient associated with the vision order. Refers to the `id` column in the `patient` table.
gUID varchar Unique identifier (GUID) for the vision order.
promotionamount_order decimal This column is not in use.
sighttest_id int This column is not in use.
sighttest_name varchar This column is not in use.
sighttest_price decimal This column is not in use.
sighttest_cptid int This column is not in use.
sighttest_taxamount decimal This column is not in use.
rounding decimal This column is not in use.
paid decimal Decimal value representing the amount paid for the vision order.
balance decimal Decimal value representing the outstanding balance for the vision order.
employerpart decimal This column is not in use.
previousbalance decimal Decimal value representing the previous outstanding balance for the vision order.
encounterid int Foreign key referring to the encounter associated with the vision order. Refers to the `id` column in the `encounter` table.
collectiondate datetime This column is not in use.
collectionstore int This column is not in use.
collectionprovider int This column is not in use.
writeoff decimal This column is not in use.
writeoff_reason_id int This column is not in use.
collectionenc int This column is not in use.
unitprice_treatment decimal This column is not in use.
promotionamount_treatment decimal This column is not in use.
taxamount_treatment decimal This column is not in use.
cashchange decimal This column is not in use.
closeorderdate datetime Date and time when the vision order was closed.
last_pair_orderid int This column stores a reference order id for this order. This column refers to column id of this column.
sendlaterhelddate datetime This column is not in use.
fixedbalance decimal This column is not in use.
acknowledgment tinyint This column stores a values 0, 1, or 2. A "0" means default, a "1" means in progress payment, and a "2" means completed payment.
dispenseFlag int This column stores values 0, 1, or 2. A “0” means default value, a “1” means the order of the status was updated from dispense for accessories, and a “2” means for repair from dispense.

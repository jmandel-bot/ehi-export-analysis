EHI Export Schema
Table bulk_batch_details
Description This table stores information about the details of Fhir R4 bulk request batches, including their status, creation and modification timestamps, and any filters applied.
Column DataType Description
id (Primary Key) int Unique identifier for each bulk batch detail record. Example: 1, 2, 3.
bulk_batch_master_id int Identifier for the associated bulk batch master record. Example: 101, 102, 103.
patient_id int Identifier for the associated patient (user) record. This column can have NULL values. Example: 1001, 1002, 1003. Foreign key relation: References `users` table on `uid` column.
fhir_resources_id tinyint Identifier for the associated FHIR (Fast Healthcare Interoperability Resources) resource. Example: 1, 2, 3.
batch_detail_status tinyint Status of the bulk batch detail, where 0 is the default value. Example: 0, 1, 2.
created_at datetime time stamp for when the bulk batch detail record was created. Example: '2021-09-01 12:34:56'.
modified_at datetime time stamp for when the bulk batch detail record was last modified. Example: '2021-09-02 14:56:32'.
apply_since_paramter tinyint Indicates whether the "since" parameter is applied, with 0 being the default value. Example: 0, 1.
type_filter varchar Filter applied to the bulk batch detail based on the type of resource. This column can have NULL values.

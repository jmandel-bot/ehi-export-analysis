EHI Export Schema
Table ip_rh_drug_admin_schedule
Description This table stores the administration schedules of each order
Column DataType Description
Id (Primary Key) int This column stores the unique identifier for each interaction record. Integer data type ensures efficient indexing and searching for records. Primary key constraint ensures uniqueness and avoids duplication.
OrderId int This column is used to store OrderId.It is a foreign key that references OrderId column ip_rh_ptmedicationorders table.
ScheduleDateTime datetime This column is used to store the scheduled date and time for the drug administration in yyyy-mm-dd hh:mm:ss format. DATETIME data type is used to store date and time data.
AdministeredDatetime datetime This column is used to store the actual date and time when the drug was administered in yyyy-mm-dd hh:mm:ss format. DATETIME data type is used to store date and time data.
AdministrationStatus varchar This column is used to store the status of the administration.Possible values: 1, 2, 3. 1 - Given 2 - Not Given 3 - Partial Given No default value is set since when drug has not been administered, the administration value stays null.
UserId int This column is used to store the ID of the user who administered the drug.It is a foreign key that references Id column users table.
AdminNote varchar This column is used to store the actual note text entered while eMAR administration.
AdministrationDose decimal This column is used to store the dose of the drug administered. Decimal data type with precision 19 and scale 4 is used to store large values with high precision.
AdministrationDoseUOM varchar This column is used to store the unit of measure for the administered dose.
AdminBagNo int This column is used to store the bag number for the drug administration.
PatientIdScanned varchar This column is used to store the scanned patient ID.
PatientIdNotScannedReason int This column is used to store the reason why the patient ID is not scanned.It is a foreign key that references Id column ip_rh_drugFormulary_reasondictionaryitems table.
DrugNotScannedReason int This column is used to store the reason why the drug is not scanned.It is a foreign key that references Id column ip_rh_drugFormulary_reasondictionaryitems table.
ObservedBy varchar This column is used to store the ID of the user who observed the drug administration.
ObservedById int This column is used to store the ID of the user who observed the drug administration.The column has a foreign key constraint that refers to the uid from the users table.
CreatedBy int This column is used to store id of user who created the record. The column has a foreign key constraint (FK_IP_rh_ drug_admin_schedule_createdby) referencing the uid from the users table.
CreatedDate datetime This column is used to store created date and time when the note was created in the format yyyy-mm-dd hh:mm:ss.
ModifiedBy int This column is used to store id of user who modified formulary common settings.The column has a foreign key constraint (FK_IP_rh_ drug_admin_schedule _modifiedBy) referencing the uid from the users table.
ModifiedDate datetime This column is used to store modified date and time in the format yyyy-mm-dd hh:mm:ss
NotGivenReason int This column is used to store the reason if the drug was not given. It is a foreign key referencing the ip_rh_drugFormulary_reasondictionaryitems table.
EpisodeEncounterId int This column is used to store the ID of the episode encounter associated with the drug administration. It is a foreign key referencing the encounterID of enc table.
verifiedBy int This column is used to store the ID of the user who verified the record. The column has a foreign key constraint that refers to the uid from the users table.
verificationFlag tinyint This column stores a value indicating if the record has been verified.Possible values: 1 - verified 0 - not verified The default value is 0 because by default record is marked as not verified.
verificationDate datetime This column stores the date and time when the record was verified.
LateAdministrationReason int This column is used to store the reason id of the reason if the drug administration was late. It is a foreign key referencing the id column of ip_rh_drugFormulary_reasondictionaryitems table.
PartialAdministrationReason int This column is used to store the reason if the drug administration was partial. It is a foreign key referencing the id column of ip_rh_drugFormulary_reasondictionaryitems table.
immunizationId int This column is used to store the ID of the immunization if the drug is an immunization drug.It is a foreign key referencing the immunizations table.
IVAction tinyint This column stores a value indicating the action taken on the IV.Possible values: "1 - START 2 - PAUSE 3 - RESUME 4 - STOP" The default value is 0.
Volume decimal This column is used to store the volume of the drug administered.
VolumeUnit varchar This column is used to store the unit of the volume of the drug administered.
Rate decimal This column is used to store the rate of the drug administration.
RateUnit varchar This column is used to store the unit of the rate of the drug administration.
Site int This column is used to store the ID of the site where the drug was administered. It's a foreign key referencing the id column of ip_rh_drugFormulary_SiteDictionary table.
EndDateTime datetime This column stores the date and time when the drug administration ended in the format yyyy-mm-dd hh:mm:ss.
EarlyAdministrationReason int This column is used to store the reason if the drug administration was early. It is a foreign key referencing the id column of ip_rh_drugFormulary_reasondictionaryitems table.
MasterEncounterID int This column is used to store the ID of the master encounter associated with the drug administration. It is a foreign key referencing the encounterID column of enc table.
LateDocumentationReason int This column is used to store the reason if the documentation of the drug administration was done late. It is a foreign key referencing the id column of ip_rh_drugFormulary_reasondictionaryitems table.
isScan tinyint This column stores a value indicating if the record is scanned. Possible values: 0 - Not scanned 1 - scanned The default value is 0.
WastageDose decimal This column is used to store the wastage dose of the drug, if any.
WastageDoseUnit varchar This column is used to store the units of wastage dose like ml, mg, etc.
WastageDispense decimal This column is used to store the wastage dispense of the drug, if any.
WastageDispenseUnit varchar This column is used to store the unit of wastage dispense like ml, mg, etc.
AdminStartSource varchar This column is used to store the source of the administration start.
docModuleEncId int This column is used to store the ID of the document module encounter associated with the drug administration. It is a foreign key referencing the encounterID of enc table.

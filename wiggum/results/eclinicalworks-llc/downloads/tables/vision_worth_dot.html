EHI Export Schema
Table vision_worth_dot
Description This table stores information about various options related to the vision worth dot system.
Column DataType Description
id (Primary Key) int This is the primary key for the table and is an auto-incrementing integer. It uniquely identifies each record in the table. Example: 1, 2, 3, etc.
encounterid int This is a foreign key that refers to the `encounterid` column in the `enc` table. It represents the unique identifier for the encounter associated with the vision worth dot record. Example: 101, 102, 103, etc.
rx_type tinyint This is a tiny integer column that stores the type of prescription. It can have values like 0 or 1, where 0 might represent a single vision prescription and 1 might represent a multifocal prescription.
distance varchar This is a varchar column with a maximum length of 50 characters. It stores the distance prescription for the vision worth dot record. Example: "OD: -1.00, OS: -1.25".
near varchar This is a varchar column with a maximum length of 50 characters. It stores the near prescription for the vision worth dot record. Example: "OD: +1.00, OS: +1.25".
comments varchar This is a varchar column with a maximum length of 255 characters. It stores any additional comments related to the vision worth dot record. Example: "Patient needs new glasses".

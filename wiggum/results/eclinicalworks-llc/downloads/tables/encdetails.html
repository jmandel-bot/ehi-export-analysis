Table encdetails
Description This table stores the details of patient encounters with doctors, including the doctor's ID, a code for the type of encounter, and a description of the encounter.
Column DataType Description
id (Primary Key) int This is the primary key for the table and is an auto-incrementing integer. It uniquely identifies each record in the table. Example: 1, 2, 3, etc.
PatientID int This column contains patient id which is linked with patients table pid. It represents id of patient involved with encounter
EncounterID int This is a foreign key that refers to the `id` column in the `encounters` table. It represents the ID of the specific encounter between the patient and the doctor. Example: 1001, 1002, 1003, etc.
DoctorId int This is a foreign key that refers to the `id` column in the `doctors` table. It represents the ID of the doctor involved in the encounter. Example: 201, 202, 203, etc.
Code char This is a code that represents the visit type of encounter.
Description varchar This is a varchar column with a maximum length of 110 characters. It stores a description of the encounter, with a default value of 'Medical Care'. Example: 'Routine Checkup', 'Emergency Visit', 'Surgery Consultation', etc.
deleteFlag tinyint This is a tinyint column that indicates whether the record has been deleted or not. A value of '0' means the record is not deleted, and a value of '1' means the record is deleted. Example: 0, 1.

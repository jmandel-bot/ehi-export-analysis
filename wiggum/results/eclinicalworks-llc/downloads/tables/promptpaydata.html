EHI Export Schema
Table promptpaydata
Description This table stores information related to patient payments, services, and billing codes.
Column DataType Description
id (Primary Key) int The unique identifier for each record in the table (e.g., 1, 2, 3, etc.).
Encid int The unique identifier for each encounter between a patient and healthcare provider. Examples include 1001, 1002, 1003, etc..
Patientid int The unique identifier for each patient. This can be a foreign key referring to the `id` column in the `patients` table. Examples include 101, 102, 103, etc..
icds varchar International Classification of Diseases (ICD) codes associated with the patient's diagnosis. It is a comma-separated list of codes (e.g., J45.9, M54.5, E11.9).
ServiceDate varchar The date when the service was provided to the patient. It is stored as a string in the format YYYY-MM-DD (e.g., 2021-08-15).
cptTIERs varchar Current Procedural Terminology (CPT) codes associated with the services provided to the patient. It is a comma-separated list of codes. Example include 99213, 99214, 99215, etc..
cptTIER8 tinyint A flag indicating if the service is associated with CPT Tier 8. It is stored as a small integer, where 1 represents 'true' and 0 represents 'false'.
Caseid int The unique identifier for each case associated with the patient. This can be a foreign key referring to the `id` column in the `cases` table. Examples include 201, 202, 203, etc..

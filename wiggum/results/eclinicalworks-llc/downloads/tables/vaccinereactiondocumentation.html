EHI Export Schema
Table vaccinereactiondocumentation
Description This table stores information about vaccine reactions documented for each immunization event.
Column DataType Description
id (Primary Key) int Unique identifier for each vaccine reaction documentation entry. Example: 1, 2, 3.
immunizationid int This column stores the immunizationid for which the reaction is being documented on Immunization Reaction Add/Edit window. This refers to the immunization.immunizationid column in the database. This column will always have a valid immunizationid value and never be NULL as the data in this column is updated from the Immunization Reaction window.
vaccineReactionid int This column storea the reactionid selected at the time of reaction documentation on the Immunization Reaction Add/Edit window. This refers to the vaccineReactionListid column of the database, so the data type must match that column. This column will always have a valid vaccine reaction ID value and never be NULL as data in this column is updated from the Immunization Reaction window.
deleteFlag tinyint This column stores the deleteFlag value for a particular immunization reaction. The default value for this column is 0 at the time of documenting a new reaction. This column is set to 1 when a user deletes a reaction from Immunization Reaction History window.
lastUpdatedBy int This column stores the userid of the logged-in user who has performed an add, update or delete action on an immunization reaction on the Immunization Reaction History window. This refers to the users.UID column of the database, so the data type must match that column. This column will always have a valid logged-in user ID value and never be NULL as data in this column is updated from the Immunization Reaction window.
vaccineReactionDate date This column stores the Reaction Date associated to an immunization reaction on the Immunization Reaction Add/Edit window. This column is populated from the EMR and is never NULL. If the optional Reaction Date field is not selected on the Reaction Add/Edit window, then the default date is applied. Date format: YYYY-MM-DD. Example: 2022-12-14.
updatedDateTime datetime This column stores the date/time when a user adds, updates, or deletes a reaction associated to an immunization on the Immunization Reaction History window. This column will always be populated with the current_time stamp when an entry is made from the EMR and is never NULL. The Date/Time format is YYYY-MM-DD hh:mm:ss. Example: 2022-03-07 18:19:11.
rxnStatus char This column stores the latest action performed on a particular immunization reaction (e.g., A - When a reaction is newly documented, U - When an existing reaction is updated, D - When a reaction is deleted). This status is sent to the Immunization Hub and state registries. This column is updated from the Immunization Reaction window; the setting defaults as A.

EHI Export Schema
Table emaillogs
Description This table stores meta data of all successful emails sent by Messenger Campaigns
Column DataType Description
id (Primary Key) int This is an auto increment id and primary key of table.
patientid int id of Patient who has received email
apUID int This column stores apUID
facilityid int facility's id who sent a email to patient. Foreign key relation to `edi_facilities` table (`id` column).
reportid int identifier for the report associated with the email log.
inserttime datetime time stamp of when the email log entry was inserted into the database. Example: '2021-08-01 12:34:56'.
campaignid int This is a Campaign Template id that the campaign/ policy belongs to
campaign varchar Name of Campaign Template that the campaign/ policy belongs to
healthMaintenance tinyint A flag indicating whether the email log is related to health maintenance (1) or not (0). Example: 0, 1.
policyid int This is a id of campaign thru which email has been sent to patient.
policyName varchar This is a name of campaign thru which email has been sent to patient.
messageid int This is id of message sent to a patient
doctorid int Doctor's id who sent a email to patient. Foreign key relation to `users` table (`UID` column).
subject text Subject of the email. Example: 'Appointment Reminder', 'Lab Results'.
body text Body content of the email. Example: 'Dear Patient, This is a reminder for your appointment...', 'Your lab results are now available...'.
dtReceived datetime time stamp of when the email was received. Example: '2021-08-01 13:45:00'.
readflag tinyint A flag indicating whether the email has been read (1) or not (0). Example: 0, 1.
dtRead datetime time stamp of when the email was read. Example: '2021-08-01 14:00:00'.

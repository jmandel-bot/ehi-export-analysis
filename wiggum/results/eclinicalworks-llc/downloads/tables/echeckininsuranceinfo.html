EHI Export Schema
Table echeckininsuranceinfo
Description This table stores information about insurance and driver's license documents uploaded by users during the e-check-in process.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
uid int User identifier, referring to the user who uploaded the document. Example: 101, 102, 103.
insuranceid int Insurance identifier, referring to the specific insurance record associated with the document. Example: 1001, 1002, 1003.
filename varchar Name of the file uploaded by the user. Example: "insurance_card.jpg", "driver_license.png".
delflag tinyint Flag indicating whether the record has been deleted (1) or not (0). Example: 0, 1.
ts_utc datetime time stamp in UTC when the record was created or updated. Example: "2021-09-01 12:34:56".
cardview varchar A short description or label for the uploaded document. Example: "Front side", "Back side".
imageguid varchar Unique identifier for the image file stored in the system. Example: "a1b2c3d4-e5f6-g7h8-i9j0-k1l2m3n4o5p6".
encounterid varchar Identifier for the specific encounter or appointment associated with the document. Example: "E123", "E124", "E125".
apuid varchar Identifier for the application user who uploaded the document. Example: "APU101", "APU102", "APU103".
websync tinyint Flag indicating whether the record has been synchronized with the web application (1) or not (0). Example: 0, 1.
STATUS smallint Status of the document, represented by a small integer. Example: 0 (Pending), 1 (Approved), 2 (Rejected).
TYPE varchar Type of the document, either 'insurance' or 'driverlicense'. Example: "insurance", "driverlicense".

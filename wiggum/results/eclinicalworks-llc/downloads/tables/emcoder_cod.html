EHI Export Schema
Table emcoder_cod
Description This table stores information about E&M Coder V2-related complexity of data computations for an encounter, along with details about who added or updated the record, and the timestamp when the record was updated. The data from this table is used when computing E&M Code using E&M Coder V2 for the encounter.
Column DataType Description
id (Primary Key) int This is the primary key for the table, which is an integer value that uniquely identifies each record. It is an auto-incrementing value. Example: 1, 2, 3, etc.
encid int This is a foreign key that refers to the encounterId column of table enc. It is an integer that represents the encounter identifier. Example: 101, 102, 103, etc.
clinicallabtest int Integer value representing the number of clinical lab tests performed during the encounter. Example: 0, 1, 2, etc.
radiologytest int Integer value representing the number of radiology tests performed during the encounter. Example: 0, 1, 2, etc.
medicinetest int Integer value representing the number of medicine tests performed during the encounter. Example: 0, 1, 2, etc.
discusswithphysician int Integer value representing the number of times the patient discussed with a physician during the encounter. Example: 0, 1, 2, etc.
obtainoldrecords int Integer value representing the number of old records obtained during the encounter. Example: 0, 1, 2, etc.
summationofoldrecords int Integer value representing the number of old records summarized during the encounter. Example: 0, 1, 2, etc.
visualizationofimages int Integer value representing the number of images visualized during the encounter. Example: 0, 1, 2, etc.
addedby int Identifier for the user who added the record. Example: 1001, 1002, 1003, etc. (No foreign key relation specified)
addeddate datetime Date and time when the record was added. Example: '2021-09-01 12:34:56'
updatedby int Identifier for the user who updated the record. Example: 1001, 1002, 1003, etc. (No foreign key relation specified)
updateddate datetime Date and time when the record was updated. Example: '2021-09-02 14:23:45'
deleteflag tinyint Tiny integer value representing whether the record is deleted or not. 0 means not deleted, 1 means deleted. Example: 0, 1

EHI Export Schema
Table ip_rh_ptmedicationorders
Description This table stores patient staging orders information which includes details like route, frequency of medications that are placed for a patient after being finalized.
Column DataType Description
medOrderId (Primary Key) int This column stores the unique identifier for each interaction record.
ptOrderId int This column is used to store ptOrder id and is foreign key references orderId column of ip_ptorders table.
ipitemid int This column is used to store item id and is foreign key references itemID column of items table.
medSigId int This is used to store Medication signature ID, this value is stored for reference, and it is not used. Deletion of medication signature is allowed so it cannot be added as foreign key.
order_dose varchar This column is used to store order dosage.This value is used from the of doseSize of ip_rh_drugformulary table
order_dose_UOM varchar This column is used to store order dosage UOM. This value is used from the of doseUoM of ip_rh_drugformulary table
RouteId int This column is used to store route id and is foreign key references routeId column of ip_drugformulary_routes table.
FrequencyId int This column is used to store frequency id and is foreign key references id column of ip_drugformulary_frequency table.
order_duration int This column is used to store order duration.
order_duration_type varchar This column is used to store order duration type like days, hours.
isHomeMed tinyint This column is used to store a home medication flag and shows whether it is converted from home medication or not. 1 - represents it is converted from home medication 0 - represent it is not converted home medication Default value is 0 ensures not home med
isPatientSupplied tinyint This column is used to store patient supplied flag it shows whether patient is supplied or not. 1 - represents patient PRis supplied 0 - represents patient is not supplied Default value is 0 that ensures patient is not supplied.
isInFormulary tinyint This column is used to store in formulary flag it shows whether it is in formulary or not.1 - represents it is in formulary 0 - represents it is not in formulary Default value is 0 that ensures item is not in formulary
rxNotes varchar This column is used to store Rx notes.
PRN tinyint This column is used to store PRN if frequency is PRN. 1 - represents PRN frequency 0 - represents Not PRN Frequency Default value of PRN is 0
PRNIndicationId int This column is used to store prn indication id and is foreign key references id column of ip_rh_drugformulary_prnindication table.
deleteflag tinyint This column is used to store in delete flag it shows whether order is deleted or not. 1 - represents order is deleted 0 - represents order is not deleted Default value is 0 ensures order is not deleted.
CreatedBy int This column is used to store the Id of the user who created the order. The column has a foreign key constraint (FK_ip_rh_ptmedicationorders_CreatedBy) referencing the uid in users table.
CreatedDateTime datetime This column is used to store create date and time in yyyy-mm-dd hh:mm:ss format.
ModifiedBy int This column is used to store the id of the user who modified order. The column has a foreign key constraint (FK_ip_rh_ptmedicationorders_ModifiedBy) referencing the uid in users table.
ModifiedDateTime datetime This column is used to store modified date and time in yyyy-mm-dd hh:mm:ss format.
orderType tinyint This column is used to store orderType value for order placed. 1 - medication order 2 - IV order 3 - complex order Default Value 1 ensures the type is medication.
isSubstitute tinyint This column is used to store substitute values like 1, 2. Default value is 0 if substitute is not specified. 0 - Substitute is not allowed 1 - Substitute is allowed
rxOrderNo varchar This column is used to store Rx order numbers, which is generated from staging order ID.
homeMedId int This column is used to store home medication ID. referencing the homemedid from the ip_rh_homemedication table.
pomFlag tinyint This column is used to store in pom flag. 1 - represents is pom 0 - represents is not pom Default value is 0 to ensure order is not pom.
disMedStatus smallint This column is used to store dispense medication status. The default value is NULL. NULL - No Action, 0 - Stop, 1 - Start, 2 - Continue, -2 - Stopped, -3 - Started, -4 - Continued
eMarEntryInstructions varchar This column is used to store eMAR Entry Instructions Default Notes populated from emar_instr column of ip_rh_drugformulary_notes
rphInstructions varchar This column is used to resident pharmacist instructions.Default Notes populated from pharmacy_instr column of ip_rh_drugformulary_notes
orderEntryInstructions varchar This column is used to store order entry Instructions.Default Notes populated from orderentry_instr column of ip_rh_drugformulary_notes
ivRate varchar This column is used to store IV rate defined in formulary for IV and complex order type.
ivRateUom varchar This column is used to store units like ml/hr, ml/min etc.
isRenewInExpiringTab tinyint This column is used to store renew in expiring flag. 1 - represents is eligible for renew in expiring 0 - represents not eligible for renew in expiring. Default value is 0 ensure order is not eligible for renew.
episodeEncId int This column is used to store episode encounter ID.referencing the encounterID from the enc table.
totalVolume decimal This is used to store total volume defined during order entry.
totalDose decimal This is used to store total dose defined during order entry.
isTitrationAllowed tinyint This column is used to store is titration or not.1 - represents titration allowed 0 - represents titration is not allowed. Default value is 0 ensure order is not titration allowed
internalNotes varchar This is used to store internal notes while adding the order. Default Notes populated from internal_notes column of ip_rh_drugformulary_notes
issFlag tinyint This column is used to store issflag enabled or not.1 - represents enbled ISS 0 - represents is not ISS flag. Default value is 0 ensure order is not ISS.
configuredISSFlag tinyint This column is used to store configured ISS flag or not. 1 - represents ISS is enabled 0 - represents no ISS flag. Default value is 0 ensure order is not configured ISS flag.
minDose varchar This is used to store min Dose value like 1ml, 200MG etc.
maxDose varchar This is used to store max Dose value like 5ml, 500MG etc.
reviewedBy int This column is used to store id of user who reviewed order.referencing the uid column in users table.
isReviewed tinyint This column is used to store reviewed or not. 1 - represents order is reviewed 0 - represents order is not reviewed Default value is 0 ensure order is not reviewed.
reviewDateTime datetime This column is used to store review date and time in yyyy-mm-dd hh:mm:ss format.
rxIncludeNow tinyint This column is used to store Rx include now or not.1 - represents first dose of Rx administration is at now 0 - represents first dose of Rx administration is at calculated datetime
infuseOverMinutes smallint This column is used to store infused over minutes.
infuseOverDisplayUOM varchar This column is used to store infuse over display UOM like Hour, Minutes, etc.
infuseOverDisplayValue varchar This column is used to store infuse over display value like 1 Hour, 1 Minute(s)
bulkflag tinyint This column is used to store bulk or not. 1 - represents order is in bulk 0 - represents order is not in bulk Default value is 0 ensure order is not in bulk.

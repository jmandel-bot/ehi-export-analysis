EHI Export Schema
Table vision_outsiderx
Description This table stores information about vision prescriptions provided by external providers, including the branch, verification status, and related notes.
Column DataType Description
id (Primary Key) int This is an auto incremented Primary Key.
visionrxid int Foreign key referring to the vision prescription id in visionrx table.
branchname varchar Name of the branch where the prescription was provided. Example: "Branch A", "Branch B", "Branch C".
notes text Long text field for storing any additional notes or comments about the prescription. Example: "Patient has astigmatism", "Prescription for reading glasses only".
recommendation text Long text field for storing recommendations provided by the external provider. Example: "Patient should wear glasses while driving", "Consider contact lenses for sports activities".
verifedby varchar Name of the person who verified the prescription. Example: "John Doe", "Jane Smith".
writtenveried tinyint A tinyint value indicating whether the prescription has been verified in writing (1) or not (0). Example: 0, 1.
televerfied tinyint A tinyint value indicating whether the prescription has been verified over the phone (1) or not (0). Example: 0, 1.
expiredate datetime Date and time when the prescription will expire. Example: "2023-01-01 00:00:00", "2024-12-31 23:59:59".
createdby int Foreign key referring to the user id who created the record. Example: 1001, 1002, 1003.
createdon datetime Date and time when the record was created. Example: "2021-01-01 12:34:56", "2021-02-02 23:45:01".
facilityid int Foreign key referring to the facility id where the prescription was provided. Example: 2001, 2002, 2003.
updatedby int Foreign key referring to the user id who last updated the record. Example: 1001, 1002, 1003.
updatedon datetime Date and time when the record was last updated. Example: "2021-01-15 12:34:56", "2021-02-10 23:45:01".
isdeletd tinyint A tinyint value indicating whether the record has been deleted (1) or not (0). Example: 0, 1.
testdate datetime Date when the vision test was conducted. Example: "2021-01-01", "2021-02-02".
providerid int This column stores the providerid int value for which vision outside Rx section is getting documented.

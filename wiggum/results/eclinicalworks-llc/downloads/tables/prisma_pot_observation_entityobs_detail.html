Table prisma_pot_observation_entityobs_detail
Description This table stores detailed information about observations made by entities in the Prisma system.
Column DataType Description
id (Primary Key) bigint A unique identifier for each record in the table. It is a bigint(20) type and is the primary key. Example: 1, 2, 3.
observationDetailId bigint A foreign key that refers to the `id` column in the `prisma_pot_observation_detail` table. It is a bigint(20) type and can be NULL. Example: 10, 20, 30.
observationId varchar A unique identifier for the observation. It is a varchar(50) type and can be NULL. Example: "OBS001", "OBS002", "OBS003".
observationTime varchar The time when the observation was made. It is a varchar(50) type and can be NULL. Example: "2021-08-01T12:00:00", "2021-08-01T12:30:00".
moodCodeValue varchar This column stores the mood code value associated with the activity. It is of type varchar(100) and can have a default value of NULL.
code varchar A code representing the type of observation. It is a varchar(100) type and can be NULL.
statusCode varchar A code representing the status of the observation. It is a varchar(100) type and can be NULL. Example: "COMPLETED", "IN_PROGRESS", "CANCELLED".
referenceText text A long text field containing reference information about the observation. It can be NULL. Example: "This observation was made during a routine checkup."
observationValue text A long text field containing the actual value of the observation. It can be NULL. Example: "98.6", "1013.25", "50%".
authorId varchar A unique identifier for the author of the observation. It is a varchar(50) type and can be NULL.
authorName varchar The name of the author of the observation. It is a varchar(100) type and can be NULL. Example: "John Doe", "Jane Smith", "Alice Johnson".

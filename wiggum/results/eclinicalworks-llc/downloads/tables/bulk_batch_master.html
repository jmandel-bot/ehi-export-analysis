EHI Export Schema
Table bulk_batch_master
Description This table stores information about Fhir R4 Bulk request which includes bulk batches, including their job ID, group ID, reference time, and status.
Column DataType Description
id (Primary Key) int This is the primary key for the table and is an auto-incrementing integer. It uniquely identifies each bulk batch record. Example: 1, 2, 3, etc.
job_id varchar This is a varchar column that stores the job ID associated with the bulk batch.
group_id varchar This is a varchar column that stores the group ID associated with the bulk batch.
since varchar This is a varchar column that stores an optional value representing a specific point in time or duration from which the bulk batch processing should start.
reference_time datetime This is a datetime column that stores the reference time for the bulk batch. Example: "2021-01-01 12:34:56", "2021-02-01 08:00:00", etc.
num_patients int This is an integer column that stores the number of patients included in the bulk batch. It has a default value of 0. Example: 0, 10, 25, etc.
batch_status tinyint This is a tinyint column that stores the status of the bulk batch. It has a default value of 0. Example: 0 (pending), 1 (processing), 2 (completed), etc.
created_at datetime This is a datetime column that stores the time stamp when the bulk batch record was created. Example: "2021-01-01 12:34:56", "2021-02-01 08:00:00", etc.
modified_at datetime This is a datetime column that stores the time stamp when the bulk batch record was last modified. Example: "2021-01-01 12:34:56", "2021-02-01 08:00:00", etc.
apply_provenence_parameter tinyint This is a tinyint column that stores a flag indicating whether to apply provenance parameters to the bulk batch. It has a default value of 0. Example: 0 (do not apply), 1 (apply), etc.

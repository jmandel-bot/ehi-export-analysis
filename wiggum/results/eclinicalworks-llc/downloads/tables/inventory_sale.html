Table inventory_sale
Description This table stores information about inventory sales transactions, including details about the patient, provider, facility, discounts, charges, taxes, and other related data.
Column DataType Description
id (Primary Key) int Unique identifier for each inventory sale record. (Example: 1, 2, 3)
Patient_id int identifier for the patient involved in the sale. Foreign key relation to the `Patient` table. (Example: 101, 102, 103)
Provider_id int identifier for the provider involved in the sale. Foreign key relation to the `Provider` table. (Example: 201, 202, 203)
Resource_id int identifier for the resource involved in the sale. Foreign key relation to the `Resource` table. (Example: 301, 302, 303)
Facility_id int identifier for the facility where the sale took place. Foreign key relation to the `Facility` table. (Example: 401, 402, 403)
Discount_id int identifier for the discount applied to the sale. Foreign key relation to the `Discount` table. (Example: 501, 502, 503)
Discount_Value decimal The value of the discount applied to the sale. (Example: 10.00, 15.50, 20.25)
ChargesAmt decimal The total amount of charges for the sale before discounts and taxes. (Example: 100.0000, 150.5000, 200.2500)
DiscountAmt decimal The total amount of discounts applied to the sale. (Example: 10.0000, 15.5000, 20.2500)
TaxesAmt decimal The total amount of taxes applied to the sale. (Example: 5.0000, 7.5000, 10.0000)
TotalAmt decimal The total amount of the sale after discounts and taxes. (Example: 95.0000, 142.0000, 190.0000)
IsDeleted tinyint A flag indicating if the record has been deleted (1) or not (0). (Example: 0, 1)
UpdatedBy int identifier for the user who last updated the record. Foreign key relation to the `User` table. (Example: 1001, 1002, 1003)
UpdatedOn datetime The date and time when the record was last updated. (Example: '2021-08-01 12:34:56', '2021-08-02 14:56:32')
CreatedBy int identifier for the user who created the record. Foreign key relation to the `User` table. (Example: 1001, 1002, 1003)
CreatedOn datetime The date and time when the record was created. (Example: '2021-07-01 10:30:45', '2021-07-02 11:45:23')
SoldBy int identifier for the user who sold the inventory item. Foreign key relation to the `User` table. (Example: 1001, 1002, 1003)
Enc_id int identifier for the encounter related to the sale. Foreign key relation to the `Encounter` table. (Example: 0, 10001, 10002)
IsBillToPt int A flag indicating if the bill is to be sent to the patient (1) or not (0). (Example: 0, 1)
IsPaymentDone int A flag indicating if the payment has been done (1) or not (0). (Example: 0, 1)
SKU int Foreign key referring to the stock keeping unit (SKU) id in another table. Example: 1001, 1002, 1003, etc.
FeeSchedule varchar The fee schedule associated with the sale. (Example: 'Standard', 'Premium', 'Basic')
DiscountType int identifier for the type of discount applied to the sale. Foreign key relation to the `DiscountType` table. (Example: 1, 2, 3)
DiscValue decimal The value of the discount applied to the sale. (Example: 10.0000, 15.5000, 20.2500)
invoiceid int identifier for the invoice related to the sale. (Example: 0, 20001, 20002)
receiptNo varchar The receipt number for the sale. (Example: 'R-10001', 'R-10002', 'R-10003')
outsideProvider varchar The name of the outside provider involved in the sale, if any. (Example: 'Provider A', 'Provider B', 'Provider C')
totalVatPercent decimal The total VAT percentage applied to the sale. (Example: 5.0000, 10.0000, 20.0000)
Currency varchar The currency used for the sale. (Example: 'USD', 'EUR', 'GBP')
DiscReason_id int identifier for the reason for the discount applied to the sale. Foreign key relation to the `DiscountReason` table. (Example: 1, 2, 3)
note varchar A note related to the sale, if any. (Example: 'First-time customer', 'Special promotion', 'Referral discount')
preferencecardmap_id int identifier for the preference card map related to the sale. Foreign key relation to the `PreferenceCardMap` table. (Example: 1, 2, 3)
ismerged tinyint A flag indicating if the record has been merged with another record (1) or not (0). (Example: 0, 1)

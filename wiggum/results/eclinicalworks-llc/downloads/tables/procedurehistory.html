EHI Export Schema
Table procedurehistory
Description This is the child table for every procedure documented for patients. Any procedure documented for a patient may have one or more entries in this table. It stores a history of changes made to the procedure over time since many dental procedures are not completed in a single encounter and progress over weeks/months. Each change to the procedure will have a new row in the table, and they all share the same ProcNum. This table has a many-to-one relationship with procedurelog.
Column DataType Description
id (Primary Key) bigint This is the auto-increment primary key column for the table.
ProcNum bigint This is a reference to the column ProcNum of the table procedurelog. This column has a many-to-one relationship with procedurelog, with entries created for each change to the procedure. This maintains a history of the changes made to the procedure.
ProcDate datetime This column stores the procedure date, which is usually the date the procedure was first documented, or the date of the encounter where the procedure was first documented.
procfee decimal This column stores the fee associated to the procedure.
Surf varchar This column stores the surfaces (if applicable) for which the procedure is applied. It's a combination of all applicable surfaces stored as a single string (example - MOD, DL, MODBL, etc).
ToothNum char This column stores the tooth number (if applicable) for which the procedure is applied.
ToothRange varchar If the procedure applied is for multiple teeth, or a specific treatment area, such as quadrant, arch, sextant, or full mouth, that information is stored in this column.
Priority bigint This column represents the priority of the procedure and is a reference to the column priorityNo of table dentalpriority.
ProcStatus tinyint This column references the id column of dentalentrystatus table and identifies the entry status of the procedure.
ProvNum bigint This column references the UID column of user table and identifies the provider id associated to the procedure.
dx int This column references the itemid column of items table and identifies one of up to four ICDs associated to the procedure.
DateEntryC datetime This column stores the completed date of the procedure.
CodeNum bigint This column references the itemid column of items table and identifies the CPT code associated to the procedure.
ModifiedByTrUserid bigint This column references the UID column of user table and identifies the id of the user who modified the procedure last.
DateModified datetime This column stores the time stamp of when the procedure was last modified.
deleteFlag tinyint This column, when set to 1, soft deletes the procedure entry. By default, when 0, the procedure is not flagged as deleted.
encounterid int This column references the encounterid column of en table and identifies the encounter id where the instance of this procedure entry was added/updated. For procedures added from the Patient Hub, this column stores -1.
procnote text This column stores the text that was documented as notes for the procedure.
migrationFlag tinyint This column indicates if the procedure was documented in eCW database through the regular use of the product, or if it was migrated from a different system to eCW database. Value is set to a non-zero value for migrated procedures.
PriorAuthStatus tinyint This column stores the prior authorization status for the procedure. A prior authorization provides advance approval for the planned service, which is generally valid for a fixed period of time. The allowed values in this column are 1 (not approved), 2 (requires approval), 3 (approved), and 0 which is the default value (no prior authorization status associated).
SelfPay tinyint This column is set to 1 when the patient opts to self-pay for a procedure. By default, the column is set to 0 which is the case when self-pay is not set.
PriorAuthNum varchar This column stores the prior authorization number obtained from the insurance company when the prior authorization has been approved for a procedure.
PriorAuthExpiryDate datetime This is the column that stores the expiry date for a prior authorization. As mentioned previously, each prior authorization is valid for a fixed period from when it is issued (typically 60 days).
dx2 int This column references the itemid column of items table and identifies one of up to four ICDs associated to the procedure.
dx3 int This column references the itemid column of items table and identifies one of up to four ICDs associated to the procedure.
dx4 int This column references the itemid column of items table and identifies one of up to four ICDs associated to the procedure.
smartnotenum int This column references the primary key column of 'smartnote' table and identifies whether a default smart note was applied to the procedure through favorite button configuration.
ptPortion decimal This column stores the patient estimate calculated for a procedure based on fee schedule and insurance.
allowedFee decimal This column stores the allowed Fee amount of the procedure based on insurance and Fee schedule.
feeUpdated tinyint This column stores value 0 or 1 to indicate whether patient estimate is calculated for the patient or not . By default the value is 0, which means patient estimate is not calculated yet. The column value changes to 1, when Treatment plan is accessed for the patient and fees gets calculated for the procedure.
feesUpdatedDate datetime This column stores the time stamp of when the procedure fees was last calculated.
resourceid int This column references the UID column of user table and identifies the id of the Resource associated to the procedure.
insid int This column stores the insurance id of the patient's Dental insurance used for calculation of Treatment plan estimates, at a procedure level. This refers to "insid" column of "insurance" table.
feeschid int This column stores the Fee schedule id of the patient used for calculation of Treatment plan estimates, at a procedure level. This refers to "id" column of "feeschlist" table.
insEstimate decimal This column stores the insurance estimate calculated for a patient based on insurance coverage percentage set for each procedure. This value depends on the dental insurance and fee schedule assigned to the patient.
insCoveragePerc decimal This column stores the insurance coverage percentage set for a procedure. Insurance coverage percentage for a procedure, under that insurance can be set either on Insurance CPT mapping screen or Dental Plans screen.
classVSurf varchar This column stores surfaces for which Class V fillings were applied. It is necessary to store this information to display on the UI the set of surfaces which have class V fillings and the set of surfaces which do not.
nonClassVSurf varchar This column stores surfaces for which non-Class V fillings were applied. It is necessary to store this information to display on the UI the set of surfaces which have class V fillings and the set of surfaces which do not.
slideDiscount decimal This column stores the Sliding Discount amount calculated for a Self-pay patient. This amount depends upon the Sliding Fee schedule assigned to the patient and is calculated either by Slide by CPT or Slide by CPT Group.
isInactive tinyint This column stores a 0 or 1 value based on procedure active or inactive status. When set to 1, the procedure is marked inactive. When set to 0, the procedure is marked active. Default value is 0.
inactiveReasonid smallint Unique identifier for the reason the procedure history record is inactive. Foreign key relation to `inactiveproceduresreasonconfig.id`.

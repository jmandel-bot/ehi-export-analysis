Table bh_intervention_referral
Description This table stores information about intervention referrals, including their relationships with structures, referral details, and the user who added the referral.
Column DataType Description
interReferralid (Primary Key) int This is the primary key, an auto-incrementing integer that uniquely identifies each intervention referral record. Example: 1, 2, 3, etc.
structMappingid int This is a foreign key integer that refers to the `bh_treatment_plan_pgoid_mapping` table's `id` column. It represents the relationship between the intervention referral and the treatment plan mapping. Example: 5, 10, 15, etc.
structid int This is a foreign key integer that refers to the `bh_treatment_plan_dictionary` table's `id` column. It represents the relationship between the intervention referral and the treatment plan structure. Example: 7, 14, 21, etc.
referralid int This is a foreign key integer that refers to the `referral` table's `Referralid` column. It represents the relationship between the intervention referral and the referral details. Example: 3, 6, 9, etc.
addedByUserid int This is a foreign key integer that refers to the `users` table's `UID` column. It represents the user who added the intervention referral. Example: 1001, 1002, 1003, etc.
addedDateTime datetime This is a datetime column that stores the date and time when the intervention referral was added. Example: '2021-09-01 12:34:56', '2021-09-02 14:30:00', etc.
deleteFlag tinyint This is a tinyint(1) column that stores a boolean value (0 or 1) indicating whether the intervention referral record has been deleted. Example: 0 (not deleted), 1 (deleted).

Table occhealth_enc_package_service_mapping
Description This table stores the mapping between occupational health encounters, medical packages, services, and employer packages, along with their completion status and additional notes.
Column DataType Description
encounterid int identifier for the encounter associated with the record. It can be a foreign key referring to the `encounterid` column in the `enc` table. Example: 101, 102, 103.
isType tinyint A tiny integer value indicating the type of the record. Example: 0, 1.
medicalPackageid int identifier for the medical package associated with the employer. Example: 201, 202, 203.
service_id int identifier for the service associated with the record. It can be a foreign key referring to the `id` column in the `occhealth_services` table. Example: 301, 302, 303.
employerPackageid int identifier for the employer package associated with the record. It can be a foreign key referring to the `id` column in the `employer_packages` table. Example: 401, 402, 403.
result text Text field containing the result of the service or package. Example: "Passed", "Failed", "Requires further evaluation".
isCompleted tinyint A tiny integer value indicating whether the service or package is completed (1) or not (0). Example: 0, 1.
certificateRestrictionid int identifier for the certificate restriction associated with the record. It can be a foreign key referring to the `id` column in the `certificate_restrictions` table. Example: 501, 502, 503.
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
notes varchar A short text field (up to 250 characters) containing any additional notes or comments about the record. Example: "Follow-up required in 6 months", "No issues detected".

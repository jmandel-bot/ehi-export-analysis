EHI Export Schema
Table coumadinflowsheet
Description This table stores information related to patients' Coumadin (Warfarin) therapy, including dosage, INR values, and other relevant details.
Column DataType Description
id (Primary Key) int This column is an auto incremented Primary Key of This column .
Patientid int This column stores patient identifier for a patient, for whom the coumadin flowsheet data is saved. It refers to column UID from user table.
Encid int This column stores an encounter id for which coumadin flowsheet data is saved. It refers to encounterid column from enc table.
TargetINR varchar This column stores an information about Target INR which is entered by the user.
Dose1 varchar This column stores the dose information which has to be taken by patient on Sunday.
Dose2 varchar This column stores the dose information which has to be taken by patient on Monday.
Dose3 varchar This column stores the dose information which has to be taken by patient on Tuesday.
Dose4 varchar This column stores the dose information which has to be taken by patient on Wednesday.
Dose5 varchar This column stores the dose information which has to be taken by patient on Thursday.
Dose6 varchar This column stores the dose information which has to be taken by patient on Friday.
Dose7 varchar This column stores the dose information which has to be taken by patient on Saturday.
CurrTabStrength varchar This column stores the the information regarding strength of Current Tablet entered by the user.
NewTabStrength varchar This column stores the the information regarding strength of New Tablet entered by the user.
CurrWeeklyDose varchar This column stores the information regarding Current Weekly Dose which has to be taken by patient per week.
NewWeeklyDose varchar This column stores the information regarding New Weekly Dose which has to be taken by patient per week.
comments text This column stores comments/notes if any entered by the user in the coumadin flowsheet.
CurrINR varchar This column stores the value of Current INR entered by the user.
CurrTabsPerWeek varchar This column stores the information regarding Current Tablets which have to be taken by patient per week.
NewTabsPerWeek varchar This column stores the information regarding New Tablets which have to be taken by patient per week.
enteredby int This column stores an identifier of user who is entering the information into coumadin flowsheet. It refers to the UID column from user table.
TargetINRHigh float Upper limit of the target INR range for the patient (e.g., 3.0).
DosageSchedule text This column stores the data regarding Dosage schedule in HTML format.
TargetINRLow float Lower limit of the target INR range for the patient (e.g., 2.0).
NextRecheckDate datetime This column stores the date which is entered by user on which rechecking is scheduled.
Indication varchar It stores the value of indication which is selected by the user. It contains one of the following values - Atrial fibrillation, Deep venous thrombosis, Pulmonary embolism, Mechanical Valve, Cerebrovascular accident, Other.
Complains varchar This column stores complain of patient which is entered by the user.
Denies varchar This column stores the reasons of denials which is entered by the user.
DiscussedPtEd tinyint This column stores the flag value if Patient Education is discussed. 0: Not Discussed ; 1: Discussed.
INRInRange tinyint This column stores a flag which indicates if the INR is within range or not. 0:Not in range 1: In range
StartDate datetime Start date of the therapy (e.g., "2021-01-01").
EndDate datetime End date of the therapy (e.g., "2021-12-31").
DiagnosisCode varchar This column stores the comma separated value of ICD Codes.
isLocked tinyint This column stores a flag which indicates if the coumadin flowsheet is locked by the user or not.1 : locked and 0 : Unlocked.
LockedBy int This column stores an identifier of user who has locked the coumadin flowsheet. It refers to the UID column from user table.
LockedOn datetime This column stores the date on which 'coumadin flowsheet' has been locked by the user.
LabSource varchar This column store encounter id for respective measures.
NextBloodDraw varchar This column stores the value of Next Blood Draw which is entered by user.
OrderingMD int This column stores the ordering MD identifier which is selected by the user and it refers to the UID column from user table.
dateofsign datetime This column stores the date of sign which is entered by user.
enrollment char This column stores the flag value for Enrollment in Anticoagulation Program. 0: Not Enrolled ; 1: Enrolled.
notes text This column stores the value of Internal notes which is entered by the user in the coumadin flowsheet.
tabsOnSun float Number of tablets to be taken on each day of the week (e.g., 1.0, 1.5).
tabsOnSat float Number of tablets to be taken on each day of the week (e.g., 1.0, 1.5).
Dose1TabStrength varchar Tablet strength for each day of the week (e.g., "2.5 mg").
Dose7TabStrength varchar Tablet strength for each day of the week (e.g., "2.5 mg").
tabsInMgOnSun float Total dosage in mg for each day of the week (e.g., 2.5, 5.0).
tabsInMgOnSat float Total dosage in mg for each day of the week (e.g., 2.5, 5.0).
CurrTabStrengthAddl varchar Additional current tablet strength for the patient (e.g., "1.0 mg").
TabsOnSunAddl float Additional number of tablets to be taken on each day of the week (e.g., 0.5, 1.0).
TabsOnSatAddl float Additional number of tablets to be taken on each day of the week (e.g., 0.5, 1.0).
tabsInMgOnSunAddl float Additional total dosage in mg for each day of the week (e.g., 1.0, 2.0).
tabsInMgOnSatAddl float Additional total dosage in mg for each day of the week (e.g., 1.0, 2.0).
dose1Addl varchar Additional dosage for each day of the week (e.g., "1.0 mg").
dose7Addl varchar Additional dosage for each day of the week (e.g., "1.0 mg").
dose1TabStrengthAddl varchar Additional tablet strength for each day of the week (e.g., "1.0 mg").
dose7TabStrengthAddl varchar Additional tablet strength for each day of the week (e.g., "1.0 mg").
createdon datetime This column stores the date on which 'coumadin flowsheet' is created.
createdby int This column stores an identifier of a user who has created the 'coumadin flowsheet'. It refers to the UID column from user table.
modifiedon datetime This column stores the date on which 'coumadin flowsheet' is modified.
modifiedby int This column stores an identifier of a user who has modified the 'coumadin flowsheet'. It refers to the UID column from user table.
DiscussedPregRisk tinyint This column stores the flag value for Discussed Preg Risk to indicate whether the risk associated with pregnancy has been discussed with the patient.
isReverted tinyint This column is used to indicate whether the coumadin flowsheet is reverted or not i.e. 1 indicates this is a reverted entry and 0 indicates this is a normal entry
labencounterid int If the patient is new and there is no lab for the patient documented and user tries to save the Anti-Coagulant Flowsheet then virtual lab encounterid is created on the Anti-Coagulant Flowsheet and is used to save in the patient document. For subsequent retrieval and prompting the user for missing document (if any) this enocunterid is referenced to get the saved document name/id - Foreign key to enc.encounterid

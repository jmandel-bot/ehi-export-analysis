Table ip_rh_ptorders_cosign
Description The table 'ip_rh_ptorders_cosign' appears to store information about co-signature actions performed on patient orders. Each record is uniquely identified by an 'id' and contains details about the order, action type, performer, and action time.
Column DataType Description
id (Primary Key) int This is the primary key for the table, providing a unique identifier for each record. It is an integer data type, auto incrementing from 1, and cannot be NULL, ensuring a unique and not NULL value for every record. This column is used to uniquely identify each co-signature action.
orderId int This is an Integer data type, cannot be NULL. This field holds the identifier for the order associated with the co-signature action. It references the orderId column in the ip_rh_ptorders table.
actionType varchar This is a varchar of size 20 and CANNOT be NULL. The default value is empty string. It holds the type of the co-signature action performed on the order.
performedBy int This is an Integer data type, cannot be NULL. This field hold the identifier of the user or entity who performed the co-signature action. It references the uId column in the users table.
performedDateTime datetime This field is a datetime data type and can be NULL. It represents the date and time the co-signature action was performed.
deleteFlag tinyint This is a tinyint, a smaller range integer. It cannot be NULL and has a default value of 0. This column indicates whether a patient order's co-sign has been marked for deletion or not. deleteFlag value 0 mark as not deleted and value 1 mark as deleted.

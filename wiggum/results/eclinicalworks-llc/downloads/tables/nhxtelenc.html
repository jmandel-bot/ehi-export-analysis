EHI Export Schema
Table nhxtelenc
Description This table stores information about inbox patient records received via P2P.
Column DataType Description
Reqid (Primary Key) int This is auto-incremented primary key in This column . This column represents each telephone encounter record attached to referral uniquely.
message text This column stores text data entered by end user or pre-defined keywords selected related to message received from or conveyed to the caller during the telephone encounter.
Assignedto varchar This column stores Full Name of provider/staff to whom the encounter is assigned to. Field acts like a search field throughout the software. User may enter letters to narrow down choices and also acts like a drop down menu.
assignedToid int This column stores UID for staff/provider whose Name is set in Assigned To filed. Field acts like a search field throughout the software. User may enter letters to narrow down choices and also acts like a drop down menu. Column refers to UID column in user table.
fromProvider varchar This column stores LastName, FirstName of the provider from whom referral with telephone encounter attached to it has been received.
fromProviderid int This column stores id of the provider from whom referral with telephone encounter attached to it has been received. This column refers to UID column in user table.
refReqid int This column stores id of requests received over P2P. This column refers to reqid column in nhxreferral table.
nhxPatientid int This column stores id of a patient for whom referral has been received over P2P. This column refers to id column in nhxpatient table.
date datetime This column stores date when referral with telephone encounter attached to it has been received/created over P2P. Date is stored in 'YYYY-MM-DD 00:00:00.000' format.
startTime datetime This column stores time when referral with telephone encounter attached to it has been received/created over P2P. Date is stored in '1900-01-01 hh:mm:ss.000' format.
priority tinyint This column is not being used.
reason varchar This column stores reason for creating a referral documented on referral which had telephone encounter of a patient attached to it.
status tinyint This column stores a digit to indicate current status of the referral received over P2P network. 0 - Referral is Open, 1 - Referral is addressed.
patientid int This column is not being used.
readStatus tinyint This column stores values 0/1 to indicate whether referral has been read by end user or not. 0 - Referral is NOT read by user, 1 - Referral is read by user.
reviewed tinyint This column stores values 0/1 to indicate whether referral has been reviewed by end user or not. 0 - Referral is NOT reviewed, 1 - Referral is reviewed.
deleteFlag tinyint This column stores values 0/1 to indicate whether the referral has been marked as deleted or not. 0 - Referral is NOT marked as deleted, 1 - Referral is marked as deleted.
requestapUID int This column stores APUID of a practice from where referral has been received from over P2P.
FromProviderP2pNPI varchar This column is not being used.
fromp2pnpi varchar This column stores National Provider identifier (NPI) number of the referring provider. This column refers to NPI column in doctors table.
top2pnpi varchar This column stores National Provider identifier (NPI) number of the referred provider. This column refers to NPI column in doctors table.
toProviderid int This column stores id of a provider to whom referral with attached telephone encounter has been sent to. This column refers to UID column in user table.
toProvider varchar This column stores LastName, FisrtName of the provider to whom referral with attached telephone encounter has been sent to.
ccP2PNPIs varchar This column stores comma separated National Provider identifiers (NPI) of providers added as CC while sending/receiving referral.
Ccp2pnames varchar This column stores comma separated names of providers added as CC while sending/receiving referral.

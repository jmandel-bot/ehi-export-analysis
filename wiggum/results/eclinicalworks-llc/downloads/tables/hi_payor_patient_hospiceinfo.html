Table hi_payor_patient_hospiceinfo
Description This table stores information about hospice care for patients, including comments, hospice status, and related notes.
Column DataType Description
id (Primary Key) bigint Unique identifier for each record in the table. Example: 1, 2, 3.
patientid int Unique identifier for the patient associated with the hospice information. This is a foreign key that refers to the `users` table's `UID` column. Example: 101, 102, 103.
comments varchar Text field for storing any comments related to the patient's hospice care. Example: "Patient prefers in-home care."
hospice tinyint A boolean flag indicating whether the patient is receiving hospice care (1) or not (0). Example: 0, 1.
hospicedate datetime The date and time when the patient started receiving hospice care. Example: '2021-09-01 12:30:00'.
hospicenotes varchar Text field for storing any additional notes related to the patient's hospice care. Example: "Patient requires special equipment."
createddate datetime The date and time when the record was created. Example: '2021-09-01 12:30:00'.
createdby int Unique identifier for the user who created the record. This is a foreign key that refers to the `users` table's `UID` column. Example: 201, 202, 203.
createdbyname varchar The name of the user who created the record. Example: "John Doe", "Jane Smith".
updateddate datetime The date and time when the record was last updated. Example: '2021-09-02 15:45:00'.
updatedby int Unique identifier for the user who last updated the record. This is a foreign key that refers to the `users` table's `UID` column. Example: 201, 202, 203.
updatedbyname varchar The name of the user who last updated the record. Example: "John Doe", "Jane Smith".
deleteflag tinyint A boolean flag indicating whether the record has been deleted (1) or not (0). Example: 0, 1.

Table jointproperty
Description This table stores information about various joint properties in a human body, such as the joint name, its position, and whether it is part of the 28 main joints.
Column DataType Description
jointPropertyid (Primary Key) int • This column will store the id of the Joint Property • The datatype of the column would be INT to accommodate large number of different Joint property that can be added. • This would be unique and auto incremented and so PRIMARYKEY constraint would be added
jointName varchar • This column will store the name of the Joint • The datatype of the column would be varchar(50) as the Joint name would normally be short and concise • NOT NULL constraint would be added as it mandatory to provide name of the Joint
position varchar • This column stores the position of the joint on the body • The datatype of the column would be enum as Possible values are ‘LEFT’, ‘RIGHT’ and ‘CENTER’
partOf28Joints tinyint • This column determines whether the joint is a part of the primary 28 joints. This property is required for determining certain scores that are used for tracking patient’s progress. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value mean joint is part of 28 joints • 1 value means joint is not part of 28 joints
deleteFlag tinyint • This column determines whether the joint is deleted. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value mean joint is not deleted • 1 value means joint is deleted

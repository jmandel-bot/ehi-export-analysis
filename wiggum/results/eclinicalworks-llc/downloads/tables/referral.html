EHI Export Schema
Table referral
Description This table stores information about patient Incoming/Outgoing referrals, including details about the referral source, destination, dates, and status.
Column DataType Description
Referralid (Primary Key) int Unique identifier for each referral record.
patientid int This column store the id of a patient for whom the referral has been created/received. This column refers to the UID in the user table.
insid int This column stores the id of an insurance being used for the referral. This column refers to insid column in the insurance table.
refFrom int This column stores the id of the provider selected under the From section in the user interface. Provider is someone who is referring the patient. This column refers to the UID column in the user table.
authNo varchar This column refers to free text string entered into Auth Code filled in the user interface. This code represents Auth Code provided by the insurance company for the referral.
date datetime This column stores date on which the referral was created for the patient. Date is stored in YYYY-MM-DD 00:00:00.000 format.
reason text This column stores reason why the referral was created. Reason could be free text or pre-configured keywords selected by the provider/staff.
diagnosis varchar This column stores an id for a diagnosis for which the referral has been created. This column refers to the itemid column in the items table. Column has been declared as VARCHAR because this column may store multiple ids for diagnosis separated by '|' (i.e., 250909|252643).
refStDate datetime This column stores start date of the referral after which the referral can be used by the patient for the visit and the date is selected by provider/staff.
refEnddate datetime This column stores the end date of the referral after which referral can not be used by the patient and the date is selected by provider/staff.
visitsAllowed int This column stores a number, which represents number of visits to referred provider will be allowed with the referral.
visitsUsed int This column stores a number, which represents the number of visits to the referred provider patient has used with the referral.
RefTo int This column stores an id of a provider under the To section in the user interface. Provider is to whom the patient has been referred to.
notes text This column stores free text notes documented by provider/staff under General Notes section under the Notes tab in the user interface.
deleteFlag int This column stores values 0/1 to indicate whether the referral has been deleted or not. 0 - Referral is not deleted, 1 - Referral is deleted.
referralType varchar This column stores a character I/O to indicate type of the referral being stored. I - Incoming Referral, O - Outgoing Referral.
priority int This column stores the priority of the referral documented by provider/staff. 0 - Routine, 1 - Urgent, 2 - Stat.
assignedto varchar This column stores name of the provider/staff to whom this referral has been assigned to in the user interface by the end user.
assignedToid int This column stores the id of a provider/staff to whom the referral has been assigned to in the user interface by the end user. This column refers to the UID column in the user table.
status varchar This column stores current status of the referral. Open - Referral is open, Pending - Consultation is Pending, Addressed - Referral is addressed, To Be Reconciled, Insurance Auth, Internal Review.
authtype varchar This column stores value of field Auth Type (Authorization Type) selected by the end user in the user interface.
procedures varchar This column stores an id for a procedure selected by provider/staff in the user interface while creating the referral. This column refers to the itemid column in the items table.
fromfacility int This column stores the id of a facility selected by the end user under the From section in the user interface while creating the referral. This column refers to the id column in the edi_facilities table.
ToFacility int This column stores the id of a facility selected by the end user under the To section in the user interface while creating the referral. This column refers to the id column in the edi_facilities table.
speciality int This column stores the id for a specialty of the practice for which the referral has been created. This column refers to the id column in the edi_speciality column.
POS int This column stores a code associated with the Place of Service. The Centers for Medicare & Medicaid Services (CMS) maintain POS codes used throughout the health care industry.
UnitType varchar This column stores values from the Unit Type field in the user interface. As per the practice, the unit type describes the nature of the visit for the referral created. The entity who receives the referral knows what kind of visit is required for that patient based on the unit type.
FrontOfficeAuth int This column stores flag 0/1 to indicate whether the referral record has been authorized by the front office staff to be uploaded to eHX or not. 0 - is not authorized, 1 -authorized by front office. This authorization is needed to send referral to eHX.
ReferralNumber varchar This column stores the Referral Number provided by a payer for prior authorization of a referral. If the Referral Number is missing and if the insurance 'requires referral,' then the claim can not be generated.
apptDate datetime This column stores the date for an appointment of a patient with referred provider.
clinicalNotes text This column stores free text notes documented by provider/staff under the Clinical Notes section under the Notes tab in the user interface.
ReceivedDate datetime This column stores date from the Received Date field in the user interface. Date when referral was received.
refEncid int identifier for the encounter associated with the referral.
ApptTime varchar This column stores the time for an appointment of a patient with referred provider associated with the referral.
extNHXApptBlockid int External appointment block identifier associated with the referral.
extNHXRefTxid int This column stores transaction id for the referral from P2P server. Transaction id is transmitted by P2P server outgoing when referral has been successfully received on the P2P server.
refReqid int identifier for the referral request. This column refers to the reqid column of the nhxreferral table.
refFromP2pNPI varchar National Provider identifier (NPI) of the referring provider.
refFromName varchar This column stores name of the referring provider.
refToP2pNPI varchar This column stores National Provider identifier (NPI) number of the referred provider. This column refers to NPI column in doctors table.
refToName varchar This column stores name of the referred provider.
uploadedToPtDocs tinyint This column stores values 0/1 to indicate whether documents received in incoming referral have been moved to the Patient Document section or not. 1 - If documents are moved to Patient Document, 0 - Otherwise.
refToCCNameList varchar This column stores comma separated names of providers added as CC while using the Send P2P Patient Record feature.
refToCCNPIList varchar This column stores comma separated National Provider identifiers (NPI) of providers added as CC while using the Send P2P Patient Record feature.
p2pDeliveryStatus tinyint This column stores delivery status of the P2P referral if the referral has been sent electronically. 1 - Success, 2 - Failed, 3 - Partial Failed, 4 - In Transit.
statusUpdateDate datetime This column stores the time stamp when the p2pDeliveryStatus is updated.
ccdaValidationStatus tinyint This column stores whether the CCDA attached to the electronic transaction satisfies MU eligibility criteria. 1 - CCDA failed MU eligibility criteria, 2 - CCDA passed MU eligibility criteria, 0 - Could not validate CCDA for MU eligibility.
fromDirectAddress varchar This column stores the direct registered health address of the provider who is sending outgoing referral out of the eClinicalWorks network.
toDirectAddress varchar This column stores the direct registered health address of the provider to whom the outgoing referral out of eClinicalWorks network is being sent to.
toDirectAddress1 varchar Alternative direct address of the receiving provider.
modifiedDate datetime This column stores the time stamp if any modification is made on the referral by the end user.
isVoiceMsgSent tinyint Flag indicating if a voice message has been sent for the referral (0 for not sent, 1 for sent).
isTextMsgSent tinyint Flag indicating if a text message has been sent for the referral (0 for not sent, 1 for sent).
quesAnsGUID varchar When SmartReferralPractice is enabled, a practice can setup the questionnaire for out-of-network referrals. When a user tries to send out-of-network referrals, he/she has to fill out the questionnaire which then is saved on P2P. The GUID for the answers is generated on P2P and saved in this column.
innetwork tinyint When SmartReferralPractice is enabled, this column stores values 0/1 to indicate if the provider to whom the referral is being sent to is in-network or not. 0 - if provider is out-of-network, 1 - if provider is in-network.
faxreferralxml text When SmartReferralPractice is enabled, if the user tries to print/fax a referral, the XML for that referral is stored in this column. This XML is then synced to P2P for reporting.
isHealowAppt tinyint This column stores 0/1 to indicate whether the referral appointment is from healow or not. 0 - Not a healow Appointment, 1 - healow Appointment.
transportationRequired tinyint This column stores flags 0/1 to indicate whether the patient would require transportation during the referral visit or not. 0 - Transportation is not required, 1 - Transportation is required. Custom column for Humana.
approvalReq tinyint This column stores values 0/1 to indicate whether the referral is submitted for approval or not. 0 - Referral is not submitted for approval, 1 - Referral is submitted for approval.
approvalstatus tinyint This column stores values 0/1 to indicate whether the referral is approved or not. 0 - Referral is not approved, 1 - Referral is approved.
EMCodes varchar This column stores Evaluation and Management Codes (EM Codes) documented by end user under procedures section of the User Interface.
subStatus varchar This column stores status from the Sub Status field in the User Interface. This is an extension to the main status of a referral. Substatuses are user defined and comes from referralstatus table.
subStatusid int This column stores id of a Status from the Sub Status field in the User Interface. 0 - No Sub Status Selected, 1 - Not Completed
isHISPTxCapable tinyint When sending electronic referral this column stores whether the receiver is a non-eCW HISP outbound receiver. 0 - Not non-eCW HISP outbound receiver, 1 - is non-eCW HISP outbound receiver. If value is 1, attachment size limitation can be adjusted accordingly.
isReceiverFTPEnabled tinyint This column store values 0/1 to indicate of FTP is enabled for the provider to whom the referral is being sent to. 1- If receiver's FTP is enabled, otherwise -0. If value is 1, attachment size limitation can be adjusted accordingly.
totalAttSize int This column stores total size of attachments to the referral in Kilo Bytes(KB).
PMReferralid varchar This column stores id for incoming referrals for which approval status was received over the referral interface. This column refers to Referralid column in pmreferrals table.(Check with Bhawna's Team)
referralsubType varchar Column stores sub type of a referral. Column stored values like "Visit", "CPT" etc
cptunitsallowed int Number of CPT (Current Procedural Terminology) units allowed for the referral.
cptunitsused int This column stores number of CPT units used for a referral.
transmissionMethod varchar Method of Transmision for Referral i.e., Print,Fax,P2P etc.
isConsultNoteAttached tinyint Flag indicating if a consult note is attached to the referral (0 for not attached, 1 for attached).
notify_portal tinyint Flag indicating if the referral should be notified on the patient portal (0 for not notified, 1 for notified).
referral360id varchar Unique identifier for the 360 referral.

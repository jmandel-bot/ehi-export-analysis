Table ip_rh_ptorders_ack_mapping
Description The table 'ip_rh_ptorders_ack_mapping' appears to store information about the mappings of acknowledgements or actions performed on patient orders. Each record is uniquely identified by an 'id' and contains details about the acknowledgement, the person who acknowledged, and the acknowledgement time.
Column DataType Description
id (Primary Key) int This is the primary key for the table, providing a unique identifier for each record. It is an integer data type, auto incrementing from 1, and cannot be NULL, ensuring a unique and not NULL value for every record. This column is used to uniquely identify each acknowledgement mapping.
ackId int This is an Integer data type, cannot be NULL. This field holds the identifier for the acknowledgement associated with the mapping. It references the Id column in the ip_rh_ptorders_ack table.
ackBy int This is an Integer data type, cannot be NULL. This field hold the identifier of the user or entity who acknowledged. It references the uId column in the users table.
ackFlag tinyint This is a tinyint, a smaller range integer, and CANNOT be NULL. The default value is 0. This column indicates whether an acknowledgement has been made or not.
ackDateTime datetime This field is a datetime data type and can be NULL. It represents the date and time the acknowledgement was made.
deleteFlag tinyint This is a tinyint, a smaller range integer. It cannot be NULL and has a default value of 0. This column indicates whether an acknowledgement mapping has been marked for deletion or not. deleteFlag value 0 mark as not deleted and value 1 mark as deleted.

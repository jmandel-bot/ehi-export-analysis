Table ip_rh_enc_macro_sentences
Description Table is used to store vitals section details.
Column DataType Description
id (Primary Key) int This column stores auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer (Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
encounterId int This column is used to identify for which patient encounter user has documented data. This column is FOREIGN KEY which refers to encounterId column of enc table. This column will store high range of numeric value hence integer is selected. This column will not always have value hence we keep NULL DEFAULT NULL.
entryId int This column stores mapping id of template and patient encounter. This column is FOREIGN KEY which refers to id column of ip_rh_enc_macro_entrytable. This column will store high range of numeric value hence integer is selected. This column will always have a value hence defined as NOT NULL.
templateId int This column store template id of template which is added for patient encounter. This column is FOREIGN KEY which refers to id column of ip_rh_clinical_sd_template table. This column will store high range of numeric value hence integer is selected. This column will always have a value hence defined as NOT NULL.
copiedFromSentenceId int This column stores the sentence id which is created from admin and is available in user facing side to document question answers.This column is FOREIGN KEY which refers to id column of ip_rh_pnsmartmacroStNotes table. This column will store high range of numeric value hence integer is selected. This column will not always have a value hence defined as NULL DEFAULT NULL.
sentenceNote varchar This column store's notes string. We are using datatype as text because users can enter large text. This column will not always have value hence defined NULL DEFAULT NULL.
sentenceOrder int This column stores integer value hence integer is selected as datatype of column. This column is used to maintain the order of items added for patient encounter. This column will always have a value hence defined as NOT NULL and default value is set to 0.
childSentence tinyint This will store 0 and 1 value. 0-not child sentence and 1-child sentence. We are using TINYINT as datatype as we just want to store 0 or 1 value. This column will always have a value hence defined as NOT NULL. By default, sentence will not be child sentence, so we keep default value is 0.
parentId int This column is FOREIGN KEY which refers to id column of ip_rh_enc_macro_sentences table. This column will store high range of numeric value hence integer is selected. This column is used to map child sentences with parent sentences. This column will not always have value hence NULL DEFAULT NULL
triggerQuestionId int This column is FOREIGN KEY which refers to id column of ip_rh_structdatadetail table. This column will store a high range of numeric value hence integer is selected. This column will not always have value hence we keep NULL DEFAULT NULL.
triggerOptionId int This column stores optionId. This column belongs to optionId column of the ip_rh_structdataoptions table. For Boolean type question this column stores 1 and 2 value and not considering ip_rh_structdataoptions table for Boolean type question. This column will not always have value hence we keep NULL DEFAULT NULL.
mandatory tinyint This will store 0 and 1 value. 0-question is not mandatory and 1- question is mandatory. We are using TINYINT as datatype as we just want to store 0 or 1 value. By default, the question is not mandatory, so we keep default value is 0
isUnAnswered tinyint This will store 0 and 1 value. 0-question is answered and 1- question is not answered. This column is used to identify whether row is in use or not. We are using TINYINT as datatype as we just want to store 0 or 1 value. By default, the question is answered, so we keep default value is 0.
deleteFlag tinyint This will store 0 and 1 value. 0-not deleted and 1-deleted. This column is used to identify whether row is in use or not. We are using TINYINT as datatype as we just want to store 0 or 1 value. This column will always have a value hence defined as NOT NULL. By default, row will be in use, so we keep default value is 0.
createdBy int This column stores user id of user. This column is used to identify which user has added this record (macro sentences) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store high range of numeric value hence integer is selected. This column will always have a value hence defined as NOT NULL.
createdDateTime datetime This Column stores datetime as we want to store a datetime when the record is added, so we are using DATETIME as data type of column. This column will always have a value hence defined as NOT NULL.
modifiedBy int This column stores user id of user. This column is used to identify which user has last updated this record (macro sentence) for patient encounter This column is FOREIGN KEY which refers to uid column of users table. This column will store high range of numeric value hence integer is selected, and this column will always not have a value hence defined as NULL DEFAULT NULL.
modifiedDateTime datetime This Column stores datetime as we want to store a datetime when the record is modified, so we are using DATETIME as data type of column. This column will always not have a value hence defined as NULL DEFAULT NULL.

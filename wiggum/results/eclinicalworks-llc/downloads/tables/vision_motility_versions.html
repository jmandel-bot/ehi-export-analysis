EHI Export Schema
Table vision_motility_versions
Description This table stores information about different versions of vision motility tests performed on patients.
Column DataType Description
id (Primary Key) int This column is an auto incremental primary key.
patientid int This column refers to the UID column in the user table. It stores patient ids for patients.
encid int This column stores an encounter id of a patient. This column refers to column encounterid from enc table.
test_type varchar This column stores an id of the item selected in the drop down option for the Versions exam in the Motility window. This column refers to column itemid from the items table.
location_target tinyint This column indicates what type of location target versions exam have. It can only hold one of the following values: (0)-Distance, (1)-Near, (2)-Intermediate. Default value is null indicating none of the radio options are selected.
location_target_value int This column stores the values for the test location target.
location_target_unit int This column stores the unit type for location_target. Values that can be stored are from 0-2. It can hold one of the following values for an unit type: (0) - feet, (1) -inches, or (2) - meters.
rx_option tinyint This column stores values for Rx option. Values that can be stored are 0-4. The default value is NULL. This column is optional and hence when a user does not enter any value, NULL will be stored in them.
rx_other_option_notes varchar This column stores Rx option notes and varchar (15) is chosen to save Rx notes.
comments varchar This column stores versions exam notes and varchar (255) is chosen to save comments.
deleteflag tinyint This column stores value that can take a value of 0 - (not deleted) or 1 - (deleted).
created_date datetime This column stores date and time when versions was created. Datetime is stored in 'YYYY-MM-DD hh:mm:ss.000' format.
last_updated_date datetime This column stores date and time when versions was last updated. Datetime is stored in 'YYYY-MM-DD hh:mm:ss.000' format.

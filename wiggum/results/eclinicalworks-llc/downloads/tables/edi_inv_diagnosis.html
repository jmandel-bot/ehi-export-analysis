EHI Export Schema
Table edi_inv_diagnosis
Description This table stores information about the diagnosis related to an invoice, including the diagnosis code, primary code, and other related details.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the edi_inv_diagnosis table. Example: 1, 2, 3.
Invoiceid int identifier for the invoice to which the diagnosis is related. This is a foreign key referring to the id column in the edi_invoice table. Example: 101, 102, 103.
Itemid int identifier for the item associated with the diagnosis. This is a foreign key referring to the id column in the item table. Example: 1001, 1002, 1003.
Code varchar The diagnosis code, typically an ICD-10 code. Example: 'J20.9', 'M54.5', 'E11.9'.
PrimaryCode tinyint A tiny integer indicating whether the diagnosis code is the primary code (1) or not (0). Example: 0, 1.
Specify varchar Additional information or specification about the diagnosis. Example: 'Acute bronchitis', 'Lower back pain', 'Type 2 diabetes'.
Notes text Text field for any additional notes related to the diagnosis. Example: 'Patient has a history of asthma', 'Pain worsens when sitting', 'Patient also has hypertension'.
icdOrder smallint A small integer representing the order of the ICD code in the list of diagnoses. Example: 1, 2, 3.
snomed varchar The SNOMED CT code for the diagnosis, an international standard for clinical terminology. Example: '195967001', '279039007', '44054006'.
POA varchar Present on Admission indicator, a 2-character code indicating whether the diagnosis was present when the patient was admitted. Example: 'Y' (Yes), 'N' (No), 'U' (Unknown).
diagnosisType varchar A 2-character code representing the type of diagnosis. Example: 'P' (Primary), 'S' (Secondary), 'T' (Tertiary).
RFV tinyint A tiny integer indicating the Reason for Visit. Example: 1 (Routine), 2 (Emergency), 3 (Referral).

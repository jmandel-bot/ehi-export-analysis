EHI Export Schema
Table edi_inv_adjustments
Description This table stores information about adjustments made to invoices in the Billing module.
Column DataType Description
id (Primary Key) int Unique identifier for each adjustment record. Example: 1, 2, 3.
Invid int The invoice identifier to which the adjustment is related. This is a foreign key that refers to the `id` column in the `edi_invoice` table. Example: 101, 102, 103.
date datetime The date when the adjustment was made. Example: '2021-09-01', '2021-09-02', '2021-09-03'.
code varchar A code representing the type of adjustment made. Example: 'ADJ01', 'ADJ02', 'ADJ03'.
amount decimal The amount of the adjustment. Example: 100.00, 50.00, -25.00.
Userid int The identifier of the user who made the adjustment. This is a foreign key that refers to the `id` column in the `users` table. Example: 1, 2, 3.
modifiedDate datetime The time stamp of the last modification made to the adjustment record. Example: '2021-09-01 12:34:56', '2021-09-02 14:30:00'.
CreatedBy int The identifier of the user who created the adjustment record. This is a foreign key that refers to the `id` column in the `users` table. Example: 1, 2, 3.
closingid int The identifier of the closing record associated with the adjustment. This is a foreign key that refers to the `id` column in the `closings` table. Example: 1001, 1002, 1003.
unpostedCPT decimal The unposted cost per transaction (CPT) for the adjustment. Example: 10.00, 5.00, 2.50.
UnpostedAmount decimal The unposted amount for the adjustment. Example: 100.00, 50.00, 25.00.
adjlock int A flag indicating whether the adjustment record is locked (1) or unlocked (0). Example: 0, 1.

EHI Export Schema
Table dental_psr
Description This table stores information of patient's dental PSR (Periodontal Screening and Recording) data which is used in the Dental Examination screen.
Column DataType Description
id (Primary Key) bigint This column is auto-incremented primary key in This column . Unique id is assigned to each PSR exam when they are created using User Interface. It is defined as NOT NULL constraint.
Providerid int This column stores identifier number of a provider while appointment is created. It refers to column UID from user table. It is defined as NOT NULL constraint. 
Patientid int This column stores identifier number of a patient, for whom the appointment is created. It refers to column pid from patients table. It is defined as NOT NULL constraint. 
USection1 varchar This column stores Upper level section 1 value from PSR. It allows NULL value as default.
USection2 varchar This column stores Upper level section 2 value from PSR. It allows NULL value as default.
USection3 varchar This column stores Upper level section 3 value from PSR. It allows NULL value as default.
LSection1 varchar This column stores Lower level section 1 value from PSR. It allows NULL value as default.
LSection2 varchar This column stores Lower level section 2 value from PSR. It allows NULL value as default.
LSection3 varchar This column stores Lower level section 3 value from PSR. It allows NULL value as default.
Pattern varchar This column stores pattern value like LL or LR. It allows NULL value as default.
Notes varchar Additional notes about the exam. Example: "Patient has mild gingivitis", "Patient needs deep cleaning".
ExamDate datetime This column stores Exam Date of PSR. It is defined as NOT NULL constraint.
StarMark varchar This column stores star mark value of PSR sections like Usection1, Usection2, Usection3, Lsection1, Lsection2, Lsection3. This column allows '0#0#0#0#0#0#' as default value.
DeleteFlag tinyint Indicates if the record has been deleted (1) or not (0). Example: 0, 1.

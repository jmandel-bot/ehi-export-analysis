Table activitydetail
Description This table stores detailed information about various activities, including their type, location, season, status, equipment, surface, and other related attributes.
Column DataType Description
id (Primary Key) int This is the primary key of This column . It is an autoincremented value.
Activityid int This stores the activity id of the activity. This refers to the id column of the activitycalendar table.
Type varchar The type of activity. Example: Running, Swimming, Cycling.
Location varchar This column stores the location code of the activity. This refers to the code column of the activitydetailinfo table.
SeasonType varchar The type of season during which the activity occurs. Example: Winter, Summer, Spring, Autumn.
Status varchar This column stores the status of the activity. Example: Complete, Reschedule.
Equipment varchar This column stores the equipment id for the activity. This refers to code column of the athletics_activity_equipment table.
Surface varchar The surface on which the activity takes place. Example: Grass, Concrete, Sand.
SurfaceType varchar This column stores the surface type id selected for the activity. This refers to code column of athletics_activity_surfaceType table.
ConditionType varchar The type of condition during the activity. Example: Sunny, Rainy, Windy.
Temperature varchar This column stores the temperature of the activity in degree fahrenheit.
Humidity varchar This column stores the humidity of the activity in percentage.
Season varchar This column stores the season code of the activity. This refers to code column of the activitydetailinfo table.
GameLocation varchar This column stores the code of home location of the activity. This refers to code column of the activitydetailinfo table.
Opponent varchar The opponent team or player for competitive activities. Example: Team X, Player Y, Club Z.
StadiumType varchar The type of stadium where the activity takes place. Example: Indoor, Outdoor, Dome.
GameWeek varchar The week of the game for team sports. Example: Week 1, Week 2, Week 3.
Notes text This column stores the notes of the activity.
deleteflag tinyint This column store values 0/1 to indicate whether the item has been marked deleted or not. 0 - item is not marked deleted, 1 - item is marked deleted.
setCurrentStatus tinyint A flag indicating if the current status should be set. Example: 0 (do not set), 1 (set).
SportId int This column is not in use anymore

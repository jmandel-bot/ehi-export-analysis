Table emcoderdmo
Description This table stores information related to medical encounters, including the encounter ID, diagnosis status, LDI status, and points associated with the encounter.
Column DataType Description
id (Primary Key) int This is the primary key for the table, which is an automatically incremented integer value. It uniquely identifies each record in the table. Example: 1, 2, 3, etc.
EncId int This is a foreign key that refers to the encounter ID in another table (not specified in the provided SQL). It is an integer value that represents the ID of the medical encounter. Example: 101, 102, 103, etc.
NewDx tinyint This is a tinyint(1) column that indicates whether the encounter has a new diagnosis or not. It is a boolean value, where 0 represents no new diagnosis and 1 represents a new diagnosis. Example: 0 or 1.
NewDxLDI tinyint This is a tinyint(1) column that indicates whether the encounter has a new diagnosis with LDI (Longitudinal Diagnostic Imaging) or not. It is a boolean value, where 0 represents no new diagnosis with LDI and 1 represents a new diagnosis with LDI. Example: 0 or 1.
Points int This is an integer column that stores the points associated with the encounter. It has a default value of 0. Example: 10, 20, 30, etc.

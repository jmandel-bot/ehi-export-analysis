Table lsm_address
Description This table stores address information for patients, including contact details and address types.
Column DataType Description
Contactid (Primary Key) int Unique identifier for each contact record. (e.g., 1, 2, 3).
Patientid int Unique identifier for the patient associated with the address (e.g., 101, 102, 103).
Name varchar Name of the contact person at the address (e.g., "John Doe", "Jane Smith").
Address1 varchar First line of the address (e.g., "123 Main St", "456 Elm St").
Address2 varchar Second line of the address, if applicable (e.g., "Apt 4B", "Suite 300").
City varchar City where the address is located (e.g., "New York", "Los Angeles").
State varchar State where the address is located. (e.g., "California", "New York").
ZipCode varchar Zip code of the address. (e.g., "10001", "90210").
Country varchar Country where the address is located (e.g., "United States", "Canada").
Phone1 varchar Primary phone number associated with the address (e.g., "555-123-4567", "555-987-6543").
Phone2 varchar Secondary phone number associated with the address, if applicable (e.g., "555-321-7654", "555-654-9872").
Ext1 varchar Extension for the primary phone number, if applicable (e.g., "123", "456").
Ext2 varchar Extension for the secondary phone number, if applicable (e.g., "789", "321").
AddressType varchar Type of address (e.g., "Billing", "Shipping", "Home", "Work").
PrimaryLocation varchar Indicates if the address is the primary location for the patient. (e.g., "Yes", "No").
DelFlag tinyint Flag to indicate if the address record has been deleted (e.g., 0 for not deleted, 1 for deleted).
Address3 varchar Third line of the address, if applicable (e.g., "Building A", "Floor 2").
IsAddressModified tinyint Flag to indicate if the address has been modified (e.g., 0 for not modified, 1 for modified).
Coordinates varchar Geographical coordinates of the address (e.g., "40.712776,-74.005974", "34.052235,-118.243683").

EHI Export Schema
Table edi_adj_codes
Description This table stores information about EDI adjustment codes, including their names, whether they adjust patient balances, their display colors, and their active/inactive status and updated by.
Column DataType Description
id (Primary Key) int Unique identifier for each adjustment code record and primary key. Example: 1, 2, 3.
Code varchar Unique code for the adjustment. Example: 'ADJ01', 'ADJ02', 'ADJ03'.
Name varchar This columns stores varchar 255 character which is describes code.
AdjustPtBalance tinyint A boolean flag (0 or 1) indicating whether the adjustment code affects the patient balance. Example: 0 (False), 1 (True).
Color varchar The color associated with the adjustment code for display purposes. Example: 'red', 'blue', 'green'.
Inactive tinyint A boolean flag (0 or 1) indicating whether the adjustment code is inactive. Example: 0 (Active), 1 (Inactive).
rcmPracticeView int An integer representing the view identifier for the RCM practice. Example: 101, 102, 103.
updatedby int An integer representing the user identifier who last updated the record. Example: 1, 2, 3.
updatedon datetime A datetime value representing the date and time when the record was last updated. Example: '2021-09-01 12:34:56'.

Table ip_rh_clinical_sd_template_question
Description This table stores the questions and sentences mapped to the corresponding template.
Column DataType Description
Id (Primary Key) int This is an auto-incremented primary key column in this table. It is used to uniquely identify the template questions. The data type is int as per business requirement and the column cannot be NULL.
tempId int This column refers to the id column in ip_rh_clinical_sd_template table. It stores the templateId for which the questions are mapped. The data type is int. This column will always have a value and hence it cannot be NULL.
questId int This column stores the id of a question which is mapped to the corresponding sentence. This column refers to id column in ip_rh_structdatadetail table. The data type is int. The column cannot be NULL as it will always have a value.
description varchar This column will store the description of the questions and sentences mapped to the template. The data type is varchar(max). As the column could store huge data as value and we have free text, therefore we have defined it as varchar(max). The default value is NULL.
modiBy int Stores the user id of the person who last modified the question mapping. It refers to uid column in users table. The datatype is int. The default value for this column is NULL.
modiTime datetime Stores the date and time the question mapping was last modified. The datetime data type allows for precise storage of date and time values. The default value is NULL.
displayIndex smallint This column stores a number to indicate in which order questions should be listed on user facing screen. The data type is smallint as compared to int is because the expected template questions displayed will not exceed the limitation provided by the smallInt datatype.
deleteFlag tinyint A flag indicating if the question is deleted (1) or not (0). The tinyint data type allows for efficient storage of binary values. The default value is 0, indicating the question is not deleted.
sentenceId int This column stores the sentence id which is id column in ip_rh_pnsmartmacroStNotes table. It stores the sentenceid mapped to the question. Foreign key is not applicable as in this table sentence Id can come from couple of tables, one when narrative paragraph is created on admin side then sentence Id come from this table 'ip_rh_pnsmartmacroStNotes' and second when this template is used on user side then entry come in this table with encounter id and sentence id from 'ip_rh_enc_macro_sentences' table.
encounterId int This column stores the encounter Id of the note when the question is documented for the template on the user facing screen. It refers to encounterId column in enc table.

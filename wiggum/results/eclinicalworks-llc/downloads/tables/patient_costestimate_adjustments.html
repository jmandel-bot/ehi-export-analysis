Table patient_costestimate_adjustments
Description This table stores adjustments and discounts related to cost estimates for patients.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
estid int Foreign key referring to the `estid` column in the `patient_costestimate_header` table. It represents the cost estimate id associated with the adjustment which is one row per costestimate . Example: 101, 102, 103.
estadjprof decimal This column store the amount of adjustment on professional fee of CPT after sliding setup applied.The datatype of the column is DECIMAL(9,2) .Cost estimator patient without sliding fee schedule does not have this adjustment so NULL is allowed.
estadjfac decimal This column store the amount of adjustment on facility fee of CPT after sliding setup applied.The datatype of the column is DECIMAL(9,2) .Cost estimator patient without sliding fee schedule does not have this adjustment so NULL is allowed.
estadjanes decimal This column store the amount of adjustment on anesthesia fee of CPT after sliding setup applied.The datatype of the column is DECIMAL(9,2) .Cost estimator patient without sliding fee schedule does not have this adjustment so NULL is allowed.
estadjtotal decimal Total adjustment amount for the cost estimate. Example: 300.00, 525.50, 750.75.
dentalDiscount decimal This column store dental discount amount for the cost estimator patient after sliding setup applied.It could be NULL if the Cost estimator patient without sliding fee schedule. Example: 10.00, 20.50, 30.75.
medicalDiscount decimal This column will store the discount amount on medical charges after sliding setup applied.It could be NULL if the Cost estimator patient without sliding fee schedule. Example: 10.00, 20.50, 30.75.
CPTDiscount decimal This column will store the discount amount on selected CPT in cost estimator after sliding setup applied.It could be NULL if the Cost estimator patient without sliding fee schedule. Example: 15.00, 25.50, 35.75.
CPTDiscountType tinyint CPT discount type identifier. Example: 1, 2, 3.
copayDiscount decimal Copay discount amount for the cost estimate. Example: 3.00, 6.50, 9.75.
copayDiscountType tinyint Copay discount type identifier. Example: 1, 2, 3.
ptBalanceLimit decimal Patient balance limit for the cost estimate. Example: 1000.00, 2000.50, 3000.75.
CPTGrpName varchar CPT group name associated with the cost estimate. Example: 'Group1', 'Group2', 'Group3'.
assignedstatus varchar This column store the assigned sliding setup status ('assigned','systemExpired','userExpired','noSlide').The datatype of the column is ENUM as 'noSlide', because by default we are not assigning any sliding adjustment on estimator.
assignedType varchar Type of the adjustment assignment. Example: 'Type1', 'Type2', 'Type3'.
povertyLevel decimal Poverty level associated with the cost estimate. Example: 1.00, 1.50, 2.00.
responsibility tinyint Responsibility identifier for the cost estimate. Example: 1, 2, 3.
expireDate datetime Expiration date of the adjustment. Example: '2022-12-31 23:59:59', '2023-01-31 23:59:59'.
hasInvalidCpts bit A bit flag indicating if the cost estimate has invalid CPTs. Possible values: b'0' (false), b'1' (true).

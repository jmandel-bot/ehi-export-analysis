EHI Export Schema
Table dental_tp_signature
Description This table stores the signature information related to dental treatment plans, including the flags, eSign flags, JSON data, dates, user IDs, and sign types for both sign1 and sign2.
Column DataType Description
id (Primary Key) int • This column will store the id of the dental_tp_signature • The datatype of the column would be INT to accommodate many different saved signature data that can be added. • This would be unique and auto incremented and so PRIMARYKEY constraint is added
TreatplanNum int • This column stores the id of the saved dental treatment plan. • This is a reference to the column treatplannum of table dentaltreatmentplan.
sign1_flag tinyint • This column is used to determine whether signature 1 is done for this treatment plan or not. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value means the signature is not done for this treatment plan. • 1 value means the signature is done for this treatment plan.
sign1_eSignFlag tinyint • This column is used to determine whether e-signature 1 is done for this treatment plan or not. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value means the e-signature is not done for this treatment plan. • 1 value means the e-signature is done for this treatment plan.
sign1_JSON text • This column is used to store signature JSON/base54 encoded string value of signature image done for this treatment plan signature 1. • The datatype of the column would be text as it can have larger value for signature image.
sign1_date datetime • This column stores the server date and time stamp of the instant the operation being performed is saved to the database. • The datatype of the column would be datetime as it is storing date.
sign1_userid int • This column stores userid of the user who has done signature for the treatment plan. • This is a reference to the column UID of table users.
sign1_signType tinyint • This column stores the sign type of signature 1. • The datatype of the column would be tinyint as it can have 0, 1or 2 value. • 0 value means the signature is done using mouse pad or e-signature for this treatment plan. • 1 value means the signature is done using scriptel web pad for this treatment plan. • 2 value means the signature is done using 10e plugin pad for this treatment plan.
sign2_flag tinyint • This column is used to determine whether signature 2 is done for this treatment plan or not. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value means the signature is not done for this treatment plan. • 1 value means the signature is done for this treatment plan.
sign2_eSignFlag tinyint • This column is used to determine whether e-signature 2 is done for this treatment plan or not. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value means the e-signature is not done for this treatment plan. • 1 value means the e-signature is done for this treatment plan.
sign2_JSON text • This column is used to store signature JSON/base54 encoded string value of signature image done for this treatment plan signature 2. • The datatype of the column would be text as it can have larger value for signature image
sign2_date datetime • This column stores the server date and time stamp of the instant the operation being performed is saved to the database. • The datatype of the column would be datetime as it is storing date.
sign2_userid int • This column stores userid of the user who has done signature for the treatment plan. • This is a reference to the column UID of table users.
sign2_signType tinyint • This column stores the sign type of signature 2. • The datatype of the column would be tinyint as it can have 0, 1or 2 value. • 0 value means the signature is done using mouse pad or e-signature for this treatment plan. • 1 value means the signature is done using scriptel web pad for this treatment plan. • 2 value means the signature is done using 10e plugin pad for this treatment plan.

EHI Export Schema
Table appealstatuscodes
Description This table stores the configuration for appeal status codes with corresponding claim status codes, so that when an appeal moves to a different status, the claim associated with the appeal will also move to the configured claim status.
Column DataType Description
id (Primary Key) int This column is auto-incremented primary key in This column . Unique id is assigned to each status (Not Started, Review Requested, Pending Review etc) when they are inserted.
statusDesc varchar Description of the appeal status code. It is a variable character string with a maximum length of 255 characters. For example: "Pending", "Approved", "Rejected".
deleteFlag tinyint Indicates whether the appeal status code is deleted or not. It is a tiny integer value with a default value of '0'. '0' means not deleted, and '1' means deleted.
isDeleteAllowed tinyint Indicates whether the appeal status code can be deleted or not. It is a tiny integer value with a default value of '0'. '0' means deletion is not allowed, and '1' means deletion is allowed.
statusOrder tinyint Represents the order in which appeal status codes should be displayed. It is a tiny integer value with a default value of '0'. For example: 1, 2, 3, etc.
claimStatus varchar Represents the claim status associated with the appeal status code. It is a variable character string with a maximum length of 64 characters. For example: "Open", "Closed", "In Progress".

Table ip_rh_order_priority_master
Description The ip_rh_order_priority_master table contains the master record of order priorities in a healthcare context.
Column DataType Description
priorityId (Primary Key) int This is an auto-incrementing integer column, serving as the primary key.
priorityName varchar This is a string (varchar) column with a maximum length of 50 characters. It cannot be NULL. This column represents the name of the order priority. The NOT NULL constraint ensures that this column must always have a value, which makes sense as every order priority should have a name.
DeleteFlag tinyint This is a tinyint, a smaller range integer. It cannot be NULL and has a default value of 0. This column indicates whether an order priority has been marked for deletion or not. deleteFlag value 0 mark as not deleted, and value 1 mark as deleted.
priorityIndex int This is an integer column with a default value of 0. This column might be used to determine the relative priority of the order priorities, with lower values indicating higher priority.
displayIndex int This is a tinyint column with a default value of 0. This column might be used to determine the order in which the order priorities are displayed in a user interface, with lower values appearing first.
criticalFlag tinyint This is a tinyint column with a default value of 0 and it cannot be NULL. This column might be used to indicate whether the order priority is critical 1 or non-critical 0.
priorityCode varchar This is a string (varchar) column with a maximum length of 50 characters. It's shipped by ecw. This column is used to store a code associated with the order priority.

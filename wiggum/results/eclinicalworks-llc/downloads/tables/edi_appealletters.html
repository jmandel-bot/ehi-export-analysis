EHI Export Schema
Table edi_appealletters
Description This table stores information about appeal letters, including their file details and submission status.
Column DataType Description
id (Primary Key) int This is the primary key for the table, which is an integer that uniquely identifies each record. It is an auto-incrementing value. Example: 1, 2, 3, etc.
appealid int This column stores identifier number of a appeal, for which document is attached. It refers to column id from edi_appealclaims table.
fileName varchar This is a varchar column that stores the name of the file associated with the appeal letter. It can be NULL. Example: "appeal_letter_1.pdf", "appeal_letter_2.docx", etc.
fileType tinyint This column stores 0/1 Indicates that this file has been generated by ECW or Vendor. 0 - Indicates that this file has been generated by ECW 1 - Indicates that this file has been requested and generated by TPS
fileStatus tinyint This column stores value 0/1/2 indicating current file status. 0 - User has requested these files from TPS 1 - The files has been downloaded to the local FTP and ready for preview 2 - The files is selected to be submitted
DocFileType int This column stores type of file or Letter to be uploaded while submitting appeal to vendor. e.g.,(coverletter, appealletter, progressnote etc.). It refers to column id from edi_appealFileType table
isSubmit tinyint This column stores value 0 / 1 indicating if the appeal is submitted to vendor or not. 0 - Not Submitted , 1 -Submitted.

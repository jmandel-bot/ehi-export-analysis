Table rpm_patient_devices_mapping
Description This table is use when provider prescribe any device to patient. So this table will be storing a mapping of the devices allocate to patient or used by patient.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT(11). This column stores integer value which could grow with number of transactions over the years hence integer(Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
patientid int This column stores unique id of a Patient which references the UID column of the user table. The data type chosen is INT. This field will always have a value hence defined as NOT NULL.
device_id int This column stores id of the Device its datatype is int. This field is a foreign key which is refer rpm_supported_devices_list table device_id column. This column stores high range numeric value hence integer is selected and this field could have NULL value. also, it not has default value. This field is a foreign key which is refer rpm_supported_devices_list table device_id column.
rpm_enrollment_id int This column stores enrollment id of rpm_enrollment_details table. Its datatype is int, this field is a foreign key which is refer rpm_enrollment_details table id column. This column stores high range numeric value hence integer is selected and this field will always have a value hence defined as NOT NULL.
tracker_mapping_id int This Column stores tracker mapping id of rpm_supported_devices_list table its datatype is int, this field is a foreign key which is refer rpm_device_tracker_mapping table id column. This column stores high range numeric value hence integer is selected and this field will always have a value hence defined as NOT NULL. This field is a foreign key which is refer rpm_supported_devices_list table id column.
isLinked tinyint This column is to check whether is patient is linked with device or not by value 0 and 1 where 0 indicates not linked and 1 indicate linked. This field will always have a value hence defined as NOT NULL also, it has default value is 0.
status_of_device varchar It will be storing a status of the devices like 'Prescribed','Activated','DeActivated' etc and its datatype is Varchar(12). This column stores characters which cannot exceed 12 characters hence Varchar(12) is selected. This field will always have a value hence defined as NOT NULL also.
order_date date This column stores current date when order completed its datatype is DATE. This column stores date hence Date is selected. This field will always have a value hence defined as NOT NULL. By default '1901-01-01'/ '0000-00-00' for MSSQL/MYSQL accordingly is chosen.
linked_date date This column stores current date when linking completed its datatype is DATE. This column stores date hence Date is selected. This field will always have a value hence defined as NOT NULL. By default '1901-01-01'/ '0000-00-00' for MSSQL/MYSQL accordingly is chosen.
setup_date date This column stores date when setup is completed its datatype is DATE. This column stores date hence Date is selected. This field will always have a value hence defined as NOT NULL. By default '1901-01-01'/ '0000-00-00' for MSSQL/MYSQL accordingly is chosen.
training_completed tinyint This field data type is tinyint and This column stores a value of is training completed or not if training completed it stores 1 otherwise 0. This field will always have a value hence defined as NOT NULL also its default value is 0. 0 - Training Not Completed  1 - Training Completed
completion_date date This column holds a date  when training was completed its datatype is DATE. This column stores date  hence DATE is selected. This field will always have a value hence defined as NOT NULL. By default, '1901-01-01'/ '0000-00-00' for MSSQL/MYSQL accordingly is chosen.
device_provider varchar It will be storing a device provider like 'Patient_Provided','Practice_Provided' etc and its datatype is Varchar(20). This column stores characters which cannot exceed 20 characters hence Varchar(20) is selected. This field could store NULL VALUE.
device_return_date date This column holds a device returning date its datatype is DATETIME. This column stores date hence DATE is selected. There is not any by default value.
device_return tinyint This column holds a value between range 0-1 if device returned successfully then its value is 1 otherwise 0. Its datatype is TINYINT. This field will always have a value hence defined as NOT NULL and its default value is 0. 0 - Device Not Return 1 - Device Return
device_serial_no varchar This Column stores a serial number of the device its datatype is VARCHAR(255). This column stores characters which cannot exceed 255 characters hence varchar(255) is selected. it has default value is NULL.
training_notes varchar This column contains a training gUIDelines or rules its datatype is VARCHAR(MAX). As the column has no limit and we have free text, therefore we have defined it as VARCHAR(MAX) / TEXT. There is no default value for this column.
modifiedBy int This column stores a UID from user table for track the modification done by which user its datatype is int. This column stores high range numeric value hence integer is selected and This field will always have a value hence defined as NOT NULL. also, it not have any default value.
modifiedDatetime datetime This column stores the date time when modification done its datatype is DATETIME. This column stores datetime hence Datetime is selected. This field will always have a value hence defined as NOT NULL.
deleteFlag tinyint Flag indicates if this mapping is deleted or not, datatype is TINYINT. This column stores binary flag i.e 0/1. 0: not deleted 1: deleted - By default column should not be in deleted status hence default 0 is chosen.
deactivationDate datetime This column stores the date when RPM Device is de activated. This column accepts Date values. There is no default value provided for this column. The value in this field can be NULL.

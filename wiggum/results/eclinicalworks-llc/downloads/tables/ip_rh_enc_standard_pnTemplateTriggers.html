Table ip_rh_enc_standard_pnTemplateTriggers
Description Table is used to store progress note section template mapping with patient encounters.
Column DataType Description
id (Primary Key) int This column stores auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer (Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
encounterId int This column is used to identify for which patient encounter user has documented data. This column is FOREIGN KEY which refers to encounterid column of enc table. This column stores a high range numeric value hence integer is selected, and this column will always have a value hence defined as NOT NULL.
triggerTemplateId int This column store template id of template which is used as trigger template id. This column is FOREIGN KEY which refers to id column of ip_rh_clinical_sd_template table. This column stores a high range numeric value hence integer is selected, and this column will always have a value hence defined as NOT NULL.
pnSectionName varchar This column stores progress note section name. We are using datatype as varchar 255. progress note section name will not be more than 255 characters, so we have set its length to 255. This column will always have value hence we keep NOT NULL.
deleteFlag tinyint This will store 0 and 1 value. 0-not deleted and 1-deleted. This column is used to identify whether row is in use or not. We are using TINYINT as datatype as we just want to store 0 or 1 value. By default, row will be in use, so we keep default value is 0.
createdBy int This column stores user id of user. This column is used to identify which user has added this record (module wise trigger template) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store high range of numeric value hence integer is selected. This column will always have a value hence defined as NOT NULL.
createdDateTime datetime This Column stores datetime as we want to store a datetime when the record is added, so we are using DATETIME as data type of column. This column will always have a value hence defined as NOT NULL.
modifiedBy int This column stores user id of user. This column is used to identify which user has last modified this record (module wise trigger template) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store high range of numeric value hence integer is selected, and this column will always not have a value hence defined as NULL DEFAULT NULL.
modifiedDateTime datetime This Column stores datetime as we want to store a datetime when the record is modified, so we are using DATETIME as data type of column. This column will always not have a value hence defined as NULL DEFAULT NULL.

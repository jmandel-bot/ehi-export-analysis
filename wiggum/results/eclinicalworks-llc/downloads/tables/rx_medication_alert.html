EHI Export Schema
Table rx_medication_alert
Description This table stores information about medication alerts for patients, including details about the medication, the user who created the alert, and the status of the alert.
Column DataType Description
id (Primary Key) int Unique identifier for each medication alert record. Example: 1, 2, 3.
itemid int This column stores id of the medication added to the patient's encounter for which alert is set. This column refers column itemId from items table.
encounterid int This column stores id of the appointment which has the medication for which alert is set. This column refers column encounterId from enc table.
rxOrderNo varchar This column stores order number fior a set of medication/drug added to patient's specific encounter. This number is generated on the application side. This column refers column rxOrderNo from oldRxMain table.
NDCCode varchar This column stores the National Drug Code of the added medication/drug which has '-' hyphen in it and is always unique for any drug.
medName varchar This column stores name of the medicine/drug of the id stored in itemId column which is added as part of the treatment and on which alert is set.
userid int This column stores id of the Provider who is treating the Patient for whom the alert is set. This column refers column uid from user table.
encDate varchar This column stores date when the encounter was created which includes the medication for which alert is set. The format used is YYYY-MM-DD.
encTime varchar This column stores time when the encounter was created which includes the medication for which alert is set. The format used is HH-MM-SS.
addedDateTime varchar This column stores the current time stamp when the medication is added as part of the patient's treatment for which alert is created. The format used is MM/DD/YYYY HH:MM:SS.
isAlertActive tinyint This column indicates if the created alert is active or not. 1 - alert is active, 0 - alert is inactive.
patientid int This column stores id of the patient for whom the alert is created. This column refers column uid from user table.
rxComment varchar This column stores one of the below values indicating status of the medication added for the patient on which alert is created. -4 = 'Unknown' , -3 = 'Discontinued', -2 = 'Not-Taking', -1 = 'Taking', 0 = 'Stop', 1 = 'Start', 2 = 'Continue', 3 = 'Increase', 4 = 'Decrease' .
userName varchar This column stores Last and First name of the Provider who has created the medication alert. The username stored is concatenation of ulname and ufname columns from user table.
oldrxid int Currently this column is not in use from This column and hence it stores the default value i.e. 0.
TYPE tinyint Type of medication alert. Example: 0 (normal), 1 (urgent).

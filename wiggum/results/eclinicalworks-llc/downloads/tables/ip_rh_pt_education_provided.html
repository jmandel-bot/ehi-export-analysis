Table ip_rh_pt_education_provided
Description Table is used to store patient education details for patient encounter.
Column DataType Description
id (Primary Key) int This column stores auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer (Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
encounterId int This column is FOREIGN KEY which refers to encounterId column of enc table. This column will store high range of numeric value hence integer is selected. This column will not always have value hence we keep NULL DEFAULT NULL
moduleEncId int This column stores module level encounter id. This column is used to identify for which patient encounter user has documented data. This column is FOREIGN KEY which refers to encounterId column of enc table. This column will store high range of numeric value hence integer is selected. This column will not always have value hence we keep NULL DEFAULT NULL
departmentId int This column stores department id. This column refers to id column of ip_rh_loc_department table. This column will store high range of numeric value hence integer is selected, and this column will always not have a value hence defined as NULL DEFAULT NULL
hwId varchar This column stores hwid which is received from response when user clicks on patient education button and system makes call to external URL to load the document. We have selected varchar(50) as data type because value of this filed will in string format and will not be more than 50 characters. This field not always have value hence defined as DEFAULT NULL
education varchar This column stores document name which is added for patient encounter. We are using datatype as varchar 255.Documnet name will not be more than 255 characters, so we have set its length to 255. This column will not always have value hence we keep NOT NULL DEFAULT ''.
mode varchar This column stores mode of document for which we have requested the document or added the document.Eg. print mode. mode will not be more than 50 characters, so we have set its length to 50. This column will not always have value hence we keep NOT NULL DEFAULT ''.
Localization varchar This column stores localization value which is received from response when user clicks on patient education button and system makes call to external URL to load the document. We have selected varchar (50) as data type because value of this filed will in string format and will not be more than 50 characters. This field not always have value hence defined as DEFAULT NULL.
version varchar This column stores version value which is received from response when user clicks on patient education button and system makes call to external URL to load the document. We have selected varchar (50) as data type because value of this filed will in string format and will not be more than 50 characters. This field not always have value hence defined as DEFAULT NULL.
contentType varchar This column stores content type value which is received from response when user clicks on patient education button and system makes call to external URL to load the document. We have selected varchar (50) as data type because value of this filed will in string format and will not be more than 50 characters. This field not always have value hence defined as DEFAULT NULL.
education_from varchar This column stores document type .Eg. custom form. Document type will not be more than 50 characters, so we have set its length to 50. This column will not always have value hence we keep NOT NULL DEFAULT ''.
comments varchar This column stores comments which is added for document. We are using datatype as varchar 255.Documnet name will not be more than 255 characters, so we have set its length to 255. This column will not always have value hence we keep DEFAULT NULL.
modifiedBy int This column stores user id of user. This column is used to identify which user has last modified this record for patient encounter. This column is FOREIGN KEY which refers to encounterId column of enc table. This column will store high range of numeric value hence integer is selected, and this column will always not have a value hence defined as NULL DEFAULT NULL
releaseDate datetime This column stores release date value which is received from response when user clicks on patient education button and system makes call to external URL to load the document. We have selected datetime as data type because value of this filed will in date format. This field not always have value hence defined as NULL DEFAULT NULL
createdDateTime datetime This Column stores datetime as we want to store a datetime when the document is added, so we are using DATETIME as data type of column. This column will always have a value hence defined as NOT NULL.
modifiedDateTime datetime This Column stores datetime as we want to store a datetime when the document details is modified, so we are using DATETIME as data type of column. This column will always not have a value hence defined as NULL DEFAULT NULL.
deleteFlag tinyint This will store 0 and 1 value. 0-not deleted and 1-deleted. This column is used to identify whether row is in use or not. We are using TINYINT as datatype as we just want to store 0 or 1 value. By default, row will be in use, so we keep NOT NULL DEFAULT 0.
episodeEncId int This column stores episodic level encountered which is used to identify record is added for which episodic encounter. This column is FOREIGN KEY which refers to encounterId column of enc table. This column stores a high range numeric value hence integer is selected, and this column will not always have a value hence defined as DEFAULT NULL.
docId int This column stores document id. This column is FOREIGN KEY which refers to docID column of document table. This column stores a high range numeric value hence integer is selected, and this column will not always have a value hence defined as DEFAULT NULL.
docurl varchar This column store external URL for which we have requested for document. We are using datatype as varchar(2000) because here url length will not be greater than 2000. This column will not always have value hence we keep DEFAULT NULL.

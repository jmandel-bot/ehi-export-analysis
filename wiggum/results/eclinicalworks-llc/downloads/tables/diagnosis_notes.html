Table diagnosis_notes
Description This table stores the diagnosis notes for patients, along with information about the user who created the note and the time stamp of the note creation.
Column DataType Description
id (Primary Key) int Unique identifier for each diagnosis note entry. It is an integer value and serves as the primary key for the table. Example: 1, 2, 3, etc.
NOTES text The actual text content of the diagnosis note. It is stored as a text data type. Example: "Patient shows signs of improvement after medication."
patientid int The unique identifier for the patient associated with the diagnosis note. It is an integer value and serves as a foreign key referring to the `pid` column in the `patients` table. Example: 101, 102, 103, etc.
delflag tinyint A flag to indicate whether the diagnosis note has been deleted or not. It is a tinyint data type with a default value of '0'. A value of '0' means the note is not deleted, and a value of '1' means the note is deleted. Example: 0 or 1.
userid int The unique identifier for the user who created the diagnosis note. It is an integer value and serves as a foreign key referring to the `UID` column in the `users` table. Example: 1001, 1002, 1003, etc.
timestamp varchar The time stamp of when the diagnosis note was created. It is stored as a varchar (40) data type. Example: "2021-08-25 14:30:00".

EHI Export Schema
Table dentalapptresourcedetails
Description This table stores information about dental appointment resource details, including provider time, resource time, chair time, and whether the appointment is for dental services or not.
Column DataType Description
id (Primary Key) int This is the primary key column, which uniquely identifies each record in the table. It is an integer value that is automatically incremented with each new record. Example: 1, 2, 3, etc.
encid int This column stores encounterid value. This column refers to encounterid column in enc table.
providerTime varchar This column stores the time allocated for the provider during the appointment. It is a varchar(250) field that can store up to 250 characters. Example: [{"startValue":0,"endValue":10}]
resourceTime varchar This column stores the time allocated for resources (e.g., equipment, staff) during the appointment. It is a varchar(250) field that can store up to 250 characters. Example: [{"startValue":10,"endValue":20}]
chairTime varchar This column stores the time allocated for the dental chair during the appointment. It is a varchar(250) field that can store up to 250 characters. Example :[{"startValue":20,"endValue":30}]
isDental tinyint This is a tinyint(1) column that indicates whether the appointment is for dental services (1) or not (0). Example: 1 (dental), 0 (non-dental).

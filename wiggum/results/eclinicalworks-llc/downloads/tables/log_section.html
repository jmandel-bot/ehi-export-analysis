Table log_section
Description This table stores information about different log sections, including their names, comments, and related log table data.
Column DataType Description
id (Primary Key) int An integer that serves as the primary key and auto-incrementing identifier for each log section. Example: 1, 2, 3, etc.
sectionName varchar A varchar(128) field that stores the unique name of the log section. Example: "UserActivity", "SystemEvents", etc.
comments varchar A varchar(255) field that stores any additional comments or descriptions about the log section. This field has a default value of an empty string. Example: "Logs user login and logout events", "Stores system error messages", etc.
logTableid int An integer that serves as a foreign key, referencing the `id` column in the `log_table`. This field establishes a relationship between the log section and the log table it belongs to. Example: 1, 2, 3, etc.
isAdminLog tinyint A tinyint(1) field that indicates whether the log section is for admin logs or not. A value of '1' means it is an admin log, and '0' means it is not. Example: 0 (not an admin log), 1 (admin log).
deleteFlag tinyint A tinyint(1) field that indicates whether the log section has been marked for deletion or not. A value of '1' means it is marked for deletion, and '0' means it is not. Example: 0 (not marked for deletion), 1 (marked for deletion).

EHI Export Schema
Table coc_contacts
Description This table stores information about users associated with a circle of care(CoC) assigned to a program for which patient is enrolled. This info is including their type, role, and other relevant details.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
contactid int Unique identifier for the contact. Example: 101, 102, 103.
type varchar Type of contact, represented as a string. Example: 'employee', 'volunteer', 'manager'.
coc_id int Unique identifier for the organization or program the contact is associated with. Example: 201, 202, 203.
displayinrp int Integer value indicating whether the contact should be displayed in the reporting portal (1) or not (0). Example: 0, 1.
codename varchar A short code name for the contact, represented as a string. Example: 'CN1', 'CN2', 'CN3'.
delflag int Integer value indicating whether the record has been deleted (1) or not (0). Example: 0, 1.
codeid int A 3-digit unique identifier for the contact's code. Example: 001, 002, 003.
priority tinyint A tiny integer value representing the priority level of the contact. Example: 0, 1.
role int Unique identifier for the role of the contact within the organization or program. This is a foreign key that refers to the `id` column in the `bh_roles` table. Example: 301, 302, 303.
ptAcoid int Unique identifier for the associated program or organization. This is a foreign key that refers to the `id` column in the `ptacoprograms` table. Example: 401, 402, 403.

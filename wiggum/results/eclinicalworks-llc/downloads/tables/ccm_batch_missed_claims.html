EHI Export Schema
Table ccm_batch_missed_claims
Description This table stores information about missed claims in a batch process, including details about the patient, batch, and error codes.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer(Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index.
patientid int Contains Primary Key column of user table to store Patient id. The value of this field cannot be NULL.
batchid int This column stores unique id of a ccm batch which references the id column of the ccm_batch table. It refers to the batch of specific billing month. The data type chosen is INT. This column does not allow a NULL value.
batchLogid int This column stores unique id of a ccm batch log which references the id column of the ccm_batch_log table. It refers to the batch log of specific billing month. The data type chosen is INT. This column does not allow a NULL value.
ptacoid int Contains Primary Key column of ptacoprograms table to store PTACO id . The datatype chosen is an INT The value of this field cannot be NULL.
claimid int When batch is processed and claim is generated , the claim id is stored in this column. This field can not have a NULL value. The default value of this field is 0. The data type chosen is INT as this field is pointing to id column of edi_invoice table which is also int.
encounterid int When batch is processed and encounter is generated , the encounter id is stored in this column. This field can not have a NULL value. The default value of this field is 0. references to to encounterid column of enc table. The data type chosen is INT as this field is pointing to encounterid column of enc table which is also int.
alertid int This is alert id which references to id column of ccm_alerts table. This field contains alert description in case if there is an alert in claim generation. The data type of this field is INT. This field cannot have a NULL value.
errorCodeid int This is error id which references to id column of ccm_errorcode table. This field contains error description in case if claim generation failed for a patient because of validation or some other issue. The data type of this field is INT. This field cannot have a NULL value.
claimDate datetime Contains date time when the batch is processed. The data type of this field is datetime. This field can have a NULL value.
BillingDate datetime Contains date of specific billing month for which the batch is processed. The data type of this field is datetime. This field can not have a NULL value.
delFlag int Flag indicates if this data is deleted or not, datatype is TINYINT. This column stores binary flag i.e 0/1. 0: not deleted 1: deleted - By default the record must not be in deleted status hence default 0 is chosen.
addedDateTime datetime Contains the date and time when the batch was run by a user for specific billing month. The datatype of this field is datetime. This field does allows a NULL value.
modDateTime datetime Contains the date and time when the batch was last modified by a user for specific billing month. The datatype of this field is datetime. This field allows a NULL value.

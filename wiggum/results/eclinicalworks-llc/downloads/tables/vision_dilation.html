EHI Export Schema
Table vision_dilation
Description This table stores information about vision dilation events during patient encounters, including the time of dilation, drug used, location, amount of the drug, and unit of measurement for the drug.
Column DataType Description
id (Primary Key) int This is the primary key for the table and is an auto-incrementing integer value. It uniquely identifies each vision dilation event. Example: 1, 2, 3, etc.
encounterid int This column stores an integer value that represents the unique identifier for the patient encounter during which the vision dilation event occurred. This column has a foreign key relation with the `enc` table's `encounterid` column. Example: 101, 102, 103, etc.
dilation_time datetime This column stores the time at which the vision dilation event occurred. It is stored as a time value. Example: '14:30:00', '10:15:00', '16:45:00', etc.
drug_used int This is an integer value representing the drug used for the vision dilation event. The specific drug can be determined by referring to an items table `itemid` column. Example: 1, 2, 3, etc.
location int This is an integer value representing the location where the vision dilation event took place. The specific location can be determined by referring to an items table `itemid` column. Example: 1, 2, 3, etc.
drug_amount decimal This column stores the amount of the drug used for the vision dilation event. It is stored as a decimal value with up to 4 digits before the decimal point and 2 digits after the decimal point. Example: 2.50, 1.25, 3.00, etc.
drug_unit varchar This column stores the unit of measurement for the drug used in the vision dilation event. It is stored as a varchar with a maximum length of 5 characters. Examples include: 'mg', 'ml', 'drops', etc.

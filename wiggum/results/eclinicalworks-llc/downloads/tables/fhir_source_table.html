EHI Export Schema
Table fhir_source_table
Description This table stores information about FHIR (Fast Healthcare Interoperability Resources) sources, including the source link, last retrieval time, and related patient information.
Column DataType Description
id (Primary Key) int This is the primary key and an auto-incrementing integer that uniquely identifies each record in the table. Example: 1, 2, 3, etc.
ecwPid int This is an integer column that stores the patient ID from the electronic health record system (EHR). It can be NULL if the patient ID is not available. Example: 101, 102, 103, etc.
lastRetrievalTime datetime This is a datetime column that stores the time stamp, when the data from this FHIR source was retrieved. Example: '2021-09-15 10:30:00', '2021-09-16 14:45:00', etc.
requestId varchar This is varchar column that stores a unique identifier of the transaction through which the data was fetched from this source. There can be multiple source per transaction.
fhirSourceLink varchar This is a varchar(255) column that stores the URL of the FHIR source. Example: 'https://fhir.example.com/patient/123', 'https://fhir.example.com/patient/124', etc.
sourcePid varchar This is a varchar(255) column that stores the patient ID from the FHIR source. Example: 'FHIR123', 'FHIR124', 'FHIR125', etc.
deleteFlag int This is an integer(1) column that stores a flag indicating whether the record has been deleted or not. A value of 0 means the record is not deleted, and a value of 1 means the record is deleted. Example: 0, 1.

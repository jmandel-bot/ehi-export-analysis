EHI Export Schema
Table structwarfarin
Description This table stores structured data related to a patient's Warfarin Flowsheet for an encounter.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the structwarfarin table. Example: 1, 2, 3.
encounterid int identifier for the patient encounter. This is a foreign key that refers to the `encounterid` column in the `enc` table. Example: 101, 102, 103.
catid int identifier for the category of the item. This is a foreign key that refers to the `itemid` column in the `items` table. Example: 1, 2, 3.
itemid int identifier for the item related to warfarin treatment. This is a foreign key that refers to the `itemid` column in the `items` table. Example: 10, 20, 30.
detailid int identifier for the detail of the item. This is a foreign key that refers to the `id` column in the `structdatadetail` table. Example: 100, 200, 300.
value varchar This column stores value of an answer to the structured question documented by end user. This column refers to column name from structdataoptions table.
valueid varchar This column stores the id of an answer selected by the end user while documenting structured questions. This column refers to column optionid from structdataoptions table.
notes varchar Not Available
chartdisplay tinyint A tinyint value indicating whether the data should be displayed on the chart (1) or not (0). Example: 1, 0.
displayIndex tinyint Stores order in which structured questions are listed on the user interface.

Table waitlistlogs
Description This table stores logs of actions performed on a waitlist, such as adding or removing items, along with details about the user, product, and changes made.
Column DataType Description
id (Primary Key) int This is the primary key and an auto-incrementing integer that uniquely identifies each log entry in the table. Example: 1, 2, 3, etc.
eventDate datetime This is a datetime column that stores the date and time when the action was performed. Example: '2021-09-15 14:30:00'.
uname varchar This is a varchar(190) column that stores the username of the user who performed the action. Example: 'john_doe'.
pid int This is an int(11) column that stores the product ID associated with the action. This could be a foreign key referring to the product's ID in another table, such as a `products` table. Example: 101, 102, 103, etc.
pname varchar This is a varchar(190) column that stores the name of the product associated with the action. Example: 'Product A', 'Product B', 'Product C', etc.
action varchar This is a varchar(50) column that stores the type of action performed on the waitlist, such as 'add', 'remove', or 'update'.
modifiedfield varchar This is a varchar(50) column that stores the name of the field that was modified during the action, if applicable. Example: 'quantity', 'price', 'status', etc.
finalvalue varchar This is a text column that stores the final value of the modified field after the action was performed. Example: '10', '25.99', 'In Stock', etc.

Table prisma_practitioner_details
Description This table stores information about practitioners, their roles, and their contact details.
Column DataType Description
ID (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
PRACTITIONER_ID varchar Unique identifier for each practitioner. Example: "PRA123", "PRA124".
PRACTITIONER_ROLE_ID varchar Unique identifier for the role of the practitioner. Example: "ROLE1", "ROLE2".
SECTION_ID bigint Identifier for the section the practitioner belongs to. Foreign key relation to `prisma_ccd_document_section` table's `id` column. Example: 101, 102.
SOURCE_ID int Identifier for the source of the practitioner's information. Example: 1, 2.
FIRST_NAME varchar First name of the practitioner. Example: "John", "Jane".
LAST_NAME varchar Last name of the practitioner. Example: "Doe", "Smith".
MIDDLE_NAME varchar Middle name of the practitioner. Example: "A", "B".
SUFFIX varchar Suffix for the practitioner's name. Example: "Jr", "Sr".
GENDER_CODE varchar Code representing the gender of the practitioner. Example: "M", "F".
GENDER_NAME varchar Full name of the gender of the practitioner. Example: "Male", "Female".
HOME_PHONE varchar Home phone number of the practitioner. Example: "(555) 123-4567", "(555) 987-6543".
WORK_PHONE varchar Work phone number of the practitioner. Example: "(555) 321-7654", "(555) 678-9012".
ORGANIZATION_REFERENCE varchar Reference identifier for the organization the practitioner belongs to. Example: "ORG1", "ORG2".
PRACTITIONER_ROLE_DISPLAY varchar Display name of the practitioner's role. Example: "Doctor", "Nurse".
PRACTITIONER_SPECIALITY varchar Speciality of the practitioner. Example: "Cardiology", "Neurology".

Table bh_transition_phm_to_bhm_status
Description This table stores the transition status and related information for patients moving from one program (PHM) to another program (BHM).
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
patientid int Unique identifier for the patient. Foreign key relation to the `users` table, `UID` column. Example: 101, 102, 103.
phmProgramid int Unique identifier for the PHM program. Foreign key relation to the `acoprograms` table, `id` column. Example: 1001, 1002, 1003.
phmPtAcoid int Unique identifier for the patient's PHM ACO program. Foreign key relation to the `ptacoprograms` table, `id` column. Example: 2001, 2002, 2003.
bhmProgramid int Unique identifier for the BHM program. Foreign key relation to the `acoprograms` table, `id` column. Example: 3001, 3002, 3003.
bhmPtAcoid int Unique identifier for the patient's BHM ACO program. Foreign key relation to the `ptacoprograms` table, `id` column. Example: 4001, 4002, 4003.
transitionStatus varchar Enum column representing the transition status of the patient. Possible values: 'Completed', 'Completed without Reviews', 'Enrolled with failed Reviews', 'Failed', 'Pending'.
transitionResult varchar Text column storing the result of the transition process. Default value is an empty string. Example: 'Success', 'Error: Invalid program'.
transitionBy int Unique identifier for the user who performed the transition. Foreign key relation to the `users` table, `UID` column. Example: 501, 502, 503.
transitionDate datetime Datetime column storing the date and time when the transition occurred. Example: '2022-01-01 12:34:56', '2022-01-02 14:30:00'.

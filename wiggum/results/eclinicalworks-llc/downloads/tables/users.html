EHI Export Schema
Table users
Description This table stores information about system users, including their personal details, contact information, and access settings.
Column DataType Description
UID (Primary Key) int Unique identifier for each user (integer).
uname varchar Username for the user (varchar, up to 60 characters).
upwd varchar Encrypted password for the user (varchar, up to 255 characters).
udesc varchar This column is not being used.
uroomno smallint This column is not being used.
umobileno varchar Mobile phone number of the user (varchar, up to 15 characters).
upagerno varchar Pager number of the user (varchar, up to 15 characters).
ufname varchar First name of the user (varchar, up to 60 characters).
uminitial varchar Middle initial of the user (varchar, up to 60 characters).
ulname varchar Last name of the user (varchar, up to 60 characters).
uemail varchar Email address of the user (varchar, up to 255 characters).
uwebpage varchar Webpage URL of the user (varchar, up to 60 characters).
upaddress varchar Primary address of the user (varchar, up to 60 characters).
upcity varchar City of the user's primary address (varchar, up to 40 characters).
upstate varchar State of the user's primary address (varchar, up to 10 characters).
upPhone varchar Primary phone number of the user (varchar, up to 30 characters).
UserType int User type identifier (integer, default value 2).
zipcode varchar Zip code of the user's primary address (varchar, up to 12 characters).
dob varchar Date of birth of the user (varchar, up to 15 characters).
ssn varchar Social Security Number of the user (varchar, up to 30 characters).
mflag varchar Marital status flag of the user (varchar, up to 10 characters).
updid varchar This column is not being used.
cdate datetime Date and time when the user was created (datetime).
mdate datetime Date and time when the user was last modified (time stamp).
sex varchar Gender of the user (varchar, up to 20 characters).
upaddress2 varchar This column contains the secondary address of the user (varchar, up to 50 characters).
suffix varchar This column contains the suffix of the user's name (varchar, up to 10 characters).
prefix varchar This column contains the refix of the user's name (varchar, up to 10 characters).
initials varchar This column contains the initials of the user (varchar, up to 10 characters).
ptDob datetime This column contains the patient's date of birth.
delFlag int This column contains a deletion flag for the user (integer, default value 0).
vmid varchar This column contains a unique identifier for the user's virtual machine (varchar, up to 60 characters).
status int This column stores flags 0 or 1 to indicate whether the user is marked as active or inactive. A "0" means active, and a "1" means inactive.
webenabled int This column stores flags 0 or 1 to indicate whether a user is web eneabled or not. A "0" means not web enabled, and a "1" means web enabled.
websync int This column stores flags 0 or 1 to indicate whether user's information has been synced to the Patient Portal or not. A "0" means not synced to the Patient Portal, and a "1" means synced to the Patient Portal.
changePass int This column stores flags "0" or "1" to indicate whether users password was reset or not. A "0" means that the password was not reset, while a "1" means the password was reset.
primaryservicelocation int This column contains the primary service location identifier for the user (integer, unsigned).
upreviousname varchar This column contains the previous name of the user (varchar, up to 60 characters).
IsResource int This column contains a resource flag for the user (integer, default value 0).
EligibilityStatus char This column contains the eligibility status of the user (char, 3 characters, default value 'V').
OSUUserType tinyint This column stores flags 0 or 1 to indicate whether the user is student or not. A "0" means not a student, a "1" means student, and a "2" means student employee. This is a practice specific column.
DefApptProvForResource int This column stores the default appointment provider for the resource (integer, default value 0).
locked tinyint This column stores flags 0 or 1 to indicate whether a user has been locked out of their account or not. A "0" means the user is not locked out, a "1" means the user is locked out of the account.
EncEligibilityStatus2 char This column stores one of the below flags when eligibility for secondary insurance is checked for a user. Flags include: E = Eligible, V = Not Verified, N = Ineligible.
EligibilityMessage2 varchar This column stores messages received from the insurance company for the eligibility request sent for the secondary insurance.
EncEligibilityStatus3 char This column stores one of the below flags when eligibility for tertiary insurance is checked for a user. Flags include: E = Eligible V = Not Verified, N = Ineligible.
EligibilityMessage3 varchar This column stores messagse received from the insurance company for the eligibility request sent for the tertiary insurance.
EncEligibilityStatus char This column stores one of the below flags when eligibility for primary insurance is checked at encounter level. Flags include: E = Eligible V = Not Verified, N = Ineligible.
EligibilityMessage varchar This column stores message received from the insurance company for the eligibility request sent for the primary insurance.
AllFacilities tinyint This column stores flags 0 or 1 to indicate whether a user has access to all facilities or not. A"0" means the user does not have access to all facilities, a "1" means a user has access to all facilities.
Emrsex varchar EMR sex of the user (varchar, up to 55 characters).
CountryCode varchar This column stores two character country codes of the user. This column refers to the code column in the countrycodes table.
transgender char This column stores values Yor N to indicate whether the user is transgender or not. A "Y" means the user is transgender,and an "N" means the user is not transgender.
Enterpriseid int This column stores the enterprise id of a user which is used for access control when the Enterprise Directory is enabled.
gestationalAge varchar This column stores a user's gestational age in weeks if a user is pregnant.
timeofbirth datetime This column stores the time of birth of the user.
LANGUAGE varchar This column contains the language preference of the user (varchar, up to 255 characters).
county varchar This column contains the county of the user's primary address (varchar, up to 60 characters).
isCredentialed tinyint This column contains a flag to indicate if the payment is processed successfully. A “1” means success, “2” means failure, “3” means response error from payment vendor, and “4”means empty response from payment vendor.
EncEligibilityStatus4 char This column stores one of the below flags when eligibility for other insurances is checked at the encounter level. Flags include: E = Eligible V = Not Verified, N = Ineligible.
EligibilityMessage4 varchar This column stores messages received from the insurance company for the eligibility request sent for any other insurance (vision, dental).
ACOUserType tinyint This column stores numbers to indicate the type of users for the CCMR module. A "1" means care manager, and a "2" means care coordinator.
credentialschange int This column stores flags 0 or 1. A “1” means the Patient Portal password of a patient is reset from the EMR, a “0” occurs once the reset password syncs to Patient Portal database.
scheduling_placeholder tinyint This column stores a digit for generic providers who are enabled for the rotation feature in the OccHealth module.
JobTitle varchar This column contains the job title of the user (varchar, up to 60 characters).
IsFreshScan char This column stores values Yes or No to indicate whether the scanning of the user's insurance card has been fetched by PM related scheduled job or not. A “Yes” means the insurance details have not been fetched by a scheduled job, and a “No” means the insurance details have been fetched.
empcode varchar This column contains the employee code of the user (varchar, up to 20 characters).
isBMHashed int This column stores flags 0 or 1 to indicate if the password of the user has been hashed using an encrypted algorithm or not. A “0” means not hashed, a “1” means hashed.
displayresourcenameonpn tinyint This displayresourcenameonpn column stores the value of 'Display Resource on Progress Note' check box in the Staff Setup window. Values stored are 1 or 0.
epcsPassword varchar This column stores the EPCS password which will be used for e-prescription instead of ADFS authorization. The datatype of the column would be varchar (255) in order to store a large password as its going to encrypt and decrypt which takes 100 to 200 characters.
newUserFlag tinyint This column contains a new user flag (tinyint, unsigned, default value 1).
RxSuffix varchar This new column stores the suffix and credentials of the provider to be sent within e-prescriptions. There is a maximum of 10 alphanumeric characters allowed to be sent in the"Suffix" tag within e-prescriptions xml as per the NCPPD script standard requirement. Hence, column length of 10 was selected and datatype of varchar.
RxJobTitle varchar This new column stores the ob title for staff to be sent within e-prescriptions. There is a maximum of 10 alphanumeric characters allowed to be sent in the "Suffix" tag within e-prescriptions xml as per the NCPPD script standard requirement. Hence, the column length of 10 was selected and datatype of varchar.
Effectivestartdate datetime This column contains the effective start date for the user.
Effectiveenddate datetime This column contains the effective end date for the user.
countycode varchar This field is used to save the county code. From the Demographics window when the user enters the state and zip code values, and if those values contain only one county associated to them, then the county details will display. Otherwise, the user must click on the county button and choose from a list. After saving the data, when the user hovers over the county button, the system displays the county code and county name.
healowPlusAccess varchar This column contains the healow plus access level for the user (enum, values: 'ehrPlus', 'healowPlus', 'ehr', default value 'ehr').
is_patientsafetyofficer bit This column contains the patient safety officer flag for the user (bit, default value 0).
userAccess varchar This column contains the user access level (enum, values: 'regular', 'emergency', default value 'regular').

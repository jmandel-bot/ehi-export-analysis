EHI Export Schema
Table empioptinconsenttagged
Description This table stores information about consent level elements (public, private, per-event) details, including items, category, and whether the consent has been signed.
Column DataType Description
id (Primary Key) int This is the primary key column, which uniquely identifies each record in the table. It is an integer value, and it cannot be NULL. Examples include: 1, 2, 3, etc.
pid int This column represents the patient id for which event level consent was applied. It refers to user.UID column.
encid int This column contain encounter id for which per event consent applied. This column refers to enc.encounterid column.
encounterDate datetime This column contains the date of an encounter for which consent was required per event.
category varchar This column contains the name of the category events marked during CCR generation and export. Possible values include: Problems, Procedures, Labs, Imaging, and Medications.
item varchar This column contains the corresponding code/name for Problems (ICD), Procedures (CPT), Labs (CPT), Imaging (CPT), and Medications (Name).
startDate datetime This column is not in use.
endDate datetime The end date of the consent period. Example: '2021-12-31', '2022-01-31', '2022-03-31'.
consentLevel char This column contains the consent level value. It will always have 1 as a possible value.
consentSigned tinyint This is the date and time when a user gives consent per event item.
timestamp datetime This column contains the date and time when an event row was inserted and updated.
deleteFlag tinyint This column represents the deletiton status for the row. A "0" means not deleted, and a "1" means deleted.

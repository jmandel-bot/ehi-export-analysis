EHI Export Schema
Table dental_carries_forms
Description This table stores information about dental caries forms for patients, including assessment dates, form data, and related codes.
Column DataType Description
id (Primary Key) int This column stores id value of the dental_carries_forms record. It is auto generated and primary key of This column . This field is kept as integer(10) so it will be consistent with all dental related tables and it is defined as NOT NULL constraint as it is a primary key.
Patientid int This column stores patienid value for patient for whom caries form created. This column refers to pid column in patients table.
Providerid int This column stores providerid value of provider who have created this form. This column refers to UID column in user table.
DeleteFlag tinyint Indicates whether the record has been deleted or not. 0 means not deleted, and 1 means deleted. Example: 0, 1.
AssessmentDate datetime Date when the dental caries assessment was performed. Example: '2022-01-01', '2022-01-02'.
FormData text This column stores JSON string value of caries form other fields that are free text and associated with different types of forms.
DateCreated datetime Date and time when the record was created. Example: '2022-01-01 12:00:00', '2022-01-02 14:30:00'.
isLocked tinyint Indicates whether the record is locked or not. 0 means not locked, and 1 means locked. Example: 0, 1.
lockedByTrUserid int This column stores providerid value of provider who have locked this form. This column refers to UID column in user table.
lockDate datetime Date and time when the record was locked. Example: '2022-01-01 15:00:00', '2022-01-02 16:30:00'.
FormItemid int This column stores form Itemid value of caries form item. This column refers to itemid column in items table.
encounterid int This column stores encounterid value for which this carries form created. This column refers to encounterid column in enc table. This field is kept as integer(11) as it referring column encounterid from enc table. Hence data type, size and constraint is kept similar.
riskvalue tinyint This column stores integer value of selected final risk in caries form.  1- Extreme 2- High 3- Moderate 4- Low.
CptCodeAdded varchar CPT (Current Procedural Terminology) code added to the dental caries form. Example: 'D0120', 'D0150'.
IcdCodeAdded varchar ICD (International Classification of Diseases) code added to the dental caries form. Example: 'K02.3', 'K02.9'.

Table rpm_supported_devices_list
Description This table maintains all the devices from different vendors which can be allocated to RPM patients. Healow supports/tracks different trackers. Different devices are availble to track steps,sleep,BMI,glucose etc. This table maintains a list of all supported devices by healow along with the vendor information. The data structure of this table is designed in reference to the healow tables.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT(11). This column stores integer value which could grow with number of supported devices added over the years hence integer(Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index.
device_id int This column stores a unique device id which is used downstream to identify the device and its information.The datatype chosen is int. This column stores high range numeric value hence integer is selected and this field will always have a value hence defined as NOT NULL.The device_id value is always unique and hence unique constraint UX_rpm_supported_devices_list_device_id is added to eliminate any duplicate device id's
deviceName varchar This column stores the device name which is primarily used for display purposes. The datatype chosen is VARCHAR(255). This column stores characters which cannot exceed 255 characters and hence VARCHAR (255) is selected. This field will always have a value hence defined as NOT NULL .
vendor_id int This column is FOREIGN KEY which refers to id column of rpm_vendors_list table that indicates which device belong to particular vendor.The data type chosen is int. This column stores high range numeric value hence integer is selected and this field will always have a value hence defined as NOT NULL.
device_description varchar This column is for giving description of particular device like what is use of given device or other related information its a VARCHAR(MAX). As the column has no limit and we have free text, therefore we have defined it as VARCHAR(MAX) / TEXT.There is no default value for this column.
deleteFlag tinyint Flag indicates if this device is deleted or not, datatype is TINYINT. This column stores binary flag i.e 0/1. Datatype can be changed to TinyInt. 0: not deleted 1: deleted - By default user must not be in deleted status hence default 0 is chosen.
cellular_device tinyint Flag indicates if this device is cellular or not, datatype is TINYINT. This column stores binary flag i.e 0/1. 0: not cellular 1: cellular.

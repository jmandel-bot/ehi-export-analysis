EHI Export Schema
Table todo_list_rx
Description This table is used to store additional information about medications
Column DataType Description
id int An auto-incremented primary key field
todo_list_id int A foreign key field referencing the 'todo_list' table's 'id' column.
ndc varchar This column stores the National Drug Code (NDC) related to medication.
rxStrength (Primary Key) varchar This column holds the strength data of given drug compendia
rxFormulation varchar This column holds the formulation data of given drug compendia
rxTake varchar This column holds the take data of given drug compendia
rxRoute varchar This column holds the route data of given drug compendia
rxFrequency varchar This column holds the frequency data of given drug compendia
rxDispense varchar This column holds the dispense data of given drug compendia
rxDuration varchar This column holds the duration data of given drug compendia
rxStartDate date This column holds the start date of given drug compendia
rxStopDate date This column holds the stop date of given drug compendia.
rxAdditionalInstruction varchar This column holds the Rx notes data of given drug compendia which gets send to the pharmacist
rxName varchar This column holds the medication name of given drug compendia, 1. MedicationStatement can hold a combination of Medication name, Take, Route, Frequency, Start date, Stop date, and Status 2. MedicationRequest can hold a combination of Medication name, Dispense, Duration, Refill
rxNorm varchar This column holds the RxNorm code of the medication
rxDetailsFromRxNorm varchar This column hold the entire XML of dose selected when match is found from for the FHIR contract sent
rxRefills tinyint This column holds the Rx refill data of given drug compendia
rxItemID int This column holds the item id of the med referenced from 'items' table of 'id'
rxStatus varchar This will hold the rxStatus which has data like active | completed | entered-in-error | intended | stopped | on-hold | unknown | not-taken, it holds a size of varchar 25 and the default value when data is not provided would be NULL
rxMedType varchar This column holds the type of med data, this is used to identify if the incoming med is of type medication statement (reconcile) or medication request (new order) and the values are 'reconcile', 'new order'.
matchingFound tinyint This column hold value 0 or 1, 0 - match does not exist in our drug database, 1 - match does exists in our drug database.

Table ip_rh_ptstagingorders
Description The table 'ip_rh_ptstagingorders' is designed to store order data, in a medical or healthcare setting. It covers comprehensive details about an order, including encounter details, order details, order processing information, and order discontinuation details.
Column DataType Description
stagingOrderId (Primary Key) int This is the primary key for the table, providing a unique identifier for each record. It is an integer data type, auto-incrementing from 1, and cannot be NULL, ensuring a unique and not NULL value for every order.
encounterId int This is an Integer data type that cannot be NULL. This field hold the unique identifier of a patient's encounter during which the order was made. It references the encounterID column in the enc table.
orderItemId int This is an Integer data type, CANNOT be NULL. The default value is 0. This field is to hold the identifier for the item ordered. It references the itemId column in the items table.
orderDateTime datetime This field is a datetime data type and can be NULL. It represents the date and time the order was made.
orderStatus int This is an int data type that can be NULL. This field hold the status of the order. It references the id column in the ip_rh_orderstatus_master table. orderStatus value mark as 1 when place a new order, orderStatus value mark as 2 when signoff order, orderStatus value mark as 3 when modify existing order, orderStatus value mark as 5 when discontinue order, orderStatus value mark as 4 when modified order signoff, orderStatus value mark as 10 when modified order discontinue.
documentedbyid int This is an Integer data type that can be NULL. This field hold the identifier of the person who documented the order. It references the uid column in the users table
documenteddatetime datetime This field is a datetime data type and can be NULL. It represents the date and time the order was documented.
startDateTime datetime This field is a datetime data type and can be NULL. It represents the start date and time of the order.
endDateTime datetime This field is a datetime data type and can be NULL. It represents the end date and time of the order.
orderedById int This is an Integer data type that can be NULL. This field hold the identifier of the person who ordered. It references the uid column in the users table
isRecurringOrder tinyint This is a tinyint, a smaller range integer. It cannot be NULL and indicate whether the order is recurring or not.
orderFrequencyId int This is an Integer data type that can be NULL. This field hold the identifier for the frequency of the order. It references the frequencyId column in the ip_rh_ptstagingorders table.
orderPriority int This is an int data type that cannot be NULL with a default value of 0. This field represents the priority level of the order.
orderInstructions varchar This is a varchar of size 255 and can be NULL. It is to hold any special instructions associated with the order.
orderEntryMethodId int Integer data type can be NULL. This field hold the identifier for the method used to enter the order. It references the Id column in the ip_rh_global_list_items table
orderReason varchar This is a varchar of size 255 and CANNOT be NULL. The default value is ''. It holds the reason for the order.
includeNow tinyint This is a tinyint, a smaller range integer. It can be NULL and indicate whether the order needs to be included immediately or not.
discontinueReasonId int This is an Integer data type that can be NULL. This field hold the identifier for the reason the order was discontinued. It references the Id column in the ip_rh_global_list_items table.
discontinueDateTime datetime This field is a datetime data type and can be NULL. It represents the date and time the order was discontinued.
onHoldReasonId int This is an Integer data type that can be NULL. This field hold the identifier for the reason the order was put on hold.
onHoldDateTime datetime This field is a datetime data type and can be NULL. It represents the date and time the order was put on hold.
deleteFlag tinyint This is a tinyint, a smaller range integer. It cannot be NULL and has a default value of 0. This column indicates whether a patient's staging order has been marked for deletion or not. deleteFlag value 0 mark as not deleted, and value 1 mark as deleted.
episodeEncId int This is an Integer data type that can be NULL. This field hold the identifier for the episode encounter associated with the order. It references the encounterId column in the enc table.
discontinuedById int This is an Integer data type that can be NULL. This field hold the identifier of the person who discontinued the order. It references the uid column in the users table.
discontinueProtocol int This is an Integer data type, CANNOT be NULL with a default value of 0. This field represents the protocol followed to discontinue the order.
treatmentPlanId int Integer data type, CANNOT be NULL with a default value of 0. This field represents the treatment plan associated with the order.
orderNotes text This is a varchar data type and cannot be NULL. It holds any notes or additional information associated with the order.
orderPlacementFlag smallint This is a smallint, a smaller range integer. It CANNOT be NULL and has a default value of 0. This column indicates whether an order has been placed or not.
retroStartDateTime tinyint This is a tinyint, a smaller range integer. It cannot be NULL and has a default value of 0. This column indicates whether a retro start datetime orders are allowed or not. 0 indicates not allowed retro orders and 1 indicates allowed retro orders.

Table rpm_device_tracker_thirdpartydata
Description This table stores information about third-party device trackers used by patients, including review details and relationships with other entities in the system.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer(Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index.
patientid int This column stores id of the Patient and its datatype is int. This column has a foreign key and it refer to user table UID column. This column could be store high range numeric value hence integer is selected and this field will always have a value hence defined as NOT NULL. Also dont have default value because of it is foreign key.
vendor_id int This column stores id of the vendor its datatype is int. This column is a foreign key and it refer to rpm_vendors_list table id column. This column could be store high range numeric value hence integer is selected and this field will always have a value hence defined as NOT NULL. Also dont have default value because of it is foreign key.
tracker_id int This column stores id of the tracker_id its datatype is int. This column is a foreign key and it refer to rpm_healow_tracker_list table id column. This column could be store high range numeric value hence integer is selected and this field will always have a value hence defined as NOT NULL. Also dont have default value because of it is foreign key.
reviewedBy int This column stores id of the userid its datatype is int. This column is a foreign key and it refer to user table UID column. This column could be store high range numeric value hence integer is selected and this field will always have a value hence defined as NOT NULL. Also dont have default value because of it is foreign key.
trackerMappingid int This column stores id of the trackerMappingid its datatype is int. This column is a foreign key and it refer to rpm_Device_tracker_mapping table id column. This column could be store high range numeric value hence integer is selected and this field will always have a value hence defined as NOT NULL. Also dont have default value because of it is foreign key.
modifiedDatetime datetime This column stores the date time when modification done its datatype is DATETIME. This column stores datetime hence Datetime is selected. This field will always have a value hence defined as NOT NULL.
reviewType varchar This column stores reviewType its datatype is VARCHAR(20). This column not support more then 20 characters hence select VARCHAR(20). This field will always have a value hence defined as NOT NULL it not have any default value but it is store a value of enum type so it has some predefine values ('Monthly','Daily') and hence CHECK constraint CHK_rpm_device_tracker_thirdPartyData_reviewType is added.
monthlyReviewDate datetime This column stores the date time when modification done its datatype is DATE. This column stores DATE hence DATE is selected. This field will always have a value hence defined as NOT NULL. By default '1901-01-01'/'0000-00-00' for MSSQL/MYSQL accordingly is chosen.
dailyReviewDate datetime This column stores the date time when modification done its datatype is DATE. This column stores date hence DATE is selected. This field will always have a value hence defined as NOT NULL. By default '1901-01-01'/'0000-00-00' for MSSQL/MYSQL accordingly is chosen.
deleteFlag tinyint Flag indicates if this data is deleted or not, datatype is TINYINT. This column stores binary flag i.e 0/1. 0: not deleted 1: deleted - By default it not-deleted hence default 0 is chosen.

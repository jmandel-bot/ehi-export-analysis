Table ip_rh_ptorders_cosign_mapping
Description The table 'ip_rh_ptorders_cosign_mapping' store information about the mappings of co-signature actions performed on patient orders. Each record is uniquely identified by an 'id' and contains details about the co-signature action, the person who co-signed, co-signature time, and the reason for co-signing.
Column DataType Description
id (Primary Key) int This is the primary key for the table, providing a unique identifier for each record. It is an integer data type, auto incrementing from 1, and cannot be NULL, ensuring a unique and not NULL value for every record. This column is used to uniquely identify each co-signature mapping.
cosignId int This is an Integer data type, cannot be NULL. This field holds the identifier for the co-signature associated with the mapping. It references the Id column in the ip_rh_ptorders_cosign table.
cosignBy int This is an Integer data type, cannot be NULL. This field hold the identifier of the user or entity who co-signed. It references the uId column in the users table.
cosignFlag tinyint This is a tinyint, a smaller range integer, and CANNOT be NULL. The default value is 0. This column indicates whether a co-signature has been made or not.
cosignDateTime datetime This field is a datetime data type and can be NULL. It represents the date and time the co-signature was made.
deleteFlag tinyint This is a tinyint, a smaller range integer. It cannot be NULL and has a default value of 0. This column indicates whether a patient orders co-sign mapping has been marked for deletion or not. deleteFlag value 0 mark as not deleted and value 1 mark as deleted.

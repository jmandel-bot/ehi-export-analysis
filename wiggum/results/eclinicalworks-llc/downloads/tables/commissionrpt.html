EHI Export Schema
Table commissionrpt
Description This table stores information about commission reports for transactions.
Column DataType Description
CommissionRptid (Primary Key) int Unique identifier for each commission report. Example: 1, 2, 3.
trans_id int Unique identifier for the transaction associated with the commission report. Example: 101, 102, 103.
code varchar This column store the 10 character of cpt code which is associated with the commission report i.e. 99201,G8103
dos datetime The date and time of service for the commission report. Example: '2021-09-01 14:30:00', '2021-09-02 10:15:00'.
PmtDate datetime The date and time of payment for the commission report. Example: '2021-09-15 16:45:00', '2021-09-16 11:30:00'.
charge decimal The charge amount for the commission report, with a precision of 9 digits and 2 decimal places. Example: 150.00, 200.50.
paid decimal The paid amount for the commission report, with a precision of 9 digits and 2 decimal places. Example: 100.00, 175.50.
employername varchar The name of the employer associated with the commission report. Example: 'Company A', 'Company B'.
facilityid int Unique identifier for the facility associated with the commission report. Example: 1, 2, 3.
tranx_fromdate datetime The start date of the transaction period for the commission report. Example: '2021-08-01', '2021-08-15'.
tranx_todate datetime The end date of the transaction period for the commission report. Example: '2021-08-31', '2021-09-14'.
createddate datetime The date when the commission report was created. Example: '2021-09-01', '2021-09-02'.
createdby int Unique identifier for the user who created the commission report. Example: 1, 2, 3.
deleteFlag int A flag indicating whether the commission report has been deleted (1) or not (0). Example: 0, 1.
AccountNo varchar A 20-character account number associated with the commission report. Example: '12345678901234567890', '09876543210987654321'.

EHI Export Schema
Table antigen_recipe_queue
Description This table stores information about the antigen recipe queue for patients, including their status, priority, and various flags related to consent, insurance verification, and cancellation.
Column DataType Description
id (Primary Key) int The column stores an id of the antigen_recipe_queue table. This is an auto incremental primary key of This column .
ENCid int This column stores encounterid associated with the recipe queue item.
PATIENTid int This column stores identifier number of a patient associated with the recipe queue. It refers to column UID from user table.
STATUS varchar This column stores status (such as OUTSTANDING, NEWFORMULATION) of the recipe queue. Outstanding - Record came from allergy skin test workflow, NEWFORMULATION - Record created using new formulation button.
UPDATE_DATE datetime This column stores update date for the recipe queue item. The format used is YYYY-MM-DD HH:MM:SS.
ASSIGNED_TO int This column stores id of the assignee to whom recipe queue is assigned.
NOTE text This column stores note of the recipe queue.
IS_INSUR_VERIFIED int This column stores is_insur_verified flag (for insurance verification detail) for the recipe. Possible values 1/0/NULL, 1 - insurance verified, 0/NULL - insurance not verified.
IS_CONSENT_SIGNED int This column stores is_consent_signed flag (for treatment consent) for the recipe. Possible values 1/0/NULL, 1 - consent signed, 0/NULL - consent not signed.
IS_PATIENT_AGREED int This column stores is_patient_agreed flag (agreement for futher treatment) for the recipe. Possible values 1/0/NULL, 1 - patient agreed, 0/NULL - patient not agreed.
IS_HIGH_PRIORITY int This column stores is_high_priority flag for the recipe. Possible values 1/0/NULL, 1 - high priority, 0/NULL - not high priority.
TRANSFER_FLAG varchar This column stores transfer flag value for the recipe. It contains values such as 'O', 'C'.
DELETEFLAG tinyint This column stores delete flag value for the recipe. Possible values 1/0 1 - deleted, 0 - not deleted.
IS_REOPEN int This column stores is_reopen flag for the recipe. Possible values 1/0/NULL, 1 - reopened, 0/NULL - not reopened.
RE_OPEN_REASON varchar This column stores is_reopen reason for the recipe queue.
COPAY varchar This column stores copay amount for the recipe queue.
DEDUCTIBLE varchar This column stores deductible amount for the recipe queue.
BOXNO varchar This column stores box number for the recipe queue.
RECIPE_NOTE text This column stores recipe note for the recipe queue.
REVIEWED int This column stores value of reviewed flag for the recipe queue.
RV_USER varchar This column stores value of reviewed by user for the recipe queue entry.
RV_TIME datetime This column stores time when user reviewed the recipe queue entry.
IS_CONSENT_SIGNED_ONARRIVAL tinyint This column stores is_consent_signed_onarrival flag (for treatment consent) for the recipe. Possible values 1/0/NULL, 1 - consent signed on arrival, 0/NULL - consent not signed on arrival.
FORM_EXP_DATE datetime This column stores expiration date of formulation.
DECLINE_REASON varchar This column stores decline reason given by patient.
CANCEL_REASON varchar This column stores cancel reason for the recipe queue entry.
CANCEL_FLAG int This column stores cancel_flag for the recipe queue entry.
VOid_FLAG int This column stores void flag for the recipe queue entry.
VOid_USERid int This column stores id of the user who has voided the recipe queue entry. It refers to column UID from user table.
VOid_USERNAME varchar This column stores name of the user who has voided the recipe queue entry.
VOid_REASON text This column stores reason for voiding the recipe queue entry.
VOid_timestamp datetime This column stores time stamp when the recipe queue was voided.
CANCEL_USERid int This column stores id of the user who has canceled the recipe queue entry. It refers to column UID from user table.
CANCEL_USERNAME varchar This column stores name of the user who has canceled the recipe queue entry.
CANCEL_timestamp datetime This column stores time stamp when the recipe queue was cancelled.
SHOT_FACid int This column stores facility id chosen by patient for the shots.
SHOT_FACNAME varchar This column stores facility name chosen by patient for the shots.
RESUBMIT_NOTE text This column stores resubmit notes for the recipe queue.
REF_PROVidER_id int This column stores referring provider id associated with the recipe queue.
REF_PROVidER_NAME varchar This column stores referring provider name associated with the recipe queue.
SHOT_FAC_TYPE tinyint This column stores shot facility type (Internal or Referring).

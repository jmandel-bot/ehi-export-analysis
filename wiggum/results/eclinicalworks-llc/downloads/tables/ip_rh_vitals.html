EHI Export Schema
Table ip_rh_vitals
Description Table is used to store vitals details.
Column DataType Description
id (Primary Key) int This column stores auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer (Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
encounterId int This column is FOREIGN KEY which refers to encounterid column of enc table. This column stores a high range numeric value hence integer is selected, and this column will always have a value hence defined as NOT NULL.
vitalID int This column stores vital id of vital which is added for patient encounter. This column is FOREIGN KEY which refers to itemId column of ip_rh_items table. This column stores a high range numeric value hence integer is selected, and this column will always have a value hence defined as NOT NULL.
VALUE varchar This column stores value added by user. We are using datatype as varchar 255. Value will not be more than 255 characters, so we have set its length to 255. This column will always have a value hence defined as NOT NULL and default value is ''.
UpdatedBy int This column stores user id of user. This column is used to identify which user has last updated this record (vital item) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column stores the userid of user who has updated the item.This column will not always have value hence we keep NULL.
unitID int This column stores unit id of vital.This column is FOREIGN KEY which refers to itemId column of ip_rh_items table. This column stores a high range numeric value hence integer is selected, and this column will not always have value hence we keep NULL DEFAULT NULL.
batchID int This column stores batch id of vital.This column is FOREIGN KEY which refers to batchId column of ip_rh_vitals_batch table. This column stores a high range numeric value hence integer is selected, and this column will always have a value hence defined as NOT NULL.
value2 varchar This column stores value being documented by end user for 'vision' vital otherwise this column is left blank for other vitals. We are using datatype as varchar 255. Value will not be more than 255 characters, so we have set its length to 255. This column will always have a value hence defined as NOT NULL and default value is ''.
qualifierSite int This column is used to store user documented site for vital. User can add/edit this value for vital at any time accordingly value get updated here. This column is FOREIGN KEY which refers to id column of ip_rh_vitalsQualifierMaster table. This column stores a high range numeric value hence integer is selected. This column will not always have value hence we keep NULL DEFAULT NULL.
qualifierPosition int This column is used to store user documented position for vital. User can add/edit this value for vital at any time accordingly value get updated here. This column is FOREIGN KEY which refers to id column of ip_rh_vitalsQualifierMaster table. This column stores a high range numeric value hence integer is selected. This column will not always have value hence we keep NULL DEFAULT NULL.
qualifierType int This column is used to store user documented type for vital. User can add/edit this value for vital at any time accordingly value get updated here. This column is FOREIGN KEY which refers to id column of ip_rh_vitalsQualifierMaster table. This column stores a high range numeric value hence integer is selected. This column will not always have value hence we keep NULL DEFAULT NULL.
UpdatedTime datetime This Column stores datetime as we want to store a datetime when the item is modified, so we are using DATETIME as data type of column. This column will always have a value hence defined as NOT NULL.
masterEncounterID int This column stores episodic level encountered which is used to identify record is added for which episodic encounter.This column is FOREIGN KEY which refers to encounterid column of enc table. This column stores a high range numeric value hence integer is selected, and this column will not always have value hence we keep NULL DEFAULT NULL.
fromMoniter tinyint This will store 0 and 1 value. 0-record from monitor and 1-record not from monitor. We are using TINYINT as datatype as we want to store number value is here. By default, record not from monitor, so we keep default value is 0.
deleteFlag tinyint This will store 0 and 1 value. 0-not deleted and 1-deleted. This column is used to identify whether record. We are using TINYINT as datatype as we want to store number value is here. By default, row will be in use, so we keep default value is 0.

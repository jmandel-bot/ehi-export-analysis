EHI Export Schema
Table vision_abnprint
Description This table stores information related to abnormal print orders in the Vision module, including details about the order, CPT codes, exception reasons, and retail prices.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. (e.g., 1, 2, 3)
Encid int Associate with patient's encounter id
Notifier text Text field containing information about the notifier for the abnormal print. (e.g., "System Alert", "Manual Entry")
Pname varchar Defines Patient Name
OrderNo varchar Unique order number for the abnormal print order. (e.g., "ORD123", "ORD456")
CPTDesc1 varchar Description of the first CPT code associated with the abnormal print order. (e.g., "Eye examination", "Visual field test")
CPTDesc2 varchar Description of the second CPT code associated with the abnormal print order, if applicable. (e.g., "Retinal imaging", "Ocular coherence tomography")
CPTDesc3 varchar Description of the third CPT code associated with the abnormal print order, if applicable. (e.g., "Corneal topography", "Fundus photography")
CPTDesc4 varchar Description of the fourth CPT code associated with the abnormal print order, if applicable. (e.g., "Fluorescein angiography", "Optical biometry")
CPTDesc5 varchar Description of the fifth CPT code associated with the abnormal print order, if applicable. (e.g., "Gonioscopy", "Pachymetry")
CPTDesc6 varchar Description of the sixth CPT code associated with the abnormal print order, if applicable. (e.g., "Visual evoked potential", "Electroretinography")
E1Reason varchar Exception reason for the first CPT code. (e.g., "Not covered by insurance", "Requires pre-authorization")
E2Reason varchar Exception reason for the second CPT code, if applicable. (e.g., "Not medically necessary", "Exceeds frequency limit")
E3Reason varchar Exception reason for the third CPT code, if applicable. (e.g., "Duplicate service", "Not a covered benefit")
E4Reason varchar Exception reason for the fourth CPT code, if applicable. (e.g., "Experimental procedure", "Provider not eligible")
E5Reason varchar Exception reason for the fifth CPT code, if applicable. (e.g., "Service not rendered", "Referral required")
E6Reason varchar Exception reason for the sixth CPT code, if applicable. (e.g., "Non-covered service", "Global period")
Retail1 decimal Retail price for the first CPT code. (e.g., 100.00, 150.00)
Retail2 decimal Retail price for the second CPT code, if applicable. (e.g., 200.00, 250.00)
Retail3 decimal Retail price for the third CPT code, if applicable. (e.g., 300.00, 350.00)
Retail4 decimal Retail price for the fourth CPT code, if applicable. (e.g., 400.00, 450.00)
Retail5 decimal Retail price for the fifth CPT code, if applicable. (e.g., 500.00, 550.00)
Retail6 decimal Retail price for the sixth CPT code, if applicable. (e.g., 600.00, 650.00)
CPTFlag varchar Which ever CPT codes present on order that flags need to add comma separated
Fac_Phone varchar Phone number of the facility associated with the abnormal print order. (e.g., "(555) 123-4567", "(555) 987-6543")
CreatedTime datetime Date and time when the record was created. (e.g., "2021-01-01 12:34:56", "2021-02-02 23:45:01")
AddedBy int Unique identifier of the user who added the record. (e.g., 1, 2, 3)
insid int Unique identifier for the insurance associated with the abnormal print order. (e.g., 1001, 1002, 1003)

Table empi_btgreasons_log
Description This table stores the log of reasons why the provider needs to access the patient record.
Column DataType Description
id (Primary Key) int This is the primary key column, which uniquely identifies each record in the table. It is an integer value, and it cannot be NULL. Examples include: 1, 2, 3, etc.
doctorid int This is a foreign key referring to the doctor's unique identifier in another table (e.g., `doctors` table). It is an integer and cannot be NULL. It refers to the doctors.doctorid column. Example: 1001, 1002, 1003, etc.
pid int This column stores the Patient id (pid). It is an integer with a maximum length of 11 digits and has a default value of '0'. It refers to the patients.pid column. Example: 101, 102, 103, etc.
reason varchar A text description of the reason for the doctor-patient relationship. Examples include: "Covering Provider", "Medication Reconciliation" etc.
ctimestamp datetime The date and time when the log entry was created. Example: '2021-09-15 14:30:00', '2021-09-16 10:15:00', etc.
delFlag tinyint A flag indicating whether the log entry has been deleted or not. A "0" means not deleted, and a "1" means deleted. Example: 0, 1.

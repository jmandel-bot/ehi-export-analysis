EHI Export Schema
Table email_disc_report
Description This table stores information about email discrepancy reports in case email ID of the patient has been overwritten
Column DataType Description
id (Primary Key) int Unique identifier for each email discrepancy report. Example: 1, 2, 3.
uid int User identifier associated with the report. This is a foreign key that refers to the `id` column in the `users` table. Example: 101, 102, 103.
pName varchar Name of the person associated with the email discrepancy report. Example: "John Doe", "Jane Smith".
pDob datetime Date of birth of the person associated with the email discrepancy report. Example: "1990-01-01", "1985-12-31".
pPhone varchar Phone number of the person associated with the email discrepancy report. Example: "+1 (123) 456-7890", "+1 (987) 654-3210".
cellphone varchar Cellphone number of the person associated with the email discrepancy report. Example: "1234567890", "9876543210".
pAddress varchar Address of the person associated with the email discrepancy report. Example: "123 Main St, New York, NY 10001", "456 Elm St, Los Angeles, CA 90001".
currentEmail varchar Current email address of the person associated with the email discrepancy report. Example: "john.doe@example.com", "jane.smith@example.com".
userSize int Number of users affected by the email discrepancy. Example: 1, 5, 10.
userList text List of user identifiers affected by the email discrepancy, stored as a text. Example: "101,102,103", "104,105".
previousEmail varchar Previous email address of the person associated with the email discrepancy report. Example: "john.doe.old@example.com", "jane.smith.old@example.com".
copyprevemail tinyint A boolean flag (0 or 1) indicating whether the previous email address should be copied to the current email address. Example: 0 (false), 1 (true).

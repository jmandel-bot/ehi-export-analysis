Table ccmr_ccm_timer
Description This table stores information about time added by staff / provider for a ccm / cpo / rpm enrolled patient including start time, end time, duration, reason, notes, recorded manualy or by timer, user who added time.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT. This column stores integer value which could grow with number of transactions added. It is AUTO_INCREMENT column and has PRIMARY KEY index
PTCO_id int This column is used to identify enrollment details of patient. This is foreign key column. It refers to primary key of ptacoprograms table. Data type is int(11).
Patient_id int This column stores unique id of a Patient which references the UID column of the user table. The data type chosen is INT. This column allows a NULL value.
START_TIMER datetime Contains date time when the timer was started and datatype is DATETIME. This column store datetime hence time stamp is selected.
END_TIMER datetime Contains date time when the timer was ended and datatype is DATETIME. This column store datetime hence time stamp is selected.
DURATION datetime Contains duration for which the timer was running and datatype is DATETIME. This column store duration hence time stamp is selected.
REASON varchar Contains the name of the timer reason and the data type is VARCHAR.Since the length of timer reasons would not exceed 255 characters it is selected as VARCHAR(255).
Notes text Contains the timer notes and the data type is TEXT.
CreatedDateTime datetime Contains date time when the timer record was created and datatype is DATETIME. This column store datetime hence time stamp is selected.
RecordedManually int Flag indicates is time is recorded manually or automaically by timer, datatype is TINYINT. This column stores integer value hence integer is selected. 0:Not manually recorded 1: Manually recorded. By default time is automatically recorded by timer hence default 0 is chosen.
ReasonItemid int Contains the itemid of the timer reason and datatype is INT. It refers the 'itemid' of items table. This field will always have a value hence defined as NOT NULL.
loggedProviderid int Contains the user id who was logged in when record was added or last updated and datatype is INT. It refers the 'UID' of user table. This field will always have a value hence defined as NOT NULL.
deleteFlag int Flag indicates reason has been deleted or not, datatype is TINYINT. This column stores integer value hence integer is selected. Datatype can be changed to tinyint - 0:NotDeleted 1: Deleted. By default timer record must not be in delete status hence default 0 is chosen.
timerstatus int Flag indicates is timer is running for the patient, datatype is TINYINT. This column stores integer value hence integer is selected. 0:Not running 1: Timer is running. By default timer is not automatically running hence default 0 is chosen.
modDateTime datetime Contains date time when the record was last modified and datatype is DATETIME. This column store datetime hence time stamp is selected.
proffessionalTime tinyint Flag indicates is time is added by provider for the patient, datatype is TINYINT. This column stores integer value hence integer is selected. 0:Not added by provider 1: Added by Provider. By default 0 is chosen.
finalBillingModule varchar This column stores the program name incase of CCM RPM concurrent billing. This field indicates that specific timer entry should be allocated to which program. The data type is varchar(50). This column allows NULL value.
billingProcessDone tinyint Flag indicates whether the cpt code decision is done considering this specific timer entry. Values 0 - billing process not done, 1 - billing process done. The data type is TINYINT. This column does not allow NULL value.

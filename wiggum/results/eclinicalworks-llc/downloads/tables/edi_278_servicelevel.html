EHI Export Schema
Table edi_278_servicelevel
Description This table stores information related to EDI (Electronic Data Interchange) 278 service level requests and responses, including details about the request, response, and involved entities.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table.
RequestCategoryCode char This column store 2 charater code that represent the category of request (e.g. 'HS' for encounter and 'SC' for referral)
CertificationTypeCode char A 2-character code representing the type of certification requested (e.g., '01' for initial certification, '02' for recertification).
ServiceTypeCode char A 2-character code representing the type of service requested (e.g., '01' for medical care, '02' for surgical care).
ReleaseOfInformationCode char A 2-character code representing the release of information status (e.g., '01' for full release, '02' for partial release).
procedureCode varchar A code (up to 30 characters) representing the specific procedure requested (e.g., '99213' for an office visit).
ServiceDate datetime The date when the service was provided, in the format 'YYYY-MM-DD'.
encounterid int Unique identifier for the encounter associated with the service level request, with a default value of '0'.
SequenceNo tinyint A 2-digit sequence number for the request.
receiverid int Unique identifier for the entity receiving the request, with a default value of '0'.
Submissionid int Unique identifier for the submission of the request, with a default value of '0'.
insuranceid int Unique identifier for the insurance associated with the request, with a default value of '0'.
responseStatus varchar A status (up to 30 characters) representing the response to the request (e.g., 'accepted', 'rejected').
patientid int Unique identifier for the patient associated with the request, with a default value of '0'.
providerid int Unique identifier for the provider associated with the request, with a default value of '0'.
facilityid int Unique identifier for the facility associated with the request, with a default value of '0'.
RequestStatus varchar A status (up to 30 characters) representing the current status of the request (e.g., 'pending', 'completed').
ST02 int An integer value associated with the ST segment of the EDI 278 transaction.
BHT03 int An integer value associated with the BHT segment of the EDI 278 transaction.
ResponseMessage varchar A message (up to 255 characters) providing additional information about the response (e.g., 'Request accepted', 'Request rejected due to insufficient information').
deleteFlag tinyint A flag (0 or 1) indicating whether the record has been deleted (0 for not deleted, 1 for deleted).
CertificationNumber varchar A number (up to 30 characters) representing the certification associated with the request (e.g., '1234567890').
RejectReason char A 3-character code representing the reason for rejection, if applicable (e.g., '001' for missing information, '002' for invalid data).
Referralid int Unique identifier for the referral associated with the request, with a default value of '0'.
refTo int Unique identifier for the entity the referral is directed to, with a default value of '0'.
FacilityTo int Unique identifier for the facility the referral is directed to, with a default value of '0'.
ServicerEntityType tinyint A 4-digit code representing the type of entity providing the service (e.g., '1' for individual, '2' for organization).
ServicerProviderid int Unique identifier for the provider providing the service. Foreign key relation to `users` table (`UID` column).
ServicerFacilityid int Unique identifier for the facility providing the service. Foreign key relation to `edi_facilities` table (`id` column).
RequesterEntityType tinyint A 4-digit code representing the type of entity requesting the service (e.g., '1' for individual, '2' for organization).
RequesterProviderid int Unique identifier for the provider requesting the service. Foreign key relation to `users` table (`UID` column).
RequesterFacilityid int Unique identifier for the facility requesting the service. Foreign key relation to `edi_facilities` table (`id` column).
RequesterType tinyint A 4-digit code representing the type of requester (e.g., '1' for individual, '2' for organization).

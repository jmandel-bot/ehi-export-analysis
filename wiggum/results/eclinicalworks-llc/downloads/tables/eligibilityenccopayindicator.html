Table eligibilityenccopayindicator
Description This table stores information about the eligibility of encounter copay indicators, including whether multiple copays are applicable for a specific encounter.
Column DataType Description
id (Primary Key) int This is the primary key for the table, which is an auto-incrementing integer value that uniquely identifies each record in the table. Example: 1, 2, 3, etc.
encounterId int This is a foreign key column that refers to the `encounterID` column in the `enc` table. It is an integer value that represents the ID of the encounter associated with the copay indicator. Example: 101, 102, 103, etc.
isMultiCopay tinyint This is a tinyint column that indicates whether multiple copays are applicable for the encounter or not. It is an unsigned value with a default of '0'. A value of '1' means multiple copays are applicable, and '0' means they are not. Example: 0, 1.

EHI Export Schema
Table dentalitems
Description This is a child table of items, containing only those CPTs which are configured as dental CPTs. In other words, an entry in this table identifies the CPT associated as a dental CPT code. The table also associates the CPT with additional properties that do not normally apply to non-dental CPT codes.
Column DataType Description
id (Primary Key) int This column stores the id for the dental CPT or procedure code. It is the auto-increment primary key column for the table.
itemid int This column references the column itemid of the table items for the dental CPT code. The dental CPTs are first created the same way as any other CPT codes with an entry in the table 'items'. They are flagged as dental codes when there is an entry for that itemid in This column .
categoryid int This is a reference to the id column dentalitemcategory table. It represents the category to which the dental CPT belongs.
treatmentAreaid int This is a reference to the id column dentaltreatmentarea table. It represents the treatment area configured for the dental CPT.
paintTypeid int This is a reference to the id column dentalpainttype table. It represents the paint type configured for the dental CPT.
procedureCode varchar This column stores the actual CPT code that corresponds to the itemid for the dental procedure.
providerTime varchar Dental appointments can be created by assigning different time slots for provider, resource and chair time (this is when the patient is on the chair and not necessarily required to have a provider/staff during this time). This column stores the details of the provider time slot in the form of a JSON string.
resourceTime varchar Dental appointments can be created by assigning different time slots for provider, resource and chair time (this is when the patient is on the chair and not necessarily required to have a provider/staff during this time). This column stores the details of the resource time slot in the form of a JSON string.
chairTime varchar Dental appointments can be created by assigning different time slots for provider, resource and chair time (this is when the patient is on the chair and not necessarily required to have a provider/staff during this time). This column stores the details of the chair time slot in the form of a JSON string.
cptDuration int This column stores the default time duration for the dental CPT. The provider, time and chair slots will be split in this time duration. The column stores the time duration as an integer in minutes. Example - a value of 60 in this column means the procedure time is set to 60 minutes.
aliasName varchar Every dental CPT can be given an alias name, which is stored in this column. When user adds that procedure to patient's appointment under "Planned procedures for today's visit", the alias name along with the certain other information will be shown on the Appointment slot. The appointment slot has limited space on the UI, which is why the alias name is limited to 7 characters. It helps identify the type of procedure using a shortened notation.

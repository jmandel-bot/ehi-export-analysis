EHI Export Schema
Table vision_refund_updated
Description This table stores information about updated vision refunds, including the associated claim and payment details.
Column DataType Description
id (Primary Key) int This is the primary key for the table, and it is an auto-incrementing integer. It uniquely identifies each record in the table. Example: 1, 2, 3, etc.
visionRefundId int This is an integer that stores the ID of the vision refund that has been updated. It can be used as a foreign key to reference the `id` column in the `vision_refund_processing` table. Example: 101, 102, 103, etc.
updatedClaimId int This is an integer that stores the updated claim ID associated with the vision refund. It has a default value of '0'. Example: 201, 202, 203, etc.
createdRefundId int This is an integer that stores the ID of the created refund associated with the updated vision refund. It can be used as a foreign key to reference the `id` column in the `refund` table. Example: 301, 302, 303, etc.
updatedPaymentId int This is an integer that stores the updated payment ID associated with the vision refund. It can be used as a foreign key to reference the `paymentId` column in the `edi_paymentDetail` table. Example: 401, 402, 403, etc.
deleteFlag int This is an integer with a default value of '0'. It is used to indicate whether the record has been deleted or not. A value of '0' means the record is not deleted, and a value of '1' means the record is deleted. Example: 0, 1.

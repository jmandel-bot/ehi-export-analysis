EHI Export Schema
Table edi_pos
Description This table stores information about Electronic Data Interchange (EDI) position codes, including their names and descriptions.
Column DataType Description
Id (Primary Key) int This is the primary key column, which uniquely identifies each record in the table. It is an integer with a maximum length of 11 digits and is automatically incremented when a new record is added. Example: 1, 2, 3, etc.
Code int This column stores the unique place of service code, which is an integer with a maximum length of 11 digits. It has a default value of '0'. Example: 1001, 1002, 1003, etc.
Name varchar This column stores the name of the place of service code. It is a variable character string with a maximum length of 60 characters and has a default value of an empty string. Example: "Home", "Office", "INPATIENT HOSPITAL", etc.
Description varchar This column stores a description of the place of service code. It is a variable character string with a maximum length of 255 characters and has a default value of an empty string. Example: "This is the description for EDI Position 1", "This is the description for EDI Position 2", "This is the description for EDI Position 3", etc.

EHI Export Schema
Table ip_rh_ptorders
Description The table 'ip_rh_ptorders' stores information about patient orders.
Column DataType Description
orderId (Primary Key) int This is auto incremented the primary key for the table.
stagingOrderId int This is an Integer data type, CANNOT be NULL. This field holds the identifier for the staging order associated with the patient order. It references the stagingOrderId column in the ip_rh_ptstagingorders_action_map table.
encounterId int This is an Integer data type, can be NULL. This field hold the unique identifier of a patient's encounter during which the order was made. It references the encounterId column in the enc table.
orderItemId int This is an Integer data type, CANNOT be NULL. This field holds the identifier for the item ordered. It references the itemId column in the items table.
orderDateTime datetime This field is a datetime data type and CAN be NULL. It represents the date and time the order was made.
orderStatus int This is an int data type, CANNOT be NULL with a default value of 0. This field represents the status of the order. orderStatus value mark as 1 when place a new order, orderStatus value mark as 2 when signoff order, orderStatus value mark as 3 when modify existing order, orderStatus value mark as 5 when discontinue order,orderStatus value mark as 4 when modified order signoff, orderStatus value mark as 10 when modified order discontinue.
documentedbyid int This is an Integer data type, can be NULL. This field hold the identifier of the person who documented the order. It references the uid column in the users table.
documenteddatetime datetime This field is a datetime data type and CAN be NULL. It represents the date and time the order was documented.
startDateTime datetime This field is a datetime data type and can be NULL. It represents the start date and time of the order.
endDateTime datetime This field is a datetime data type and can be NULL. It represents the end date and time of the order.
orderedById int This is an Integer data type, can be NULL. This field hold the identifier of the person who ordered. It references the uid column in the users table.
isRecurringOrder tinyint This is a tinyint that stores 0/1. This column indicates whether the order is recurring or not.
orderFrequencyId int This is an Integer data type, cannot be NULL. This field hold the identifier for the frequency of the order. It references the orderFrequencyId column in the ip_rh_frequency_list table.
orderPriority tinyint This is a int, a smaller range integer. It cannot be NULL and represent the priority of the order.
orderInstructions varchar This is a varchar of size 255 and can be NULL. It holds any special instructions associated with the order.
orderEntryMethodId int This is an Integer data type, can be NULL. This field hold the identifier for the method used to enter the order. It references the Id column in the ip_rh_global_list_items table.
discontinueReasonId int This is an Integer data type, can be NULL. This field hold the identifier for the reason the order was discontinued. It references the Id column in the ip_rh_global_list_items table.
discontinueDateTime datetime This field is a datetime data type and can be NULL. It represents the date and time the order was discontinued.
onHoldReasonId int This is an Integer data type, can be NULL. This field hold the identifier for the reason the order was put on hold.
onHoldDateTime datetime This field is a datetime data type and can be NULL. It represents the date and time the order was put on hold.
orderReason varchar This is a varchar of size 255 and CANNOT be NULL. The default value is ''. It holds the reason for the order.
deleteFlag tinyint This is a tinyint, a smaller range integer. It cannot be NULL and has a default value of 0. This column indicates whether a patient's order has been marked for deletion or not. deleteFlag value 0 mark as not deleted and value 1 mark as deleted.
episodeEncId int This is an Integer data type, cannot be NULL. This field hold the identifier for the episode encounter associated with the order. This FK that stores encounter column in enc table.
discontinuedById int This is an Integer data type, can be NULL. This field hold the identifier of the person who discontinued the order. It references the uId column in the users table.
treatmentPlanId int This is an Integer data type, CANNOT be NULL with a default value of 0. This field represents the treatment plan associated with the order.
lastGeneratedScheduleDate datetime This field is a datetime data type and can be NULL. It represents the last date a schedule was generated for the order.
orderNotes text This is a varchar data type and cannot be NULL and default is empty. It holds any notes or additional information associated with the order.
includeNow tinyint This is a tinyint, a smaller range integer. It cannot be NULL with a default valule 0 and indicate whether the order needs to be included immediately or not.
retroStartDateTime tinyint This is a tinyint, a smaller range integer. It cannot be NULL and has a default value of 0. This column indicates whether a retro start datetime orders are allowed or not. 0 indicates not allowed retro orders and 1 indicates allowed retro orders.

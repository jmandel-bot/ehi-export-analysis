Table ip_rh_chiefcomplaints
Description Table is used to store chief complaint details for patient encounter.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer (Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
encounterID int This column is used to identify for which encounter what chief complaint item is documented. This column is used to map chief compliant items with patient encounter. This column is FOREIGN KEY which refers to encounterid column of enc table. This column stores a high range numeric value hence integer is selected, and this column will always have a value hence defined as NOT NULL.
reason varchar This column stores chief complaint item added or selected by user. We are using datatype as VARCHAR (255). item will not be more than 255 characters, so we have set their length to 255. This column will always not have a value hence defined as NULL DEFAULT NULL.
reasonItemId int This column stores item id of chief complaint added for patient encounter. This column is FOREIGN KEY which refers to itemId column of items table. This column stores a high range numeric value hence integer is selected, and this column will not always have a value hence defined as NULL DEFAULT NULL.
displayIndex int This column stores integer value hence integer is selected as datatype of column. This column is used to maintain the order of items added for patient encounter. This column will always have a value hence defined as NOT NULL and default value is set to 1.
deleteFlag tinyint This will store 0 and 1 value. 0-not deleted and 1-deleted. This column is used to identify whether row is in use or not. We are using TINYINT as datatype as we just want to store 0 or 1 value. This column will always have a value hence defined as NOT NULL. By default, row will be in use, so we keep default value is 0.
createdBy int This column stores user id of user. This column is used to identify which user has added record (chief complaint) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store high range of numeric value hence integer is selected. This column will always have a value hence defined as NOT NULL.
createdDateTime datetime This Column stores datetime as we want to store a datetime when the item is added, so we are using DATETIME as data type of column. This column will always have a value hence defined as NOT NULL.
modifiedBy int This column stores user id of user. This column is used to identify which user has last modified record (chief complaint) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store a high range of numeric value hence integer is selected, and this column will always not have a value hence defined as NULL DEFAULT NULL.
modifiedDateTime datetime This Column stores datetime as we want to store a datetime when the record is modified, so we are using DATETIME as data type of column. This column will always not have a value hence defined as NULL DEFAULT NULL.

EHI Export Schema
Table edi_appealclaims
Description This table stores information about appeal claims related to invoices and insurance.
Column DataType Description
id (Primary Key) int This column is auto-incremented primary key in This column . Unique id is assigned to claim moved to appeal after insurance/clearinghouse denial for payment.
invoiceid int This column stores identifier number of a invoice, for which appeal is created. It refers to column id from edi_invoice table.
insid int This column stores identifier number of a Insurance, against which claim is appealed. It refers to column insid from insurance table.
appealStatus int This column stores identifier number of a appeal status code, appeal is in. It refers to column id from appealstatuscodes table.
createdDate datetime Date when the appeal claim record was created. Example: '2021-09-01', '2021-09-02', '2021-09-03', etc.
submittedDate datetime Date when the appeal claim was submitted, can be NULL if not submitted yet. Example: '2021-09-10', '2021-09-11', NULL, etc.
deleteFlag tinyint This column stores value 0 / 1 indicating if the appeal is deleted or not, this is soft delete. 0 - Not Deleted , 1 -Deleted.
Eobattached tinyint This column stores value 0 / 1 indicating if the EOB file is attached or not. 0 - Not attached , 1 - file is attached.
Patienteligibility tinyint This column stores value 0 / 1 indicating if the patient is eligible for appeal or not. 0 - no , 1 - yes.
Progressnotes tinyint This column stores value 0 / 1 indicating if the progress note is attached or not. 0 - Not attached , 1 - file is attached.
customNote varchar A custom note related to the appeal claim, stored as a string with a maximum length of 512 characters. Example: 'Patient has a special condition', 'Additional documents required', '', etc.

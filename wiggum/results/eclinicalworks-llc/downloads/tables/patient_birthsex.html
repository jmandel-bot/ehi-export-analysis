EHI Export Schema
Table patient_birthsex
Description This table stores a patient's sex at birth for the SOGI feature.
Column DataType Description
id (Primary Key) int This is auto-incremented primary key in This column .
patientid int This column stores id of a patient whose birth sex is being documented by end user. This column refers to UID column in user table.
modified_by int This column stores id of a user who added/modified patient's birth sex information. This column refers to UID column in user table.
birthsex varchar This column stores a letter M/F/UNK for patient's birth sex. M - Male, F - Female and UNK - Unknown
isactive tinyint This column stores values 0/1 to indicate whether current record of patient's birth sex is active or not. 0 - Record is Inactive, 1 - Record is Active.
deleteflag tinyint This column stores values 0/1 to indicate whether the patient's birth sex record is marked as deleted or not. 0 - Record in NOT deleted, 1 - Record is deleted.
added_on datetime This column stores date and time when patient's birth sex record was documented by end user. Date and time is stored in 'YYYY-MM-DD hh:mm:ss.sss' format.
inactive_date datetime This column stores date and time when patient's birth sex record was updated to a new value. Date and time is stored in 'YYYY-MM-DD hh:mm:ss.sss' format.
action varchar This column stores a string to indicate action performed by end user on patient's birth sex record. 'added' - When end user add new record or update an old record new record is inserted with this action, 'deleted' - When end user update existing record, old record is marked as action 'deleted'.
modality varchar This column stores string to indicate from which eCW product patient's birth sex record was updated/added. 'WEB' - If record was added/updated from eCW Web Application, 'EXE' - If record was added/updated from eCW EXE Application, 'KIOSK' - If record was added/updated from eCW KIOSK Application, 'INTERFACE' - If record was added/updated from interface

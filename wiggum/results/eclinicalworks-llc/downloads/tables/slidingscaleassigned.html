Table slidingscaleassigned
Description This table stores information about sliding scale assignments for patients based on their income, dependants, and other factors.
Column DataType Description
id (Primary Key) int Unique identifier for each sliding scale assignment record. (e.g., 1, 2, 3)
patientid int Unique identifier for the patient associated with the sliding scale assignment refers foreign key on column pid of table patient. (e.g., 101, 102, 103)
Income decimal The patient's income in a double format with two decimal places. (e.g., 45000.00, 55000.00, 65000.00)
Dependants tinyint The number of dependants the patient has between 1 to 99.
PovertyLevel decimal The poverty level percentage for the patient in a double format with two decimal places. (e.g., 100.00, 150.00, 200.00)
AssignedType char A 3-character code representing the type of sliding scale assignment. (e.g., 'A', 'B', 'C')
AssignedDate datetime The date when the sliding scale assignment was made. (e.g., '2021-01-01', '2021-02-01', '2021-03-01')
ExpiryDate datetime The date when the sliding scale assignment expires. (e.g., '2022-01-01', '2022-02-01', '2022-03-01')
DocProof tinyint A flag indicating whether the patient provided documentation proof for their income. (e.g., 0 for No, 1 for Yes)
FeeSchid int Unique identifier for the fee schedule associated with the sliding scale assignment. (e.g., 1, 2, 3)
FeeSchedule varchar A varchar(50) field representing the name of the fee schedule. (e.g., 'Schedule A', 'Schedule B', 'Schedule C')
deleteFlag tinyint A flag indicating whether the sliding scale assignment record has been deleted. (e.g., 0 for Not Deleted, 1 for Deleted)
Unit varchar A varchar(15) field representing the unit of measurement for the sliding scale assignment. (e.g., 'Percentage', 'Fixed')
MedicalDiscount decimal The medical discount amount in a double format with two decimal places. (e.g., 10.00, 20.00, 30.00)
DentalDiscount decimal The dental discount amount in a double format with two decimal places. (e.g., 5.00, 10.00, 15.00)
Expired tinyint A flag indicating whether the sliding scale assignment has expired. (e.g., 0 for Not Expired, 1 for Expired)
CopayDiscount decimal The copay discount amount in a double format with two decimal places. (e.g., 2.00, 4.00, 6.00)
CopayDiscountType tinyint A tinyint(1) field representing the type of copay discount. (e.g., 0 for Percentage, 1 for Fixed)
IncomeDetailid int Unique identifier for the income detail record associated with the sliding scale assignment refers foreign key on column id of table slidingscaleptincomedetails. (e.g., 1, 2, 3)
NonProofOfIncome tinyint A flag indicating whether the patient has not provided proof of income. (e.g., 0 for No, 1 for Yes)
userExpiredDate datetime The date and time when the user marked the sliding scale assignment as expired. (e.g., '2021-12-01 12:34:56', '2021-12-02 14:56:32')
userid int Unique identifier for the user who marked the sliding scale assignment as expired. (e.g., 1, 2, 3)

EHI Export Schema
Table test_followup
Description This table stores information about follow-up actions for patients, including the status of the follow-up, due dates, and alerts.
Column DataType Description
id (Primary Key) int Unique identifier for each follow-up record. Example: 1, 2, 3.
patientid int identifier for the patient associated with the follow-up. This could be a foreign key referring to the `id` column in a `patients` table. Example: 101, 102, 103.
itemid varchar This column stores an id of the item selected which is used to render the patient's test followup. This column refers column itemid from items table.
followupDate datetime Date when the follow-up is scheduled. Example: '2022-01-01', '2022-01-15', '2022-02-01'.
DueDate varchar Due date for the follow-up action, stored as a string. Example: '2022-01-10', '2022-01-20', '2022-02-05'.
STATUS varchar Status of the follow-up action, stored as a string. Example: "Pending", "Completed", "Cancelled".
donedate datetime Date when the follow-up action was completed. If not completed, it has a default value of '0000-00-00'. Example: '2022-01-10', '0000-00-00', '2022-02-05'.
alertid int identifier for the alert associated with the follow-up. This could be a foreign key referring to the `id` column in an `alerts` table. Example: 201, 202, 203.
alertType tinyint This column stores one of the below values indicating type of the alert. Values: BillingAlert, Immunizations, Labs, Other, Procedures, XRays.

EHI Export Schema
Table report_writer_template
Description This table stores information about report writer templates, including their names, categories, access settings, and related user information.
Column DataType Description
templateid (Primary Key) int Unique identifier for each report writer template. Example: 1, 2, 3.
templateName varchar This column is used to store the name of the template. This column is mandatory in UI. Hence, not NULL constraint is suitable here. As per requirement, user can add maximum 100 characters. Hence, varchar(100) is suitable here.
categoryid int This column is used to store template category. The default value of this column is 1 which is for DI. This column should have associated category. Hence, not NULL constraint is suitable here. All the template should be by default associated with DI category. Hence, added 1 as default value. This column can have value as 0, 1, 2. Hence, TINYINT is suitable here.
publicAccess tinyint This column is used to store access of template. Default value of this column is 0 which is private if the column value is 1. This column should have 0 or 1 value. Hence, not NULL constraint is suitable here. As per requirement, all the templates should be private by default. Hence, added 0 as default value. This column can have 0 or 1 value. Hence, TINYINT is suitable here.
defaultTemplateType tinyint This column is used to store type of template. Default value of this column is 0 which means its for all DI orders. This column should have 0 or 1 value. Hence, not NULL constraint is suitable here. By default all template should be associated with all DI orders. Hence, added 0 as default value. 0: Template is for all DI orders  1: Template is for selected DI Category  2: Template is for selected DI This column can have value as 0, 1 or 2. Hence, TINYINT is suitable here.
defaultCategoryid int This column is used to store categoryid. This column is a foreign key that references to labcategories (id). The default value of this column is NULL. It's storing category id of labcategories table when defaultTemplateType is 1. DefaultCategoryid will have data only when defaultTemplateType is 1. Hence, added this column type as NULL.
defaultDIid int This column is used to store DI id. The default value of this column is NULL because it will have data only when defaultTemplateType is 2. This column stores itemid of item table when defaultTemplateType is 2. This column is a foreign key that references to items (itemid). Itemid column of items is INT type. Hence, INT datatype is suitable here
userid int This column is used to store the userid who created template. Template should have created the user id. Hence, not NULL constraint is suitable here. This column is a foreign key that references to users (UID).
deleteFlag tinyint This column is used for soft deleting records, the value should be 1 or 0. This column should have 0 or 1 value. Hence, NOT NULL constraint is suitable here. 1 : Deleted  0 : Not Deleted By default, all template should not be deleted. Hence, added 0 as default value. This column can have 0 or 1 value. Hence, TINYINT is suitable here.
status tinyint This column is used to store the status of template. This column should have 0 or 1 value. Hence, not NULL constraint is suitable here. 0: Template is active. 1: Template is Inactive. By default all template should be active. Hence, added 0 as default value. This column can have 0 or 1 value. Hence, TINYINT is suitable here.

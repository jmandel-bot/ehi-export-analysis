Table cpocertification
Description This table stores information about patient certification in care plan oversight program. It tores information including provider selected, agency selected, Home Health or Hospice service type, plan of care document signed, initial certification or re certification, claim Id, encounterId.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of over the years hence integer(Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
ptacoid int Contains Primary Key column of ptacoprograms table to store PTACO id . The datatype chosen is an INT The value of this field cannot be NULL. This field references id column of ptacoprograms table.
patientid int Contains Primary Key column of user table to store Patient id. The value of this field cannot be NULL. This field references UID column of user table.
providerid int This column stores the Provider id who is going to provide care to the patient Contains the Primary Key column id of user table. It stores the unique id of a Provider. The value of this field can not be NULL. This field references to UID column of user table. The data type chosen is INT as this field is pointing to UID column of user table which is also int.
facilityid int Contains Primary Key column of TCM_Hospitals table to store facility id. The data type of this field is INT. The value of this field cannot be NULL.
serviceType tinyint Contains type of service provided to the patient. Possible values - 1 - Home Health, 2 - Hospice. The data type is TINYINT and this field cannot have NULL values.
planOfCareSignedOff tinyint Contains a flag if plan of care document is signed by the provider or not. Values - 1 - Plan of care signed , 0 - Plan of care not signd. The data type is TINYINT.
certificationType tinyint Contains type of type of certification for the patient. Possible values - 1 - Initial Certification, 2 - Re Certification. The data type is TINYINT and this field cannot have NULL values.
certificationCompleted tinyint Contains a flag if the certification is completed or not. Values - 1 - Certification completed, 0 - Certification not completed. The data type is TINYINT.
startCareDate datetime Contains date time when the care is started. The data type is Datetime and the value cannot be NULL.
certificationStartDate datetime Contains date time when the certification is started. The data type is Datetime and the value cannot be NULL.
certificationEndDate datetime Contains date time when the certification is ended. The data type is Datetime and the value cannot be NULL.
delflag tinyint Flag indicates if this data is deleted or not, datatype is TINYINT. This column stores binary flag i.e 0/1. 0: not deleted 1: deleted - By default the record must not be in deleted status hence default 0 is chosen.
notes text Contains textual notes added by the user while documenting certification. The data type is text and this can have NULL value.
certEnded tinyint Contains a flag if the certification is discontinued or not. Values - 1 - Certification ended, 0 - Certification not ended. The data type is TINYINT.
certEndedBy int This column is used to store user id to identify which user ended the certification. It refers to primary key UID of user table. Data type is int(11).
certEndedCreateClaim tinyint Contains a flag if claim should be created after disenrolling the patient or not. Values - 1 - Create claim, 0 - do not create claim. The data type is TINYINT.
certEndedDate datetime Contains date time when the certification is ended manually by the user. The data type is Datetime and the value cannot be NULL.
certEndedNotes text Contains textual notes added by the user while ending a certification. The data type is text and this can have NULL value.
episodeid int Contains unique id of cpo certification which is the first certification of the series in case of multiple cpo certifications of a patient.
nextCertid int Contains unique id of cpo certification which is the next certification in case of multiple cpo certifications of a patient.
Claimid int When batch is processed and claim is generated , the claim id is stored in this column. This field can not have a NULL value. The default value of this field is 0. The data type chosen is INT as this field is pointing to id column of edi_invoice table which is also int.
Encounterid int When batch is processed and encounter is generated , the encounter id is stored in this column. This field can not have a NULL value. The default value of this field is 0. references to to encounterid column of enc table. The data type chosen is INT as this field is pointing to encounterid column of enc table which is also int.
ClaimDate datetime Contains date time when the batch is processed. The data type of this field is datetime. This field can have a NULL value.
claimFailureReason text Contains error description in case of claim failure. The data type is text and it can have a NULL value.
attachmentid int Contains unique id of document table to store the plan of care document attached by the user. The data typs is INT and it can have a NULL value.
addedDateTime datetime Contains the date and time when the record was first added by a user. The datatype of this field is datetime. This field does allows a NULL value.
modDateTime datetime Contains the date and time when the record was last modified by a user. The datatype of this field is datetime. This field does allows a NULL value.

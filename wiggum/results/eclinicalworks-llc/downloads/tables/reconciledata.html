EHI Export Schema
Table reconciledata
Description This table stores information about reconciled data related to patients, including file details and reconciliation dates.
Column DataType Description
ReconcileId (Primary Key) int This is the primary key for the table and is an integer that uniquely identifies each record in the table. It is auto-incremented, so each new record will have a unique value. Example: 1, 2, 3, etc.
PatientId int This is a foreign key that refers to the `Patient` table's primary key (`PatientId`). It is an integer that represents the unique identifier of the patient associated with the reconciled data. Example: 101, 102, 103, etc.
FileName varchar This is a varchar column that stores the name of the file associated with the reconciled data. Example: "patient_data_101.csv", "patient_data_102.csv", etc.
InterfaceName varchar This is a varchar column that stores the name of the interface used for the reconciliation process. Example: "Interface1", "Interface2", etc.
Description text This is a text column that stores a detailed description of the reconciled data. Example: "Reconciled patient data from Hospital A and Hospital B."
ReconciledDate datetime This is a date column that stores the date when the data was reconciled. Example: "2021-08-01", "2021-08-02", etc.
FilePath varchar This is a varchar column that stores the file path where the reconciled data file is stored. Example: "/reconciled_data/patient_data_101.csv", "/reconciled_data/patient_data_102.csv", etc.
deleteflag int This is an integer column that indicates whether the record has been deleted or not. A value of 1 means the record is deleted, and a value of 0 means the record is not deleted. Example: 0, 1.

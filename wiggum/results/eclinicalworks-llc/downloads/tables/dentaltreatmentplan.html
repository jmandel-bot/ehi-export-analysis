EHI Export Schema
Table dentaltreatmentplan
Description This table stores information about dental treatment plans for patients, including details about the plan, its status, and any associated signatures.
Column DataType Description
TreatplanNum (Primary Key) int • This column will store the treatplannum of the dental treatment plan• The datatype of the column would be INT to accommodate many treatment plan data that can be added. • This would be unique and auto incremented and so PRIMARYKEY constraint is added
patNum int • This column stores the id of the patient. • This is a reference to the column pid of table patients.
dateTP datetime • This column stores the dental treatment plan start date selected on UI. • The datatype of the column would be datetime as it is storing date.
heading varchar • This column stores treatment plan heading entered on UI. • The datatype of the column would be varchar as it is storing alphanumeric characters.
notes varchar This column stores notes entered on UI. • The datatype of the column would be varchar as it is storing alphanumeric characters.
signature tinyint • This column is used to determine whether patient signature is done for this treatment plan or not. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value means the signature is not done for this treatment plan. • 1 value means the signature is done for this treatment plan.
deleteFlag tinyint • This column determines whether saved treatment plan record is deleted or not. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value means the treatment plan record is not deleted. • 1 value means the treatment plan record is deleted.
signatureJSON varchar • This column is used to store signature JSON/base54 encoded string value of patient signature image done for this treatment plan. • The datatype of the column would be text as it can have larger value for signature image.
loginUserid int • This column stores userid of the user who has done created this treatment plan. • This is a reference to the column UID of table users.
dateSigned datetime • This column is used to store the date when signature is done for the treatment plan. • The datatype of the column would be datetime as it is storing date.
signType tinyint • This column stores the sign type of signature. • The datatype of the column would be tinyint as it can have 0, 1or 2 value. • 0 value means the signature is done using mouse pad or e-signature for this treatment plan. • 1 value means the signature is done using scriptel web pad for this treatment plan. • 2 value means the signature is done using 10e plugin pad for this treatment plan.
encid int • This column stores the encounter id for which this treatment plan created. • This is a reference to the column encounterid of table enc.
datemodified datetime • This column stores the server date and time stamp of the instant the operation being performed is saved to the database.
copyofTPid int Not Available
createdByUserid int • This column stores userid of the user who has created for this treatment plan. • This is a reference to the column UID of table users.
modifiedByUserid int • This column stores userid of the user who has last modified for this treatment plan. • This is a reference to the column UID of table users.
enddateTP datetime This column is not used in V12 version.
copiedTPDate datetime • This column stores treatment plan date selected on UI while it is copied from other treatment plan. • The datatype of the column would be datetime as it is storing date.
copyofTPName varchar • This column stores treatment plan name entered on UI while it is copied from other treatment plan. • The datatype of the column would be varchar as it is storing alphanumeric characters.
setCompleteManually tinyint This column is not used in V12 version.
statusid int This column store dental treatment plan status id selected on UI. This column refers to statusid of dentaltreatmentplanstatus table
signSettingid int This column store signature setting id which is active when this treatment plan is created. This column refers to id of dental_tp_sign_settings table
isArchive tinyint • This column determines whether the treatment plan is archived or not. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value means the treatment plan is not archived . • •1 value means the treatment plan is archived.
highlightApplySlides tinyint • This column determines whether the apply slide button is highlighted on this treatment plan or not • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value means the apply slide button is not highlighted . •1 value means the apply slide button is highlighted .
highlightApplyBenefits tinyint • This column determines whether the apply benefits button is highlighted on this treatment plan or not • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value means the apply benefits button is not highlighted . •1 value means the apply benefits button is highlighted .
appliedBenefits tinyint • This column determines whether the insurance benefits is applied on this treatment plan or not • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value means the apply benefits is not applied . •1 value means the apply benefits is applied .
appliedSlides tinyint • This column determines whether the insurance slide calculation is applied on this treatment plan or not • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value means the slide calculation is not applied . •1 value means the slide calculation is applied .

EHI Export Schema
Table billingdata
Description This tables stores a patient's billing related items to an encounter.
Column DataType Description
id (Primary Key) int This column is a Primary Key and is auto-incremented.
Encounterid int This column stores encounterid which the Billing Information is documented for. This column refers column encounterid from enc table.
itemid int This column stores itemid of the added Procedures / Immunizations in the Billing information of the patient. This column refers to column itemid from items table.
date datetime This column stores start date of service for a specific medical, diagnostic, and surgical services represented by (CPT Code). The format used is YYYY-MM-DD.
endDate datetime This column stores end date of service for a specific medical, diagnostic, and surgical services represented by (CPT Code). The format used is YYYY-MM-DD.
endData datetime This column is not used by application anymore, hence it stores NULL which is the default value.
providerid int This column stores id of the Provider for whom created claims are submitted. This column refers column UID from user table.
Notes text This column stores notes being documented by end user related to the added procedures / immunizations in the Billing Information of the Patient.
displayIndex int This column stores sequence number of the added Procedures / Immunizations to be displayed in the Billing Information of the Patient.
deleteFlag int This column stores value 0. 0 - Added Procedure / Immunization in Billing Information of Patient is not deleted.
Mod1 varchar This column stores modifier's code which adds more information to the selected procedure and indicates the selected procedure has been altered due to some specific reason. This column refers column Code in modifier_validcodestable.
Mod2 varchar This column stores modifier's code which adds more information to the selected procedure and indicates the selected procedure has been altered due to some specific reason. This column refers column Code in modifier_validcodestable.
Mod3 varchar This column stores modifier's code which adds more information to the selected procedure and indicates the selected procedure has been altered due to some specific reason. This column refers column Code in modifier_validcodestable.
units decimal This column stores the number of times added Procedure is conducted as Units.
codeFlag varchar This column stores Flags like 'C' & 'E'. 'C' - indicates Procedure code is added using Add CPT button. 'E' - indicates Procedure code is added using E&M Tool.
Icd1 varchar This column stores the sequence number of the added Assessment as the first Assessment in order to map it with the selected Procedure.
Icd2 varchar This column stores the sequence number of the added Assessment as the second Assessment in order to map it with the selected Procedure.
Icd3 varchar This column stores the sequence number of the added Assessment as the third Assessment in order to map it with the selected Procedure.
Icd4 varchar This column stores the sequence number of the added Assessment as the fourth Assessment in order to map it with the selected Procedure.
icdcode1 varchar This column stores the Assessment code i.e. ICD code for the Assessment referred in column Icd1. This column refers to column value in itemdetails table.
icdcode2 varchar This column stores the Assessment code i.e. ICD code for the Assessment referred in column Icd2. This column refers to column value in itemdetails table.
icdcode3 varchar This column stores the Assessment code i.e. ICD code for the Assessment referred in column Icd3. This column refers to column value in itemdetails table.
icdcode4 varchar This column stores the Assessment code i.e. ICD code for the Assessment referred in column Icd4. This column refers to column value in itemdetails table.
vitalCpt tinyint This column stores flags 0 or 1. 1 - CPT code is linked to the documented vital , 0 - otherwise.
Mod4 varchar This column stores modifier's code which adds more information to the selected procedure and indicates the selected procedure has been altered due to some specific reason. This column refers column Code in modifier_validcodestable.
ToothNumber varchar This column stores tooth number or the pre defined portions of the dental arch on the claim submitted for the encounter. Dental Arch: Upper Arch, Lower Arch, UL Quad, UR Quad, LL Quad, LR Quad, FM, Sextant 1, Sextant 2, Sextant 3, Sextant 4, Sextant 5, Sextant 6.
Surface varchar This column stores surface related pre defined tooth characters on the claim submitted for the encounter M, O, I, D, B, F, L. M- Mesial, O - Occlusal, I- Incisal, D - Distal, B- Buccal, F - Facial, L - Lingual.
documentedTime int This column is not used by application anymore hence it stores 0 which is the default value.
ndccode varchar This column stores National Drug Code of the drugs submitted on a claim. The NDC code has '-' hyphen in it.

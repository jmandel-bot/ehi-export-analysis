Table prisma_socialhistory_details
Description This table stores the social history details of patients, including information about observations, categories, and values.
Column DataType Description
id (Primary Key) bigint Unique identifier for each record in the table. Example: 1, 2, 3.
sectionId bigint Identifier for the section the social history detail belongs to. Foreign key relation with `prisma_ccd_document_section.id`. Example: 101, 102, 103.
obsId varchar Identifier for the observation related to the social history detail. Example: "OBS001", "OBS002", "OBS003".
categoryname varchar Name of the category for the social history detail. Example: "Smoking", "Alcohol Consumption", "Exercise".
starttime varchar Start time for the social history detail. Example: "2021-01-01", "2021-06-01", "2021-09-01".
endtime varchar End time for the social history detail. Example: "2021-12-31", "2021-06-30", "2021-09-30".
code varchar Code for the social history detail. Example: "SH001", "SH002", "SH003".
codename varchar Name of the code for the social history detail. Example: "Smoking Status", "Alcohol Intake", "Physical Activity".
value varchar It is the snomed code for the valuename.
valuename varchar Name of the value for the social history detail. Example: "Cigarettes per day", "Drinks per week", "Minutes of exercise per day".
valuetype varchar Type of the value for the social history detail. Example: "CD", "PQ"
unit varchar Unit of the value for the social history detail.
codeSystem varchar Code system used for the social history detail. Example: "ICD-10", "SNOMED-CT", "LOINC".
valueCodeSystem varchar Code system used for the value of the social history detail. Example: "ICD-10", "SNOMED-CT", "LOINC".
externalSystemId varchar Identifier for the external system related to the social history detail. Example: "EXT001", "EXT002", "EXT003".
status varchar Status of the social history detail. Example: "Active", "Inactive", "Completed".
issuedDate datetime Date when the social history detail was issued. Example: "2021-01-01 00:00:00", "2021-06-01 00:00:00", "2021-09-01 00:00:00".
encounterId bigint Identifier for the encounter related to the social history detail. Foreign key relation with `prisma_encounter_details.id`. Example: 201, 202, 203.
isException tinyint Flag indicating if the row has incomplete data to be eligible for showing to the user
exceptionMessage varchar list of ids of the exception messages indicationg missing data points for the row to be eligible for showing to the user
sourcePriority int Source priority to be considered while duplicating entries
codeSystemPriority int Source code priority for deduplication for the social history detail. Example: 1, 2, 3.
duplicatedBy int Identifier for the record that duplicates the current record. Example: -1 (not duplicated), 2, 3.

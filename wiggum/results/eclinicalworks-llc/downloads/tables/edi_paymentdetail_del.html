EHI Export Schema
Table edi_paymentdetail_del
Description This table stores additional payment details for the encounter payment.
Column DataType Description
PmtDetailId (Primary Key) int Unique identifier for each payment detail record. Example: 1, 2, 3.
paymentId int It stores paymentid associted with edi_inspayments table.
encounterId int It stores encounter id associted with enc table
invoiceId int It stores invoice id associted with edi_invoice table
adjustment decimal Adjustment amount applied to the payment detail. Example: 10.00, 25.50.
allowed decimal Allowed amount for the payment detail. Example: 100.00, 200.00.
deduct decimal Deducted amount from the payment detail. Example: 5.00, 10.00.
coins decimal Coinsurance amount for the payment detail. Example: 20.00, 30.00.
memberBalance decimal Member's balance for the payment detail. Example: 50.00, 75.00.
paid decimal Paid amount for the payment detail. Example: 80.00, 120.00.
withheld decimal Withheld amount for the payment detail. Example: 10.00, 15.00.
MsgCode varchar Message code associated with the payment detail. Example: 'A01', 'B02'.
timestamp datetime time stamp of the payment detail record creation or update. Example: '2021-08-01 12:34:56'.
eraAdjustment decimal Adjustment amount related to Electronic Remittance Advice (ERA). Example: 5.00, 10.00.
closingId int Identifier for the closing associated with this payment detail. Foreign key referring to the closing table's primary key.
claimTaxAmt decimal Claim tax amount for the payment detail. Example: 2.00, 4.00.
CalculatedAdjustment decimal Calculated adjustment amount for the payment detail. Example: 8.00, 12.00.
ClaimInsId int It stores insuranceid associted with insurance table
seqno int Sequence number for the payment detail. Example: 1, 2, 3.
RecreatedFlag tinyint Flag indicating whether the payment detail record has been recreated. Example: 0 (not recreated), 1 (recreated).
ERAClaimStatusCLP02 int ERA claim status code. Example: 1, 2, 3.
ERACASPostedAsIs tinyint Flag indicating whether the ERA CAS (Claim Adjustment Segment) is posted as is. Example: 0 (not posted as is), 1 (posted as is).
hl7id varchar Identifier for the HL7 message associated with this payment detail. Example: '123abc', '456def'.

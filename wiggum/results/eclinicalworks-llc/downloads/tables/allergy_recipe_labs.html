Table allergy_recipe_labs
Description This table stores information about allergy recipes, their lab processing, and related details such as pickup, mailing, and quality checks.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table.
QUEUE_id int identifier for the queue the allergy recipe belongs to, can be NULL.
NOTE text Text field for any additional notes or comments about the allergy extract
ASSIGN_TO int identifier for the user assigned to process the allergy extract, can be NULL.
ADDRESSED tinyint A boolean flag (0 or 1) indicating if the allergy extract has been addressed.
ADD_VALUE varchar Additional value or information related to the allergy extract, can be NULL.
STATUS tinyint A boolean flag (0 or 1) indicating the status of the allergy extract, can be NULL.
NOTIFIED tinyint A boolean flag (0 or 1) indicating if the user has been notified about the extract.
NOTIFIEDTIME datetime time stamp of when the user was notified, can be NULL.
EXPECTEDSTDATE datetime This column stores the expected start date of the first shot date
ADD_TIME datetime time stamp of when the allergy extract was added, can be NULL.
NOTIFIEDBY varchar Name of the person who notified the user, can be NULL.
REVIEWED int A boolean flag (0 or 1) indicating if the allergy extract has been reviewed, can be NULL.
REV_TIME datetime time stamp of when the allergy extract was reviewed, can be NULL.
REV_BY varchar Name of the person who reviewed the allergy extract, can be NULL.
SUP_PHY int identifier for the supervising physician, can be NULL.
MIXFAC varchar Name of the facility where the allergy recipe is mixed, can be NULL.
SHOTFAC varchar Name of the facility where the allergy shot is given, can be NULL.
MAILTO_TYPE int identifier for the type of mailing, can be NULL.
MAILTO varchar Name of the person to mail the allergy recipe to, can be NULL.
MAILTO_ADDRESS1 varchar First line of the mailing address, can be NULL.
MAILTO_ADDRESS2 varchar Second line of the mailing address, can be NULL.
MAILTO_CITY varchar City of the mailing address, can be NULL.
MAILTO_STATE varchar State of the mailing address, can be NULL.
MAILTO_ZIPCODE varchar Zip code of the mailing address, can be NULL.
MIXFAC_id int identifier for the mixing facility, can be NULL.
SHOTFAC_id int identifier for the shot facility, can be NULL.
ADD_VALUEid int identifier for the additional value, can be NULL.
PICKUP tinyint A boolean flag (0 or 1) indicating if the allergy extract is to be picked up, can be NULL.
PICKUPBY varchar Name of the person picking up the allergy extract, can be NULL.
PICKUPBYRELATION varchar Relation of the person picking up the allergy extract to the patient, can be NULL.
PICKUPDATE datetime Pickup Date selected on extract screen
PICKUPFAC_id int identifier for the pickup facility, can be NULL.
PICKUPFAC_NAME varchar Name of the pickup facility, can be NULL.
CONSENTSIGNED tinyint A boolean flag (0 or 1) indicating if consent has been signed, can be NULL.
CONSENTSIGNEDBY varchar Name of the person who signed the consent, can be NULL.
CONSENTSIGNEDTIME datetime time stamp of when the consent was signed, can be NULL.
MAILTO_DATE datetime Date when the allergy recipe was mailed, can be NULL.
SHOT_FAC_TYPE tinyint A boolean flag (0 or 1) indicating the type of shot facility, default is 0.
MAILTO_TEL varchar Telephone number for the mailing recipient, can be NULL.
QUALITYCHECKED tinyint A boolean flag (0 or 1) indicating if the allergy extract has been quality checked, default is 0.
CHECKED_BY varchar Name of the person who performed the quality check, can be NULL.
CHECKED_BY_id int identifier for the person who performed the quality check, can be NULL.
CHECKED_TIME datetime time stamp of when the quality check was performed, can be NULL.
QUALITYCHECK_STATUS tinyint A boolean flag (0 or 1) indicating the status of the quality check, can be NULL.
ISNONBILLABLE tinyint A boolean flag (0 or 1) indicating if the allergy extract is non-billable, can be NULL.
dataWithInventoryIntegration tinyint A boolean flag (0 or 1) indicating if the record has inventory integration, default is 0.

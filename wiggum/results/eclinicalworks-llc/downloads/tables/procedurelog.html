EHI Export Schema
Table procedurelog
Description This is the parent table for every procedure documented for patients. Any procedure documented for a patient will have exactly one entry in this table. The information stored does not change over time such as the created date, created by user ID, patient ID, etc.

deleteFlag tinyint This column identifies whether a procedure is flagged as deleted (soft-deleted) or not. When set to 1, the procedure is not shown for the patient and is treated as a non-existing entry. When 0, the procedure is treated as being present and valid.
encounterid int This column is unused and will be removed in a future build.

procnote text This column is used for documentation of some details on migration when the procedure is migrated from a 3rd party system to eCW database.
isBilled tinyint This column is unused going forward, but will be retained for backward compatibility. The column used to flag a procedure as billed if set to 1 and flag the procedure as unbilled if set to 0. This task is now accomplished by the column billingdataid, which stores the reference to the column billingdata that can provide more details on the billing of the procedure.
autoCodeNum int Dental favorite procedures can be configured to use an auto code. Auto codes are rules that automatically select CPT codes based on certain conditions (like tooth type, surface type, etc). When a procedure is documented through such a favorite button, the id of the auto code is stored in this column. This way, future updates to the procedure will still follow the same auto code rules. When a procedure is documented without auto codes and a CPT is used directly, this column will store '0' (default value).
CreatedByTrUserid int This column is a reference to the 'UID' column of table 'users'. It stores the id of the user who created/documented the procedure. The procedure may later be modified by other users or the same user. This value will never change for a procedure once it has been recorded.
billingdataid int This column is a reference to id column of the table billingdata. When a procedure is billed, it is linked to billingdata table.
isGroupNote tinyint This column is set to 1 if the entry corresponds to a group note and not a dental procedure. If 0, it corresponds to a dental procedure.
isLocked tinyint This column stores a flag identifying whether a procedure is locked or unlocked. When locked, the column value is set to 1. When unlocked, it is set to 0 or NULL.
lockedByTrUserid int This column is a reference to the UID column of user table, and identifies the user who performed the last lock/unlock on the procedure.
lockDate datetime This column identifies the date/time of when the last lock/unlock operation was performed on the procedure.
proctpVisitid int This column stores the value of the new procedure added from Appointment Treatment Plan screen. It is the reference to the column "encProcTPNum" of "proctp_visit" table.
rowChangeStatus tinyint This column stores value 0 or 1. When a procedure's priority, tooth number, surface or D Code is changed , the column value updates to 1.
codeNumChangeStatus tinyint This column stores value 0 or 1. This column is specific to D code changes on a procedure. When a D code is changed manually or by an autocode , the column value changes to 1.

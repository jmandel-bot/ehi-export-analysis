EHI Export Schema
Table task
Description This table stores information about tasks assigned to doctors for their patients.
Column DataType Description
taskID (Primary Key) int An integer that uniquely identifies each task in the table. It is the primary key and is auto-incremented. Example: 1, 2, 3, etc.
description varchar A varchar(50) column that stores a brief description of the task. Example: "Check blood pressure", "Administer medication", etc.
taskDate varchar A varchar(20) column that stores the date when the task is scheduled. Example: "2022-08-15", "2022-08-16", etc.
taskTime datetime A time column that stores the time when the task is scheduled. Example: "14:30:00", "09:15:00", etc.
patientID int An integer that stores the unique identifier of the patient associated with the task. It is a foreign key that refers to the `patientID` column in the `patient` table. Example: 1, 2, 3, etc.
doctorID int An integer that stores the unique identifier of the doctor assigned to the task. It is a foreign key that refers to the `doctorID` column in the `doctor` table. Example: 1, 2, 3, etc.
status int An integer that stores the status of the task. Example: 0 (Not started), 1 (In progress), 2 (Completed), etc.

Table prisma_organization_details
Description This table stores information about various organizations, including their names, contact details, addresses, and other related information.
Column DataType Description
ID (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
ORGANIZATION_ID varchar Unique identifier for the organization. Example: "ORG123", "ORG456".
SECTION_ID bigint Identifier for the section associated with the organization. Foreign key relation: `prisma_ccd_document_section.id`. Example: 101, 102, 103.
SOURCE_ID int Identifier for the source of the organization information. Example: 1, 2, 3.
ORGANIZATION_NAME varchar Name of the organization. Example: "Acme Corporation", "Global Industries".
ALIAS_NAME varchar Alternative name or alias for the organization. Example: "Acme Corp", "Global Ind".
CONTACT_NUMBER varchar Contact phone number for the organization. Example: "+1-555-123-4567", "+1-555-987-6543".
PART_OF_ORGANIZATION varchar Identifier for the parent organization, if applicable. Example: "PARENT_ORG123", "PARENT_ORG456".
ADDRESS_LINE1 varchar First line of the organization's address. Example: "123 Main St", "456 Elm St".
ADDRESS_LINE2 varchar Second line of the organization's address, if applicable. Example: "Suite 100", "Apt 4B".
ADDRESS_LINE3 varchar Third line of the organization's address, if applicable. Example: "Building A", "Floor 3".
CITY varchar City where the organization is located. Example: "New York", "Los Angeles".
ORGANIZATION_STATE varchar State where the organization is located. Example: "NY", "CA".
POSTAL_CODE varchar Postal code for the organization's address. Example: "10001", "90001".
COUNTRY varchar Country where the organization is located. Example: "United States", "Canada".
ORGANIZATION_TYPE_DISPLAY varchar Display name for the type of organization. Example: "Corporation", "Non-profit".
ACTIVE tinyint Indicates if the organization is active (1) or inactive (0). Example: 1, 0.

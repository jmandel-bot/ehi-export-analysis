EHI Export Schema
Table smsmsglogs
Description This table stores information about SMS messages, their status, and related details for various types of messages such as appointments, labs, and patient statements.
Column DataType Description
ID (Primary Key) int This column is auto-incremented primary key in This column .
ToSMSName varchar This column stores the name to whom SMS is sent.
ToSMSNumber varchar This column stores the number to which SMS is sent.
ToSMSProvider varchar Not Available
Retry int This column store no of SMS retry has been done.
SMSMessageID varchar This column stores reference id indicating records on the messenger server.
SMSMESSAGEBODY text This column stores the actual SMS has been sent.
SmsMessageStatus varchar This column stores SMS message current status. The status can be 'Logged' 'Sending', 'Pending', 'Completed', 'Failed', 'Uploading', 'Cancelled', 'Reply Pending', 'Replied', 'HealthMaintenance', 'queued', 'sent', 'Provider exclude', 'Facility exclude', 'VisitType exclude', 'Resource exclude', 'Family msg', 'RxFax exclude'
smsMessageType varchar This column stores the type of SMS. The type can be 'appointment', 'labs', 'PtStatements', 'General', 'unknown', 'Rx', 'TelEnc', 'HealthMaintenance', 'No-Show', 'Birthday', 'Messenger Consent', 'Referral', 'OTP', 'Televisit'.
SMSSubject varchar This column stores subject for the SMS.
sentdate varchar This column stores the date on which SMS has been sent.
SentTime datetime This column stores the time on which SMS has been sent.
ScheduledDate datetime The date the SMS message was scheduled to be sent.
ScheduledTime datetime The time the SMS message was scheduled to be sent.
FromSMSName varchar This columns store name who is sending the SMS.
FromSMSNumber varchar This columns store then number from which the SMS has been sent.
FromSMSProvider varchar This columns store name of the provider who is sending the SMS.
ClientId int column not in use/ inactive
Priority int This column stores values 0/1 to indicate whether SMS priority High or Low. 0 - SMS priority is Low, 1 - SMS priority is High.
EcwWebSMSUser varchar column not in use/ inactive
ErrorId int This column stores the error Id which sent from the messenger server in case of any failure.
ErrorMsg varchar This column stores the error message which is been shown to user in case of any failure.
DateCreated datetime This column stores the date and time of message creation.
isDeleted int This column stores values 0/1 to indicate whether the SMS has been marked as deleted or not. 0 - If SMS is NOT marked as deleted, 1 - If SMS is marked as deleted.
EncounterID int This column stores the encounter id against which SMS has be been sent. This column refers to encounterID column in enc table.
LabID int This column stores the lab id against which SMS has be been sent. This column refers to ReportId column in labdata table.
PatientId int This column stores the patient id to whom message has been sent. This column refers to pid column in patients table.
FacilityId int This column stores the id of the facility from where the message has been sent. This column refers to Id column in edi_facilities table.
PtStatementId int The identifier for the patient statement associated with the SMS message.
Language varchar This column stores the language in which message has been sent. The language value can be English, Spanish etc.
smsmsguser varchar This column stores the username of the messenger server user account through which message has been uploaded on messenger server.
smsmsgpwd varchar This column stores the password of the messenger server user account through which message has been uploaded on messenger server.
AppServMachineName varchar This column stores the hostname of the messenger job server through message has be sent.
AppServMachineIP varchar This column stores the ip address of the messenger job server through message has be sent.
AppServMacAddr varchar This column stores the Mac address of the messenger job server through message has be sent.
sentbylastname varchar This column stores the last name who has initiated SMS.
SentByFirstName varchar This column stores the first name who has initiated SMS.
SentBy int This column stores the user id who has initiated SMS. This column refers to uid column in user table.
VendorMsgId varchar The identifier for the vendor message.
VendorSMSresponse text column not in use/ inactive
TemplateId int This column stores the id of the SMS template used send SMS. This column refers to id column in voicemsgTemplateText table.
uploadlocation varchar This column stored the URL of messenger server on which message has ben uploaded.
DoctorId int This column stores the provider id. This column refers to uid column in user table.
RequiredConfirm tinyint This column stores values 0/1 to indicate whether SMS confirmation required from patients or not. 0 - If No confirmation required, 1 - If confirmation required.
ConfirmNumber int This column stores random number used by patient to send their confirmation through SMS.
smsmsgreply varchar The reply to the SMS message.
smsmsgreply_id int This column stores the id for the replied SMS from the patient.
parentid int This stores parent message id while resending the SMS.
ActualMessage text This column stores the actual SMS has been sent.
ResourceId int This column stores ID of resource such as anyone on the staff (except a physician), devices, or even rooms.
msgSource varchar This column stores the source of the message creation like Manual Message etc.
ecwversion varchar This column stores the EMR version through which the SMS has been sent.
ecmversion varchar This column stores the eClinicalMessenger version through which the SMS has been sent.
AutoUpgradeKey varchar This column stores the APU id of the practice.
FaxLogId int This column stored fax log id. This column refers to FaxLogId column in FaxLogs table.
eRxReqID int This column stores the eRx Request ID. This column refers to ReqID column in surescript_eprescription or surescript_requestLog table.
eRxReFillReqID int This column stores the eRx refill Request ID. This column refers to ReqID column in surescript_eprescription or surescript_requestLog table.
campaignid int This column stores the campaign id for appointment reminder, Rx reminder, Lab reminder etc.
campaign varchar column not in use/ inactive
MsgExpireTime datetime This column store the expire date and time for the particular SMS.
customreportid int column not in use/ inactive
policyid int This column stores policy id for all the reminders like appointment, rx, lab etc.
ApptStatus varchar This column stores the appointment status for the encounter. This column refers to Status column in enc table.
slotid int This column stores the slot id. This column refers to slotid column in reservedslots table.
ptmulogid int This column stores patient demographics log id. This column refers to ptmulogid column in logpatientdemographicsxml_mulog table.
smsguid varchar This column stores the random GUID for the particular SMS.
updateflag tinyint A flag indicating whether the SMS message has been updated (0 for not updated, 1 for updated).
VendorSmsId varchar column not in use/ inactive
measureid varchar The identifier for the measure associated with the SMS message.
msgkeywords varchar This column stores json string of contains all the resolved keywords used in SMS body.
vststsupdated tinyint This column stores values 0/1/2 to indicate based on reply message. 0 - If replied with 'yes', 1 - If replied with 'no' 2 - If replied with something else
responseupdatedtime datetime This columns stores the date and time for the update message reply.
rscancflag tinyint This column stores values 0/1 to indicate whether the rescheduled the cancelled message or not. 0 - Do not rescheduled the cancelled message, 1 - Rescheduled the cancelled message.
enablearrival tinyint A flag indicating whether the arrival feature is enabled (0 for not enabled, 1 for enabled).
arrivaleng varchar This column stores patient arrival message in english which patient do echeck-in for appointment.
arrivalspanish varchar This column stores patient arrival message in Spanish which patient do echeck-in for appointment.
prearrduration int This column stores the duration before the start of the appointment at which the "I have arrived" button should be available to patients in appointment reminders.
csdata varchar This column stores the extra information needed for other module to send their notification.
ptemail varchar This column stores the patient Email Address to whom SMS is send.
ScheduledDateTime datetime The scheduled date and time for the SMS message.

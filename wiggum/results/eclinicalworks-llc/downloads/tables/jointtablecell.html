EHI Export Schema
Table jointtablecell
Description This table stores information about joint table cells, which are used to represent the relationship between encounters, joint properties, and joint conditions.
Column DataType Description
JointTableCellid (Primary Key) int • This column will store the id of the Joint Table Cell data • The datatype of the column would be INT to accommodate large number of different Joint Table Cell data that can be added. • This would be unique and auto incremented and so PRIMARYKEY constraint would be added
encounterid int • This column stores the encounter id where the condition and value are being documented. T • This is a reference to the column encounterid of table enc
jointPropertyid int • This column stores the id of the joint being documented. • This is a reference to the column jointPropertyid of table JointProperty
jointConditionid int • This column stores the id of the condition being saved. • This is a reference to the column jointConditionid of table JointCondition.
jointConditionValueid int • This column stores the id of the value being saved. • This is a reference to the column jointConditionValueid of table JointConditionValue.
jointConditionValue varchar • This column stores the value being documented for those values that are free text or numeric. • This column is needed for cases where jointConditionValueid is not applicable.
modifiedByUserid int • This column stores the id of the user performing the operation. • This is a reference to the column UID of table users
modifiedDateTime datetime • This column stores the server date and time stamp of the instant the operation being performed is saved to the database.
deleteFlag tinyint • This column determines whether the joint table cell data is deleted. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value mean joint table cell data is not deleted • 1 value means joint table cell data is deleted
importedFromEncid int • This column stores the encounterid if the value is imported from a different encounter. • This is a reference to the column encounterid of table enc. If the row values are not imported, this column will be set to NULL.

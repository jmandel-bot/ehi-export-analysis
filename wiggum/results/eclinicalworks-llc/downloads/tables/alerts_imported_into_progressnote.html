EHI Export Schema
Table alerts_imported_into_progressnote
Description This table stores information about alerts that have been imported into the Progress Notes, including their type, status, and related encounter details.
Column DataType Description
id (Primary Key) int This column is an auto incremented primary key.
AlertType varchar This column stores the type of an alert. Possible values are CDSS, Generic, patient specific, Rx specific, Dx specific, or multisatisfied.
Alertid int This column stores an id of the alert from the AlertType column.
Encounterid int This column stores the id of the encounter to which alert is imported. This column refers to the encounterid from the enc table.
AlertName varchar This column stores the name of the alert merged in the Progress Notes.
DueDate datetime This column indicates the alert's due date at the time it was imported to the Progress Notes. The format used is YYYY-MM-DD 00:00:000. In case if there is no due date, NULL is stored.
Status varchar This column indicates the status of the alert. Possible values are Complaint, Suppressed, Excluded, Non-Complaint, or pending results.
LastDoneDate datetime This column stores the date on which the test was ordered for a particular patient. The format used is YYYY-MM-DD 00:00:000. In case if there is no last done date, NULL is stored.
ActionTaken text This column indicates if the auto imported alert was removed or manually imported. '-' for manually imported and 'removed auto imported' for auto imported removal.
ImportedDate datetime This column stores a time stamp when the alert was imported to a patient's Progress Notes. The format used is YYYY-MM-DD HH:MM:SS.
DeleteFlag tinyint This column stores a flag 0 or 1. 1 - when the auto imported alert is removed manually and the removal is undone or when the alert is added manually, and then removed manually or 0 - otherwise.
RemovedDate datetime This column stores a time stamp when the auto imported alert was removed manually. The format used is YYYY-MM-DD HH:MM:SS.
orderStatusIndicator tinyint This column is linked to the LastDoneDate column. This column stores 0, 1 or 2. 0 - There is no order. 1 - There is a test is ordered, 2- There is a test received.

Table rpm_device_tracker_mapping
Description This table stores the mapping between devices and trackers, indicating which tracker is associated with a specific device.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT(11). This column stores integer value which could grow with number of transactions over the years hence integer(Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
device_id int This column contains "device id" referenced from device_id column of rpm_supported_devices_list table which is used for mapping and it is a foreign key. This column could be store high range numeric value hence integer is selected and this field will always have a value hence defined as NOT NULL.Also, it not has any default value because it is a foreign key.
tracker_id int This column contains tracker id referenced from tracker_id column of rpm_healow_tracker_list table which maintains a foreign key relationship. It is always expected to have a value and must not be NULL. This column could store high range numeric value hence integer is selected and this field will always have a value hence defined as NOT NULL.
deleteFlag tinyint Flag indicates if this mapping is deleted or not, datatype is TINYINT. This column stores binary flag i.e 0/1. 0: not deleted 1: deleted - By default device tracker maping must not be in deleted status hence default 0 is chosen.

EHI Export Schema
Table ip_rh_allergies
Description Table is used to store allergy details for patient encounter
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT (11). This column stores integer value which could grow with number of transactions over the years hence integer (Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
patientID int This column is used to identify for which patient user has documented allergies. This column is FOREIGN KEY which refers to uid column of users table. This column stores high range numeric value hence integer is selected, and this column will always have a value hence defined as NOT NULL.
encounterId int This column is used to identify for which patient encounter user has documented allergies This column is FOREIGN KEY which refers to encounterid column of enc table. This column stores a high range numeric value hence integer is selected, and this column will always have a value hence defined as NOT NULL.
drug varchar This column stores agent/drug/substance name which is added for patient encounter. We are using datatype as varchar 255.Drug name will not be more than 255 characters, so we have set its length to 255. This column will not always have value hence we keep NOT NULL DEFAULT ''.
allergy varchar This column stores reaction/allergy name which is added for patient encounter. We are using datatype as varchar 255. Allergy name will not be more than 255 characters, so we have set its length to 255. This column will always have a value hence defined as NOT NULL and default value is ''.
itemID int This column is used to store item id of added allergy to patient encounter. This column is FOREIGN KEY which refers to itemID column of items table. This column stores itemid of agent/drug/substance which is added for patient encounter.This column will not always have value hence we keep NULL DEFAULT NULL.
displayIndex int This column stores integer values hence integer data type is selected. This column is used to maintain order of drugs added for patient encounter. This column will always have a value hence defined as NOT NULL and default value is 1.
SMDamConceptID int This is a legacy column.This column is not being used as of now. Value is always set to 0.
SMDamConceptIDDesc varchar This is a legacy column.This column is not being used as of now.
ndc_code varchar This column stores ndc code of reaction/allergy name which is added for patient encounter. We are using datatype as varchar 255. Ndc code will not be more than 255 characters, so we have set its length to 255. This column will not always have a value hence defined as NULL and default value is NULL.
allergytype smallint This column stores the id of reaction/allergy which is added for patient encounter. This column belongs to the id column of allergytypes table. This column stores a small range numeric value hence SMALLINT datatype is selected. This column will not always have value hence we keep NULL DEFAULT NULL.
type tinyint This column stores 1 or 2 number value.1-existing agent/substance .2- user entered value. The data type for this column is char (1) selected with default value is ''.
status varchar This column will store "Active" and "Inactive" text as value. We are using VARCHAR (10) as data type because the value of this column will not be more than 10 characters. This column will always have value hence we defined it as NOT NULL DEFAULT 'Active'.
criticality smallint This column stores the id of allergy criticality which is added for patient encounter. This column belongs to the id column of allergycriticality table. This column stores a small range numeric value hence SMALLINT is selected as datatype. This column will not always have value hence we defined it as NULL DEFAULT NULL.
onsetdate varchar This column stores date string value. We are using VARCHAR (10) as data type because the value of this column will not be more than 10 characters. This column will not always have value hence we defined it as NULL DEFAULT NULL.
deleteFlag tinyint This will store 0 and 1 value. 0-not deleted and 1-deleted. This column is used to identify whether row is in use or not. We are using TINYINT as datatype as we just want to store 0 or 1 value. By default, row will be in use, so we keep NOT NULL DEFAULT 0.
createdBy int This column stores user id of user. This column is used to identify which user has added record(allergy) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store high range of numeric value hence integer is selected. This column will always have a value hence defined as NOT NULL.
createdDateTime datetime This Column stores datetime as we want to store a datetime when the record is added, so we are using DATETIME as data type of column. This column will always have a value hence defined as NOT NULL.
modifiedBy int This column stores user id of user. This column is used to identify which user has last updated/modified the added record(allergy) for patient encounter. This column is FOREIGN KEY which refers to uid column of users table. This column will store high range of numeric value hence integer is selected, and this column will not always have a value hence defined as NULL DEFAULT NULL.
modifiedDateTime datetime This Column stores datetime as we want to store a datetime when the record is modified, so we are using DATETIME as data type of column. This column will always not have a value hence defined as NULL DEFAULT NULL.
uId varchar This column stores vm id string value. We are using VARCHAR (60) as data type because this column's value will not be more than 60 characters. This column will always have a value hence defined as NOT NULL.

EHI Export Schema
Table rpm_vendors_list
Description This table stores information about remote patient monitoring (RPM) vendors, their display names, supported modalities, and other related attributes.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT(11). This column stores integer value which could grow with number of RPM vendors added over time and hence integer(Range: 0-2,147,483,647) is selected . It is AUTO_INCREMENT column and has PRIMARY KEY index.
name varchar Contains the name of the vendor and the data type is VARCHAR(255). This column stores characters which cannot exceed 255 characters hence VARCHAR(255) is selected. This field will always have a value hence defined as NOT NULL. This field must have a unique name hence UNIQUE constraint UX_rpm_vendors_list_name  is added.
vendor_display_name varchar Contains the display name of the vendor and the datatype is VARCHAR(255). This column stores characters which cannot exceed 255 characters hence VARCHAR(255) is selected This field will always have a value hence defined as NOT NULL.
modality varchar This column contains the modality of the vendor.Vendors send data either through API's into healow which is passed to EMR or the readings are accessed through the PDF. Some of the vendors support both API/PDF. Hence the classification. The data type chosen for this column is VARCHAR(20). This field will always have a value hence defined as NOT NULL with Default value API.Since the column value does not exceed 20 characters VARCHAR(20) is selected. The modality can have values ('PDF','API','BOTH') only and hence CHECK constraint CHK_rpm_vendors_list_name is added.
isLinked tinyint This column specifies if the vendor is linked and data type is TINYINT. This field will always have a value hence defined as NOT NULL with Default value 0. This column could contain two columns  0- If the vendor is not linked on healow  1- If the vendor is linked on healow 
deleteFlag tinyint Flag indicates vendor is Deleted or not, datatype is tinyint - 0: not deleted 1: deleted. This field will always have a value hence defined as NOT NULL. By default, vendor should not be in inactive status hence default 0 is chosen.
support_cellular_device tinyint This column stores a boolean value (0 or 1) indicating whether the RPM vendor supports cellular devices or not. For example: 0 for not supporting cellular devices, 1 for supporting cellular devices.

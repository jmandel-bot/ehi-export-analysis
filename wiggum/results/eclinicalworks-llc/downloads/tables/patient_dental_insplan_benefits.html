EHI Export Schema
Table patient_dental_insplan_benefits
Description This table stores information about the dental insurance plan benefits for each patient.
Column DataType Description
id (Primary Key) int • This column will store the id of the patient_dental_insplan_benefits table. • The datatype of the column would be INT to accommodate large number of different patient insurance benefits data can be added. • This would be unique and auto incremented and so PRIMARYKEY constraint is added
patientid int • This column stores the id of the patient • This is a reference to the column pid of table patients.
insid int • This column stores the id of the insurance • This is a reference to the column insid of table insurance.
patientInsid int • This column stores the id of the patient insurance. • This is a reference to the column id of table insurancedetail.
annualMaximumRemaining_individual decimal • This column will store individual annual maximum remaining value captured on UI. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field.
annualMaximumRemaining_family decimal • This column will store family annual maximum remaining value captured on UI. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field
deductibleRemaining_individual decimal • This column will store individual deductible remaining value captured on UI. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field
deductibleRemaining_family decimal • This column will store family deductible remaining value captured on UI. • The datatype of the column would be decimal as it can have 0 to 9999999.99 values. • This column is -1.00 by default to show blank value in field
anniversaryMonth_individual tinyint • This column will store anniversary month present for selected insurance plan. • The datatype of the column would be tinyint as it can have 0 to 12 values. • This column is 0 by default to show blank value in field
deleteflag tinyint • This column determines whether the patient dental insurance benefits data is deleted. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value mean patient dental insurance benefits data is not deleted • 1 value means patient dental insurance benefits data is deleted
endDate datetime • This column stores the benefits end date. This date is calculated based on anniversary month selected for the insurance plan.
updatedDate datetime • This column stores the server date and time stamp of the instant the operation being performed is saved to the database.

Table jointcondition
Description This table stores information about various joint conditions, their value types, and display settings.
Column DataType Description
jointConditionid (Primary Key) int • This column will store the id of the Joint Condition • The datatype of the column would be INT to accommodate large number of different Joint Condition that can be added. • This would be unique and auto incremented and so PRIMARYKEY constraint would be added
conditionName varchar • This column will store the name of the Joint Condition • The datatype of the column would be varchar(50) as the Joint Condition name would normally be short and concise • NOT NULL constraint would be added as it mandatory to provide name of the Joint condition
valueType tinyint • This column will store the value type of the Joint Condition • The datatype of the column would be tinyint as it will be having values only 1, 2 or 3. Following are these values meaning 1 – Structured Text 2 – General Text 3 – Numeric
itemOrder tinyint • This column stores the order in which the condition appears in the list. • The datatype of the column would be tinyint as it will be having numeric values only. • By default, this column value will be zero
deleteFlag tinyint • This column determines whether the joint condition is deleted. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value mean joint condition is not deleted • 1 value means joint condition is deleted
hiddenFlag tinyint • This column determines whether the joint condition is hidden. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value mean joint condition is not hidden • 1 value means joint condition is hidden
readOnlyFlag tinyint • This column determines whether the joint condition is read-only or can be updated on the configuration screen. • The datatype of the column would be tinyint as it can have 0 or 1 value. • 0 value mean joint condition is not read only. • 1 value means joint condition is readonly
UOM varchar • This column determines the unit of measurement for the condition. • The datatype of the column would be varchar(100) as the condition UOM would normally be short

Table vision_eligibility_info
Description This table stores information about the eligibility of individuals for vision-related services, including contact details and call timestamps.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
eligdeterminedby varchar The entity responsible for determining eligibility. Example: "Insurance Company A", "Government Agency".
eligContactName varchar The name of the contact person responsible for eligibility determination. Example: "John Doe", "Jane Smith".
elighwauth varchar The hardware authorization used for eligibility determination. Example: "HW123", "Auth567".
eligcall varchar The call identifier for the eligibility determination process. Example: "Call001", "Call002".
eligphone1 varchar The primary phone number used for contacting the eligibility determination entity. Example: "+1-555-123-4567", "+1-555-987-6543".
eligphone2 varchar The secondary phone number used for contacting the eligibility determination entity. Example: "+1-555-234-5678", "+1-555-876-5432".
DateofCall datetime The date when the eligibility call was made. Example: '2021-10-01 00:00:00', '2021-10-02 00:00:00'.
Timeofcall datetime The time when the eligibility call was made. Example: '2021-10-01 14:30:00', '2021-10-02 09:15:00'.
deleteFlag smallint A flag indicating whether the record has been deleted (1) or not (0). Example: 0, 1.
STATUS tinyint The status of the eligibility determination process. Example: 0 (Pending), 1 (Approved), 2 (Denied).
Insdetailid int The identifier of the insurance detail record associated with the eligibility. This is a foreign key that refers to the `id` column in the `insurance_details` table.
planid int The identifier of the insurance plan associated with the eligibility. This is a foreign key that refers to the `id` column in the `insurance_plans` table.

EHI Export Schema
Table externalicd10inbound
Description This table stores information about inbound ICD-10 codes, their descriptions, and related data from external sources.
Column DataType Description
Id (Primary Key) int Unique identifier for each record in the table. Example: 1, 2, 3.
intId int This column refer to intlist table
hl7msgId int This column refer to ptformsdatahl7content table
patientId int An integer identifier for the patient associated with the ICD-10 code, which can be NULL. This can be a foreign key referring to the `Id` column in the `patients` table.
ICD10Code varchar A string representing the ICD-10 code for a specific diagnosis or condition. Example: "A00.0", "B01.1", "C02.2".
ICD10CodeDescription varchar A string describing the ICD-10 code in detail, which can be NULL. Example: "Cholera due to Vibrio cholerae 01, biovar cholerae", "Varicella meningitis", "Malignant neoplasm of the tongue".
SendingApplication varchar A string representing the name of the application that sent the ICD-10 code, which can be NULL. Example: "App1", "App2", "App3".
SendingFacility varchar A string representing the name of the facility that sent the ICD-10 code, which can be NULL. Example: "Hospital1", "Clinic2", "Lab3".
receivedDateTime datetime A datetime value representing when the ICD-10 code was received, which can be NULL. Example: "2022-01-01 12:34:56", "2022-02-02 14:30:00", "2022-03-03 16:45:30".
status int An integer representing the status of the record (e.g., processed, unprocessed), which can be NULL. Example: 0 (unprocessed), 1 (processed).
deleteFlag tinyint A tiny integer representing whether the record has been deleted or not. Example: 0 (not deleted), 1 (deleted).
source varchar A string representing the source of the ICD-10 code, which can be NULL. Example: "Source1", "Source2", "Source3".

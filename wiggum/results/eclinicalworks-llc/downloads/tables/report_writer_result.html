Table report_writer_result
Description This table stores the results of report writers, including notes, report and template IDs, document IDs, and whether the report is editable or not.
Column DataType Description
id (Primary Key) int This is the primary key, an integer that uniquely identifies each report writer result. Example: 1, 2, 3.
notes varchar This column is used to store the final result of report writer. As per requirement, user can add up to 80,000 characters. Hence, added this column as Text/Varchar(Max). As per requirement, user can not enter NULL value from UI. Hence, not NULL constraint is suitable here.
reportid int This column is used to store reportid of associated DI. Report writer result can associate with other entity apart from reportid. Hence, added this column as NULL. This column is a foreign key that references to labdata (reportid). Reportid column of labdata is INT type. Hence, INT datatype is suitable here.
templateid int This column is used to store template id of associated template. Report writer result should be associated with specific template. Hence, not NULL constraint is suitable here. This column is a foreign key that references to report_writer_template (templateid). Templateid column of report_writer_template is INT type. Hence, INT datatype is suitable here.
docid int This column is used to store generated Document id once user click on Publish Report button to generated PDF. By default, value of this column will be NULL. When user click Publish Report, the system will generate document and add associated docid in this column. Hence, added this column as NULL. This column is a foreign key that references to document (docid), docid column of document is INT type. Hence, INT datatype is suitable here.
isReportEditable tinyint This column is used to maintain flag to update Report Writer details. 0: Report Writer details not editable 1: Report Writer details editable The default value of this column is 1, so Report Writer tab will be editable. Once the user clicks Publish Report button and the PDF is generated, this column is 0 which will make the Report Writer details disabled. Once the user clicks Edit Report, this will change to 1. This column can have only 0 or 1 value. Hence, TINYINT is suitable for this column.

Table birth_vitals
Description This table stores unique records for a patient for capturing birth vitals information with additional notes.
Column DataType Description
id (Primary Key) int This is the primary key, an auto-incrementing integer that uniquely identifies each record in the `birth_vitals` table. Only 1 record can exists for a patient indicating information captured during birth.
patientid int This is a foreign key that refers to the `pid` column in the `patients` table. It is an integer that represents the unique identifier of the patient whose birth vitals are being recorded. Example: 101, 102, 103, etc. Foreign Key Relation: `patients` (`pid`)
notes varchar This is a varchar(255) column that stores any additional notes or comments about the patient's birth vitals. It has a default value of an empty string (''). Example: 'Normal birth weight', 'Low birth weight', etc.
ispndisplay smallint Flag indicates whether birth vitals should be displayed on Progress Notes, datatype is smallint. Default value is -1 : which means till the age of 2 months, birth vitals will be displayed on PN, value is 0 : birth vitals will not be displayed on PN, value is 1 : birth vitals will be displayed on PN,

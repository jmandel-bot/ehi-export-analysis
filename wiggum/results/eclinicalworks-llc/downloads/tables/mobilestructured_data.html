EHI Export Schema
Table mobilestructured_data
Description This table stores structured data related to mobile encounters, including notes, transcription information, and user details.
Column DataType Description
id (Primary Key) int A text column that stores the data associated with the synchronization action. This could be the data being inserted, updated, or deleted, depending on the action type.
encounterid int A varchar(500) column that stores the SQL WHERE clause used to filter the records affected by the synchronization action. This column is optional and can be empty if no filtering is needed. Example: "user_id = 5 AND status = 'active'".
notes text A datetime column that stores the time stamp when the synchronization action was created. This column is optional and can be NULL if the creation time is not needed. Example: '2021-09-15 12:34:56'.
transcriptionid int A smallint(6) column that indicates whether the synchronization action has been performed on the web server or not. It has a default value of '0'. Example: 0 (not synced), 1 (synced).
userid int A tinyint(2) column that stores the number of times the synchronization action has failed. It has a default value of '0'. Example: 0, 1, 2, etc.
visittype varchar This is an integer column that represents the unique identifier of the P2P (Peer-to-Peer) Network Provider Interface for the recipient. Example: 1001, 1002, 1003.
scribedstatus tinyint This is a varchar(700) column that stores the name of the P2P provider for the recipient. Example: 'ProviderA', 'ProviderB', 'ProviderC'.
searchKey text Unique identifier for each multisatisfiedalert record. Example: 1, 2, 3, etc.
created datetime A datetime value indicating when the mobile structured data record was created. It can be NULL. Example: '2021-08-01 12:34:56', '2021-08-02 14:30:00'.

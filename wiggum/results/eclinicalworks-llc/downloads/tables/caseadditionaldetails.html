EHI Export Schema
Table caseadditionaldetails
Description This table stores additional details related to a specific case, such as disability dates, employer note dates, and return to duty dates.
Column DataType Description
id (Primary Key) int This is the primary key for the table, which is an integer value that uniquely identifies each record. It is an auto-incrementing value. Example: 1, 2, 3, etc.
caseid int This is a foreign key that refers to the `CaseId` column in the `case_details` table. It represents the unique identifier of the case that the additional details are associated with. Example: 101, 102, 103, etc.
disabilitybegundate datetime This is a datetime column that stores the date and time when the disability associated with the case began. Example: '2021-09-01 10:30:00'
employernotedate datetime This is a datetime column that stores the date and time when the employer provided a note related to the case. Example: '2021-09-02 14:45:00'
disabilityenddate datetime This is a datetime column that stores the date and time when the disability associated with the case ended. Example: '2021-09-30 18:00:00'
modifiedbegandate datetime This is a datetime column that stores the date and time when the modified disability period began, if applicable. Example: '2021-09-15 09:00:00'
modifiedenddate datetime This is a datetime column that stores the date and time when the modified disability period ended, if applicable. Example: '2021-09-29 17:00:00'
returndutydate datetime This is a datetime column that stores the date and time when the individual returned to duty after the disability period. Example: '2021-10-01 08:00:00'
deleteflag tinyint This is a tinyint column that stores a boolean value (0 or 1) indicating whether the row has been deleted or not. A value of 0 means the row is not deleted, and a value of 1 means the row is deleted. Example: 0 or 1

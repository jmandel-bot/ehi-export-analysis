EHI Export Schema
Table case_careteam_map
Description This table stores the mapping between cases and care teams, including information about the primary care team, role, care type, and the relationship with the COC table.
Column DataType Description
id (Primary Key) int Unique identifier for each record in the case_careteam_map table. Example: 1, 2, 3.
caseid int identifier for the case associated with the care team. Example: 101, 102, 103.
careteamid int identifier for the care team associated with the case. Example: 201, 202, 203.
isprimary tinyint A boolean flag indicating if the care team is the primary team for the case. 1 represents true (primary), and 0 represents false (not primary). Example: 0, 1.
role int identifier for the role of the care team in the case. Example: 301, 302, 303.
delflag tinyint A boolean flag indicating if the record has been deleted. 1 represents true (deleted), and 0 represents false (not deleted). Example: 0, 1.
caretype varchar A string representing the type of care provided by the care team. Example: 'emergency', 'routine', 'specialist'.
coc_id int identifier for the coc (Chain of Custody) record associated with the case_careteam_map. This is a foreign key that refers to the `id` column in the `coc` table. Example: 401, 402, 403.

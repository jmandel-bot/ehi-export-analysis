EHI Export Schema
Table empiccrvoidrequest
Description This tables stores the data for those encounter where lab results published after progress note exported already, we use entries in table to voidccr so it can export progress note again in next cycle
Column DataType Description
id (Primary Key) int This is the primary key for the table and is an auto-incrementing integer. It uniquely identifies each void request. Example: 1, 2, 3, etc.
encid int This is a foreign key that refers to the "encounterid" column in the "enc" table. It represents the encounter id associated with the void request. Example: 101, 102, 103, etc.
voidEncTime datetime This column stores the date and time when the encounter was voided. It is of datetime type and can have a default value of NULL if the encounter has not been voided yet. Example: '2021-09-15 14:30:00', '2021-09-16 10:15:00', etc.
deleteFlag tinyint This is a tinyint column that indicates whether the void request has been deleted or not. It has a default value of '0' (not deleted). A value of '1' would indicate that the void request has been deleted. Example: 0, 1.

Table ip_rh_waitList_followUpData
Description This table is used to store the follow-up details entered while adding patient into wait list. This is an extension of ip_rh_sch_patient_waitList. When user added follow up for patient while adding/edit patient's entry into waitlist then those details will get save in this table.
Column DataType Description
id (Primary Key) int Contains auto incremented id, datatype is INT. This column stores integer value which could grow with number of transactions over the years hence integer (Range: 0-2,147,483,647) is selected. It is AUTO_INCREMENT column and has PRIMARY KEY index
waitListId int This column stores the wait list id which is a foreign key of ip_rh_sch_patient_waitList. The data type chosen is int. This column stores high range numeric value hence integer (Range: 0-2,147,483,647) is selected, and this field will always have a value hence defined as NOT NULL.
addedOn datetime The purpose of this column is to store date time value for follow-up added into Add/Edit wait list screen.
followUpCallDueDateTime datetime The purpose of this column is to store date time value for which users can follow-up with patient into Add/Edit wait list screen.
followUpCallAttemptDateTime datetime This column stores date time value for which users can followed with patient into Edit wait list screen. The datatype chosen is DATETIME. Default value is null
notes nvarchar This column is for storing notes of follow-up details, it's a NVARCHAR (MAX). As the column could store huge data, we have defined it as NVARCHAR (MAX).
addedBy int The purpose of this column is to store the id of a user who has added follow up details on the wait list screen.
followUpIn int This column stores the follow up in, in hours added while adding follow up details. This column stores high range numeric value hence integer is selected. Default value is NULL because if user has selected follow Up call due date directly on follow up screen, then this column will have no value and if user has added number of hours in this column, then follow Up call due date will get calculated from current date time and added into follow Up call due date column.
deleteFlag tinyint Flag indicates if this log is deleted or not. The datatype chosen is tinyint with length 1. 0: follow up details not deleted 1: follow up details deleted Default 0 is chosen for this column because by default follow up details must be undeleted.

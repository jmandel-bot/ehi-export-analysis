EHI Export Schema
Table asc_vitals_batch
Description This table is used to store vitals data.
Column DataType Description
batchid (Primary Key) int used to store Autoincrement id for the table.
encounterid int used to store encounter id ,which maps to encounterid from enc table.
reportedBy int used for storing reported by user id, which maps to UID from user table.
reportedTime datetime used for storing reported date and time.
notes text This field is not in use.
deleteFlag tinyint its value can be 0 or 1, which indicates if item is deleted or not.1 indicates the item is deleted.0 indicates the item exists.
isVitalFromDevice tinyint By default its value is 0.Its value can be 0 or 1.It will be 1,if data is documented through device. Otherwise it will be 0.
importedBy int An integer representing the unique identifier of the user who imported the batch of vital signs. This column has a foreign key relation with the `users` table's `UID` column. Example: 2001, 2002, 2003.
isImported tinyint A tinyint(1) value indicating whether the batch of vital signs was imported or not. A value of 0 means the batch was not imported, and a value of 1 means it was. NOT NULL DEFAULT '0'. Example: 0, 1.

EHI Export Schema
Table proctp
Description This table stores information procedures those are part of saved treatment plan of the patients.
Column DataType Description
procTPNum (Primary Key) int This is the auto-increment primary key column for the table.
treatPlanNum int This column stores identifier number of a Treatment Plan. It refers to column TreatplanNum from dentaltreatmentplan table.
patNum int This column stores the patient identifier reference for whom the procedure is being documented. It is a reference to the column 'UID' of table 'users'.
procNumOrig int This is a reference to the column ProcNum of the table procedureHistory. This column has a many-to-one relationship with procedureHistory, with entries created for each change to the procedure.
deleteFlag tinyint This column identifies whether a procedure is flagged as deleted (soft-deleted) or not for treatment plan. When set to 1, the procedure is not shown for the patient Treatment plan and is treated as a non-existing entry. When 0, the procedure is treated as being present and valid.
priority smallint This column represents the priority of the procedure and is a reference to the column priorityNo of table dentalpriority.
ToothNumTP varchar This column stores the tooth number (if applicable) for which the procedure is applied.
Surf varchar This column stores the surfaces (if applicable) for which the procedure is applied. It's a combination of all applicable surfaces stored as a single string (example - MOD, DL, MODBL, etc).
ProcCode varchar This column stores the Procedure code value. It is a reference to the column 'procedurecode' of table 'dentalitems'.
Descript varchar This column stores the Procedure description. It is a reference to the column 'itemname' of table 'items'.
feeamt decimal This column stores the fee associated to the procedure.
PriInsAmt decimal This column stores the primary insurance amount associated to the procedure.
SecInsAmt decimal This column stores the secondary insurance amount associated to the procedure.
patamt decimal This column stores the patient estimate amount of the procedure.
discount decimal This column used to store the discount value for each procedure on Individual Treatment Plan screen. This column stores decimal value max=9999999.99. Column does not expect NULL values as it stores decimal value. So, default value is set to 0.00.
Prognosis varchar This column is unused and will be removed in a future build.
Dx varchar This column references the itemid column of items table and identifies one of up to four ICDs associated to the procedure.
entrystatus varchar This column references the id column of dentalentrystatus table and identifies the entry status of the procedure.
dx2 varchar This column references the itemid column of items table and identifies one of up to four ICDs associated to the procedure.
dx3 varchar This column references the itemid column of items table and identifies one of up to four ICDs associated to the procedure.
dx4 varchar This column references the itemid column of items table and identifies one of up to four ICDs associated to the procedure.
ToothRange varchar If the procedure applied is for multiple teeth, or a specific treatment area, such as quadrant, arch, sextant, or full mouth, that information is stored in this column.
areaName varchar This column value is set to ("") blank and no longer used. so this is unused and will be removed in a future build.
inactive tinyint This column is used to identify that procedure is active or inactive. Value is 0 =active. 1 = inactive.
completedDate datetime Date when the treatment plan procedure was completed. (e.g., '2021-01-01', '2021-02-01', '2021-03-01')
allowedFee decimal Allowed fee amount for the treatment plan procedure. (e.g., 90.00, 135.00, 180.00)
feesUpdatedDate datetime Date when the fees were last updated for the treatment plan procedure. (e.g., '2021-01-01', '2021-02-01', '2021-03-01')
insid int This column stores the insurance id of the patient's Dental insurance used for calculation of Treatment plan estimates, at a procedure level. This refers to "insid" column of "insurance" table.
feeschid int This column stores the Fee schedule id of the patient used for calculation of Treatment plan estimates, at a procedure level. This refers to "id" column of "feeschlist" table.
insEstimate decimal Insurance estimate for the treatment plan procedure. (e.g., 80.00, 120.00, 160.00)
insCoveragePerc decimal Insurance coverage percentage for the treatment plan procedure. (e.g., 80, 90, 100)
insEstModified tinyint Indicates if the insurance estimate was modified (1) or not (0). (e.g., 0, 1)
ptEstModified tinyint Indicates if the patient estimate was modified (1) or not (0). (e.g., 0, 1)
rowChangeStatus tinyint This column stores value 0 or 1. When a procedure's priority, tooth number, surface or D Code is changed , the column value updates to 1.
codeNumChangeStatus tinyint This column stores value 0 or 1. This column is specific to D code changes on a procedure. When a D code is changed manually or by an autocode , the column value changes to 1.
itemid int This column stores identifier number of a dental items. It refers to column itemid from dentalitems table.
annualMaxRemUsed decimal Annual maximum remaining used for the treatment plan procedure. (e.g., 1000.00, 900.00, 800.00)
deductibleRemUsed decimal Deductible remaining used for the treatment plan procedure. (e.g., 50.00, 40.00, 30.00)
annualMaxRemExhausted tinyint Indicates if the annual maximum remaining is exhausted (1) or not (0). (e.g., 0, 1)
deductibleApplied tinyint Indicates if the deductible was applied (1) or not (0). (e.g., 0, 1)
deductibleFlag tinyint Indicates if the deductible flag is set (1) or not (0). (e.g., 0, 1)
coPay decimal This new column is introduced to store the Co-Pay($) value for each procedure on Individual Treatment Plan screen. This column stores decimal value max=999999.99. Column does not expect NULL values as it stores decimal value. So, default value is set to 0.00.

IRFPAI_ABSTRACTIONS
Description:
IRFPAI_ABSTRACTIONS stores information for RDI records with a type of IRF-PAI.

Primary Key
Column Name Ordinal Position
REGISTRY_DATA_ID 1

Column Information
Name Type Discontinued?
1 REGISTRY_DATA_ID NUMERIC No
The unique identifier (.1 item) for the registry data record.
2 IRFPAI_CMG_CODE VARCHAR No
Stores CMG code associated with IRF-PAI RDI record.
3 IRFPAI_CMG_DATE DATETIME No
Stores the CMG date. CMG date is the date IRF-PAI is accepted by CMS.
4 PRES_COMPLIANCE_ADM_YN VARCHAR No
Stores if the patient qualifies as presumptive compliant or not based on the admission date.
May contain organization-specific values: No
Category Entries:
No
Yes
5 ADJ_COMPLIANCE_YN VARCHAR No
Stores if the patient qualifies as adjusted compliant or not.
The category values for this column were already listed for column: PRES_COMPLIANCE_ADM_YN
6 PRES_COMPLIANCE_DIS_YN VARCHAR No
Stores if the patient qualifies as presumptive compliant or not based on the discharge date.
The category values for this column were already listed for column: PRES_COMPLIANCE_ADM_YN
7 MEDICARE_PAT_YN VARCHAR No
Stores whether the IRF-PAI is for a Medicare patient. This is determined by what is entered in the Payer Information section of the IRF-PAI.
The category values for this column were already listed for column: PRES_COMPLIANCE_ADM_YN
8 PAT_ADMISSION_DATE DATETIME No
Stores the date that the patient was admitted to the IRF. This is determined by what is entered in the Identification Information section of the IRF-PAI.
9 REHAB_IMPAIRMENT_CATEGORY_C_NAME VARCHAR No
This is the Rehab Impairment Category (RIC) for this IRF-PAI abstraction.

IB_MESSAGES
Description:
The IB_MESSAGES table contains basic information about In Basket messages.

Primary Key
Column Name Ordinal Position
MSG_ID 1

Column Information
Name Type Discontinued?
1 MSG_ID VARCHAR No
The unique ID of the In Basket Message.
2 CREATE_TIME DATETIME (Local) No
The time the message record was created.
3 REGARDING_TOPIC VARCHAR No
A short description of the message contents.
4 PAT_ID VARCHAR No
The unique ID of the patient record for this row. This column is frequently used to link to the PATIENT table.
5 PAT_ENC_DATE_REAL FLOAT No
The encounter date of the patient associated with this message.
6 SENDER_USER_ID VARCHAR No
The unique ID of the sender (networked to the User master file).
7 SENDER_USER_ID_NAME VARCHAR No
The name of the user record. This name may be hidden.
8 NAME_OF_CALLER VARCHAR No
The name of the caller.
9 TRANSCRIPTION_LINE VARCHAR No
This is the ID of the dictation transcription for a transcription created outside of Epic and sent to In Basket through an interface. It is networked to the Notes (HNO) master file.
10 ORDER_PROC_ID NUMERIC No
The unique ID of the order procedure (networked to the Orders master file).
11 ORDER_MED_ID NUMERIC No
The unique ID of the order procedure (networked to the Orders master file).
12 ORD_DATE_REAL FLOAT No
The internal date of the ordered procedure.
13 PAT_EMAIL_RESP_YN VARCHAR No
Indicates if the patient wants an e-mail response to their In Basket message. Y indicates that the patient does want an e-mail response and N indicates that the patient does not want an e-mail response.
May contain organization-specific values: No
Category Entries:
YES
NO
14 APPT_STATUS_C INTEGER No

15 CANCEL_REASON_C INTEGER No
Appointment Notification Email Message Type - If the APPT_STATUS_C column is canceled, this column contains the link to the reason for cancellation
16 PAT_ENC_CSN_ID NUMERIC No
A unique serial number for this encounter. This number is unique across all patients and encounters in the system. This column should be replaced with IB_MESSAGE_PAT_ENCS.PAT_ENC_CSN_ID instead. Historical records will still have this data populated, as well as data moving forward, but it may not contain all of the encounter contact serial numbers (CSNs) as the new column and table would
17 PAT_ORDENC_DT_REAL FLOAT No
The new orders only encounter associated with this message. This will be used if a reflex action is performed on a result and the original contact is owned elsewhere.
18 DICT_HNO_ID VARCHAR No
This is the ID of the dictation transcription for a dictation transcribed in Epic (not in a third party system). It is networked to the notes (HNO) master file.
19 TASK_ID_ID VARCHAR No

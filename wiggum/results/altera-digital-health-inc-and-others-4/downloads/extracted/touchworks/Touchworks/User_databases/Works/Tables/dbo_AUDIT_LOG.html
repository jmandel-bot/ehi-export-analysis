RecordIDnumeric(16,0)9NOT NULLStores the recordID being touched. In some cases depending on the RecordType this will be the same as the PatientID. In other cases such as Patient Query it will be a 0 as you don't have a record in context yet because you are searching for one. Then for some events such as Medication/Order/Problems updates the recordID will map out to Activity Header ID in order_activity_header.id and problem_header.id. It can also be  task ID for a Task record type.
